# inbox-inna/audio/ — куда класть аудиофайлы для транскрипции

Сюда Инна кладёт сырые аудиофайлы, которые нужно превратить в текст:

- Zoom-записи (mp4 / m4a)
- Голосовые из Telegram / WhatsApp (ogg / opus)
- Диктофон с телефона (m4a / mp3 / wav)
- Любой готовый аудиофайл

## Что происходит дальше

1. Кладёшь файл сюда.
2. В чате говоришь Director-у: «обработай аудио <имя_файла>» (или просто «транскрибируй» — он найдёт самый свежий файл).
3. Запускается скилл [`/transcribe`](../../.claude/skills/transcribe/SKILL.md) — Слой 1 пайплайна.
4. Транскрипт уезжает в:
   - `clients/<имя>/transcripts/<дата>-<тема>.md` — если ты сказала, что это про клиента
   - `transcripts/<дата>-<источник>-<тема>.md` — если общий поток
5. **Исходный файл остаётся здесь** — для возможного переслушивания. Удаляй вручную когда тема закрыта.

## Что НЕ коммитится в git

Аудиофайлы (`.mp3`, `.wav`, `.m4a`, `.ogg`, `.opus`, `.mp4`) — в `.gitignore` (тяжёлые). Только текстовые `.md` транскрипты попадают в репо.

## Имена файлов — необязательно структурированные

Пиши как удобно — `Запись звонка 2026-05-08.m4a` или `audio_2026-05-08.ogg`. Скилл всё равно спросит у тебя короткую тему для имени итогового транскрипта.
