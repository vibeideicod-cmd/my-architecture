# agent-runtime/shared — промежуточные данные между агентами

Папка для обмена данными между агентами через файлы, а не через текст в диалоге.

## Зачем

Когда один агент закончил работу и нужно передать данные следующему:
- **Не копируем текст в диалог** — он съедается контекстом
- **Пишем в файл** — следующий агент читает его

## Как использовать

### Агент-источник (тот кто собрал данные)
```
Сохрани результат в agent-runtime/shared/<название>.md или .json
Верни Директору путь к файлу: agent-runtime/shared/<название>.md
```

### Агент-получатель (тот кто использует данные)
```
Прочитай agent-runtime/shared/<название>.md
Работай на основе этих данных
```

## Форматы файлов

- `.md` — тексты, анализы, исследования, отчёты
- `.json` — структурированные данные (котировки, списки, таблицы)

## Соглашение по именам

```
<дата>-<агент>-<тема>.md
```

Примеры:
- `2026-04-29-analytics-competitors.md` — анализ конкурентов от Аналитики
- `2026-04-29-marketer-vk-research.md` — ресёрч VK от Маркетолога
- `2026-04-29-content-drafts.md` — черновики постов от Content

## Жизненный цикл файлов

Файлы в shared/ **временные**. После завершения задачи:
- Финальный результат → переносится в `outputs/`
- Промежуточные → можно удалить или оставить для архива

## Важно

Эта папка в `.gitignore` — файлы НЕ попадают в git. Это рабочий стол, не архив.
