## Что уже наработано по HUB MVP

### Сценарий MVP HUB v0.1 — закрытый просмотр для Инны и Ирины

**Где живёт:** `ideidlyabiznesa1913.ru/preview/` (на текущем = Иринином Beget-аккаунте, тестово). Не корень `/`, чтобы не убирать Beget-заглушку до момента публичного запуска.

**Что входит в MVP (минимум):**
- Главная (Hero + плитка 5 болей + краткая С.С.С.Р. + футер) — полностью свёрстана в брендбуке Инны
- 1 лендинг боли (предлагаю «Нет клиентов» как универсальный) — длинная страница продажи: hero → симптомы → решение → продукты → AI-помощник в углу → форма
- Каталог продуктов (заглушка) — список из `inna/product-line.md`, без подробных страниц каждого продукта
- Кейсы (заглушка с 1 кейсом) — страница `/cases/` с фильтром-кнопками 8 ниш + 1 готовый кейс
- Блог (заглушка со структурой) — страница `/blog/` с 1-2 черновыми постами
- AI-помощник (заглушка-форма) — виджет в углу, реальный AI подключаем во второй итерации
- Шапка + футер — меню + контакты + ссылки на политику и оферту (даже если оферта-заглушка)

**Что НЕ делаем в MVP:**
- Остальные 4 лендинга боли (тестируем шаблон на одном)
- AI-помощник реальный (заглушка-форма пока)
- Лид-магниты `/podarok/` (после первой итерации)
- Полные страницы продуктов в каталоге
- Любые поддомены (`inna.*`, `mini.*`, `demo.*` — отдельные сессии)

### Команда и шаги (когда вернёмся)

| Сессия | Что делаем | Кто ведёт | Длительность |
|---|---|---|---|
| 1 | Бриф под HUB MVP — `brief.md` | Product-Builder + Copywriter | 30-60 мин |
| 2 | Тексты главной + 1 лендинга | Copywriter | 1-2 часа |
| 3 | Вёрстка | Websites | 1 день (vibe coding) |

Реалистичный срок до показа = **2-3 дня работы команды** без пауз.

4. **Стартуем сразу с Сессии 1 (бриф под HUB MVP) или ещё что-то уточнить?**

## Recovery-протокол (по `agents/director.md`)

Этот файл ведётся по правилу:
> Задача > 5 шагов или > 15 минут → создавай `agent-runtime/_progress-<тема>.md` (зона .gitignore).
> При продолжении в новой сессии (после «продолжи»):
> 1. Сначала читай `_progress-<тема>.md` если есть
> 2. Затем `git log --oneline -5`
> 3. Затем `git status`
> 4. Только после этого — продолжай работу.

При завершении задачи (HUB запущен публично) — **удалить этот файл**.
