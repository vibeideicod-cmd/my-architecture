# Research: Лендинг-портфолио цифрового специалиста

Исследование для разработки лендинга специалиста по сайтам, приложениям и ИИ-агентам.

## Часть 1. Примеры портфолио — что цепляет

### 1. Brittany Chiang — brittanychiang.com
**Hero:** Тёмный фон, зелёный акцент, терминальная эстетика. Имя крупно, должность, краткое описание.
**Секции:** Hero → About → Experience → Work → Contact
**Стиль:** Тёмно-синий (#0a192f) + мятно-зелёный (#64ffda). Моноширинные шрифты.
**Фишка:** Навигация с подсветкой активной секции — справа чёрточки показывают где ты находишься.

### 2. Bruno Simon — bruno-simon.com
**Hero:** Весь экран — 3D-игра на Three.js. Управляешь грузовиком по полигональному миру.
**Секции:** Одна страница = игровой мир. Проекты — объекты в 3D-пространстве.
**Стиль:** Пастельные low-poly пейзажи (розовый, лиловый, жёлтый).
**Фишка:** Весь сайт — playable experience. Невозможно уйти, не поиграв.

### 3. Tobias van Schneider — vanschneider.com
**Hero:** Большая типографика, имя на весь экран. Минимум элементов. Список избранных проектов.
**Секции:** Hero → Selected Work → About → Writing → Contact
**Стиль:** Белый/светло-серый фон, чёрный текст, красный акцент. Швейцарский editorial стиль.
**Фишка:** Hover на проектах — изображение следует за курсором. Магнетизм на меню.

### 4. Lynn Fisher — lynnandtonic.com
**Hero:** Иллюстрированный персонаж, меняющийся при resize браузера. Каждый год сайт перерисовывается.
**Секции:** Hero → Projects → Side Projects → Talks
**Стиль:** Меняется каждый год. Иллюстративный, яркий, SVG-рисунки.
**Фишка:** Responsive design как арт-объект — персонаж меняется при изменении ширины окна. Чистый CSS.

### 5. Josh Comeau — joshwcomeau.com
**Hero:** Яркий, неоновые акценты, 3D-иконки. Нестандартная вёрстка с перекрывающимися элементами.
**Секции:** Hero → Latest Posts → Courses → Featured Articles
**Стиль:** Тёмный фон + неоновые розово-пурпурные акценты. «Candy dark» эстетика.
**Фишка:** Кастомный курсор, световые эффекты при движении мыши, интерактивные компоненты в статьях.

### 6. Stripe — stripe.com
**Hero:** Анимированный градиентный фон (фиолетово-синие переливы), чёткий заголовок, мокап продукта.
**Секции:** Hero → Social proof → Features → How it works → Developer experience → Pricing → Footer CTA
**Стиль:** Тёмно-фиолетовый/синий, белый текст, светло-голубой акцент. Технологично, дорого.
**Фишка:** Живой mesh-градиент в hero — мягко движется, создаёт ощущение живого продукта.

### 7. Linear — linear.app
**Hero:** Кино-тёмный интерфейс, тонкая типографика, скриншот продукта с невероятной детализацией.
**Секции:** Hero → Features (scroll-анимации) → Workflow → Integrations → Testimonials → Pricing
**Стиль:** Почти чёрный (#0f0f0f), тонкий белый текст, фиолетово-синий акцент. Ультраминимализм.
**Фишка:** Интерфейс продукта «строится» при скролле — каждый feature появляется через reveal.

### 8. Paco Coursey — paco.me
**Hero:** Абсолютный минимализм. Только имя, должность, три строки текста, список ссылок.
**Секции:** Hero → Projects → Writing → Craft
**Стиль:** Белый или очень тёмный фон. Системные шрифты. Zero decoration.
**Фишка:** Раздел «Craft» — коллекция крошечных интерактивных UI-экспериментов. Показывает скилл лучше резюме.

## Часть 2. Идеальная структура лендинга (8-10 блоков)

```
[1] HERO
[2] SOCIAL PROOF
[3] УСЛУГИ
[4] КАК Я РАБОТАЮ
[5] ПРОЕКТЫ
[6] ОТЗЫВЫ
[7] СТЕК
[8] ОБО МНЕ
[9] FAQ
[10] КОНТАКТ / CTA
```

### Блок 1 — HERO
**Цель:** Захватить внимание за 3 секунды. Дать понять кто ты и чем поможешь.
**Содержимое:**
- Сверхкраткое представление (1-2 слова: кто ты)
- Заголовок-ценность (что получает клиент, не что делаешь ты)
- Подзаголовок (технологии, аудитория, подход)
- Два CTA: «Обсудить проект» + «Смотреть работы»
- Маленький социальный сигнал прямо под кнопками: «5 проектов сдано за последние 3 месяца»
- Визуальный элемент: анимация / 3D / мокап / абстракция

**Почему важен:** Решает bounce rate. Не зацепил за 3 секунды — человек ушёл.

### Блок 2 — SOCIAL PROOF
**Цель:** Снять первичный скептицизм сразу после hero.
**Содержимое:** Логотипы клиентов (5-8) или ключевые цифры (проектов, лет, клиентов) или одна сильная цитата.
**Почему важен:** Сигналы доверия снижают тревогу и увеличивают время на сайте.

### Блок 3 — УСЛУГИ
**Цель:** Клиент должен узнать себя в твоих услугах.
**Содержимое:** 3-6 карточек. Каждая: название → боль клиента → что получает.
**Почему важен:** Без этого блока теряются и нецелевые, и целевые посетители.

### Блок 4 — КАК Я РАБОТАЮ
**Цель:** Убрать страх неизвестности.
**Содержимое:** 3-5 шагов: Бриф → Прототип → Разработка → Запуск → Поддержка. Опционально — сроки.
Добавить акцент: «от тебя нужно минимум» — снимает страх перегрузки клиента.
**Почему важен:** Люди охотнее покупают, когда понимают что их ждёт.

### Блок 5 — ПРОЕКТЫ
**Цель:** Доказать делом. Показать реальный результат.
**Содержимое:** 3-6 проектов. Каждый: визуал + название + тип + результат (цифры).
Формат кейса: **Задача → Решение → Результат в цифрах**.
Hover на карточке открывает описание снизу вверх — не уводит со страницы.
**Почему важен:** Главный аргумент для решения. Либо доверяют, либо нет.

### Блок 6 — ОТЗЫВЫ
**Цель:** Социальное доказательство от тех, кто уже прошёл путь клиента.
**Содержимое:** 3-6 отзывов с фото, именем, должностью. Каждый закрывает конкретное возражение.
**Почему важен:** Один отзыв убеждает сильнее десяти параграфов текста.

### Блок 7 — СТЕК
**Цель:** Показать профессиональную глубину.
**Содержимое:** Иконки технологий по категориям: Frontend / Backend / AI / Tools.
**Почему важен:** Для техклиентов — фильтр совместимости. Для нетехнических — сигнал серьёзности.

### Блок 8 — ОБО МНЕ
**Цель:** Humanize. Клиент нанимает человека, не резюме.
**Содержимое:** Живое фото + 2-3 абзаца (кто, путь, что отличает) + личные детали + ссылки.
**Почему важен:** Решает chemistry. Часто именно этот блок — последнее перед тем как написать.

### Блок 9 — FAQ
**Цель:** Закрыть возражения без переписки.
**Содержимое:** 5-8 реальных вопросов: цена, сроки, удалённость, гарантии.
Для AI-направления добавить: «Что такое ИИ-агент и нужен ли он мне?»
**Почему важен:** Клиент, получивший ответы заранее, пишет увереннее.

### Блок 10 — КОНТАКТ / CTA
**Цель:** Конвертировать интерес в действие.
**Содержимое:** Сильный заголовок + короткая форма или кнопка в Telegram/WhatsApp + время ответа.
**Почему важен:** Без чёткого CTA пользователь теряется. Должен быть простым и приглашающим.

## Часть 3. 12 трендов 2025-2026

### Визуальные тренды

**1. Bento Grid**
Сетка неравных карточек в стиле Apple — разные размеры, единый ритм. Каждая карточка = один факт.
Применить: блок услуг или проектов в асимметричный CSS Grid с colspan.
Сложность: средняя.

**2. Dark Mode как основа**
Не просто чёрный фон, а глубокие тёмно-синие (#0d0d0f, #0a0a14) с градиентами между тёмными оттенками.
Применить: избегать чистый #000000, работать с глубиной через оттенки.
Сложность: низкая.

**3. Fluid Typography с clamp()**
Шрифты плавно масштабируются от мобильного к десктопу без медиазапросов.
Применить: `font-size: clamp(2rem, 5vw, 6rem)` для hero-заголовков.
Сложность: низкая.

**4. Noise Texture + Glassmorphism**
Тонкий зернистый шум (3-8% opacity) + полупрозрачные карточки с backdrop-filter blur.
Применить: SVG-фильтр или PNG-оверлей + `background: rgba(255,255,255,0.05)` на карточках.
Сложность: низкая / средняя.

**5. Большая акцентная типографика**
Слово или два на весь экран. Иногда текст намеренно обрезается краями. Шрифты с характером.
Применить: `font-size: clamp(4rem, 15vw, 14rem)`, overflow hidden на контейнере.
Сложность: низкая.

**6. Organic / Blob-формы**
Плавные неправильные формы для фонов, аватаров, секций. Противовес жёсткой сетке.
Применить: `border-radius: 60% 40% 30% 70% / 60% 30% 70% 40%` или CSS clip-path.
Сложность: низкая.

### Анимации и интерактивность

**7. Kinetic Typography**
Текст — главный визуальный элемент. Заголовки появляются по словам (slide-up), буквы перелистываются как табло аэропорта (flip-эффект). Без иллюстраций — только шрифт и анимация.
Применить: hero-заголовок появляется по словам через CSS `@keyframes` + JS IntersectionObserver.
Сложность: высокая (эффект wow-уровня).

**7а. Typewriter эффект**
Текст «печатается» посимвольно. Мигающий курсор через CSS `::after { animation: blink 1s step-end infinite }`.
Применить: для смены ролей в подзаголовке hero — «Разработчик. AI-инженер. Ваш партнёр.»
Сложность: низкая (~15 строк JS).

**7б. Scroll Progress Bar**
Тонкая полоска вверху страницы показывает прогресс прокрутки.
Применить: `window.scrollY / document.body.scrollHeight`, обновляет `width` элемента. Цвет — акцентный (оранжевый).
Сложность: низкая.

**7в. Scroll-triggered Reveal (CSS-only)**
С 2025 года браузеры поддерживают `animation-timeline: scroll()` — элементы анимируются при скролле без JS.
Применить: `@keyframes` + `animation-timeline: scroll()` для прогресс-баров навыков.
Сложность: низкая (только CSS), но пока не все браузеры.

**8. Scroll-triggered Reveal**
Элементы появляются при скролле — снизу вверх, с fade, с лёгким смещением.
Применить: JS Intersection Observer + CSS transition. ~20 строк кода.
Сложность: низкая.

**8. Marquee / бесконечная строка**
Логотипы, теги, цитаты едут горизонтально в бесконечном цикле.
Применить: CSS animation с `translateX` и дублированием контента. Без JS.
Сложность: низкая.

**9. Cursor-следящие эффекты**
Кастомный курсор, magnetic effect (элементы тянутся к мышке), spotlight где свет идёт от курсора.
Применить: JS mousemove → CSS custom property → radial-gradient как «фонарик».
Сложность: средняя.

**10. Hover с image reveal**
При наведении на ссылку проекта появляется превью-изображение, следующее за курсором.
Применить: абсолютно позиционированный img + JS добавляет координаты через transform.
Сложность: средняя.

**11. Stagger animations**
Карточки появляются с задержкой — одна за другой. Ощущение живого ритма.
Применить: CSS `animation-delay` с шагом 0.1s или JS setTimeout на каждый элемент.
Сложность: низкая.

**12. Smooth scroll + секционная навигация**
Плавная прокрутка + фиксированный индикатор текущей секции (точки или чёрточки сбоку).
Применить: `scroll-behavior: smooth` + IntersectionObserver меняет активную точку.
Сложность: средняя.

## Часть 4. Позиционирование AI/Digital специалиста

### 5 сильных заголовков для hero

1. **«Превращаю бизнес-задачи в работающий цифровой продукт»**
   Фокус на результат. Понятно любому клиенту.

2. **«Создаю сайты, приложения и ИИ-агентов — от идеи до запуска»**
   Конкретный список + полный цикл работы.

3. **«Ваш технический партнёр — без корпоративной бюрократии»**
   Против крупных агентств. Для тех, кто обжигался на студиях.

4. **«Автоматизирую рутину, создаю продукты, которые работают без вас»**
   Акцент на ИИ и автоматизацию. Для предпринимателей, ценящих время.

5. **«Full-stack разработчик × AI-инженер — строю умные цифровые решения»**
   Технический вариант. Для аудитории, знакомой с терминами.

### Варианты позиционирования

- **«Цифровой мастер»** — широко, понятно, без перегруза. Для любой аудитории.
- **«Full-stack разработчик и AI-инженер»** — точно, технически. Для стартапов.
- **«Независимый продуктовый разработчик»** — для тех кто строит продукты, не просто пишет код.
- **«Веб-разработчик + AI-интегратор»** — конкретный стек для малого бизнеса.
- **«Строю ИИ-помощников и сайты для [ниша]»** — самый конвертирующий вариант при наличии опыта в конкретной отрасли (рестораны, клиники, e-commerce). Узкое позиционирование даёт лучшие лиды.

### 10 услуг для лендинга

1. Сайт-лендинг под ключ
2. Многостраничный корпоративный сайт
3. Веб-приложение (SaaS)
4. Мобильное приложение (iOS/Android)
5. ИИ-агент для бизнеса
6. Автоматизация рутины (n8n/Make/Zapier)
7. Интеграция AI в существующий продукт
8. Аудит и ускорение существующего сайта
9. Техподдержка и развитие (monthly)
10. Консультация по технологиям и архитектуре

### Типичные боли клиентов (по категориям)

**Про время:**
- «Нет времени разбираться в технологиях самому»
- «Уже месяц откладываю запуск — не знаю с чего начать»

**Про деньги и агентства:**
- «Агентство просит огромную сумму и ещё 3 месяца ждать»
- «Заплатил фрилансеру — пропал после предоплаты»
- «Не понимаю, за что столько платить»

**Про технологии:**
- «Слышал об ИИ, но не понимаю как это применить у меня»
- «Есть сайт, но он не приносит клиентов»
- «Хочу автоматизировать процессы, но не знаю что возможно»

**Про коммуникацию:**
- «Объясняю задачу, а разработчик делает что-то своё»
- «Каждый вопрос — отдельный счёт»

### Примеры социального доказательства

**Цифры:** «17 проектов за 2 года», «Средний срок — 3 недели», «8 из 10 клиентов возвращаются», «70% заказчиков возвращаются с новыми задачами»

**Отзывы — формула:** имя + должность/компания + результат в цифрах:
- «Запустили сайт за 2 недели вместо обещанных 2 месяцев у другого разработчика — Алексей, владелец кофейни»
- «Бот закрывает стандартные вопросы клиентов 24/7 — менеджер освободил 6 часов в неделю»

**Кейсы в 3 строки (Задача → Решение → Результат):**
- «Конверсия лендинга выросла с 1.2% до 3.8% после редизайна»
- «Автоматизация сэкономила 40 часов в месяц»

**До/после:** скриншоты старого и нового сайта — самый убедительный формат.

## Итог: рекомендации для лендинга

**Структура:** 10 блоков в порядке выше. Начать с минимума — Hero + Услуги + Проекты + Контакт.

**Стиль:** Тёмный фон (#0a0a0f или #0d0d1a), оранжевый или горчичный акцент, Montserrat + Playfair Display.

**Анимации для старта (низкая сложность):**
- Scroll reveal на всех секциях
- Marquee для логотипов клиентов или стека технологий
- Stagger на карточках услуг и проектов

**Фишки для wow-эффекта (средняя сложность):**
- Spotlight-курсор в hero
- Hover image reveal на проектах
- Bento Grid для услуг

**Позиционирование:** «Создаю сайты, приложения и ИИ-агентов — от идеи до запуска» — максимально конкретно и понятно.

## Часть 4. Экспертная оценка лендинга-портфолио цифрового мастера

Проект: лендинг специалиста по сайтам, приложениям и ИИ-агентам.
Оценка с трёх точек зрения: UX-дизайнер → маркетолог → клиент.

---

### 🎨 Эксперт 1: UX-дизайнер (10 лет опыта)

#### Типичные ошибки новичков в структуре

**1. Hero говорит о тебе, а не о клиенте.**
Новички пишут: "Я — Инна, веб-разработчик с 5-летним опытом". Клиенту не важно кто ты — важно что он получит. Правильно: "Сайт, который приводит клиентов — не просто красиво выглядит."

**2. Портфолио без контекста.**
Показывают скриншоты без объяснения: что была за задача, что сделали, какой результат. Клиент видит картинку — и не понимает, применимо ли это к его задаче.

**3. Слишком много разделов навигации.**
6–8 пунктов меню = когнитивная перегрузка. На лендинге-портфолио достаточно 3–4: Работы / Услуги / Обо мне / Контакт.

**4. Форма контакта в самом низу и только там.**
Пользователь готов написать уже после второго-третьего блока. Если кнопки нет — уходит. CTA должен появляться минимум трижды.

**5. Неадаптивные проекты.**
Портфолио выглядит хорошо на десктопе, разваливается на телефоне. Именно с телефона часто смотрят "на первый взгляд".

**6. Отсутствие визуальной иерархии.**
Всё одного размера и цвета — взгляду не за что зацепиться. Нет главного — нет порядка чтения.

#### Что ОБЯЗАТЕЛЬНО должно быть

| Элемент | Зачем |
|---------|-------|
| Заголовок-ценность в Hero | Объясняет кто ты за 3 секунды |
| 2–3 работы с описанием | Доказывает компетентность |
| Процесс работы (3–5 шагов) | Снимает страх "а как это вообще происходит" |
| Кнопка CTA минимум 3 раза | Не теряет готового клиента |
| Мобильная версия | 60–70% трафика — телефон |
| Один чёткий способ связи | Telegram-ссылка лучше формы |
| Скорость загрузки до 3 сек | Каждая секунда = потеря 7% конверсии |

#### Что раздражает и заставляет уйти

- Автовоспроизведение видео или музыки
- Анимации, которые нельзя пропустить (блокируют контент)
- Горизонтальный скролл на мобильном
- Текст мельче 16px или серый-на-тёмном (особенно для 40+)
- "Сайт в разработке" или пустые заглушки в портфолио
- Форма с 5+ полями (имя, фамилия, телефон, email, бюджет, сроки)
- Отсутствие цен или хотя бы ориентира стоимости — создаёт тревогу
- Слишком много всплывающих окон

### 📊 Эксперт 2: Маркетолог, специалист по конверсии

#### Оффер, который зацепит

Слабый оффер: "Делаю сайты, приложения и ИИ-агентов"
Сильный оффер строится по формуле: **[Кому] + [Что получит] + [За счёт чего] + [Без чего]**

Примеры для разных аудиторий:

> **Для малого бизнеса:**
> "Сделаю сайт, который превращает посетителей в заявки — без технических заморочек с вашей стороны"

> **Для тех, кто хочет автоматизацию:**
> "Настрою ИИ-агента, который отвечает клиентам вместо вас — 24/7, без найма сотрудника"

> **Широкий оффер:**
> "Создаю цифровые инструменты для бизнеса — от лендинга за неделю до ИИ-автоматизации под ключ"

Ключевое: **оффер должен решать конкретную боль**, а не описывать ваши навыки.

#### Триггеры доверия — по приоритету

**Уровень 1 — критически важно:**
- Реальные работы с результатами (не просто "красивые скриншоты")
- Имя и фото — живой человек, не анонимное агентство
- Конкретные цифры: "5 проектов за 3 месяца", "сайт запустили за 9 дней"

**Уровень 2 — сильно помогает:**
- Отзывы с именем, должностью, компанией (не "Анна К., предприниматель")
- Процесс работы — клиент понимает, что его не бросят
- FAQ с честными ответами на "а сколько стоит" и "как быстро"

**Уровень 3 — дополнительный вес:**
- Логотипы компаний-клиентов
- Упоминания в медиа, телеграм-каналах, профессиональных сообществах
- Стек технологий (показывает уровень, не только широту)

#### Путь посетителя от "зашёл" до "оставил заявку"

```
Зашёл по рекламе / рекомендации / поиску
        ↓
[Hero] — понял что ты делаешь и для кого (3 сек)
        ↓
[Social Proof / цифры] — первичное доверие (5 сек)
        ↓
[Услуги] — "о, это именно то, что мне нужно" (20 сек)
        ↓
[Проекты] — "вот это похоже на мою задачу" (30 сек)
        ↓
[Процесс] — "понятно, как это происходит, не страшно" (10 сек)
        ↓
[Отзывы/кейсы] — "другим помогло, поможет и мне" (20 сек)
        ↓
[CTA] — нажимает "Обсудить проект" или пишет в Telegram
        ↓
Заявка
```

Суммарно: клиент принимает решение за 1.5–3 минуты. Каждый блок работает на одну задачу. Лишние блоки — потеря внимания.

**Правило:** убрать всё, что не двигает клиента вперёд по этому пути.

#### Частые ошибки конверсии

- CTA "Написать мне" — слишком абстрактно. Лучше: "Обсудить проект" / "Получить оценку стоимости"
- Кнопка одного цвета с фоном — теряется визуально
- Нет мессенджера — часть аудитории не любит формы, уходит
- Блок "Обо мне" раньше блока "Услуги" — сначала клиент, потом вы

### 👤 Эксперт 3: Потенциальный клиент — владелец бизнеса

*Внутренний монолог типичного клиента при просмотре лендинга*

#### Первые 5 секунд — что он видит и думает

Открывает страницу. Смотрит на Hero.

**Вопросы, которые возникают мгновенно:**
- "Это вообще про меня? Мне здесь помогут?"
- "Это серьёзно или очередной фрилансер?"
- "Куда нажать, если захочу написать?"

Если на эти три вопроса нет ответа — он уходит. Не потому что плохо, а потому что непонятно.

**Что цепляет взгляд за 5 секунд:**
1. Заголовок — самый крупный текст на экране
2. Кнопка CTA — если она контрастная
3. Пример работы или фото — что-то конкретное, не абстракция

#### Что заставит остаться

- **Узнал себя в описании.** "Сайт для малого бизнеса" → "это я". "Лендинг за 7 дней" → "нужно быстро — это вариант".
- **Увидел понятный результат.** Не "разработка сайтов", а "лендинг, который собирает заявки".
- **Почувствовал, что его поймут.** Простой язык без терминов, описание процесса "для обычного человека".
- **Нашёл похожий проект в портфолио.** "О, они делали для кафе — у меня тоже кафе."
- **Увидел реальные цифры.** Стоимость или хотя бы диапазон — "от 30 000 руб." снимает тревогу неизвестности.

#### Что заставит закрыть вкладку

- Непонятно чем отличается от всех остальных ("делаю сайты" — это делают все)
- Нет примеров — только слова
- Слишком сложно: технические термины, которые клиент не понимает (REST API, WebSocket, Docker)
- Не видно цены — сразу думает "дорого, раз не пишут"
- Трудно связаться — не нашёл кнопку или форму за 10 секунд
- Страница долго грузится — закрыл, не дождавшись
- Ощущение "шаблона" — типовой дизайн, который видел 100 раз

#### Главные инсайты от клиента

> "Мне не нужен самый технически продвинутый специалист. Мне нужен тот, кто **поймёт задачу**, **сделает в срок** и **не бросит после запуска**."

> "Если я не понимаю что написано — ищу другого. У меня нет времени разбираться."

> "Первый вопрос — сколько это стоит. Второй — как быстро. Третий — вы уже такое делали?"

### Общий список рекомендаций

Сводный чеклист на основе трёх экспертных точек зрения.

#### Структура и контент

- [ ] Hero: заголовок о выгоде клиента, не о навыках исполнителя
- [ ] Под заголовком — подзаголовок с уточнением: для кого, что именно, как быстро
- [ ] CTA на первом экране: одна главная кнопка + альтернатива (Telegram-ссылка)
- [ ] Social proof сразу после Hero: цифры или логотипы (не в конце страницы)
- [ ] Услуги: 3–4 карточки с болью клиента → что получит
- [ ] Портфолио: минимум 3 работы, каждая с задачей → решением → результатом
- [ ] Процесс: 3–5 шагов, на языке клиента, первый шаг = бесплатно/без обязательств
- [ ] Отзывы: с именем, компанией, конкретным результатом (не "всё отлично!")
- [ ] FAQ: "сколько стоит", "как долго", "что мне нужно сделать", "что если не понравится"
- [ ] Финальный CTA: другими словами, не копия первого

#### UX и дизайн

- [ ] Мобильная версия проверена на реальном телефоне, не только в DevTools
- [ ] Шрифт основного текста — минимум 16px
- [ ] Кнопки CTA — минимум 48px высота, контрастный цвет
- [ ] Страница грузится за 2–3 секунды (WebP-изображения, preconnect для шрифтов)
- [ ] Нет горизонтального скролла
- [ ] Анимации не блокируют контент и не раздражают

#### Конверсия

- [ ] Оффер: конкретная боль клиента + конкретный результат
- [ ] CTA появляется минимум 3 раза (Hero / после проектов / финал)
- [ ] Рядом с CTA — снятие возражения: "Без предоплаты" / "Ответ за 2 часа"
- [ ] Ценовой ориентир: хотя бы "от X руб." — убирает тревогу
- [ ] Telegram/WhatsApp как альтернатива форме
- [ ] Нет лишних полей в форме — максимум имя + телефон/мессенджер

#### Позиционирование

- [ ] Чёткий ответ: чем отличаетесь от "просто сделать сайт"?
- [ ] ИИ-агенты — выделить отдельно, это редкая компетенция в 2025 году
- [ ] Специализация по нише (если есть): "работаю с кафе и ресторанами" — сильнее чем "работаю со всеми"
- [ ] Личность: живое фото, 2–3 строки о себе как о человеке — не резюме
