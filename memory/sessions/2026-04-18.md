## Открытое — вернуться при реструктуризации архитектуры

**Безопасность push в main.** Сейчас Claude Code блокирует прямой push в main (индустриальный дефолт), хотя CLAUDE.md проекта разрешает. Инна решила не клеить защиту на старую архитектуру — учтём при реструктуризации.

Когда будем переделывать — спроектировать слоями:
1. **Git pre-commit hook** в репо — ловит секреты (`.env`, API-ключи по паттернам `sk-...`/`ghp_...`/32+ hex), большие файлы (>10 МБ). Блокирует опасный коммит до push.
2. **Политика push для Claude** в CLAUDE.md — чёткие границы: что пушу сама (доки, knowledge, brief/research), что спрашиваю (`.sh`, `.env.example`, код клиента, >10 файлов в diff, правки корневого CLAUDE.md или director.md), что не пушу никогда (rebase-merge-конфликты, сломанный QA).
3. **Permission rule** узко для `my-architecture/` в `.claude/settings.local.json` — разрешить push только здесь, чтобы в других проектах (vps-manager, будущие) защита сохранялась.

**Риски если пустить push в main без защиты:** секрет в public git-истории, сломанный деплой у клиента (у нас автодеплой с main через `deploy-*.sh`), засорённая история, потеря последнего контроля Инны перед production.

**Сейчас на временно:** коммит делает Claude, push делает Инна вручную.
