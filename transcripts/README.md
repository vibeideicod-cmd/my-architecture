# transcripts/ — общий склад сырых транскриптов

Сюда попадают результаты Слоя 1 пайплайна (`/transcribe`), когда транскрипт **не привязан к конкретному клиенту**:

- Встречи с Ириной (визитка, бот, сайт, обсуждение продуктов)
- Мастергруппа — встречи с кураторами
- Ученики Димы — ежедневные встречи
- Практикумы и мастер-классы
- Диктофон Инны
- Голосовые в Telegram (если не клиентские)

## Когда транскрипт идёт НЕ сюда

Если аудио про конкретного клиента → транскрипт уезжает в `clients/<имя>/transcripts/`, не сюда. Решает скилл `/transcribe` по подсказке Инны.

## Формат имени файла

```
<YYYY-MM-DD>-<источник>-<тема>.md
```

Источник — короткая метка (`ирина`, `mastergroup`, `ученики`, `диктофон`, `irina-zoom`).
Тема — 2-4 слова сути.

Примеры:
- `2026-05-08-ирина-визитка-обсуждение.md`
- `2026-05-09-mastergroup-кураторы-неделя.md`
- `2026-05-09-ученики-дима-вторник.md`
- `2026-05-10-диктофон-идея-нового-продукта.md`

## Структура файла

В каждом транскрипте:
- frontmatter (дата, источник, длительность, формат, число спикеров)
- `## Саммари` — 5-10 строк выжимки
- `## Транскрипт` — полный текст (с метками Speaker 0/1 если несколько участников)

## Как пользоваться складом

1. **Положила аудио → запустила `/transcribe` → транскрипт здесь.**
2. Когда нужен **разбор на артефакты** — запускаешь `/audio-debrief <путь к транскрипту>` (Слой 2). Скилл сам разложит блоки по местам: `clients/`, `knowledge/`, `ideas/`, `inna/`.
3. Если транскрипт уже не нужен — удали вручную или оставь как архив.
