## Принцип

По auto-memory `feedback_no_deletion_archive_first`:
1. Спорное/устаревшее уезжает в `_архив-возможно-удалить/` через `mv` (не `rm`)
2. Архив фиксируется записью в `_архив-возможно-удалить/README.md`
3. Финальное `rm -rf` — только Инной, отдельной командой, не автоматически

В этом аудите **физических операций не делаем** — только списки и рекомендации.

## Категория 1 — безопасный мусор macOS

### `.DS_Store` файлы (14 штук в репо)

```bash
find . -name ".DS_Store" -not -path "./node_modules/*" -not -path "./.git/*"
```

- **Что это:** служебные файлы Finder с метаданными отображения папок (иконки, позиции, превью).
- **Зачем чистить:** засоряют `git status`, попадают в diff, никакой полезной нагрузки.
- **Рекомендация:** удалить все через `find . -name ".DS_Store" -delete`. Добавить `.DS_Store` в `.gitignore` если ещё не добавлен.
- **Когда делать:** отдельной командой, чтобы коммит был чисто «чистка .DS_Store». Безопасно.
