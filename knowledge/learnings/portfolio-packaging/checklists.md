# Чек-листы и фреймворки: упаковка портфолио

> **Назначение:** готовые рабочие чек-листы и фреймворки из урока. Применяются как есть или адаптируются под клиента.

---

## Как пользоваться

1. Выбери чек-лист или фреймворк под свою задачу
2. Скопируй структуру в свой документ
3. Заполни под клиента
4. После использования — отметь в frontmatter методички (`использовано-в:`)

---

## Чек-листы

### 1. Чек-лист «Готов ли я выйти с портфолио»
<!-- LIBRARIAN-CHECKLIST: ready-with-portfolio -->

**Когда применять:** перед запуском нового специалиста / нового сегмента / нового продукта в публичную плоскость.

**Шаги:**

- [ ] Выбрал **одну конкретную нишу** — не «все подряд»
- [ ] Провёл исследование рынка через Claude (Промпт 1 в `prompts.md`)
- [ ] Прошёл интервью для позиционирования — выбрал лучший вариант (Промпт 2)
- [ ] Создал **демо-продукт** для своей ниши (рабочий прототип, не финальный продукт)
- [ ] Собрал **портфолио-страницу** через Claude (Промпт 3)

**Если хотя бы 1 пункт не сделан** — портфолио рано публиковать. Дозревёт.

---

### 2. Чек-лист «Что должно быть в портфолио — 5 элементов»
<!-- LIBRARIAN-CHECKLIST: portfolio-five-elements -->

**Когда применять:** при создании портфолио-страницы, лендинга специалиста, презентации, визитки.

| Элемент | Зачем | Что показать | Готово? |
|---|---|---|---|
| **Кто я** | Доверие | Ниша + опыт + подход к работе | ☐ |
| **Что решаю** | Ценность | 3-4 конкретные боли клиентов в нише | ☐ |
| **Примеры работ** | Доказательство | 3-5 проектов: боль → решение → результат | ☐ |
| **Демо** | Убедительность | Ссылка на рабочий прототип или видео | ☐ |
| **Контакт** | Конверсия | Telegram, способ написать прямо сейчас | ☐ |

**Антипаттерны:**
- ❌ Кейсы без формулы «боль → решение → результат» = это отчёт, а не кейс
- ❌ Контакт «info@example.com» = не личный, не для разговора
- ❌ Нет демо = клиент видит слова, не работу
- ❌ «Что решаю» через инструменты, не через боли = клиент не узнаёт свою проблему

---

### 3. Чек-лист «Позиционирование сформулировано правильно»
<!-- LIBRARIAN-CHECKLIST: positioning-correct -->

**Когда применять:** после получения вариантов позиционирования из Промпта 2 — для финальной валидации перед использованием.

- [ ] **Нет слов** «разработчик», «программист», «вайбкодер», «дизайнер», «специалист»
- [ ] **Нет названий инструментов** (Claude, Tilda, Figma, Canva)
- [ ] **Есть конкретная ниша** или сегмент клиентов
- [ ] **Есть конкретная боль**, которую решает
- [ ] **Есть результат**, который получает клиент
- [ ] **Длина:** 1-2 предложения максимум
- [ ] **Тон:** как у эксперта в нише, не как у технаря

**Хорошие примеры формулировок:**
- ✅ «Помогаю салонам красоты не терять клиентов из-за пропущенных записей»
- ✅ «Делаю онлайн-школам платформы, на которых ученики не бросают курс»
- ✅ «Слышу, какого человек цвета — перевожу хаос в чёткие, красивые, ясные картинки» (Ирина)
- ✅ «Архитектор автосистем и коммуникаций для тех, кто работает 24/7» (Инна)

**Плохие примеры:**
- ❌ «Разработчик с экспертизой в Claude Code и Cursor»
- ❌ «Специалист по автоматизации бизнеса»
- ❌ «Делаю красивые сайты на Tilda»

## Фреймворки

### Фреймворк «Боль → Решение → Результат» (формула кейса)
<!-- LIBRARIAN-FRAMEWORK: pain-solution-result -->

**Назначение:** правильно сформулировать каждый кейс в портфолио, чтобы клиент думал «этот человек понимает мою проблему».

**Структура:**

```
Кейс: [короткий цепляющий заголовок]

🔴 БОЛЬ КЛИЕНТА
   Что было плохо. Конкретно. Без воды.
   Например: «Салон терял по 5-7 записей в день из-за пропущенных
   звонков в нерабочее время.»

🛠 РЕШЕНИЕ
   Что я сделал. Без технических терминов.
   Например: «Сделал бот в Telegram, который записывает клиентов
   в любое время, синхронизируется с журналом мастеров и
   отправляет напоминания.»

🟢 РЕЗУЛЬТАТ
   Что получил клиент. С цифрой если есть.
   Например: «Через месяц количество записей выросло на 30%,
   ноль пропущенных запросов.»
```

**Применение в портфолио Ирины:**

```
Кейс: Брендбук для онлайн-школы «X»

🔴 БОЛЬ КЛИЕНТА
   У школы было 4 преподавателя — каждый делал презентации в своём
   стиле. Ученики не узнавали бренд от урока к уроку.

🛠 РЕШЕНИЕ
   Изготовила дизайн-систему: 8 цветов, 2 шрифта, 12 шаблонов
   слайдов, библиотеку иконок. Передала с гайдом «как пользоваться».

🟢 РЕЗУЛЬТАТ
   Все презентации теперь визуально единые. Ученики стали узнавать
   школу с первого слайда. Преподаватели сэкономили час на каждом уроке.
```

### Фреймворк «3 источника ниши»
<!-- LIBRARIAN-FRAMEWORK: three-niche-sources -->

**Назначение:** найти свою нишу через пересечение трёх измерений.

**Структура:**

```
Источник 1: ПРОФЕССИОНАЛЬНЫЙ ОПЫТ ДО КУРСА
   Где я разбираюсь лучше среднего? В каких темах меня уже считают
   экспертом или к кому приходят с вопросами?

Источник 2: ПРОЕКТЫ, СДЕЛАННЫЕ НА КУРСЕ
   Какие из проектов получились лучше всего? Что было в кайф делать?

Источник 3: ОКРУЖЕНИЕ
   В каком бизнесе работают ваши знакомые? Чьи боли вы понимаете
   изнутри, потому что слышите их каждый день?

ПЕРЕСЕЧЕНИЕ (то, где все три источника говорят одно)
   = ваша ниша
```

**Применение для Ирины (пример заполнения):**

```
1. ПРОФ.ОПЫТ:
   - Графический дизайн (Photoshop, Illustrator) с большим опытом
   - Иллюстрация
   - Опыт работы с малым бизнесом

2. ПРОЕКТЫ НА КУРСЕ:
   - Лендинги с использованием AI
   - Мини-приложения в Telegram (TMA)
   - Брендбуки, упакованные через Claude Code

3. ОКРУЖЕНИЕ:
   - Партнёрство с Инной (архитектура автосистем) — клиенты
     самозанятые эксперты и малый бизнес
   - 8 ниш: инфобизнес, финансы, бьюти, коучи, фитнес, торговля,
     риэлторы, санатории

ПЕРЕСЕЧЕНИЕ:
   Дизайн + интерактивные продукты для самозанятых экспертов и
   малого бизнеса в 8 нишах
```

### Фреймворк «Структура портфолио-страницы (HTML)»
<!-- LIBRARIAN-FRAMEWORK: portfolio-page-html-structure -->

**Назначение:** готовый каркас HTML-страницы портфолио, генерируемой через Промпт 3.

**Структура:**

```html
<section class="hero">
  <h1>{Заголовок: что делаю и для кого}</h1>
  <p>{Подзаголовок: позиционирование через результат}</p>
  <a href="...">Связаться</a>
</section>

<section class="solve">
  <h2>Что я решаю</h2>
  <div class="pain">{Боль 1}: {как проявляется}</div>
  <div class="pain">{Боль 2}: ...</div>
  <div class="pain">{Боль 3}: ...</div>
  <div class="pain">{Боль 4}: ...</div>
</section>

<section class="cases">
  <h2>Проекты</h2>
  <div class="case">
    <h3>{Название кейса}</h3>
    <p class="pain">🔴 {боль}</p>
    <p class="solution">🛠 {решение}</p>
    <p class="result">🟢 {результат}</p>
  </div>
  <!-- 3-5 кейсов -->
</section>

<section class="demo">
  <h2>Попробовать</h2>
  <a href="{ссылка-на-демо}">{название демо-продукта}</a>
</section>

<section class="cta">
  <h2>Расскажи про свой проект</h2>
  <a href="https://t.me/...">Написать в Telegram</a>
</section>
```

## Шаблоны

### Шаблон «Описание проекта в портфолио»
<!-- LIBRARIAN-TEMPLATE: project-description -->

**Когда нужен:** при формулировке каждого проекта для портфолио.

**Шаблон:**

```
[Название кейса — короткое, цепляющее, без технических деталей]

Клиент: {ниша + опционально имя если разрешил}

🔴 Боль:
{1-2 предложения. Что было плохо. Конкретно.}

🛠 Что сделала:
{1-2 предложения. Без технического жаргона. Через результат.}

🟢 Результат:
{1 предложение. Цифра если есть.}

[Опционально: 1-2 скриншота / ссылка на демо]
```

## Связь с нашими стандартами

В нашей системе у Ирины уже есть:
- **Часть 5 «Шаблон кейса»** в `clients/irina/product-line.md` — таблица из 6 блоков (Клиент / Боль / Задача / Услышала / Создала / Передала / Результат)

**Сравнение:**

| Элемент | Наш шаблон | Шаблон обучения | Решение |
|---|---|---|---|
| Кто клиент | ✅ есть | частично («ниша + опционально имя») | Берём наш — глубже |
| Боль | ✅ есть | ✅ есть | Совпадает |
| Что сделала | ✅ «Услышала + Создала + Передала» (3 блока) | ✅ «Решение» (1 блок) | Берём наш — лучше показывает процесс Ирины |
| Результат | ✅ есть | ✅ есть | Совпадает, добавим «цифра если есть» |
| Демо | ❌ нет | ✅ есть | **Добавляем в наш шаблон** ссылку на демо |

**Итоговое решение:** усилить наш шаблон — **добавить блок «Демо»** (ссылка на работающий пример или скриншоты в высоком разрешении). Остальное оставить как есть.
