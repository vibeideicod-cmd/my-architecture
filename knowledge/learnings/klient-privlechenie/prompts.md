# Готовые промпты: клиент-привлечение

> **Назначение:** промпты, скрипты диалогов и шаблоны сообщений из 4 спринтов по теме привлечения клиентов и продаж. Готовы к копированию и применению.
> **Принцип:** скрипты — рабочий инструмент, не «правда в последней инстанции». Адаптируйте под свою нишу, ToV и контекст клиента.

---

## Как пользоваться

1. Найди скрипт под свою задачу
2. Скопируй блок целиком
3. Замени переменные `{в_фигурных_скобках}` под клиента / нишу
4. Если это промпт для AI — отправь в Claude / ChatGPT
5. Если это скрипт диалога — адаптируй под свой ToV и используй в переписке/созвоне

## БЛОК 1 — Создание офферов (Дмитрий Ледовских, спринт «Оффер»)

### 1.1. Промпт «Найти идею для продукта»
<!-- LIBRARIAN-PROMPT: find-product-idea -->

**Когда использовать:** новичок не знает что создавать, или есть продукт, но нужна идея следующего проекта.

**Промпт:**

```
Ты помогаешь новичкам найти идею для первого продукта на вайбкодинге.
Человек будет создавать приложение через Claude Code — без программирования,
только промптами. Сначала задай мне ровно 3 вопроса — по одному, жди ответа на каждый:

Вопрос 1: «Чем ты занимаешься в жизни или работе? (можно коротко, пару слов —
например: продаю металл, веду соцсети, учу английский)»

Вопрос 2: «Что тебя раздражает или отнимает время каждую неделю? (любая мелкая
рутина — не обязательно по работе)»

Вопрос 3: «Ты хочешь сделать что-то для себя, или чтобы потом продавать /
показывать другим?»

После моих ответов — предложи ТОП-3 идеи продукта. Формат каждой идеи:

• Название (короткое, понятное)
• Что делает: одно предложение
• Кому полезно: одна строка
• Почему можно сделать без кода: одна строка
• Примерное время создания через AI: X минут / часов

В конце спроси: «Какая идея тебе ближе? Напиши цифру 1, 2 или 3 — и я дам
тебе готовый промпт для создания этого продукта».
```

---

### 1.2. Промпт «Реализовать идею» (превратить идею в готовый промпт для разработки)
<!-- LIBRARIAN-PROMPT: idea-to-build-prompt -->

**Когда использовать:** есть идея, нужно превратить в готовый промпт для AI-Studio / Claude / Lovable.

**Промпт:**

```
Я выбрал идею для своего первого продукта на вайбкодинге.
Идея: {ВСТАВЬ ВЫБРАННУЮ ИДЕЮ}.

Твоя задача — создать готовый промпт, с которым я пойду в Google AI Studio или Claude,
чтобы создать рабочее приложение. Промпт для создания должен быть написан по этой структуре:

1. ЧТО СОЗДАТЬ — точное название и тип (веб-приложение, бот, инструмент)
2. ЧТО ДЕЛАЕТ — функции по шагам, как пользователь взаимодействует
3. ИНТЕРФЕЙС — как выглядит, какие кнопки/блоки нужны
4. СТИЛЬ — простой, современный, без лишнего
5. ЯЗЫК — русский (если не указано иное)

Правила для промпта:
• Написан от первого лица: «Создай приложение которое...»
• Конкретно и подробно — никаких размытых формулировок
• Без технических терминов (не писать API, backend, deploy)
• Длина: 150-250 слов

После промпта добавь блок: КАК ИСПОЛЬЗОВАТЬ:
• Скопируй промпт выше
• Отправь в Google AI Studio (aistudio.google.com), Claude или Lovable
• Вставь промпт и нажми отправить
• Если что-то не так — напиши AI: «Измени [что именно]»
• Когда всё готово — напиши: «Дай мне код для публикации»
```

### 1.3. Шаблон тёплого оффера (для знакомых)
<!-- LIBRARIAN-PROMPT: warm-offer-template -->

**Когда использовать:** аудитория уже знает вас (друзья, коллеги, бывшие клиенты).

**Шаблон:**

```
{Имя}, привет.
Сейчас внедряю {ЧТО — конкретный результат за конкретное время / цифру}.
Беру одного человека под внедрение под ключ — {честная причина}.
Поэтому цена {условие}.
Кому из твоего окружения это могло бы быть актуально?
```

**Пример:**

```
Женя, привет.
Сейчас внедряю AI-агента, который делает это вместо тебя — закидываешь любые источники,
каждое утро получаешь готовый дайджест за 2 минуты.
Беру одного человека под внедрение под ключ — хочу показать на живом примере, что это реально работает.
Поэтому цена минимальная.
Кому из твоего окружения это могло бы быть актуально?
```

---

### 1.4. Шаблон холодного оффера (для незнакомых)
<!-- LIBRARIAN-PROMPT: cold-offer-template -->

**Когда использовать:** человек вас не знает. Акцент на боль и решение.

**Шаблон:**

```
{Имя}, {вопрос про их ежедневную боль/рутину}?
Я внедряю {ЧТО} — {описание как это устраняет боль}.
Каждый {день/неделю} получаешь {результат} за {время}.
Беру одного человека под внедрение под ключ — {честная причина}.
Поэтому цена {условие}.
Кому из твоего окружения это могло бы зайти — можешь посоветовать?
```

**Пример:**

```
Женя, сколько времени ты тратишь на просмотр каналов каждый день?
Я внедряю агента, который делает это вместо тебя — закидываешь любые источники,
каждое утро получаешь готовый дайджест за 2 минуты.
Беру одного человека под внедрение под ключ — хочу показать на живом примере, что это реально работает.
Поэтому цена минимальная.
Кому из твоего окружения это могло бы зайти — можешь посоветовать?
```

---

### 1.5. Шаблон оффера для соцсетей (ультракороткий)
<!-- LIBRARIAN-PROMPT: social-offer-template -->

**Правила:** максимум 6-8 строк, разговорный тон, конкретика, дефицит в первые 2 строки.

**Шаблон:**

```
{Имя}, привет.
Ты {боль через действие — например «каждый день читаешь каналы руками»} —
я могу сделать так, чтобы {решение от первого лица}.
Беру одного человека под внедрение под ключ — {честная причина}.
Поэтому цена минимальная.
Кому из знакомых это могло бы зайти — можешь посоветовать?
```

## БЛОК 2 — Каналы трафика (Дмитрий Ледовских, спринт «Каналы»)

### 2.1. Скрипт первого контакта по личному кругу
<!-- LIBRARIAN-PROMPT: warm-circle-first-contact -->

**Когда использовать:** старт работы, нет портфеля. Цель — узнать есть ли задача, не продать.

**Скрипт:**

```
{Имя}, привет!
Я тут прошёл обучение и теперь занимаюсь разработкой приложений
и автоматизацией для бизнеса — с помощью ИИ.

Хотел посоветоваться: ты в своей работе сталкиваешься с задачами,
которые делаешь вручную каждый день? Или может, знаешь кого-то,
кому это было бы интересно?
```

**Принцип:** не «куплю» / не «я продаю» — только вопрос и приглашение к диалогу. Человек либо сам скажет «о, мне как раз нужно!», либо вспомнит знакомых.

---

### 2.2. Скрипт массового сообщения (когда уже есть портфель)
<!-- LIBRARIAN-PROMPT: bulk-mass-message -->

**Когда использовать:** есть первые проекты — можно уже прямо.

**Скрипт:**

```
Привет всем! Теперь я вайб-кодер, принимаю заказы.
Вас интересуют такие технологии: приложения, веб-сайты,
каналы в соцсетях, чат-боты, видеоредактирование?

Пишите — помогу!
```

---

### 2.3. Главное правило (напоминание себе перед каждым сообщением)
<!-- LIBRARIAN-PROMPT: first-contact-rule-reminder -->

```
Цель первого контакта — НЕ продать.
Цель — узнать, есть ли задача.

Если задача есть — предложить созвон.
Сделки в переписке не делаются.
```

---

### 2.4. Промпт «Создать 5 сторис вайб-кодера на неделю»
<!-- LIBRARIAN-PROMPT: 5-stories-vibecoder-prompt -->

**Когда использовать:** нужно наполнить Telegram-канал / Stories системным контентом.

**Промпт:**

```
Создай 5 сторис для моего Telegram-канала на эту неделю по моей нише {НИША}.
Структура — 5 типов сторис:

1. Напоминание — рассказываю чем сейчас занимаюсь
2. Инсайт — зачем эта тема сейчас нужна
3. Кейс — показываю конкретный результат
4. Актуализация — что будет если это не внедрить;
   что классного можно сделать; где применить
5. Предложение — для первых 5 особые условия

Для каждой сторис:
- Заголовок (один экран Stories)
- Первые 1-2 строки (крючок)
- Основная мысль (3-5 строк)
- CTA в конце

Тон: дружеский, без пафоса. Русский язык.
```

## БЛОК 3 — Минимальный старт (Екатерина Суханова)

### 3.1. Скрипт холодного сообщения (диагностика, НЕ продажа)
<!-- LIBRARIAN-PROMPT: cold-msg-diagnostic -->

**Когда использовать:** массовая рассылка по холодной базе. Цель — диагностика, не продажа.

**Скрипт:**

```
Привет, {Имя}! Я заметил, что вы работаете в {область}.

У меня вопрос: какая сейчас самая большая боль в {конкретная область проблемы}?

Я собираю инсайты и могу поделиться тем, что видим мы.
Минут 15 на звонок?
```

---

### 3.2. Формула коммуникации — 4 блока
<!-- LIBRARIAN-PROMPT: communication-4-blocks -->

**Структура любого сообщения / поста / письма:**

```
1. Контекст: «Я знаю, что вы {описание ситуации}»

2. Проблема: «И часто это приводит к {боль, которая резонирует}»

3. Решение: «Мы нашли способ {краткое описание решения}»

4. Выгода и CTA: «Это помогает {результат}.
   Хотите 15 минут разобраться вместе?»
```

---

### 3.3. Шаблон письма первому клиенту
<!-- LIBRARIAN-PROMPT: first-client-email -->

**Когда использовать:** холодное письмо потенциальному клиенту с конкретным расчётом.

**Шаблон:**

```
Тема: «Потенциальная экономия {сумма} в месяц для {компания}»

Привет {Имя},

Я наблюдал, как компании в вашей нише тратят примерно {X} в месяц на {процесс}.
Мы разработали способ сократить это вдвое, сохраняя качество.

Конкретный пример: компания {похожая компания} экономила {результат}
в течение {период}.

Нет смысла в долгом письме — лучше поговорить.
Есть ли у вас 15 минут на звонок в четверг после 17:00?
```

---

### 3.4. Скрипты преодоления возражений (мягкий подход)
<!-- LIBRARIAN-PROMPT: objection-handling -->

**Возражение «Это дорого»:**

```
Понимаю. Большинство изначально думали так же.
Но когда они считают экономию от {результат},
обычно окупается за {период}.
Могу показать расчёт?
```

**Возражение «Нужно подумать»:**

```
Отлично. О чём именно вы хотите подумать?
[Слушаем]
Может быть, я смогу помочь разобраться прямо сейчас?
```

**Возражение «У меня нет времени»:**

```
Я ценю вашу занятость.
Именно поэтому это экономит время.
Может быть, в четверг вечером 15 минут?
```

## БЛОК 4 — Воронка диалога 7 этапов (Ярослав Ефремов)

### 4.1. Этап 1 — Контакт (установление раппорта)
<!-- LIBRARIAN-PROMPT: stage-1-contact -->

**Задача:** расположить человека и снять напряжение.
**Важно:** вопрос простой, без давления, без продажи.

**Примеры вопросов:**

```
Как у вас сейчас дела?
Как у вас сейчас обстановка в работе/проекте?
Удобно сейчас говорить?
Как вообще у вас сейчас идёт {контекст: продажи / клиенты / проекты}?
```

**При переписке:**

```
Как у тебя сейчас дела с {тема}?
```

---

### 4.2. Этап 2 — Ситуация (контекст)
<!-- LIBRARIAN-PROMPT: stage-2-situation -->

**Задача:** собрать общую картину без давления.

**Примеры вопросов:**

```
Расскажи, как у вас сейчас выстроен процесс {продаж}?
Сколько сейчас у вас клиентов / заявок?
Есть ли уже какие-то инструменты / подрядчики?
Расскажи, как сейчас у тебя обстоят дела с {фото / соцсети / продвижением}?
Делал(а) ли ранее что-то подобное?
```

---

### 4.3. Этап 3 — Проблема (боль)
<!-- LIBRARIAN-PROMPT: stage-3-problem -->

**Задача:** найти, что не работает. Здесь появляется причина покупки.

**Примеры вопросов:**

```
Что сейчас не устраивает?
Где чувствуешь, что теряешь?
Что не получается так, как хотелось бы?
С какими сложностями сталкиваешься?
Почему хочешь это изменить?
```

---

### 4.4. Этап 4 — Усиление (важность проблемы)
<!-- LIBRARIAN-PROMPT: stage-4-amplification -->

**Задача:** показать важность проблемы, эмоциональное включение.

**Примеры вопросов:**

```
Сколько ты на этом теряешь?
Как это влияет на результат?
Если ничего не менять, что будет дальше?
Сколько времени уходит впустую?
Как ты себя чувствуешь из-за этого?
Насколько это для тебя важно сейчас?
Что будет, если оставить как есть?
Сколько примерно теряете в деньгах из-за этого?
```

---

### 4.5. Этап 7 — Договорённость (вопросы-переводы)
<!-- LIBRARIAN-PROMPT: stage-7-transition -->

**Задача:** связать проблему и решение через подтверждение.

**Примеры вопросов:**

```
Правильно понимаю, что тебе важно {X}?

Если это решить таким образом — это закроет задачу?
```

---

### 4.6. Лайфхак на короткий ответ собеседника
<!-- LIBRARIAN-PROMPT: hack-6-second-pause -->

```
{ЕСЛИ ОТВЕТ КОРОТКИЙ ИЛИ ОДНОСЛОЖНЫЙ}

→ ПОДОЖДИТЕ 6 СЕКУНД
→ В переговорах и продажах выигрывает тот, кто умеет выдерживать паузу
→ Часто человек сам продолжит и расскажет больше
```

---

### 4.7. Лайфхак — переводим разговор на клиента
<!-- LIBRARIAN-PROMPT: hack-talk-about-client -->

**Когда вас просят рассказать о себе:**

```
Давайте больше поговорим о вас и вашей проблеме,
чтобы я понял(а), как тебе помочь.
Какие есть проблемы?
```
