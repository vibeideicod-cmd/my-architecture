# Спецификация агента-библиотекаря (черновик, растёт по ходу)

> **Статус:** агент **не создан**. Этот документ — рабочая спека под будущее создание.
> **Триггер создания:** ≥5 тем в `learnings/` ИЛИ Инна 2+ раза спросила «где у меня лежит про X».
> **Создан:** 2026-05-02

---

## Назначение

Агент знает что лежит в `knowledge/learnings/`, по запросу находит релевантные принципы и выдаёт выжимку под конкретную задачу. Не «прочитать всю папку», а «вот тебе 3 ключевых принципа из 2 разных обучений, которые применимы к твоей задаче».

---

## Что агент должен уметь

### 1. Навигация по базе

- Знать список всех тем (читает таблицу из `learnings/README.md`)
- Знать ключевые принципы каждой темы (по комментариям `<!-- LIBRARIAN-PRINCIPLE: ... -->` в методичках)
- Знать теги и связи между темами (из frontmatter методичек)

### 2. Поиск под задачу

**Вход:** «нужна помощь с задачей X»
**Выход:** релевантные принципы из `learnings/` с указанием источника

Пример:
- **Вход:** «Делаю портфолио для Ирины — какие принципы есть в базе?»
- **Выход:** «В `learnings/portfolio-packaging/methodology.md` есть принцип «3-2-1 подачи кейса» (показывать каждый кейс в 3 масштабах). В `learnings/vibe-coding/methodology.md` есть принцип «технологическая глубина в продакшене» (если технические работы — показывать процесс, не только результат). Полные тексты — по ссылкам.»

### 3. Логирование использования

Когда агент-команды берёт принцип из `learnings/` — библиотекарь добавляет запись в frontmatter методички:

```yaml
использовано-в:
  - clients/irina/portfolio.md (2026-05-02)
  - clients/X/Y.md (2026-XX-XX)
```

Это даёт картину «какие принципы реально работают в практике, а какие лежат мёртвым грузом».

### 4. Сравнение «база vs команда» по запросу

**Вход:** «как этот принцип отличается от методологии нашего Branding-агента?»
**Выход:** короткое сопоставление + рекомендация (брать / игнорировать / синтезировать).

Это **не автоматический процесс** — только по явному запросу.

---

## Триггеры активации

Агент вызывается, когда:

- В запросе явно: «возьми что-то из learnings», «есть ли в базе про X», «что у нас в обучениях по теме Y»
- Агент-команды собирается работать над задачей, для которой в `learnings/` есть прямо релевантная тема (например, портфолио → есть `portfolio-packaging/`). В таком случае Director может сам подсказать команде «загляни в библиотеку перед стартом».

**Не вызывается** когда:
- Задача стандартная и решается базовой методологией команды
- Инна явно сказала «не используй внешние материалы»

---

## Что агент НЕ должен делать

- ❌ **Не переписывает методологии команды.** Только подсказывает источники.
- ❌ **Не «обучает» агентов команды на лету.** Если из обучения что-то реально сильнее — это повод для апгрейда, который согласует Инна.
- ❌ **Не сохраняет ничего вне `learnings/`.** Это его зона.
- ❌ **Не читает оригиналы PDF в каждом запросе.** Только методички (.md). PDF — это архив.

---

## Источники, которые читает агент

В порядке приоритета:

1. `knowledge/learnings/README.md` — общий навигатор
2. `knowledge/learnings/<тема>/methodology.md` — выжимки (с frontmatter и LIBRARIAN-маркерами)
3. `knowledge/learnings/<тема>/prompts.md` — готовые промпты
4. `knowledge/learnings/<тема>/checklists.md` — чек-листы и фреймворки

PDF-исходники в `<тема>/source.pdf` агент **не читает** — это архив.

---

## Формат вывода агента

**По умолчанию (быстрый):**

```
Релевантные принципы из learnings/:

1. [Принцип] — источник: <тема>/methodology.md
   Краткая суть в 1-2 предложениях.

2. [Принцип] — источник: <тема>/methodology.md
   Краткая суть.

Промпты по теме (если есть): <тема>/prompts.md
Чек-листы по теме (если есть): <тема>/checklists.md
```

**Углублённый (по запросу):**

Полная выжимка из методички с цитатами оригинала (если важна формулировка).

---

## Что роадмап-агент развивает по ходу

Эта спека **черновик**. Она растёт по мере того, как:

- Появляются реальные кейсы использования базы (мы понимаем какие функции реально нужны)
- Инна формулирует новые ожидания («хочу чтобы агент делал ещё X»)
- Skill-Auditor находит расхождения между ожиданиями и реальностью

**Не делаем фичи «на всякий случай»** — только под живой запрос.
