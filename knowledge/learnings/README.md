# База знаний из обучений

> **Назначение:** хранилище методологий из обучений, в которых Инна участвует (vibe coding, кураторская система университета, упаковка портфолио и другие).
> **Принцип:** это **справочник, не источник истины**. Команда агентов работает по своей методологии (С.С.С.Р., архитектура Инны, брендбук Ирины) — материалы из этой папки используются **по необходимости** или **по явной команде Инны**, не автоматически.
> **Создан:** 2026-05-02

---

## 🔒 Правило независимости агентов команды

**Жёсткое правило:**

1. **Базовая логика агентов = их собственная методология.** Не из этой папки.
2. **Когда агент использует материал из `learnings/`** — он **явно указывает это** в результате работы. Например: «Решение опирается на принцип «3-2-1 подачи кейса» из `learnings/portfolio-packaging/methodology.md`».
3. **Решение об апгрейде агента на основе обучения принимает Инна**, не агент сам по себе. Если в обучении нашли принцип сильнее нашего — это повод обсудить апгрейд, а не молча перенять.
4. **Сравнения и точечные апгрейды** фиксируются в `agent-runtime/shared/` как заметки `<дата>-<тема>-comparison.md`. Skill-Auditor пробегается раз в месяц и рекомендует апгрейды.

**Что нарушает правило:**
- ❌ Агент молча начинает работать по чужой методологии без уведомления
- ❌ Агент-файл переписывается «под обучение» без обсуждения с Инной
- ❌ Принципы из обучений переезжают в `knowledge/methodology/` напрямую (там — наша С.С.С.Р., не чужие материалы)

## 📂 Структура папки

Каждое обучение = отдельная подпапка. В подпапке — три типа файлов:

```
learnings/
├── README.md                       ← этот файл — навигатор
├── _meta/                          ← мета-инфраструктура
│   ├── librarian-spec.md           ← спецификация будущего агента-библиотекаря
│   ├── template-methodology.md     ← шаблон выжимки методологии
│   ├── template-prompts.md         ← шаблон файла с промптами
│   └── template-checklists.md      ← шаблон файла с чек-листами
├── portfolio-packaging/            ← пример: упаковка портфолио
│   ├── source.pdf                  ← оригинал презентации (опционально)
│   ├── methodology.md              ← выжимка ключевых принципов
│   ├── prompts.md                  ← готовые промпты из обучения
│   └── checklists.md               ← фреймворки и чек-листы
├── vibe-coding/                    ← (когда подгрузим)
└── ...
```

**Принцип:** одна папка = одна тема. Не «всё в кучу».

## ➕ Как добавить новое обучение

1. **Положить исходник** в `inbox-inna/` (PDF, презентация, методичка)
2. **Сказать Кодычу:** «обработай обучение по [тема]»
3. **Кодыч создаёт папку** `learnings/<тема>/` по шаблонам из `_meta/`
4. **Заполняет три файла:**
   - `methodology.md` — выжимка принципов
   - `prompts.md` — готовые промпты (если есть)
   - `checklists.md` — чек-листы и фреймворки (если есть)
5. **Обновляет таблицу выше** в этом README

## 🤖 Roadmap агента-библиотекаря

**Сейчас:** ручная навигация через этот README + метаданные (frontmatter) в каждой методичке.

**Триггер для создания агента:**
- ≥5 тем в базе **ИЛИ**
- Инна 2+ раза подряд спрашивает «а где у меня лежит про X?»

**Что агент будет уметь:**
- Поиск по темам и тегам
- Возврат выжимки под конкретную задачу (не вся папка)
- Логирование «использовано в задаче такой-то»
- Сравнение «что в базе» vs «что у агента команды» по запросу

**Спецификация:** `_meta/librarian-spec.md` (растёт по мере использования базы)

**Что в файлах уже сделано под будущего агента:**
- В каждой методичке — заголовочный блок с метаданными (тема, теги, ключевые принципы, связанные темы)
- В ключевых местах — комментарии-маркеры `<!-- LIBRARIAN-PRINCIPLE: ... -->` для индексации
- Структура файлов одинаковая для всех тем (по шаблонам из `_meta/`)

Когда триггер сработает — агент создаётся за час над уже готовой структурой.

## 💡 Бонус: эта база как продукт

Потенциально `learnings/` — это **актив самой Инны**. Со временем (10+ тем) это может стать тарифом Платформы С.С.С.Р.: «Доступ к моей библиотеке методологий из X+ обучений». Это отдельный разговор — но структура папки делается с прицелом на это.
