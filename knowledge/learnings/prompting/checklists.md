# Чек-листы и фреймворки: промтинг

> **Назначение:** готовые рабочие чек-листы и фреймворки по теме промтинга. Применяются как есть или адаптируются под задачу.
> **Источники:** см. `sources/` в этой же папке.

---

## Как пользоваться

1. Выбери чек-лист или фреймворк под свою задачу
2. Скопируй структуру в свой документ
3. Заполни под клиента
4. После использования — отметь в frontmatter методички (`использовано-в:`)

---

## Чек-листы

### 1. Чек-лист «Готов ли мой промпт» — CRAFT-проверка
<!-- LIBRARIAN-CHECKLIST: craft-readiness -->

**Когда применять:** перед тем как отправить промпт в нейросеть.

- [ ] **C** — Контекст описан (ситуация, ЦА, особые условия)
- [ ] **R** — Роль AI задана (профессия + опыт + специализация)
- [ ] **A** — Действие конкретное (задача + требования + критерии успеха)
- [ ] **F** — Формат указан (тип контента + параметры + особенности)
- [ ] **T** — Тон сформулирован (стиль + эмоция + опционально метафора)

**Если хотя бы 1 пункт пуст** — промпт даст слабый результат, дополни.

---

### 2. Чек-лист «Качество ответа от AI» — оценка по 4 критериям
<!-- LIBRARIAN-CHECKLIST: result-evaluation -->

**Когда применять:** после получения первого ответа от AI.

| Критерий | Оценка 1-10 |
|---|---|
| Соответствие задаче | ☐ |
| Полнота ответа | ☐ |
| Точность формата | ☐ |
| Правильность тона | ☐ |

**Если хотя бы один критерий ниже 8** — запускаем итерации.

---

### 3. Чек-лист «3 ошибки промтинга — не сделал ли я их?»
<!-- LIBRARIAN-CHECKLIST: prompt-errors -->

- [ ] В моих формулировках **нет** многозначностей и двусмысленностей
- [ ] Я создал **новый чат** для этой задачи (не использую один диалог под разные задачи)
- [ ] Я **не беру первый результат** — обязательно делаю минимум 2 итерации

---

### 4. Чек-лист «Применил ли я фишки усиления»
<!-- LIBRARIAN-CHECKLIST: spices-applied -->

Опционально для важных задач:

- [ ] Угрозы («если текст слабый — тебя уволят»)
- [ ] Шкала качества («оцени по 1-10, переписывай пока не 9/10»)
- [ ] Денежная мотивация («бонус 1000 долларов за лучший результат»)
- [ ] Английский язык (для сложных аналитических задач)
- [ ] Поиск свежих данных в интернете

## Фреймворки

### Фреймворк CRAFT — пошаговая сборка промпта
<!-- LIBRARIAN-FRAMEWORK: craft-step-by-step -->

**Назначение:** собрать промпт с нуля по 5-элементной формуле.

```
Шаг 1. КОНТЕКСТ
   ┌─ Что происходит? (ситуация)
   ├─ Кто аудитория? (ЦА, боли, что важно)
   └─ Особые условия? (ограничения, требования, предыстория)

Шаг 2. РОЛЬ (метод супергероя)
   ┌─ Суперсила (в чём AI лучший)
   ├─ Происхождение (опыт, годы, проекты)
   └─ Миссия (какие проблемы решает)

Шаг 3. ДЕЙСТВИЕ
   ┌─ Основная задача (что конкретно сделать, в каком объёме)
   ├─ Дополнительные требования (обязательные элементы, ограничения)
   └─ Критерии успеха (как измерить результат)

Шаг 4. ФОРМАТ
   ┌─ Тип (пост / статья / письмо / презентация)
   ├─ Технические параметры (объём, структура, разделы)
   └─ Особые требования (стиль оформления, элементы)

Шаг 5. ТОН (метод актёра)
   ┌─ Характер (формальный/неформальный)
   ├─ Эмоции (что передать)
   └─ Метафора (опционально — как Х в стиле Y)
```

### Фреймворк «Цепочка из 3 шагов» (Chain prompting)
<!-- LIBRARIAN-FRAMEWORK: chain-three-steps -->

**Назначение:** разбить большую задачу на 3 промпта.

```
Шаг 1. ИЗВЛЕЧЬ
   «Назови 3 главные [проблемы / страхи / преимущества / возражения] клиентов
   при выборе [продукт]»

Шаг 2. РАСКРЫТЬ
   «Как наш [бизнес] решает каждый пункт из шага 1? Конкретно, с цифрами и фактами»

Шаг 3. СОБРАТЬ
   «Используя пункты из шагов 1-2, создай [пост / лендинг / письмо].
   Начни с главной проблемы. Используй конкретные цифры»
```

**Критерии успеха цепочки:**
- В каждом шаге есть конкретные цифры
- Нет общих фраз типа «высокое качество»
- Есть измеримый результат

### Фреймворк «6 контекстных якорей»
<!-- LIBRARIAN-FRAMEWORK: six-anchors -->

**Назначение:** привязать промпт к конкретному клиенту/ситуации.

```
🎯 КТО — Идеальный клиент (пол, возраст, роль)
⏰ КОГДА — Время (утро / день / вечер / выходные)
📍 ГДЕ — Локация (дом / офис / транспорт)
😣 ПРОБЛЕМА — Главная боль прямо сейчас
🌡️ СОСТОЯНИЕ — Эмоция в момент чтения
📐 ФОРМАТ — Где будет жить контент (Instagram / Telegram / VK / лендинг)
```

**Чем больше якорей — тем точнее попадание в цель.**

---

### Фреймворк «Итеративное улучшение» (3 итерации)
<!-- LIBRARIAN-FRAMEWORK: three-iterations -->

**Назначение:** довести первый результат до уровня «безупречно».

```
Итерация 1 — АНАЛИЗ
   «Проанализируй текст как маркетолог с 10-летним опытом.
   Что улучшить по структуре, конкретике, эмоциональному воздействию?»

Итерация 2 — УСИЛЕНИЕ
   «Переработай с учётом замечаний. Усиль доказательную базу,
   призывы к действию, работу с возражениями»

Итерация 3 — ФИНАЛ
   «Сравни две версии. Что стало лучше? Проверь логику,
   усиль ключевые сообщения, оптимизируй заключение»
```

---

### Фреймворк «Эксперт наоборот» (критика → улучшение)
<!-- LIBRARIAN-FRAMEWORK: expert-inverse-framework -->

**Назначение:** найти и закрыть слабые места.

```
Шаг 1. РАЗНОСИМ
   «Ты [роль] с 15-летним опытом. Найди ВСЕ слабые места,
   неточности, упущенные возможности. Будь максимально критичен»

Шаг 2. ПЕРЕДЕЛЫВАЕМ
   «Зная все эти проблемы, перепиши текст так, чтобы он был безупречным.
   Каждое замечание должно быть устранено»

Шаг 3. СВЕРЯЕМ (опционально)
   «Сверь новую версию с замечаниями из шага 1. Все ли устранены?»
```

---

### Фреймворк «Few-shot» (с примерами)
<!-- LIBRARIAN-FRAMEWORK: few-shot-framework -->

**Назначение:** дать AI пример успешного контента, а не «с нуля».

```
Шаг 1. ИЗВЛЕЧЬ ФОРМУЛУ
   «Вот пример успешного [тип контента] в моей нише: [текст].
   Что делает его эффективным? Выдели 5 ключевых элементов»

Шаг 2. ПРИМЕНИТЬ К СВОЕМУ
   «Используя эти 5 элементов, напиши [тип контента] для моего [продукт].
   Сохраняй стиль и структуру, но контент свой»
```

**Если есть 2+ примера** — даём оба, AI обобщит сильнее.

## Шаблоны

### Шаблон «Универсальный промпт для бизнеса» (CRAFT + якоря + few-shot)
<!-- LIBRARIAN-TEMPLATE: universal-business-prompt -->

**Когда нужен:** максимально мощный промпт для важной задачи.

```
=== РОЛЬ ===
Ты [профессия] с [опыт] лет, специализация [конкретная зона],
успешные кейсы [результаты, награды].

=== КОНТЕКСТ ===
Ситуация: [что происходит]
ЦА: [портрет с 6 якорями: кто/когда/где/проблема/состояние/формат]
Бизнес: [что делаем, чем отличаемся, кейсы]

=== ПРИМЕР УСПЕШНОГО РЕШЕНИЯ ===
[вставить 1-2 успешных примера в нише с пометкой результатов]

=== ЗАДАЧА ===
[конкретное действие]
Обязательно: [список требований]
Критерии успеха: [как измерим]

=== ФОРМАТ ===
[тип контента, объём, структура, особенности]

=== ТОН ===
[стиль, эмоция, опционально метафора]

=== ФИШКИ ===
- Оцени каждый блок по шкале 1-10
- Переписывай пока не будет минимум 9/10
- Если результат слабый — этот текст не пойдёт в работу
```

### Шаблон «Анализ данных»
<!-- LIBRARIAN-TEMPLATE: data-analysis -->

```
Анализ [что]:

ЧТО СМОТРИМ: [конкретные метрики, период]
ЧТО ИЩЕМ: [аномалии, тренды, возможности роста, болевые точки]
КАК ПОКАЗАТЬ: [таблица + 3 главных вывода]
ФОРМАТ: маркированный список с цифрами и процентами
```

---

### Шаблон «Креативная задача»
<!-- LIBRARIAN-TEMPLATE: creative-task -->

```
Создай [акция / идея / концепция]:

ЦЕЛЕВАЯ ЭМОЦИЯ: [радость / удивление / срочность]
МЕХАНИКА: [простая, как в [известном бренде]]
БЮДЖЕТ: до [сумма]
ДЛИТЕЛЬНОСТЬ: [период]
ОБЯЗАТЕЛЬНО: [ключевые элементы]
```

---

### Шаблон «Принятие решения»
<!-- LIBRARIAN-TEMPLATE: decision-making -->

```
Помоги принять решение: [вопрос]

КРИТЕРИИ УСПЕХА:
- [метрика 1]
- [метрика 2]
- [метрика 3]

ЧТО ПРОВЕРИТЬ:
- [фактор 1]
- [фактор 2]

ФОРМАТ ОТВЕТА:
✓ Плюсы (с цифрами)
✗ Минусы (с рисками)
! Риски (что может пойти не так)
→ Рекомендация (одна фраза)
```

## Связь с нашими стандартами

В нашей системе у Инны и Ирины **нет** аналога CRAFT-формулы в `knowledge/standards/`. **Точечный апгрейд:** добавить CRAFT в Director-агента или в новый файл `knowledge/standards/prompt-engineering.md` после согласования с Инной (не автоматически).

В `agents/copywriter.md` и `agents/content.md` — частично используют структурированные промпты, но без явной 5-элементной формы. Возможный апгрейд: явно опереться на CRAFT при написании в этих агентах.
