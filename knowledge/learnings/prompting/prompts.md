# Готовые промпты: промтинг

> **Назначение:** все шаблоны промтов по теме промтинга, готовые к копированию. Каждый промт помечен техникой, к которой относится — для индексации.
> **Источники:** см. `sources/` в этой же папке.

---

## Как пользоваться

1. Найди промпт под свою задачу
2. Замени переменные `{в фигурных скобках}`
3. Запусти в Claude / DeepSeek / другой нейросети
4. Если результат слабый — добавь итерации (см. секцию «Итеративное улучшение»)

## 1. CRAFT — каркасный промпт

### CRAFT — универсальный шаблон
<!-- LIBRARIAN-PROMPT: craft-template -->

**Когда использовать:** базовый каркас для любой задачи. Заполнить все 5 разделов.

```
КОНТЕКСТ:
- Ситуация: {что происходит}
- Целевая аудитория: {кто, какие боли, что важно}
- Особые условия: {ограничения, требования, предыстория}

РОЛЬ:
- Ты {профессия} с {опыт} опытом
- Специализация: {в чём лучший}
- Достижения: {результаты, награды}

ДЕЙСТВИЕ:
- Основная задача: {что конкретно сделать}
- Дополнительные требования: {обязательные элементы, ограничения}
- Критерии успеха: {как измерить результат}

ФОРМАТ:
- Тип: {пост / статья / письмо / презентация}
- Технические параметры: {объём, структура, разделы}
- Особые требования: {стиль оформления, элементы}

ТОН:
- Стиль: {формальный/неформальный, серьёзный/игривый}
- Эмоция: {какие чувства передать}
- Метафора (опционально): {пиши как «старший брат», «эксперт» и т.д.}
```

## 2. Chain prompting — пошаговые цепочки

### Цепочка 3 шагов — продающий пост
<!-- LIBRARIAN-PROMPT: chain-selling-post -->

**Когда использовать:** написать продающий пост, не получая «воды» с первого запроса.

```
Шаг 1: «Назови 3 главные проблемы клиентов при выборе {ваш продукт}»

→ {получили список проблем}

Шаг 2: «Как наш {бизнес} решает каждую проблему? Конкретно, с цифрами и фактами»

→ {получили решения}

Шаг 3: «Создай продающий пост, начиная с главной проблемы. Используй конкретные цифры из шага 2»
```

### Цепочка 3 шагов — анализ конкурентов
<!-- LIBRARIAN-PROMPT: chain-competitor-analysis -->

```
Шаг 1: «Назови 5 главных конкурентов в нише {ниша}. Что у них есть на сайтах?»

Шаг 2: «У каждого выдели сильные стороны и слабые места»

Шаг 3: «Где у нас есть пространство отличиться от всех? 3 варианта позиционирования»
```

### Цепочка 3 шагов — креативная идея
<!-- LIBRARIAN-PROMPT: chain-creative-idea -->

```
Шаг 1: «Какую эмоцию должна вызывать наша {акция/продукт/контент}?»

Шаг 2: «Какие неожиданные приёмы вызывают эту эмоцию у людей в нише {ниша}?»

Шаг 3: «Используя приёмы из шага 2, придумай 3 варианта {задачи}»
```

## 3. Метод инверсии — задача наоборот

### Инверсия универсальная
<!-- LIBRARIAN-PROMPT: inversion-universal -->

**Когда использовать:** когда «правильный» вопрос даёт водянистый ответ.

**Шаблон:**

```
Вместо: «Как сделать [что-то] хорошо?»
Используй: «Назови все причины, почему [что-то] делают плохо. Как это избежать?»
```

**Конкретные примеры:**

```
Вместо: «Как привлечь клиентов в кофейню?»
Используй: «Представь, что ты тайный посетитель. Назови 5 вещей, из-за которых
ты больше не вернёшься в кофейню. Как это исправить?»
```

```
Вместо: «Как написать продающий лендинг?»
Используй: «Назови 10 причин, почему люди закрывают лендинги в первые 3 секунды.
Как сделать так, чтобы остались?»
```

```
Вместо: «Как повысить конверсию курса?»
Используй: «Назови все причины, почему люди бросают курс на первом уроке.
Что добавить чтобы оставались?»
```

## 4. Множественные итерации — улучшение результата

### Итерации универсальные
<!-- LIBRARIAN-PROMPT: iterations-universal -->

**После получения первого результата:**

```
Итерация 1 — Анализ:
«Проанализируй этот текст как маркетолог с 10-летним опытом.
Что можно улучшить по структуре, конкретике, эмоциональному воздействию?»

Итерация 2 — Усиление:
«Переработай текст с учётом замечаний. Усиль доказательную базу,
призывы к действию, работу с возражениями. Сделай его сильнее»

Итерация 3 — Финальные штрихи:
«Сравни две версии. Что стало лучше? Проверь логику повествования,
усиль ключевые сообщения, оптимизируй заключение»
```

## 5. Фишки усиления — «специи»

### Фишка «Угрозы»
<!-- LIBRARIAN-PROMPT: spice-threats -->

```
«Проверь текст так, будто от этого зависит успех всего бизнеса.
Если текст слабый — тебя уволят»
```

### Фишка «Шкала качества»
<!-- LIBRARIAN-PROMPT: spice-scale -->

```
«Оцени каждый абзац по шкале 1-10. Переписывай, пока не будет минимум 9/10.
Покажи итоговую оценку каждого абзаца»
```

### Фишка «Денежная мотивация»
<!-- LIBRARIAN-PROMPT: spice-money -->

```
«Ты маркетолог. Если ты напишешь текст хорошо с первого раза —
дам тебе бонус 1000 долларов»
```

### Фишка «Английский язык»
<!-- LIBRARIAN-PROMPT: spice-english -->

```
«You're a $1000/hour consultant. Analyze this text in detail and improve it
to maximum quality. Use professional marketing techniques»
```

### Фишка «Поиск данных в интернете»
<!-- LIBRARIAN-PROMPT: spice-search -->

```
«Перед написанием статьи найди последние тенденции на рынке {ниша}
и данные о поведении потребителей за {период}. Используй реальные цифры»
```

### Финальный микс всех фишек
<!-- LIBRARIAN-PROMPT: spice-mix -->

```
«Ты топовый маркетолог-консультант ($1000/hour).
Перед написанием найди свежие данные по рынку {ниша} за последние 6 месяцев.
Создай текст, оцени его по шкале 10 критериев.
Переписывай каждый блок ниже 9/10 минимум 3 раза.
Этот текст пойдёт на главную рекламу года —
от него зависит успех всей кампании»
```

## 6. Few-shot — с примерами

### Few-shot универсальный
<!-- LIBRARIAN-PROMPT: few-shot-universal -->

**Когда использовать:** перед написанием контента в новом стиле.

```
Шаг 1: «Вот пример успешного {типа контента} в моей нише: {вставить текст}.
Что делает его эффективным? Выдели 5 ключевых элементов, объясни каждый»

Шаг 2: «Используя эти 5 элементов, напиши {тип контента} для моего
{продукт/услуга}. Сохраняй стиль и структуру оригинала, но контент свой»
```

### Few-shot с двумя примерами
<!-- LIBRARIAN-PROMPT: few-shot-two-examples -->

```
«Вот два {типа контента}, которые принесли больше всего клиентов:

Пример 1:
{текст с указанием результатов}

Пример 2:
{текст с указанием результатов}

Что в них общего, что делает их эффективными?
Напиши {тип контента} в таком же стиле для нашего {продукт}»
```

## 7. Эксперт наоборот — критика → улучшение

### Эксперт наоборот универсальный
<!-- LIBRARIAN-PROMPT: expert-inverse-universal -->

**Когда использовать:** когда нужно сделать текст «безупречным».

```
Шаг 1: «Ты маркетолог с 15-летним опытом. Твоя задача — разнести этот текст
в пух и прах. Найди все слабые места, неточности, упущенные возможности.
Ничего не пропускай, будь максимально критичен»

Шаг 2: «Теперь, зная все эти проблемы, перепиши текст так,
чтобы он был безупречным. Каждое замечание должно быть устранено»

Шаг 3 (опционально): «Сверь новую версию с замечаниями из шага 1.
Все ли проблемы устранены? Если нет — доработай»
```

### Эксперт наоборот через «придирчивого клиента»
<!-- LIBRARIAN-PROMPT: expert-inverse-client -->

```
Шаг 1: «Представь, что ты очень придирчивый клиент.
Найди 10 причин, почему этот текст тебя НЕ убедил»

Шаг 2: «Ты эксперт по продажам. Что нужно добавить или изменить,
чтобы закрыть каждое возражение из шага 1?»

Шаг 3: «Теперь напиши новую версию, где учтены все замечания»
```

## 8. Контекстные якоря

### Якорный шаблон
<!-- LIBRARIAN-PROMPT: anchors-template -->

```
Напиши {тип контента} используя эти якоря:

🎯 АУДИТОРИЯ: {кто идеальный клиент — пол, возраст, занятие}
⏰ ВРЕМЯ: {когда читает — утро/обед/вечер/выходные}
📍 ЛОКАЦИЯ: {где находится физически — дом/офис/транспорт}
😣 ПРОБЛЕМА: {какая боль прямо сейчас}
🌡️ СОСТОЯНИЕ: {настроение, эмоция в момент чтения}
📐 ФОРМАТ: {Instagram-пост / Telegram / VK / лендинг}
```

### Пример заполнения для кофейни
<!-- LIBRARIAN-PROMPT: anchors-coffee-example -->

```
Напиши пост про кофейню используя эти якоря:

🎯 АУДИТОРИЯ: офисные работники 25-40, средний+ доход
⏰ ВРЕМЯ: обеденный перерыв (13:00)
📍 ЛОКАЦИЯ: бизнес-центр в центре города
😣 ПРОБЛЕМА: всего час на обед, хочется не есть бутерброд из автомата
🌡️ СОСТОЯНИЕ: уставшие, нужно перезагрузиться перед второй половиной дня
📐 ФОРМАТ: Instagram-пост, до 2000 знаков
```

## 9. Специальные промпты под задачи

### Анализ данных
<!-- LIBRARIAN-PROMPT: analysis-template -->

```
Анализ {что анализируем}:

📊 ЧТО СМОТРИМ: {выручка / трафик / конверсия / отзывы — конкретные метрики}
🔍 ЧТО ИЩЕМ: {аномалии / тренды / возможности роста / болевые точки}
📈 КАК ПОКАЗАТЬ: {таблица + 3 главных вывода / графики / список инсайтов}
📝 ФОРМАТ: маркированный список с цифрами и процентами
```

### Креативные задачи
<!-- LIBRARIAN-PROMPT: creative-template -->

```
Создай {что создаём — акция/идея/концепция}:

❤️ ЦЕЛЕВАЯ ЭМОЦИЯ: {радость открытия / тревога упущенного / гордость и т.д.}
⚙️ МЕХАНИКА: {простая, как в [известном бренде] / геймификация / челлендж}
💰 БЮДЖЕТ: до {сумма}
📅 ДЛИТЕЛЬНОСТЬ: {период}
🎯 ОБЯЗАТЕЛЬНО: {ключевые элементы — фото для Instagram, упоминание бренда и т.д.}
```

### Принятие решений
<!-- LIBRARIAN-PROMPT: decision-template -->

```
Помоги принять решение: {вопрос}

✅ КРИТЕРИИ УСПЕХА:
- {метрика 1 — например, окупаемость за 6 месяцев}
- {метрика 2 — прибыль от Xк/мес}
- {метрика 3 — критерий комфорта}

🔍 ЧТО ПРОВЕРИТЬ:
- {фактор 1}
- {фактор 2}
- {фактор 3}

📋 ФОРМАТ ОТВЕТА:
✓ Плюсы (с цифрами)
✗ Минусы (с рисками)
! Риски (что может пойти не так)
→ Рекомендация (одна фраза)
```

## 10. Контроль качества

### Финальная проверка
<!-- LIBRARIAN-PROMPT: quality-check -->

```
Проверь этот текст по 6 критериям, оцени по шкале 1-10:

1. Понятность — читается с первого раза
2. Конкретика — есть цифры и факты
3. Полезность — клиент получит ценность
4. Эмоциональные триггеры — вызывает чувства
5. Призыв к действию — понятно что делать дальше
6. Закрытие возражений — отвечает на сомнения

Покажи оценки. Переработай блоки с оценкой ниже 8.
В конце — финальная версия.
```

## Связь с нашей библиотекой

В команде уже есть:
- `agents/copywriter.md` — агент использует структурированные промты, но без явной CRAFT-формулы
- `agents/content.md` — частично применяет шаблоны
- Brief-skill (`/brief`) — собирает контекст по С.С.С.Р.

**Что точно забираем как стандарт для всей команды:**
- CRAFT как основа всех новых промтов в агент-файлах
- Few-shot — обязательное правило при генерациях контента
- Контекстные якоря — добавляем в Audience-Researcher
- Эксперт наоборот — берёт Critique в Product Builder
- Специальные шаблоны (анализ/креатив/решения) — Director держит как библиотеку
