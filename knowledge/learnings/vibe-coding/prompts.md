# Готовые промпты: вайбкодинг

> **Назначение:** промпты, которые автор курса предлагает использовать на 5-шаговом алгоритме (Идея → Исследование → Критика → План → Реализация), при настройке рабочего места и при сборке Паспорта проекта. Готовы к копированию.
> **Принцип:** промпты — рабочий инструмент, не «правда в последней инстанции». Адаптируйте под клиента или замените на свои.

---

## Как пользоваться

1. Найди промпт под свою задачу в списке ниже
2. Скопируй блок целиком
3. Замени переменные `{в_фигурных_скобках}` на конкретные значения под клиента/нишу
4. Используй в нужной нейросети — Claude Code (для кода), Google AI Studio / ChatGPT (для текстов и плана), Lovable (для красивых сайтов)

## БЛОК 1 — Идея и выбор продукта

### 1.1. Помощь с идеей цифрового продукта (Урок 2)
<!-- LIBRARIAN-PROMPT: idea-help -->

**Когда использовать:** в самом начале, когда есть ниша, но нет конкретной идеи продукта.

**Целевая нейросеть:** Claude / Google AI Studio / ChatGPT.

**Промпт:**

```
Я хочу выбрать идею для своего цифрового продукта.
Моя сфера: {впишите свою нишу или интерес}

Предложи 5 идей продуктов (лендинг / квиз / калькулятор / бот / приложение),
которые я могу создать без кода с помощью ИИ.

Для каждой идеи напиши:
- Что создаём (тип продукта)
- Для кого (аудитория)
- Какую проблему решает

Выбери из 5 идей одну лучшую и объясни почему.
```

---

### 1.2. Генерация 10 идей продуктов под нишу (Урок 1)
<!-- LIBRARIAN-PROMPT: 10-product-ideas -->

**Когда использовать:** для подбора витрины продуктов под клиента в конкретной нише.

**Промпт:**

```
Ты — опытный продакт-менеджер, который создаёт цифровые продукты без кода для разных ниш.

Моя ниша: {впишите вашу нишу или вид деятельности, чем вы занимаетесь сейчас}

Задача: напиши 10 идей цифровых продуктов, которые я могу создать с помощью ИИ для моей ниши,
и эти продукты будут полезны моим клиентам и помогут дать им решения.

Для каждого продукта укажи:
1. Название
2. Для кого (целевая аудитория)
3. Какую проблему решает
4. Формат (лендинг / квиз / калькулятор / бот / приложение)
5. Примерная стоимость на рынке

Формат: таблица.
```

---

### 1.3. Уточнение формулировки идеи
<!-- LIBRARIAN-PROMPT: refine-idea-formulation -->

**Когда использовать:** есть черновая формулировка, нужно её отполировать.

**Промпт:**

```
Я хочу создать [НАЗВАНИЕ] для [АУДИТОРИЯ].

Вот моя формулировка идеи:
Я создаю ЧТО для КОГО, чтобы решить проблему КАКУЮ

Уточни эту формулировку так, чтобы она была конкретнее.
```

## БЛОК 2 — Исследование рынка

### 2.1. Поиск 5-7 успешных аналогов (Урок 2)
<!-- LIBRARIAN-PROMPT: research-analogs -->

**Когда использовать:** Шаг 2 (Исследование). После того как сформулировали идею.

**Промпт:**

```
Я — аналитик цифровых продуктов. Найди 5-7 успешных IT-продуктов в нише: {ВАША НИША},
которые можно воссоздать через вайбкодинг (Cursor, Lovable, Claude Code, n8n и т.д.).

По каждому продукту дай:
- Название + что делает (1 фраза)
- Ключевой функционал и путь пользователя
- Модель монетизации
- Сложность: простое / среднее / сложное
- Как это можно сделать через Claude Code или Lovable и сколько времени на MVP

В конце: ТОП-3 продукта для быстрого старта с пошаговым планом MVP на 1-3 дня.
Важно: реальные примеры, фокус на быстрый запуск без навыков программирования.
```

---

### 2.2. Базовый промпт исследования примеров (Урок 2)
<!-- LIBRARIAN-PROMPT: simple-research -->

**Промпт:**

```
Покажи мне 5-7 примеров успешных проектов для {ваша ниша}.
Для каждого опиши:
- Какие секции есть
- Что делает его эффективным
- Какие элементы можно взять для моего проекта
```

---

### 2.3. Глубокое исследование по 5 направлениям (Урок 5)
<!-- LIBRARIAN-PROMPT: deep-research-5-directions -->

**Когда использовать:** перед созданием паспорта проекта. Запрашиваем у Claude разносторонний обзор за один проход.

**Промпт:**

```
Проведи исследование для моего проекта.
Исследуй 5 направлений:

1. КОНКУРЕНТЫ — найди 5 примеров похожих продуктов.
   Для каждого: название, что умеет, плюсы, минусы.

2. ФУНКЦИИ — какие функции обычно есть в подобных продуктах?
   Список из 15-20. Отметь, какие для старта, какие потом.

3. UX И СЦЕНАРИИ — как пользователь обычно взаимодействует?
   Опиши 3 типичных сценария. Что удобно, что бесит.

4. ТЕХНОЛОГИИ — какие технологии обычно используют?
   Сравни 2-3 варианта: плюсы, минусы, цена.

5. МОНЕТИЗАЦИЯ — как подобные проекты зарабатывают?
   Подписка, комиссия, реклама, партнёрства.

Создай файл с исследованием в проекте.
```

## БЛОК 3 — Критика идеи

### 3.1. Базовый промпт критики (Урок 2)
<!-- LIBRARIAN-PROMPT: basic-critique -->

**Когда использовать:** Шаг 3 (Критика). После исследования, до плана.

**Промпт:**

```
Я хочу создать {ваша идея из Шага 1}.

Проанализируй мою идею и ответь:
- Что может не сработать?
- Чего не хватает для успеха?
- Какие секции/элементы обязательно должны быть?
- Какие типичные ошибки делают в таких проектах?
- Как улучшить мою идею?
```

---

### 3.2. Полный промпт «Исследование + Критика + Улучшение» (домашнее задание Урока 2)
<!-- LIBRARIAN-PROMPT: full-research-critique-improve -->

**Когда использовать:** один промпт, который проводит идею через 3 этапа сразу.

**Промпт:**

```
Ты — опытный продуктовый менеджер.
Вот моя идея проекта: {вставьте свою формулировку идеи}

Выполни 3 задачи:

1. ИССЛЕДОВАНИЕ:
   Найди 5 похожих продуктов, которые уже существуют.
   Для каждого напиши: что хорошо, что плохо, что можно взять.

2. КРИТИКА:
   Раскритикуй мою идею честно:
   - 3 слабых места
   - Что я забыл учесть
   - Кому это точно НЕ нужно

3. УЛУЧШЕНИЕ:
   Предложи улучшенную версию моей идеи.
   Сформулируй в одном предложении:
   Я создаю [ЧТО] для [КОГО] чтобы решить проблему [КАКУЮ]
```

---

### 3.3. «Раскритикуй» — глазами пользователя (Урок 5)
<!-- LIBRARIAN-PROMPT: critique-as-user -->

**Когда использовать:** результат готов, нужно проверить с точки зрения пользователя.

**Промпт:**

```
Представь, что ты обычный пользователь, который впервые открыл этот {продукт/трекер/калькулятор/сайт}.
Что тебе непонятно? Что неудобно? Чего не хватает?
Дай 5 конкретных рекомендаций по улучшению.
```

---

### 3.4. «Раскритикуй» — варианты под разные роли (Урок 5)
<!-- LIBRARIAN-PROMPT: critique-as-roles -->

```
Посмотри глазами пользователя — что неудобно?
```

```
Посмотри как опытный UI/UX-дизайнер — что выглядит непрофессионально?
```

```
Какие минусы у этого решения? Что может пойти не так?
```

```
Посмотри как владелец конкурирующего бизнеса — что бы ты сделал лучше?
```

## БЛОК 4 — Документация проекта

### 4.1. Создание `project.md` (Паспорт проекта) (Урок 5)
<!-- LIBRARIAN-PROMPT: create-project-md -->

**Когда использовать:** есть идея + результаты исследования, нужно собрать паспорт.

**Промпт (этап 1 — рассказать идею):**

```
Я хочу сделать {описание идеи простыми словами как другу}.

Идея такая: {опишите подробно}.

Аудитория — {кто пользователи}.

Бюджет небольшой, поэтому начну с минимума: {функции MVP}.
{Дополнительные функции на потом}.

Помоги превратить это в документацию проекта.
Создай файл с описанием: что это, для кого, какую проблему решает,
основные функции, чего НЕ будет на старте.
```

**Промпт (этап 2 — собрать полный паспорт):**

```
На основе моей идеи и результатов исследования —
создай файл project.md, полный паспорт проекта.

Включи:
- Название проекта
- Что это (2-3 предложения)
- Для кого (целевая аудитория)
- Какую проблему решает
- Основные функции (MVP — что делаем первым)
- Функции на будущее (что добавим позже)
- Конкуренты (краткий обзор 3-5 штук)
- Технологии (что будем использовать)
- Чего НЕ будет на старте
- Первые шаги (с чего начать)

Посмотри на этот документ критически — что я мог упустить?
```

---

### 4.2. Создание `CLAUDE.md` (правила для ИИ) (Урок 5)
<!-- LIBRARIAN-PROMPT: create-claude-md -->

**Когда использовать:** один раз в начале проекта, чтобы Claude в каждом новом чате читал правила автоматически.

**Промпт:**

```
Создай файл CLAUDE.md в корне проекта.
Опиши правила работы с этим проектом:

- Все страницы делаем в одном HTML-файле, без внешних библиотек
- Дизайн: тёмный фон, светлый текст, акцентный цвет — оранжевый
- Адаптивная вёрстка — всё должно хорошо выглядеть на мобильных
- Стиль текста: простой, дружеский, без канцелярита
- Перед созданием нового файла — сначала расскажи, что планируешь сделать

{Добавьте свои правила: технологии, чего нельзя, как проверять}
```

**Проверка работы CLAUDE.md:** откройте новый чат и дайте короткий запрос без упоминания стиля:

```
Создай страницу pricing.html — страницу с тарифами для моего проекта. Три тарифа.
```

Если Claude сделал тёмный фон + оранжевый акцент без подсказки — `CLAUDE.md` работает.

---

### 4.3. Создание `PLAN.md` (план работ) (Урок 5)
<!-- LIBRARIAN-PROMPT: create-plan-md -->

**Промпт:**

```
Создай файл PLAN.md — план развития проекта.

Формат — чек-лист:
- [x] Визитка (готово)
- [x] Лендинг (готово)
- [ ] Страница тарифов
- [ ] Страница контактов
- [ ] Форма обратной связи
- [ ] Портфолио с примерами работ

Когда я буду отмечать задачи выполненными — помогай мне обновлять этот файл.
```

**Использование позже:** «Добавь в `PLAN.md` задачу — сделать страницу FAQ» / «Посмотри `PLAN.md`. Что у нас следующее?»

---

### 4.4. Лайфхак: попросить Claude составить план реализации сам для себя (Урок 5)
<!-- LIBRARIAN-PROMPT: claude-plans-itself -->

**Когда использовать:** паспорт проекта готов, нужен пошаговый план MVP.

**Промпт:**

```
Изучи паспорт проекта (project.md) и исследование.
На основе этого составь подробный промпт для самого себя —
пошаговый план реализации MVP этого проекта.

Учти:
- Код пишешь ты, не человек
- Начни с самого простого — чтобы через 10 минут уже был результат
- Каждый шаг должен давать видимый результат
- После каждого шага — проверка: работает ли то, что сделали
- Комментируй, что делаешь — я новичок и хочу учиться

Составь план и выполни первый шаг.
```

## БЛОК 5 — Создание страниц и сайтов (Урок 3)

### 5.1. Визитка одним сообщением
<!-- LIBRARIAN-PROMPT: vizitka-html -->

**Когда использовать:** первый практический шаг новичка после установки Claude Code.

**Промпт:**

```
Создай интерактивный файл vizitka.html — персональную визитку. Данные:
- Имя: {Ваше имя и фамилия}
- Специализация: {Чем вы занимаетесь}

- Тёмный фон (#0a0a0a)

- Кнопка «Написать мне» с анимацией
- Красиво и на компьютере, и на телефоне

Всё в одном HTML-файле, без внешних зависимостей. Для иконок используй SVG.
```

**Когда использовать:** Mini App работает, но бот не «знает» пользователей, которые зашли по прямой ссылке. Нужен ненавязчивый сбор подписок.

**Промпт:**

```
Добавь экран-оффер в Mini App.
Появляется при первом открытии,
показывать один раз
(запоминать в localStorage —
хранилище браузера).

Структура:

1. Эмодзи — крупная, по центру
   [🎁]

2. Заголовок — главная ценность
   [{ВАШ ОФФЕР — например "Скидка 15% на первую запись"}]

3. Подзаголовок — что нужно сделать
   [Подпишитесь на бота —
    получите промокод в личное сообщение]

4. Буллеты — выгоды подписки
   [• Напомним о записи за день]
   [• Первыми узнаёте о свободных окошках]
   [• Эксклюзивные акции для подписчиков]

5. Кнопка — яркая, на всю ширину
   Текст: [{ВАШ CTA — "Получить скидку 15%"}]
   Ссылка: https://t.me/{username_бота}?start=from_app

6. Под кнопкой — «Пропустить»

Дизайн:
- Модалка с полупрозрачным фоном
- Карточка по центру, скруглённые углы
- Кнопка — акцент (#2AABEE), минимум 48px
- Заголовок — жирный, 20-24px
- Плавная анимация появления
```

⚠️ **Внимание:** замените `{username_бота}` на username вашего бота (тот, что на `_bot`). Иначе ссылка не работает.
