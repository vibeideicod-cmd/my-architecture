# Чек-листы и фреймворки: вайбкодинг

> **Назначение:** готовые рабочие чек-листы, фреймворки и шаблоны из Модуля 1 курса. Применяются как есть или адаптируются под задачу.

---

## Как пользоваться

1. Выбери чек-лист или фреймворк под свою задачу
2. Скопируй структуру в свой документ или прогони по пунктам
3. Заполни под клиента / нишу
4. После использования — отметь в frontmatter методички (`использовано-в:`)

---

## ФРЕЙМВОРК 1 — 5-шаговый алгоритм (GPS вайбкодера)

<!-- LIBRARIAN-FRAMEWORK: 5-step-algorithm -->

**Назначение:** универсальная последовательность для любого цифрового продукта.

**Структура:**

```
ИДЕЯ → ИССЛЕДОВАНИЕ → КРИТИКА → ПЛАН → РЕАЛИЗАЦИЯ
 💡         🔍              🤔        📋        ⚡
Что,    Смотрим      Находим    Разбиваем  Даём задачи
для     лучших,       дыры ДО    на шаги    ИИ по одной
кого,   берём за      начала
зачем   основу        работы
```

**Применять всегда в этом порядке.** Перепрыгивать через шаг — повторять провал 70% IT-стартапов.

## ФРЕЙМВОРК 2 — Формула идеи

<!-- LIBRARIAN-FRAMEWORK: idea-formula -->

**Шаблон:**

```
Я создаю {ЧТО} для {КОГО}, чтобы решить проблему {КАКУЮ}
```

**Проверка идеи:**

- [ ] Конкретный тип продукта? (не «приложение», а «квиз» или «калькулятор»)
- [ ] Конкретная аудитория? (не «все люди», а «репетиторы» или «владельцы кофеен»)
- [ ] Конкретная проблема? (не «хочу помочь», а «нет сайта для записи»)

---

## ФРЕЙМВОРК 3 — Формула 5 элементов хорошего промпта

<!-- LIBRARIAN-FRAMEWORK: prompt-5-elements -->

**Назначение:** структура промпта для Claude Code (и любой нейросети) при работе с кодом.

**Структура:**

| # | Элемент | Что | Пример |
|---|---|---|---|
| 1 | Контекст | Что за проект, какая ситуация | «Я делаю сайт для кофейни» |
| 2 | Роль | Кем должен быть ИИ | «Ты опытный веб-дизайнер» |
| 3 | Задача | Что именно сделать | «Создай главную страницу» |
| 4 | Формат | Как получить результат | «Один HTML-файл, адаптивный» |
| 5 | Ограничения | Чего не делать | «Без внешних библиотек» |

**Хороший vs плохой промпт:**

❌ Плохой: `Сделай трекер привычек`

✅ Хороший:
```
Создай файл habits.html — трекер привычек на неделю.
5 привычек: вода, зарядка, чтение, сон до 23:00, без сахара.
Таблица 5×7 с чекбоксами. При клике — галочка с анимацией.
Прогресс-бар внизу: сколько процентов выполнено за день.
Тёмный фон, неоновые акценты, адаптивный дизайн.
```

## ФРЕЙМВОРК 4 — Цикл вайбкодера (6 шагов)

<!-- LIBRARIAN-FRAMEWORK: vibecoder-cycle -->

**Структура:**

| Шаг | Что делаем | Пример |
|---|---|---|
| 1 | Представьте образ конечного результата | «Хочу трекер привычек с прогресс-баром» |
| 2 | Опишите его словами (или попросите Claude помочь) | Подробный промпт |
| 3 | Посмотрите результат в браузере | Открыли файл, проверили |
| 4 | Раскритикуйте глазами пользователя | «Что неудобно? Чего не хватает?» |
| 5 | Улучшите итерациями | Короткие уточнения по одному |
| 6 | Повторяйте до идеала | Снова критика → снова улучшение |

**Формула:** Создал → Раскритиковал → Улучшил → Снова раскритиковал.

## ФРЕЙМВОРК 5 — Стратегия экономии (3 инструмента, 3 этапа)

<!-- LIBRARIAN-FRAMEWORK: free-then-paid -->

**Структура:**

| Этап | Что | Инструмент | Стоимость |
|---|---|---|---|
| 1 | Идея + план + тексты | Google AI Studio | Бесплатно |
| 2 | Быстрый красивый набросок | Lovable | $25/мес |
| 3 | Полноценный продукт | Claude Code | $20/мес |

**Главный принцип:** сначала бесплатно, потом за деньги.

**Формула экономии:**
1. План и тексты в AI Studio (бесплатно)
2. Набросок сайта в AI Studio → перенос в Lovable
3. Готовое из галереи Lovable → Remix вместо создания с нуля
4. Серьёзная работа в Claude Code → полный контроль

## ЧЕК-ЛИСТ 1 — Настройка рабочего места (Урок 3)

<!-- LIBRARIAN-CHECKLIST: workspace-setup -->

**Когда применять:** перед стартом первого проекта вайбкодера.

**Шаги:**

- [ ] **VS Code установлен** (code.visualstudio.com → Download)
- [ ] **Windows: галочка «Добавить в PATH»** при установке (без неё Claude Code не работает)
- [ ] **Создана папка `Documents/Projects/`** для всех проектов
- [ ] **Создана папка проекта** внутри Projects (латиница, без пробелов, строчными)
- [ ] **Папка открыта в VS Code** (File → Open Folder)
- [ ] **Расширение Claude Code установлено** (магазин расширений → «Claude Code» от Anthropic → Install)
- [ ] **Авторизован в Claude Code** через браузер
- [ ] **Подписка Pro $20/мес** оформлена (для старта хватит)
- [ ] **Проверка:** «Привет! Ты меня слышишь?» в чат Claude Code → ответил
- [ ] **Выбран режим: Ask Before Edit** (для новичков, рекомендуется)
- [ ] **GitHub-аккаунт зарегистрирован** (github.com → Sign Up)
- [ ] **Email подтверждён** на GitHub
- [ ] **Проект подключён к GitHub** (промпт: «Подключи проект к GitHub»)

## ЧЕК-ЛИСТ 6 — 3 критерия личного проекта

<!-- LIBRARIAN-CHECKLIST: personal-project-3-criteria -->

- [ ] **Кому-то это нужно** — есть аудитория, которая заплатит или будет пользоваться

## ЧЕК-ЛИСТ 9 — Цикл улучшения документации

<!-- LIBRARIAN-CHECKLIST: documentation-improvement -->

Прогнать документацию через 2-3 прохода:

- [ ] **Перечитал** как будто впервые. Понятно ли человеку со стороны? Нет ли противоречий?
- [ ] **«Ты уверен?»** проверены ключевые утверждения
- [ ] **Критический взгляд:** «Какие слабые места? Что может пойти не так? Какие функции обычно забывают?»
- [ ] **Пересмотрел** — обновил с учётом найденного

**Когда останавливаться:** новые правки мелкие и косметические. Нет явных дыр.

## ЧЕК-ЛИСТ 10 — Ритм коммитов (рефлекс)

<!-- LIBRARIAN-CHECKLIST: commit-rhythm -->

| Когда | Действие |
|---|---|
| Сделали что-то рабочее | `Сделай коммит с сообщением — {что добавили}` |
| Сделали 2–3 коммита | `Отправь на GitHub` |
| Уходите от компьютера | `Сделай коммит и пуш` |
| Каждые 15–30 минут | `Сделай коммит и пуш` ← рефлекс! |

**Главное правило:** регулярные коммиты — страховка. Если что-то сломалось, можно откатиться к последнему рабочему коммиту за 30 секунд.

## ШАБЛОН 2 — `CLAUDE.md` (правила для ИИ)

<!-- LIBRARIAN-TEMPLATE: claude-md -->

**Когда нужен:** один раз в начале проекта. Claude читает автоматически в каждом новом чате.

**Шаблон:**

```markdown
# Правила работы с этим проектом

## Стиль и подход
- Все страницы делаем в одном HTML-файле, без внешних библиотек
- Дизайн: тёмный фон, светлый текст, акцентный цвет — оранжевый
- Адаптивная вёрстка — всё должно хорошо выглядеть на мобильных
- Стиль текста: простой, дружеский, без канцелярита

## Что НЕ делать
- Не подключать сторонние JS-библиотеки и CSS-фреймворки
- Не использовать кириллицу в названиях файлов и переменных

## Как работать с проектом
- Перед созданием нового файла — сначала расскажи, что планируешь сделать
- После каждой правки — предложи закоммитить

## Проверка результатов
- Адаптивность проверяется на 375px, 768px, 1440px
- Контраст текста должен быть достаточным для чтения

## Общение
- Я новичок, объясняй простым языком
- Если делаешь что-то нестандартное — кратко объясняй зачем
```

## ТАБЛИЦА: Хаотичный vs Системный подход (Урок 2)

<!-- LIBRARIAN-FRAMEWORK: chaotic-vs-systemic -->

| Аспект | Хаотичный | Системный |
|---|---|---|
| Начало | «Хочу приложение» | Чёткая формулировка идеи |
| Исследование | Не смотрит, что есть | Изучает лучшие, берёт основу |
| Критика | Не проверяет | Ищет слабые места ДО создания |
| План | Нет плана | Разбивает на малые шаги |
| Результат | Бросает через 3 дня | Готовый продукт за 2-5 часов |

## СЛОВАРЬ ТЕРМИНОВ (из шпаргалки Урока 4)

<!-- LIBRARIAN-FRAMEWORK: glossary -->

| Термин | Что это простыми словами | Аналогия |
|---|---|---|
| Фронтенд | Внешняя часть сайта — кнопки, тексты, картинки | Витрина магазина |
| Бэкенд | Внутренняя часть — логика, данные, вход | Склад и кухня за витриной |
| API | Способ, которым программы общаются между собой | Официант — берёт заказ и приносит блюдо |
| Хостинг | Место в интернете, где живёт ваш сайт | Аренда квартиры для сайта |
| Домен | Адрес сайта (например, mysite.ru) | Название улицы и номер дома |
| Деплой | Процесс публикации сайта в интернет | Открытие магазина |
| Токены | Единицы работы AI. Каждое слово стоит токенов | Минуты на телефоне |
| Промпт | Команда, которую вы пишете AI | Рецепт для повара — чем точнее, тем вкуснее |
| MVP | Самая простая версия продукта, которая уже работает | Самокат вместо автомобиля — уже едет |
| Remix | Копирование чужого проекта в Lovable для переделки | Взять чертёж дома и перестроить по-своему |
| Localhost | Адрес вашего компьютера — черновик, видный только вам | Черновик в столе |
| Commit | Сохранение на своём компьютере | Сохранение в видеоигре |
| Push | Загрузка сохранения в облако (GitHub) | Загрузка фото в облако |

## АНТИЧЕК-ЛИСТ — Чего НЕ делать

<!-- LIBRARIAN-FRAMEWORK: anti-checklist -->

- ❌ Открывать ИИ и писать «сделай мне приложение»
- ❌ Делать всё за один промпт — не разбивая на этапы
- ❌ Кириллица или пробелы в названиях папок
