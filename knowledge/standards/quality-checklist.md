# Чеклист качества

Используется агентом QA перед каждой сдачей результата.

---

## Универсальные требования (все направления)

- [ ] Задача из брифа выполнена полностью
- [ ] Нет ничего лишнего, что не просили
- [ ] Язык простой, без канцелярита и шаблонных фраз
- [ ] Нет орфографических ошибок в заметных местах (заголовки, кнопки, имена)

---

## Сайты и лендинги

- [ ] Нет внешних библиотек и CDN
- [ ] Все стили внутри `<style>` в том же HTML-файле
- [ ] Адаптив: мобильный (до 480px), планшет (768px), десктоп (1024px+)
- [ ] Используется `clamp()` для размеров текста
- [ ] Цвета соответствуют дизайн-системе (`/knowledge/standards/design-system.md`)
- [ ] Шрифты: только Google Fonts или системные
- [ ] Семантический HTML: h1-h3, section, main, footer, header
- [ ] Кнопки CTA присутствуют и ведут на правильный адрес
- [ ] Изображения-заглушки обозначены явно (если нет финальных фото)

---

## Контент и соцсети

- [ ] Целевая аудитория учтена в тексте
- [ ] Есть чёткий призыв к действию
- [ ] Нет шаблонных фраз: "уникальный", "команда профессионалов", "мы рады предложить"
- [ ] Тон соответствует голосу бренда клиента
- [ ] Если стратегия — есть рубрики с объяснением и примерами тем

---

## Брендинг

- [ ] Каждое решение (цвет, шрифт) обосновано
- [ ] Цветовая палитра задокументирована с HEX-кодами
- [ ] Шрифтовая пара названа и обоснована
- [ ] Tone of voice описан
- [ ] Рекомендации применимы на практике

---

## Системы для бизнеса

- [ ] Архитектура решения описана понятно для нетехнического человека
- [ ] Стек инструментов объяснён (почему именно они)
- [ ] Шаги внедрения расписаны с ответственными и сроками
- [ ] Указаны зависимости (что нужно от клиента) и риски
- [ ] Есть критерий готовности — как понять, что система работает
