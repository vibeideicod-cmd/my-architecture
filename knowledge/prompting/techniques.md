# Техники структурирования промтов

Методы для задач, где CRAFT недостаточно — когда задача сложная, ответ получается общим, или нужно улучшить уже готовый результат.

---

## Chain Prompting — цепочка шагов

**Принцип:** Большая задача → маленькие последовательные запросы. Каждый шаг строится на результате предыдущего. Вместо одного размытого ответа — конкретика на каждом уровне.

**Когда использовать:**
- Задача сложная и многосоставная
- Первый результат получается общим и без деталей
- Нужно глубоко разобраться в теме перед тем как что-то создать

**Структура:**
```
Шаг 1: Выяви проблему / собери информацию
Шаг 2: Реши каждую часть / углубись в детали
Шаг 3: Собери финальный результат на основе шагов 1–2
```

---

### Примеры для каждого направления

#### Контент и соцсети

```
Шаг 1:
«Назови 3 главных страха клиентов [тип бизнеса] при выборе [продукт/услуга]»

Шаг 2:
«Как конкретно [наш бизнес] решает каждый из этих страхов?
Приведи точные факты, цифры, примеры»

Шаг 3:
«Теперь напиши продающий пост для Instagram, начиная с главного страха.
Используй конкретные цифры из шага 2»
```

#### Сайты и лендинги

```
Шаг 1:
«Изучи этот бриф: [вставить бриф].
Назови 5 ключевых возражений, которые посетитель может иметь перед покупкой»

Шаг 2:
«Для каждого возражения — как лендинг должен его закрывать?
Какие блоки, тексты, доказательства нужны?»

Шаг 3:
«Напиши структуру лендинга с заголовками и описанием каждого блока,
закрывающую все 5 возражений»
```

#### Приложения и инструменты (вайб-кодинг)

```
Шаг 1:
«Я хочу сделать [описание инструмента]. Какие 3 главных сценария использования
будут у пользователя? Опиши каждый пошагово»

Шаг 2:
«Для каждого сценария — какие экраны / состояния / компоненты нужны?
Что должно происходить при каждом действии пользователя?»

Шаг 3:
«Теперь напиши структуру HTML-страницы / компонентов приложения
на основе этих сценариев. Начни с MVP — только самое важное»
```

#### Брендинг

```
Шаг 1:
«Вот информация о бренде: [описание]. Кто конкурент №1?
Назови 5 вещей, которые делают его визуальный стиль узнаваемым»

Шаг 2:
«Как наш бренд может отличаться от конкурента визуально?
Предложи 3 разных направления: цвет, типографика, стиль иллюстраций»

Шаг 3:
«Опиши визуальную систему для нашего бренда: палитра, шрифты, элементы стиля.
Оформи как мини-брендбук для дизайнера»
```

## Метод инверсии

**Принцип:** Переворачиваешь вопрос наоборот. Вместо «как сделать хорошо» — «почему это работает плохо». Решения становятся очевидными, когда видишь корень проблемы.

**Когда использовать:**
- Не знаешь с чего начать
- Результат получается шаблонным
- Нужно найти слабые места до того как они стали проблемой

**Алгоритм:**
```
1. Возьми свой главный запрос
2. Переверни его: вместо «как достичь X» → «почему X не работает / почему люди уходят»
3. Получи список проблем
4. Спроси: «Как это исправить?»
5. Сравни с тем, что планировал изначально
```

---

### Примеры для каждого направления

#### Контент и соцсети

| Прямой вопрос | Инверсия |
|---|---|
| Как привлечь подписчиков? | Почему люди отписываются от похожих аккаунтов? |
| Как сделать вирусный пост? | Какие посты люди точно не будут репостить? |
| Как повысить вовлечённость? | Почему люди скроллят мимо не останавливаясь? |

**Пример запроса:**
```
«Представь, что ты подписчик аккаунта [тип бизнеса].
Назови 5 вещей, из-за которых ты бы отписался в первую неделю»
```

#### Сайты и лендинги

| Прямой вопрос | Инверсия |
|---|---|
| Как сделать конвертирующий сайт? | Почему люди закрывают сайт за 3 секунды? |
| Как повысить конверсию? | Почему люди не нажимают на кнопку? |
| Как выстроить доверие? | Что заставляет усомниться в компании? |

**Пример запроса:**
```
«Ты потенциальный клиент, зашёл на сайт [описание бизнеса].
Назови 5 причин, из-за которых ты закроешь страницу не оставив заявку»
```

#### Приложения и инструменты (вайб-кодинг)

| Прямой вопрос | Инверсия |
|---|---|
| Как сделать удобный интерфейс? | Что раздражает в похожих приложениях? |
| Как сделать, чтобы пользователь вернулся? | Почему люди бросают приложения после первого использования? |
| Как написать надёжный код? | Где этот код сломается при реальном использовании? |

**Пример запроса для кода:**
```
«Вот моя функция: [вставить код].
Представь, что ты злобный тестировщик. Назови 5 способов как её сломать —
нестандартный ввод, крайние значения, неожиданное поведение пользователя»
```

**Пример запроса для UI:**
```
«Вот макет экрана: [описание].
Представь, что ты человек, который первый раз видит этот интерфейс.
Назови 5 вещей, которые тебя запутают или разочаруют»
```

#### Брендинг

| Прямой вопрос | Инверсия |
|---|---|
| Как создать запоминающийся бренд? | Почему некоторые бренды забываются через неделю? |
| Как выглядеть профессионально? | Что сигнализирует «это несерьёзно» при первом взгляде? |

## Комбинация: Chain + Инверсия

Самый мощный вариант — начать цепочку с инверсии:

```
Шаг 1 (инверсия):
«Назови 5 главных причин почему [продукт/сайт/контент] не работает у конкурентов»

Шаг 2:
«Как мы можем избежать каждой из этих проблем?
Предложи конкретное решение для каждого пункта»

Шаг 3:
«Теперь создай [результат] с учётом всех решений из шага 2»
```

## Техника "Раскритикуй" (для кода и интерфейсов)

**Принцип:** Просишь AI посмотреть на готовый результат глазами конкретного человека. Находит проблемы до того, как их найдёт пользователь.

**Когда использовать:** После создания сайта, страницы, инструмента — перед сдачей клиенту.

**Готовые промты по ролям:**

```
Представь, что ты обычный пользователь, который впервые открыл [продукт].
Что непонятно? Что неудобно? Чего не хватает? Где запутаешься?
Дай 5 конкретных рекомендаций по улучшению.
```

```
Посмотри как опытный UI/UX-дизайнер.
Оцени: цвета, отступы, типографику, контраст, визуальную иерархию.
Для каждого пункта: что не так и как исправить.
```

```
Какие минусы у этого решения? Что может пойти не так?
Посмотри как владелец конкурирующего бизнеса — что бы ты сделал лучше?
```

```
Посмотри как тестировщик. Попробуй сломать это: нестандартный ввод,
крайние значения, неожиданные действия пользователя.
Назови 5 сценариев где это сломается.
```

**Цикл:** Создал → Раскритиковал → Исправил → Снова раскритиковал → Готово.

## Мета-техника: AI пишет промт для себя

**Принцип:** Когда не знаешь как сформулировать задачу — попроси AI самому поставить себе задачу на основе твоего описания.

```
Я хочу сделать [описание идеи].
Подготовь подробный промт для самого себя — что создать,
какая структура, какой дизайн, какие элементы, какие ограничения.
Потом выполни этот промт.
```

**Когда использовать:**
- Не можешь сформулировать детали задачи
- Хочешь получить более продуманный результат с первого раза
- Задача сложная и многосоставная

**Вариант для проверки ответа:**
```
Ты уверен?
```
Используй когда AI уверенно говорит что-то важное — особенно про технические детали. Не нужно проверять всё, но ключевые утверждения стоит перепроверить.

## Чеклист применения техник

**Chain Prompting:**
- [ ] Задача разбита на 3 шага, каждый строится на предыдущем
- [ ] Шаг 1 собирает информацию, шаг 3 создаёт результат
- [ ] Каждый шаг — отдельный запрос, не один большой

**Инверсия:**
- [ ] Вопрос перевёрнут: не «как сделать хорошо», а «почему плохо»
- [ ] Получен список проблем
- [ ] Проблемы конвертированы в конкретные решения

**Раскритикуй:**
- [ ] Выбрана роль для критики (пользователь / дизайнер / конкурент / тестировщик)
- [ ] Получен список конкретных проблем (не абстрактных)
- [ ] Каждая проблема превращена в короткую задачу на исправление
