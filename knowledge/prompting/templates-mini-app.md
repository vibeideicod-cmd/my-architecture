**Источник:** Урок 11 «Собираем и подключаем» + Урок 12 «Улучшаем и публикуем».

## 1. Стартовый one-shot промпт — создать Mini App из brief.md + research.md

**Когда:** Фаза 4 (Build), первый запуск кодинга, есть готовые `brief.md` и `research.md`.

**Перед применением:** Директор обязательно задал вопрос «Beget или Vercel?». Ответ записан в PLAN.md.

```
Ты — фронтенд-разработчик, специалист по Telegram Mini App, 10+ лет опыта с HTML/CSS/JS.

Прочитай файлы в проекте:
- brief.md — чертёж экранов и логики
- research.md — что есть у конкурентов, что взять
- CLAUDE.md — правила проекта (если уже есть)
- knowledge/standards/design-system.md — наш брендбук

Создай Telegram Mini App на основе brief.md и research.md.

Технические требования:
- Только HTML + CSS + JavaScript, без фреймворков, без CDN
- Подключи Telegram Web App SDK первым скриптом:
  <script src="https://telegram.org/js/telegram-web-app.js"></script>
- Все файлы положи в папку clients/<project>/tg-app/
- Точка входа — clients/<project>/tg-app/index.html
- Адаптация к теме Telegram: использовать переменные window.Telegram.WebApp.themeParams
  через CSS var(--tg-bg-color), var(--tg-text-color) и т.д.
- Поддержка светлой и тёмной темы Telegram автоматически

Дизайн:
- Стиль нативного Telegram-приложения: закруглённые углы, мягкие тени
- Выглядит как ПРИЛОЖЕНИЕ, не как сайт в рамке
- Плавные переходы между экранами (fade + slide, 200–300ms)
- Брендбук Инны (если продукт внутренний): светлый нейтральный фон,
  горчичный/золотой/зелёный акценты, тёмно-серый текст
- Тема клиента (если продукт клиентский): взять из brief.md

Мобильная адаптация:
- Ширина экрана 320–430px (мобильный only)
- Кнопки минимум 44px по высоте (тапаются пальцем)
- Шрифт минимум 14px
- Нет горизонтальной прокрутки ни на одном экране
- Касания отзываются визуально (active state, HapticFeedback.impactOccurred)

Контент:
- Русский язык интерфейса
- Реальные данные из brief.md, не «Lorem ipsum» и не «Мастер Анна»
- Если данных нет — оставить placeholder «TBD: [описание]» и пометить в PLAN.md

Структура файлов в tg-app/:
- index.html — все экраны внутри, переключение через CSS-классы is-active
- css/style.css — все стили
- js/data.js — данные (моки или обёртки над Supabase)
- js/app.js — вся логика, навигация, TG SDK обёртки

Дополнительно — создай CLAUDE.md в корне tg-app/:
- Какие файлы за что отвечают
- Карта навигации между экранами
- Где менять данные (какой файл, какая константа)
- Как добавлять новый экран

Все комментарии в коде — на русском.

Задавай мне вопросы по ходу работы, если в brief.md что-то не хватает.
НЕ придумывай факты о клиенте — только из brief.md и research.md.
```

## 2. `.env` + `.env.example` + `.gitignore` — обязательная тройка

**Когда:** как только появляется первый секрет в проекте (токен бота, ключ API, пароль БД).

```
Создай обязательную тройку файлов для хранения секретов:

1. .env — настоящие значения (для моего компьютера)
   - BOT_TOKEN=<пусто — заполню сам>
   - [другие переменные по необходимости]

2. .env.example — шаблон БЕЗ реальных значений, с комментариями откуда брать
   - BOT_TOKEN=ваш_токен_от_BotFather
   - [другие переменные с подсказками]

3. .gitignore — список того, что не публикуется
   - .env
   - node_modules/
   - .DS_Store

Правило: .env НИКОГДА не коммитим в git, .env.example — коммитим,
.gitignore — коммитим.

Проверь что .env уже в .gitignore (если файл был раньше).
```

**Важно:** после создания `.env` с BOT_TOKEN=<пусто> — напомнить Инне:
> ⚠️ Открой `.env` в VS Code и вручную вставь токен от BotFather. Не забудь нажать **Cmd+S** чтобы сохранить!

Это обязательный шаг, Инна предпочитает Способ 2 (ручная вставка) из Урока 11 для максимальной безопасности. См. `feedback-token-step.md` и `feedback-save-reminder.md` в памяти.

## 3. Создать бота в BotFather (ручной шаг Инны)

Это не промпт — это чеклист, который Директор показывает Инне:

```
1. Telegram → @BotFather → Start
2. Напиши: /newbot
3. BotFather спросит имя. Введи: «[название продукта]»
4. Потом спросит username. Введи: [проект]_bot
   (должен заканчиваться на _bot)
5. Скопируй токен — длинную строку из букв и цифр
6. Открой .env в VS Code → замени BOT_TOKEN=<пусто> на настоящий токен
7. Cmd+S
```

После этого Кодыч сможет настроить бота через Bot API (см. пункт 6).

## 4. Добавить tab bar — нижнее меню

**Когда:** Фаза 4 или 6 (после тестирования, если пользователи не понимают навигацию — это самый частый фидбек).

```
Добавь нижнее меню (tab bar) с вкладками:
[Каталог] [О мастере] [Портфолио] [Мои записи]

Требования:
- Фиксированное положение внизу экрана (position: fixed; bottom: 0)
- Равномерные ширина вкладок (flex: 1)
- Активная вкладка выделена (цвет, индикатор сверху)
- Цвета — из темы Telegram через var(--tg-*), чтобы работала тёмная тема
- Видно только на верхнеуровневых tab-экранах, скрыто на flow-экранах
  (screen-details, screen-booking, screen-confirm, screen-success)
- При клике — переход на соответствующий экран + haptic feedback
- Высота кнопок минимум 44px
- Не ломай текущую навигацию

Обнови updateTabbar() в app.js чтобы он вызывался из navigate().

После этого обнови CLAUDE.md — добавь секцию «Tab bar»: какие вкладки,
где они видны, как добавлять новую.
```

**Варианты вкладок под нишу:**

| Ниша | Вкладки |
|---|---|
| Бьюти-мастер | Каталог / О мастере / Портфолио / Мои записи |
| Ресторан | Меню / О нас / Акции / Мои заказы |
| Репетитор | Услуги / Обо мне / Расписание / Мои уроки |
| Фотограф | Портфолио / Обо мне / Услуги / Контакты |
| Мастер-группа | Программа / О мастере / Чат / Мои задания |

## 5. Screen-offer — модалка для `/start`-капчи

**Когда:** Фаза 4, если Mini App будет распространяться через прямую ссылку (`t.me/<bot>/app`), а не только через Menu Button.

**Зачем:** если человек зашёл через прямую ссылку, не нажав `/start` — бот НЕ сможет потом писать ему первым. Screen-offer решает: модалка при первом открытии, кнопка ведёт на бота с параметром `?start=from_app`, пользователь видит ценность, жмёт — `/start` срабатывает автоматически, бот получает право на сообщения.

```
Добавь экран-оффер в Mini App.

Поведение:
- Появляется при первом открытии приложения
- Показывается один раз, запоминается в localStorage (ключ: <project>_offer_shown)
- Можно закрыть по «Пропустить» — не показывается снова

Структура:

1. Эмодзи — крупная, по центру (🎁 или другая, взять из brief.md)

2. Заголовок — главная ценность
   Пример для бьюти: «Скидка 15% на первую запись»
   Пример для мастер-группы: «Первая встреча — бесплатно»

3. Подзаголовок — что нужно сделать
   «Подпишитесь на бота — получите [ценность] в личное сообщение»

4. Буллеты — 3 выгоды подписки
   • [Напомним о записи за день]
   • [Первыми узнаете о свободных окошках]
   • [Эксклюзивные акции для подписчиков]

5. Кнопка — яркая, на всю ширину
   Текст: «Получить [ценность]»
   Ссылка: https://t.me/<BOT_USERNAME>?start=from_app

6. Под кнопкой — ссылка «Пропустить» серым

Дизайн:
- Модалка с полупрозрачным фоном (backdrop)
- Карточка по центру, скруглённые углы (16-20px)
- Кнопка — высота минимум 48px, акцент брендбука
- Заголовок — жирный, 20–24px
- Плавная анимация появления (fade 300ms + scale от 0.95)

ВАЖНО: замени <BOT_USERNAME> на настоящий username бота перед деплоем.
Параметр ?start=from_app — это метка, по которой бот потом поймёт
что пользователь пришёл из приложения и отправит специальное приветствие.
```

## 6. One-shot настройка бота через Bot API

**Когда:** Фаза 6 (QA + Deploy), бот создан в BotFather, токен в `.env`, Mini App задеплоен.

```
Ты — backend-разработчик. Настрой Telegram-бота полностью через Bot API,
используя токен из .env (BOT_TOKEN).

Бот открывает Mini App: [описание в 1-2 предложения что делает приложение].

Через Telegram Bot API выполни следующие запросы:

1. setMyDescription
   Текст: 2-3 предложения, отвечающие на три вопроса:
   - ЧТО — что делает этот бот
   - ЗАЧЕМ — какую пользу получит человек
   - КУДА НАЖАТЬ — чёткое следующее действие
   Формат HTTP-запроса:
   POST https://api.telegram.org/bot<TOKEN>/setMyDescription
   body: { "description": "<текст>" }

2. setMyShortDescription
   Одно предложение для поиска в Telegram.
   POST https://api.telegram.org/bot<TOKEN>/setMyShortDescription
   body: { "short_description": "<текст>" }

3. setMyCommands
   Команды:
   - /start — Начать работу
   - /help — Помощь и контакты
   - /about — О сервисе (опционально)
   POST https://api.telegram.org/bot<TOKEN>/setMyCommands
   body: { "commands": [{"command":"start","description":"..."}, ...] }

4. setChatMenuButton (для Menu Button)
   Привязать Mini App к кнопке меню:
   POST https://api.telegram.org/bot<TOKEN>/setChatMenuButton
   body: {
     "menu_button": {
       "type": "web_app",
       "text": "<текст кнопки>",
       "web_app": { "url": "<HTTPS ссылка Mini App>" }
     }
   }

Покажи мне каждый запрос перед отправкой. После каждого — покажи ответ Telegram.

Язык интерфейса — русский, тон дружелюбный, без пафоса.
```

## 8. Точечные правки — «один промпт, одна задача»

Формула: **Что именно / Где именно / Что не трогать**.

Примеры:

```
Сделай кнопки на экране каталога крупнее (минимум 48px)
и добавь отступ 12px между ними.
Больше ничего не меняй.
```

```
Замени фото-заглушки на экране services на emoji-иконки из brief.md.
Не трогай остальные экраны. Не меняй размеры контейнеров.
```

```
Экран details слишком пустой между заголовком и описанием.
Добавь блок с 3 короткими фактами о мастере.
Только этот экран. Остальные не трогай.
```

```
На экране success кнопка «Поделиться с подругой» плохо видна.
Сделай её акцентной (золотой градиент). Остальное оставь.
```

**Правило:** если в голове «надо поменять 5 вещей» — отправь 5 отдельных промптов, не один большой. Большой = путаница и ошибки.
