# Библиотека готовых промт-шаблонов

Готовые команды для всех типов задач. Подставь свои данные вместо `{...}` и отправляй.

---

## Сайты и лендинги

### Лендинг (одностраничник)
```
Создай лендинг для {продукт/услуга}.
Целевая аудитория: {кто}.
Структура:
- Первый экран: заголовок с УТП + кнопка
- Блок проблемы: {какую боль решаем}
- 3 преимущества с иконками
- Отзывы (3 штуки)
- Тарифы
- Финальный призыв + кнопка
Стиль: {тёмный/светлый, акцентный цвет, настроение}.
Адаптивная вёрстка. Один HTML-файл.
```

### Портфолио
```
Создай сайт-портфолио.
Структура:
- Шапка с именем и специализацией
- Сетка проектов (карточки: превью, название, описание)
- При клике — модальное окно с деталями проекта
- Фильтры по категориям
- Секция «Обо мне»
- Контакты и соцсети
Стиль: {описать}.
```

### Многостраничный сайт
```
Создай сайт для {бизнес/проект} из 4 страниц:
1. Главная — представление + УТП
2. Услуги — карточки с ценами
3. О нас — история, команда
4. Контакты — форма + карта
Общая навигация на всех страницах.
Единый стиль: {описать}.
Адаптивный дизайн.
```

### Интернет-магазин (витрина)
```
Создай витрину интернет-магазина {тематика}.
- Сетка товаров (карточки: фото-заглушка, название, цена, кнопка)
- Фильтры: по категории, по цене
- Поиск по названию
- Корзина: добавление товаров, подсчёт суммы
- Адаптивный дизайн
Стиль: {описать}.
```

## Приложения и инструменты

### Трекер привычек
```
Создай трекер привычек на неделю.
{N} привычек: {перечислить}.
Таблица с чекбоксами. При клике — галочка с анимацией.
Прогресс-бар: процент выполнения за день.
Кнопка «Новая неделя» для сброса.
Данные сохраняются в браузере.
Стиль: {описать}.
```

### Калькулятор стоимости
```
Создай калькулятор стоимости {услуги}.
Параметры:
- {Параметр 1: ползунок / выпадающий список}
- {Параметр 2}
- {Параметр 3}
Итог считается мгновенно при изменении.
Кнопка «Оставить заявку» под результатом.
Стиль: {описать}.
```

### Квиз / тест
```
Создай квиз из {N} вопросов на тему {тема}.
Один вопрос на экране, варианты ответа.
Прогресс-бар сверху. Кнопка «Далее» после выбора.
В конце — результат с рекомендацией и кнопкой {действие}.
Стиль: {описать}.
```

### Список задач (To-Do)
```
Создай приложение для списка задач.
- Поле ввода + кнопка добавления
- Задачи списком с чекбоксами и кнопкой удаления
- Фильтры: все / активные / выполненные
- Счётчик: выполнено / всего
- Данные сохраняются в браузере
Стиль: {описать}.
```

### Дашборд / панель управления
```
Создай дашборд для {проект/бизнес}.
Блоки:
- 4 карточки с метриками (число + тренд вверх/вниз)
- График (столбчатый или линейный)
- Таблица последних {событий/заказов/клиентов}
- Боковое меню навигации
Стиль: {описать}.
```

## Дизайн и интерфейс

### Изменить цветовую схему
```
Измени цветовую схему:
- Фон: {цвет}
- Текст: {цвет}
- Акцент (кнопки, ссылки): {цвет}
Убедись, что контраст достаточный для чтения.
```

### Добавить анимации
```
Добавь анимации:
- Элементы появляются при прокрутке (плавно снизу)
- Кнопки увеличиваются при наведении
- Карточки приподнимаются при наведении
- Плавные переходы на всех элементах
```

### Навигация
```
Добавь навигацию:
- Логотип слева, пункты меню справа
- Пункты: {перечислить}
- На мобильном: меню-бургер, при клике — выезжает сбоку
- Фиксированная шапка (прилипает при прокрутке)
```

### Тёмная/светлая тема
```
Добавь переключатель темы.
Кнопка (солнце/луна) в правом верхнем углу.
Тёмная тема: {цвета}.
Светлая тема: {цвета}.
Плавный переход. Запоминать выбор в браузере.
```

## Критика и проверка

### Глазами пользователя
```
Представь, что ты обычный пользователь, который впервые открыл этот проект.
Что непонятно? Что неудобно? Чего не хватает? Где запутаешься?
Дай 5 конкретных рекомендаций по улучшению.
```

### Глазами UI/UX-дизайнера
```
Посмотри как опытный UI/UX-дизайнер.
Оцени: цвета, отступы, типографику, контраст, визуальную иерархию.
Для каждого пункта: что не так и как исправить.
```

### Глазами конкурента
```
Посмотри как владелец конкурирующего бизнеса.
Что бы ты сделал лучше? Где твой продукт выигрывает?
Дай 5 конкретных улучшений.
```

### Проверка на мобильных
```
Проверь на разных экранах: 375px, 390px, 768px, 1440px.
Что ломается? Что обрезается? Что налезает друг на друга?
Исправь все проблемы.
```

## Исправление ошибок

### Ошибка в консоли
```
Вот ошибка: {скопировать текст ошибки}.
Объясни простыми словами, что пошло не так.
Исправь. Покажи, что изменил и почему.
```

### Выглядит не так
```
{Что не так}. Ожидаю: {как должно быть}. Вижу: {что вижу сейчас}.
Исправь, чтобы выглядело как ожидаю.
```

### Всё сломалось
```
После последних изменений перестало работать {что именно}.
До изменений работало нормально.
Найди, что сломалось, и верни в рабочее состояние.
```

## Планирование и исследование

### Выбор подхода
```
Мне нужно сделать {задача}.
Какие есть способы? Сравни 3 варианта.
Для каждого: плюсы, минусы, сложность.
Дай рекомендацию для моего случая: {контекст}.
```

### Разбивка проекта на шаги
```
Мне нужно сделать {проект}.
Разбей на 5-7 простых шагов.
Для каждого: что делаем, какой файл создаём, какой результат увидим.
Начни с самого простого — чтобы через 10 минут уже был видимый результат.
```

### Claude пишет промт для себя (мета-техника)
```
Я хочу сделать {описание идеи}.
Подготовь подробный промт для самого себя — что создать,
какая структура, какой дизайн, какие элементы, какие ограничения.
Потом выполни этот промт.
```
Когда использовать: когда не знаешь как сформулировать задачу. Claude сам разберётся что спросить и что сделать.

## Публикация и обратная связь

### Проверка проекта перед деплоем
Запускай ДО `./deploy-beget.sh` — убедись что всё работает.
```
Проверь мой проект перед публикацией:
- Убедись, что все страницы открываются без ошибок
- Проверь, что формы, кнопки и интерактивные элементы работают
- Убедись, что проект выглядит нормально на мобильных (375px, 768px, 1440px)
- Если есть ошибки в консоли — исправь их
После проверки скажи: что работает, что нужно поправить.
```

### Исправления после обратной связи
После того как показал сайт клиенту или знакомым — фиксируем замечания и деплоим.
```
Мне дали обратную связь по сайту. Вот что нужно исправить:
1. {что не так — своими словами}
2. {что нужно добавить}
3. {что убрать или поменять}

Исправь всё, сделай git commit и задеплой на Beget.
```

## Финализация сайта (три шага от ТЗ до готового продукта)

### Шаг 1 — Создаём сайт из ТЗ
```
Прочитай файл brief.md — это техническое задание на лендинг.
Создай файл index.html — полноценный лендинг по этому ТЗ.
Требования:
- Всё в одном HTML-файле (стили и скрипты внутри)
- Современный, стильный дизайн
- Адаптивная вёрстка — красиво на телефоне и компьютере
- Плавные анимации при прокрутке
- Рабочая навигация по секциям
- Заглушки для фото (placeholder-картинки)
- Градиенты, тени, скругления
Чистый HTML + CSS + JavaScript. Без внешних библиотек.
```

### Шаг 2 — Вау-эффекты
```
Посмотри на index.html и добавь вау-эффекты:
- Анимации появления — каждый блок плавно появляется при прокрутке (fade-in снизу вверх)
- Hover-эффекты — карточки приподнимаются при наведении, кнопки меняют цвет с плавным переходом
- Плавная прокрутка — клик по меню скроллит к секции
- Градиентный фон заголовка — анимированный, меняет цвета
- Параллакс-эффект — фон двигается медленнее при прокрутке
Сделай так, чтобы сайт выглядел дорого и профессионально.
```

### Шаг 3 — Интерактивность
```
Добавь в index.html интерактивные элементы:
- Кнопка «наверх» — появляется при прокрутке, плавно возвращает наверх
- Форма обратной связи — имя, email, сообщение. При «Отправить» — красивая анимация успеха (реальная отправка не нужна, только визуал)
- Счётчики — числа плавно «накручиваются» при появлении на экране
- Мобильное меню — гамбургер-иконка раскрывает меню
Проверь работу на мобильных устройствах. Файл должен открываться при клике из любого мессенджера. После проверки выдай отчёт.
```
