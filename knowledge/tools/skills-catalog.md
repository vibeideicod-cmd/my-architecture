# Каталог скиллов

Скиллы — это готовые роли и сценарии, которые активируются командой `/имя` в Claude Code. Каждый скилл — как отдельный специалист с чёткой задачей.

---

## Наши скиллы (уже установлены)

| Команда | Что делает | Когда вызывать |
|---|---|---|
| `/brief` | Собирает бриф по системе СССР — маркетинг, контент, сайт, бренд | Старт любого клиентского проекта |
| `/discovery` | Глубокое интервью → техническое задание для нового продукта | Создание бота, инструмента, приложения, квиза |
| `/deploy` | Деплой на Beget или другой хостинг | После завершения верстки |
| `/seo` | SEO-проверка страницы | Перед публикацией или по запросу клиента |
| `/social-research` | Анализ ЦА, конкурентов и трендов в соцсетях — исследование перед контент-стратегией | Перед `/vk-pack` или `/tg-pack`, перед контент-планом |
| `/vk-pack` | Полный пакет контента для VK: стратегия, шапка, посты, хештеги | Запуск или перезапуск VK-присутствия клиента |
| `/tg-pack` | Полный пакет контента для Telegram: стратегия, описание канала, посты | Запуск или перезапуск Telegram-канала клиента |
| `/competitor-research` | Глубокий анализ 5+ конкурентов: позиционирование, цены, белые пятна рынка | Перед стратегией для нового клиента, перед запуском продукта |
| `/client-audit` | Полный аудит нового клиента: бизнес, продукт, аудитория, каналы, боли | Начало работы с новым клиентом перед брифом |
| `/digital-audit` | Анализ всего цифрового присутствия: сайт, соцсети, реклама, коммуникации | Когда клиент говорит «что-то не работает» или перед цифровой стратегией |
| `/market-research` | Анализ рынка: объём, тренды, барьеры входа, вывод «входить / не входить» | Перед запуском нового продукта в незнакомой нише |
| `/monitoring-check` | Проверка инфраструктуры: домены, SSL, хостинг, боты, деплои | Перед важным запуском, при сбоях, плановая проверка |

### Разница между `/brief` и `/discovery`

| | `/brief` | `/discovery` |
|---|---|---|
| Для чего | Маркетинговые задачи | Новые цифровые продукты |
| Результат | Бриф для специалиста | Полное техническое задание |
| Глубина | 7 вопросов | 5 блоков + сценарии + стек |
| Когда | Сайт, контент, бренд | Бот, инструмент, приложение |

## Служебные скиллы (для Директора и Claude)

### graphify — knowledge graph по папке проекта
**Источник:** [safishamsi/graphify](https://github.com/safishamsi/graphify), PyPI `graphifyy`
**Что делает:** Индексирует папку (код + доки + PDF + картинки) и строит knowledge graph — карту проекта с god-nodes, связями и «неочевидными» узлами. На выходе `GRAPH_REPORT.md`, `graph.html`, `graph.json`.
**Когда применяем:** Только точечно по чек-листу — 15+ файлов, смешанный корпус, долгосрочный клиент, фаза 3+. Полные критерии и процесс — в [graphify.md](./graphify.md).
**Для чего конкретно:** (1) быстро возвращаться в контекст клиента, (2) видеть дырки для до-продажи, (3) экономить токены Claude на крупных корпусах (5–70x).
**Статус:** не установлен. Ставить в конкретную клиентскую папку когда она дорастёт — по решению Директора.

## Скиллы к рассмотрению (для финансовой стороны бизнеса)

> Эти три скилла рассмотреть когда дойдём до монетизации Beauty Визитки и/или соберём юнит-экономику портфеля. Источники нужно уточнить у автора рекомендации (коллега Инны).

### startup-financial-modeling
**Что делает:** Финансовое моделирование стартапа — проекции выручки, burn rate (скорость расхода денег), runway (на сколько хватит), точка безубыточности.
**Когда нам:** Beauty SaaS перед запуском платных подписок. Финансовое здоровье агентства в целом.

### startup-metrics-framework
**Что делает:** Юнит-экономика — CAC (стоимость привлечения клиента), LTV (сколько клиент принесёт за всё время), окупаемость продукта.
**Когда нам:** Любой SaaS-продукт. Решение «делать или не делать» для идей из [ideas.md](../../ideas.md).

### market-sizing-analysis
**Что делает:** Размер рынка — TAM (весь рынок), SAM (доступный нам), SOM (реалистично возьмём).
**Когда нам:** Перед стартом нового продукта — оценить «есть ли вообще кому продавать». Особенно для идей расширения Beauty в другие ниши.

**Статус:** Не установлены. Нужны ссылки на источник от автора рекомендации перед оценкой качества.

## Как устанавливать скиллы

Все скиллы хранятся в `.claude/skills/` как `.md` файлы.

**Наши скиллы** — создаются вручную в этой папке.

**Внешние скиллы** — устанавливаются командой в терминале:
```
claude skills add [ссылка]
```
После установки активируются сразу, перезагрузка не нужна.

```
Готова страница, нужно задеплоить              → /deploy
Нужна SEO-проверка                             → /seo
Анализ ЦА / конкурентов перед контентом        → /social-research
Нужен контент-пакет для VK                     → /social-research → /vk-pack
Нужен контент-пакет для Telegram               → /social-research → /tg-pack
Кто конкуренты, какие цены, где дыры           → /competitor-research
Что-то не работает в онлайне клиента           → /digital-audit
Стоит ли вообще заходить в эту нишу           → /market-research
Перед запуском — проверить инфраструктуру      → /monitoring-check
Нужен уникальный дизайн с WOW-эффектом        → Frontend Design skill
Клиент просит полноценное веб-приложение       → Fullstack Developer skill
```
