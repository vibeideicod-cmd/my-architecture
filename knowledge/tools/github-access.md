# Работа с GitHub с любого устройства

Доступ к репо `vibeideicod-cmd/*` — не только через VSCode. Вот три способа для ситуаций «я не за своим маком».

## 1. GitHub Mobile (iPhone/Android)

- Установить: App Store / Google Play → «GitHub»
- Залогиниться: тем же аккаунтом `vibeideicod-cmd` (пароль или через Google OAuth)

**Что можно с телефона:**
- Просматривать все репо и файлы
- Редактировать отдельные файлы (тап по файлу → карандаш)
- Создавать issues (задачи)
- Коммитить правки прямо из приложения
- Смотреть коммиты и историю

**Что нельзя удобно:** массовая заливка файлов, работа с большими папками.

## 2. Веб-браузер с любого компьютера

Открыть [github.com](https://github.com) → войти через Google (OAuth) или пароль.

**Загрузка файлов через веб:**
1. Открыть нужный репо (например, `showcases`)
2. Кнопка **«Add file»** → **«Upload files»**
3. Перетащить файлы с рабочего стола
4. Внизу — поле «Commit changes» → написать описание → **«Commit changes»**

**Редактирование файла в браузере:**
1. Открыть файл → иконка карандаша справа сверху
2. Править прямо в окне браузера
3. Внизу — commit message → Commit

Работает с любого устройства: чужой комп, iPad, даже рабочий компьютер.

## 3. Passkey — вход без пароля

Чтобы не вводить пароль при входе с разных устройств — настроить passkey (отпечаток / Face ID):

1. github.com → Settings → **Password and authentication** → **Passkeys**
2. **«Add a passkey»** → следовать подсказкам
3. На iPhone — Face ID, на Mac — Touch ID, на Windows — Windows Hello

После этого при входе — просто подтверждение биометрией, никаких паролей.

## Аккаунт

- **Логин:** `vibeideicod-cmd`
- **Привязан:** через Google (gmail) — логинься любым Google-аккаунтом, где есть связь
- **Активные репо:** my-architecture, showcases, neurobabki, my-vps-manager, vibeideicod-cmd (профиль)

## Когда что использовать

| Ситуация | Что использовать |
|---|---|
| Работаю дома за Mac | VSCode + Claude Code как обычно |
| В дороге, нужно посмотреть README | GitHub Mobile |
| Срочно поправить опечатку с телефона | GitHub Mobile, редактирование |
| На чужом ноуте, нужно залить файл | github.com через браузер, Upload files |
| На чужом ноуте, нужно только посмотреть | github.com в браузере, без логина для public-репо |
