## Хостинг и публикация

> **Важно:** Мы работаем с российским хостингом **Beget**. Vercel, Netlify и другие зарубежные платформы в наших проектах не используются — если в каком-то материале упоминается Vercel, адаптируй под Beget.

| Инструмент | Назначение | Когда |
|---|---|---|
| **Beget** | Основной хостинг | Все наши сайты и сайты клиентов |
| `deploy-beget.sh` | Скрипт деплоя | Запускать через `/deploy` или `agents/deployer-beget.md` |
| GitHub | Хранение кода | Все проекты в репозитории |

**Схема публикации:**
```
Изменения в коде → git commit → git push → ./deploy-beget.sh → сайт обновлён на Beget
```

**Данные для деплоя** хранятся в `.env` (не коммитить!):
- `BEGET_USER` — логин
- `BEGET_HOST` — SSH-хост
- `BEGET_PATH` — путь на сервере
- `DOMAIN` — домен сайта

## Принцип выбора

1. Простота прежде всего — не перегружаем клиента сложными инструментами
2. Учитываем, есть ли у клиента технический человек
3. Предпочитаем инструменты с русским интерфейсом для оффлайн-бизнеса
4. Стараемся минимизировать ежемесячные расходы клиента на старте
