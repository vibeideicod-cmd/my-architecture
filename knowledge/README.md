# База знаний команды

Здесь хранится всё, что помогает работать быстрее и стабильнее: типы клиентов, стандарты дизайна, правила текстов, инструменты.

Агенты обращаются сюда перед работой над задачей.

---

## Структура

### /clients — всё о клиентах
- [client-types.md](clients/client-types.md) — профили типичных клиентов: санаторий, магазин, сервис
- [intake-questions.md](clients/intake-questions.md) — вопросы для первичного брифинга
- [typical-problems.md](clients/typical-problems.md) — типичные боли оффлайн-бизнеса
- [ai-solutions.md](clients/ai-solutions.md) — AI-решения по типам клиентов и готовые AI-продукты

### /standards — стандарты работы
- [design-system.md](standards/design-system.md) — цвета, шрифты, кнопки, карточки, адаптив
- [html-rules.md](standards/html-rules.md) — правила вёрстки
- [content-voice.md](standards/content-voice.md) — тон и стиль текстов
- [post-types.md](standards/post-types.md) — три типа постов (экспертный, вовлекающий, продающий) с промт-шаблонами
- [content-plan.md](standards/content-plan.md) — система контент-планирования: формула 40/30/30, триггеры, 6-шаговый промт, 4-недельная структура
- [quality-checklist.md](standards/quality-checklist.md) — чеклист QA перед сдачей

### /tools — инструменты
- [tech-stack.md](tools/tech-stack.md) — что используем и когда
- [workflow.md](tools/workflow.md) — как устроен рабочий процесс
- [ai-tools.md](tools/ai-tools.md) — AI-инструменты по категориям: текст, картинки, видео, аналитика, автоматизация
- [skills-catalog.md](tools/skills-catalog.md) — каталог скиллов: /brief, /discovery, /deploy, /seo + внешние
