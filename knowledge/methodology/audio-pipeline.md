# Audio pipeline — транскрибация аудио через Deepgram

Общая методика для трёх скиллов, которые работают с аудио:

| Слой | Скилл | Что делает |
|---|---|---|
| 1 | [`/transcribe`](../../.claude/skills/transcribe/SKILL.md) | Аудио → текст. Без анализа. Сырой транскрипт в склад |
| 2 | `/audio-debrief` *(в разработке)* | Разбор готового транскрипта на блоки → раскладка артефактов в N мест |
| 3 | `/client-onboarding-audio` *(в разработке)* | Транскрипт + жёсткая раскладка по 6 файлам нового клиента |

Все три используют общий конвейер `ffmpeg → Deepgram REST API`, описанный ниже.

---

## Зачем это нужно

Раньше Инна транскрибировала аудио вручную через сайт `playground.deepgram.com` — загружала файл, копировала текст, складывала. Этот пайплайн автоматизирует ручной процесс: положила файл в `inbox-inna/audio/` → получила готовый `.md` транскрипт.

## Архитектура потоков

```
ИСТОЧНИКИ АУДИО                    ОБРАБОТКА                    СКЛАД ТРАНСКРИПТОВ
──────────────                     ─────────                    ──────────────────
Zoom-встречи (mp4/m4a)             ffmpeg → wav 16k mono        transcripts/<дата>-<тема>.md
                                   ↓                            (общий склад, если без клиента)
Telegram/WhatsApp голосовые        Deepgram REST API
(ogg/opus)                         ↓                            clients/<имя>/transcripts/<дата>.md
                                   JSON-ответ                   (если указан клиент)
Диктофон телефона (m4a/mp3)        ↓
                                   Парсинг → .md                inbox-inna/audio/  ← исходник
YouTube видео (mp4 после curl)                                  остаётся для переслушивания

Аудио клиента (Zoom/voice)
```

Аудио-файлы (тяжёлые) — в `.gitignore`, транскрипты (`.md`, лёгкие) — коммитим.

## Ключ Deepgram — Связка ключей macOS

Ключ хранится в Связке ключей под именем `deepgram-api-key` (по аналогии с `claude-backup-key`, см. auto-memory `reference_backup_setup.md`).

**Извлечение в bash:**
```bash
DEEPGRAM_API_KEY=$(security find-generic-password -s deepgram-api-key -w)
```

Если команда выдала пусто или ошибку — ключ не положен. Скилл должен на этом остановиться и попросить Инну положить ключ через Keychain Access (Файл → Создать новый элемент пароля → Имя `deepgram-api-key`, в поле «Пароль» вставить значение).

**Никогда:**
- Не писать ключ в репо, в `.env`, в `.claude/settings.json`
- Не показывать ключ в чате (даже фрагмент)
- Не коммитить файл с упоминанием конкретного значения

## Шаг 1 — Подготовка аудио через ffmpeg

Deepgram принимает большинство форматов напрямую (mp3, wav, m4a, ogg, opus, flac, mp4 с аудиодорожкой). Но конвертация в `wav 16kHz mono` снижает размер запроса в 2-5 раз и улучшает точность распознавания на длинных файлах.

**Универсальная команда:**
```bash
ffmpeg -i "$AUDIO_INPUT" -vn -ar 16000 -ac 1 -c:a pcm_s16le -y "$AUDIO_WAV"
```

| Флаг | Зачем |
|---|---|
| `-i "$AUDIO_INPUT"` | Входной файл (любого формата) |
| `-vn` | Отбросить видеодорожку (для mp4 Zoom-записей) |
| `-ar 16000` | Сэмплрейт 16 кГц — оптимально для STT |
| `-ac 1` | Моно — Deepgram-ам канала достаточно |
| `-c:a pcm_s16le` | Несжатый PCM 16-bit |
| `-y` | Перезаписывать без вопросов |

Промежуточный файл — в `/tmp/`, чтобы не загромождать репо: `/tmp/audio-$(date +%s).wav`.

## Шаг 2 — Запрос к Deepgram REST API

**Эндпоинт:** `https://api.deepgram.com/v1/listen` (prerecorded — для готовых файлов).

**Базовая команда:**
```bash
curl -sS -X POST "https://api.deepgram.com/v1/listen?model=nova-3&language=ru&punctuate=true&smart_format=true&paragraphs=true&diarize=true&utterances=true" \
  -H "Authorization: Token $DEEPGRAM_API_KEY" \
  -H "Content-Type: audio/wav" \
  --data-binary @"$AUDIO_WAV" \
  -o /tmp/deepgram-response.json
```

**Параметры query:**

| Параметр | Значение | Зачем |
|---|---|---|
| `model` | `nova-3` | Топовая модель Deepgram (точность для русского лучше, чем у nova-2) |
| `language` | `ru` | По умолчанию у нас русский. Если аудио на английском — `en`. Если смешанный — `multi` |
| `punctuate` | `true` | Расставит точки/запятые/вопросы — без этого текст сплошняком |
| `smart_format` | `true` | Числа, даты, валюты в человекочитаемом виде («2026 год», «5 тысяч рублей»), не «две тысячи двадцать шесть») |
| `paragraphs` | `true` | Разбивка на абзацы по смыслу — удобно читать |
| `diarize` | `true` | Метки спикеров (Speaker 0, Speaker 1...) — критично для встреч с несколькими участниками |
| `utterances` | `true` | Разбивка на отдельные высказывания с таймкодами |

**Стоимость:** Nova-3 примерно $0.0043/мин на момент 2026-05. Free tier $200 кредитов = ~770 часов транскрибации. Для 1-часовой Zoom-встречи — ~$0.26.

## Шаг 3 — Парсинг JSON-ответа

Deepgram возвращает структуру:
```json
{
  "results": {
    "channels": [{
      "alternatives": [{
        "transcript": "Полный текст одной строкой",
        "paragraphs": {
          "transcript": "Текст с абзацами\n\nВторой абзац"
        }
      }]
    }],
    "utterances": [
      {"speaker": 0, "transcript": "Первая фраза", "start": 0.5, "end": 3.2},
      {"speaker": 1, "transcript": "Ответ", "start": 3.5, "end": 6.1}
    ]
  }
}
```

**Извлечение через `jq` (стандартно стоит на macOS):**

| Что нужно | Команда |
|---|---|
| Полный текст с абзацами | `jq -r '.results.channels[0].alternatives[0].paragraphs.transcript' /tmp/deepgram-response.json` |
| Текст с метками спикеров | `jq -r '.results.utterances[] \| "Speaker \(.speaker): \(.transcript)"' /tmp/deepgram-response.json` |
| Длительность аудио | `jq -r '.metadata.duration' /tmp/deepgram-response.json` |

**Если в аудио один спикер** (диктофон, голосовое сообщение) — `paragraphs.transcript` достаточно.
**Если несколько спикеров** (Zoom-встреча, разговор с Ириной) — используем `utterances` с метками `Speaker 0/1/2`.

## Шаг 4 — Сохранение транскрипта в .md

**Шаблон файла:**
```markdown
---
дата: 2026-05-08
источник: <имя_файла>
длительность: <минуты>
формат: <input_ext>
спикеров: <число>
---

# <тема — заполняется при сохранении>

## Саммари

<5-10 строк выжимки: о чём речь, кто говорит, ключевые факты, главные решения>

## Транскрипт

<полный текст: для одного спикера — paragraphs, для нескольких — utterances со Speaker 0/1...>
```

**Имя файла:** `<YYYY-MM-DD>-<источник>-<тема>.md`. Источник — короткая метка (`ирина`, `mastergroup`, `ученики`, `<имя_клиента>`). Тема — 2-4 слова сути.

**Куда сохраняется:**
- Если Инна указала клиента → `clients/<имя>/transcripts/<имя_файла>`
- Иначе → `transcripts/<имя_файла>`

## Обработка ошибок

| Ошибка | Что значит | Что делать |
|---|---|---|
| `security find-generic-password` пусто | Ключ не положен в Связку | Стоп, попросить Инну положить ключ |
| `ffmpeg: command not found` | ffmpeg не установлен | Стоп. Команда для Инны: `brew install ffmpeg` |
| Deepgram ответил `401` | Неверный ключ | Проверить корректность значения в Keychain Access |
| Deepgram ответил `400` | Файл повреждён или неподдерживаемый формат | Показать Инне ответ Deepgram, попросить другой файл |
| Деpegram ответил `402` | Закончились кредиты | Стоп, попросить Инну пополнить баланс |
| `paragraphs.transcript` пусто | Аудио тихое или без речи | Показать Инне `metadata.duration` и `confidence`, спросить хочет ли продолжить |
