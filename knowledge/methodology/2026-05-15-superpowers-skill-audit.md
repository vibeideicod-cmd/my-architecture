## Проверка по протоколу Skill-Auditor

### 1. Триггеры

Статус: чисто после уточнения.

`/superpowers` включается для инженерных задач: код, баги, деплой, API, боты, Mini App, AI-консультанты, клиентские цифровые продукты.

Уточнение внесено: чистый статичный лендинг без JS, форм, калькуляторов, квизов и риска клиентского продакшна не запускает mini/full-процесс; для него обязателен smoke test + обычный QA.

### 2. Антитриггеры

Статус: чисто.

Не включается как полный процесс для текстов, постов, сортировки входящих, исследований без реализации, простых правок документации и личной операционной переписки.

### 3. QA-overlap

Статус: чисто.

Канон явно говорит: «передай в QA» = один вызов `agents/qa.md`. В `agents/qa.md` уже есть Шаг 3.5 для Superpowers. Двойной QA не нужен.

### 4. Тесты vs сценарии

Статус: чисто после правки `systems.md`.

Канон и `systems.md` согласованы:
- желательный максимум — failing test;
- обязательный минимум — воспроизводимый сценарий;
- если тестов нет — это техдолг, фиксируется в `project-log.md`.

### 5. Handoff

Статус: чисто.

Цепочка закрыта:

`Director → /superpowers → Product Builder/Websites/Systems/AI Builder → QA → DevOps/Git → project-log или memory`

### 6. Измеримость

Статус: чисто.

Есть метка `#superpowers-applied` с полями:
- режим;
- задача;
- помог;
- мешал;
- проверка.

Этого достаточно для калибровки 2026-05-22 и sunset-аудита 2026-06-15.

### 7. Failure mode

Статус: чисто.

Если Superpowers мешает, Director может перейти в `off`/vibe-режим и обязан зафиксировать, почему методология была лишней. Это защищает систему от ритуала ради ритуала.

## Найденные проблемы и исправления

### 🟡 Важно — статичные лендинги могли трактоваться двояко

Проблема: канон говорил «новый цифровой продукт: сайт, лендинг...» как обязательный триггер, а `routing-table.md` уже уточнял, что для чистой статичной вёрстки нужен только smoke test.

Решение: канон и локальный скилл уточнены:
- mini/full включается для сайта/лендинга только при изменении поведения, JS, формах, калькуляторах, квизах, Mini App или клиентском продакшне;
- чистая статика = smoke test + обычный QA.

### 🟢 Рекомендация — глоссарий живёт только в каноне

Глоссарий оставлен в `knowledge/methodology/superpowers-integration.md`, а не продублирован в `SKILL.md`, чтобы скилл не разрастался. Это нормально: `SKILL.md` ссылается на канон как источник правды.

Если Инна начнёт часто спрашивать термины во время работы, можно добавить в `SKILL.md` короткий раздел «если термин непонятен — смотри глоссарий в каноне».
