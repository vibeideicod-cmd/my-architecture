# Стандарт исследовательской работы

Универсальный стандарт для всех агентов-исследователей: [analyst](../../agents/analyst.md), [analytics-head](../../agents/analytics-head.md), [audience-researcher](../../agents/audience-researcher.md), [product-analyst](../../agents/product-analyst.md). И для скиллов которые делают research: `/competitor-research`, `/market-research`, `/social-research`, `/digital-audit`, `/client-audit`.

## Правило 1 — факт vs предположение

Каждое утверждение в исследовании должно быть помечено по статусу:

- **Подтверждено** — есть конкретный источник (URL, документ, кейс, цифра)
- **Предположение** — логичный вывод, но без прямого источника
- **Требует проверки** — нужны дополнительные данные, прежде чем опираться на это

**Никогда не выдавай предположение за факт.** Если не уверен — пометь.

Пример формулировки:
- Подтверждено: «По данным <источник>, средний чек в нише — 8 000₽»
- Предположение: «Целевая аудитория готова платить 5-7 000₽, исходя из похожих ниш — требует проверки на интервью»
- Без пометки: «Аудитория платит 5-7 000₽» — так писать нельзя, это выдача предположения за факт

## Правило 2 — противоречие источников

Если два источника говорят разное — **указывай оба варианта**, не выбирай молча.

Формулировка:
> «По данным <источник 1>, рынок растёт на 15% в год. По <источник 2>, рост 5%. Вероятная причина расхождения — разные определения сегмента. Рекомендую опираться на источник 1, потому что его методология ближе к нашему сегменту.»

Молчаливый выбор одного варианта = скрытое предположение под видом факта.

## Правило 3 — структура ответа

Любой исследовательский результат — в этой структуре:

1. **Факты** — что получили из источников (с пометками подтверждено/предположение)
2. **Источники** — где взяли (URL, документ, имена, даты)
3. **Выводы** — что данные значат для нашей задачи
4. **Рекомендации** — что делать

Это не четыре строки — это четыре отдельных блока. Не сваливай факты и выводы в один абзац.

## Правило 4 — уточнение темы одним вопросом

Перед стартом research:
- Один уточняющий вопрос если тема нечёткая
- Не анкета из 7 пунктов
- Если непонятно вообще ничего — задай вопрос про **главное**, не про всё сразу

Если контекст есть в [inna/architecture.md](../../inna/architecture.md), `clients/{имя}/`, [knowledge/](../) — сначала прочитай, потом спрашивай только то чего нет.
