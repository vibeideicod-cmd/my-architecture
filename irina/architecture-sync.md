## 🚦 КРИТИЧНЫЕ ПРАВИЛА ПЕРЕНОСА (соблюдаются в любой итерации)

1. **Партнёрство Инна↔Ирина НЕ упоминается** в публичной части ни в одном Слое Ирининой архитектуры. Симметричного раздела «Партнёр: Инна» у Ирины быть НЕ должно
2. **Vibe coding / Claude Code / Cursor / нейросети как инструменты** Ирины — НЕ упоминаются. У Ирины — графические редакторы (Figma и др.), редакторы вёрстки и текстов
3. **8 ниш у Ирины = 8 ниш Инны** (1в1)
4. **Зеркальные ОКВЭД-формулировки** — у Инны «аудит → рекомендации → сопровождение внедрения», у Ирины «изготовление дизайна / разработка веб-приложения / разработка чат-бота» (через виды деятельности самозанятой)
5. **У Ирины 1 контур работы** (своя коммерческая практика). НЕ переносить раздел «1.4 Три контура работы» из master Инны
6. **У Ирины НЕТ кураторской системы** (Mastergroup) — это Контур 2 Инны
7. **Подбренд «Нейро Бабки»** у Ирины переосмысляется — это её собственный творческий актив, не подбренд Инны (Фаза 2)
