# Архив — возможно удалить (создан 11.05 v1)

> **Создал:** Director, 2026-05-11, по правке Инны в аудите my-architecture.
> **Назначение:** место для документов, которые **возможно** устарели / дублируются / отвергнуты. Финальное решение «удалять / вернуть / оставить в архиве» принимает **только Инна** после ручного разбора каждого файла.

---

## Главное правило

🛑 **НИЧЕГО НЕ УДАЛЯЕМ.** Document сюда переезжает командой `git mv` (с сохранением истории), не `rm`. Из архива в основной репо тоже только `git mv`, не «копировать и удалить».

🛑 **Удаляет только Инна, по факту разбора.**

✅ **Папка не пустеет автоматически.** Если все документы изучены и возвращены / решены — она остаётся пустой, на случай следующих архивных кандидатов.

---

## Как заполнять (когда придёт фаза «двигаем сюда устаревшее»)

Каждый перенос — отдельная строка в таблице ниже + директива в файл `notes.md` (если содержит критичный контекст).

| Дата | Файл | Откуда (полный путь) | Причина | Возможный возврат | Решение Инны |
|---|---|---|---|---|---|
| _пусто, документов пока не перемещали_ | | | | | |

**Столбцы:**
- **Дата** — когда перенесено, формат `ДД.ММ`
- **Файл** — имя файла или папки в архиве
- **Откуда** — путь до перемещения, чтобы можно было вернуть на место
- **Причина** — кратко: «устарел», «отвергнут», «дубль», «черновик», «версия заменена»
- **Возможный возврат** — что в этом документе может быть полезным (один-два пункта). Это чтобы Инна при разборе понимала, что не зря лежит
- **Решение Инны** — заполняется когда она просмотрит: «удалить» / «вернуть в `<путь>`» / «оставить в архиве»
