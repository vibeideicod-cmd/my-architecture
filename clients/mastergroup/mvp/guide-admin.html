Инструкция по МГ-админке

Как модерировать анкеты, контролировать потоки и где что искать. Если что-то забыл — возвращайся сюда.

📌 Быстрые ссылки
  • 🔑 Админка /mg/admin (пароль nbcccp-2026)
  • 🏠 Публичный лендинг /mg/
  • 📝 Анкета /mg/join (что видит мастер)
  • 📘 Гайд для мастера /mg/guide — шли эту ссылку мастерам

Три точки входа в систему

Платформа работает для трёх типов пользователей — каждому нужно своё.

1. Мастер (участник МГ)

Подаёт анкету → ждёт одобрение → собирает страницу в конструкторе → публикует → делится ссылкой с другими участниками МГ.

2. Участник мастера

Другой участник мастер-группы получает ссылку от мастера → открывает публичную страницу → нажимает кнопку записи → пишет в Telegram/WhatsApp или оставляет заявку через форму.

3. Ты (админ)

Контролируешь всё: модерируешь анкеты, следишь за собранными страницами, видишь заявки от участников — всё по адресу /mg/admin, вход по паролю.

🎛 Админка — 4 вкладки

🆕 Вкладка 1 — «Анкеты»

Очередь заявок от мастеров, которые подали форму через /mg/join. Realtime — новая анкета всплывает сама, без F5.

Что делаешь с карточкой:

  1. Клик на карточку — раскрываются все поля (имя, контакт, ниша, чего хочет от приложения)
  2. При желании — пишешь комментарий модератора
  3. Жмёшь ✅ Одобрить → откроется модалка с готовым сообщением для Telegram + кнопка «Скопировать»
  4. Или ❌ Отклонить с причиной — мастер увидит её у себя в статусе
💾 Ссылка на конструктор сохраняется в админке. Даже если закроешь модалку — в любой момент можно зайти во вкладку «📄 Страницы», нажать «📋 Ссылка на конструктор» и получить ту же самую ссылку. Мастер не теряет прогресс, если потерял ссылку — пришли её заново.

📄 Вкладка 2 — «Страницы»

Все страницы одобренных мастеров. Видишь статус: draft (ещё собирает) или published (опубликовано).

Зачем:

  • Смотришь, кто застрял на конструкторе и не допубликовал — можно написать и поддержать
  • Открываешь страницу и проверяешь качество перед тем, как мастер начнёт её распространять
  • Быстрый переход на публичную страницу любого мастера

📨 Вкладка 3 — «Заявки участников»

Заявки от участников мастер-группы — когда участник заполняет форму на публичной странице другого мастера (если этот мастер выбрал CTA = «форма на странице»). Realtime — новая заявка всплывает сама.

Что видишь по каждой заявке:

  • К какому мастеру пришёл участник
  • Имя и контакт
  • Сообщение
  • Время

Фильтр сверху — можно смотреть заявки только по одному мастеру.

Зачем тебе это: ты видишь весь поток записей внутри мастер-группы. Если у мастера пошли записи от других участников — ты это увидишь первой. Можешь прямо из админки сообщить мастеру: «тебе пришла заявка от [имя], контакт такой-то».

📘 Как работать с гайдом для мастера

В шапке админки три кнопки:

  • 📘 Гайд для мастера — открывает публичный гайд в новой вкладке. Посмотри сама что там написано
  • 📋 Скопировать ссылку на гайд — кладёт URL гайда в буфер обмена. Вставляешь в Telegram мастеру перед тем как он сядет за конструктор
  • 📕 Моя инструкция — открывает эту самую страницу

Рекомендованный сценарий: когда одобряешь анкету, в Telegram-сообщении для мастера к ссылке на конструктор добавляй ещё и ссылку на гайд:

«Привет, [имя]! Твоя анкета одобрена 🎉 Вот ссылка на конструктор: [...]. А вот пошаговая инструкция — пройди перед тем как садиться: [гайд]. 20 минут — и у тебя живая ссылка.»

🔄 Типичные сценарии

Новая анкета — что делать

  1. Открываешь админку → вкладка «Анкеты» (уже открыта по умолчанию)
  2. Видишь новую карточку (всплывает сама без F5)
  3. Кликаешь → читаешь ответы
  4. Жмёшь «Одобрить» → копируешь готовое сообщение → вставляешь мастеру в Telegram

Мастер потерял ссылку на конструктор

Одним кликом — вкладка «📄 Страницы» → находишь мастера → кнопка 📋 Ссылка на конструктор. В буфер скопируется готовое сообщение для Telegram с той же самой ссылкой, которую отправляли изначально. Ничего не теряется — всё, что мастер уже заполнил в конструкторе, остаётся на месте.

Как это работает: raw-токен хранится в колонке token_plain таблицы mg_master_pages, доступен только через admin RPC mg_admin_get_master_link. Для старых страниц, созданных до этого апдейта, кнопка предложит перевыпустить токен — старая ссылка при этом перестанет работать.

Хочешь посмотреть, как выглядит чья-то страница

Вкладка «Страницы» → находишь мастера → кнопка «Открыть» или «Превью» (для draft).

Участник написал тебе, а не мастеру

Зайди во вкладку «Заявки участников» → фильтр по мастеру → найди заявку. Если её нет — значит человек не проходил через форму на публичной странице. Скажи мастеру, пусть настроит CTA = Telegram/WhatsApp для прямых контактов или «форма на странице» для централизации.

🔐 Безопасность

  • Пароль админки: nbcccp-2026 (хранится как hash в RPC, в коде только фронтовый hash для локального gate)
  • Токены мастеров: в БД лежит и SHA-256 hash (для верификации), и raw-токен (для восстановления), raw доступен только через admin RPC
  • RLS: закрыта полностью, все операции идут через RPC security definer — прямой INSERT/UPDATE невозможен
  • Ключ Supabase в коде — публичный publishable key, ничего не даёт кроме вызова RPC

🆘 Если что-то сломалось

  • Анкета не всплывает realtime — обнови страницу (F5). Проверь что publication на mg_applications живой
  • Лид не всплывает realtime — то же самое, publication на mg_leads
  • RPC возвращает «Неверный админ-ключ» — sessionStorage сбросился, выйди и войди заново
  • Всё сломалось — пиши Кодычу, разберёмся
