📕 Моя инструкция

🛠A. Подготовка

Открыть Telegram на телефоне (или Desktop)
Открыть браузер на компьютере (для админки)
Подготовить тестовые данныеИмя: Тест Тестов · Контакт: @test_master · Ниша: Психология · Описание: «Помогаю людям находить баланс» · Цель: «Хочу одну ссылку для участников»

📝B. Тест роли «Мастер» — подача анкеты

Точка входа: t.me/mastergruppa_bot/mg_platform или /mg/join
Открыть Mini App t.me/mastergruppa_bot/mg_platform в TelegramДолжно открыться во весь экран с заголовком «Расскажи о себе и своём деле»
Видна ссылка «Подробная инструкция →» в описании — она ведёт на /mg/guide
Заполнить все обязательные поля (ФИО, контакт, ниша, описание бизнеса, цель, согласие на ПД)
Нажать «Отправить анкету»
Увидеть экран «🌿 Спасибо, [имя]!» с ссылкой статуса и 4 способами сохранитьВ Telegram Mini App должна быть золотая кнопка «📥 Сохранить в Telegram» — остальные тоже видны
Скопировать ссылку на статус (нажать 📋 или 📥)
Открыть ссылку статуса — увидеть «⏳ На рассмотрении»

🔑C. Тест роли «Админ» — модерация

Точка входа: /mg/admin · Пароль: nbcccp-2026
Открыть /mg/admin, ввести пароль nbcccp-2026
Вкладка «🆕 Анкеты» — видна тестовая анкета (должна всплыть без F5 — realtime)Если не всплыла — обнови страницу F5. Если и после этого нет — проблема с realtime publication
Кликнуть на карточку анкеты — раскрываются все поля (контакт, ниша, бизнес, цель)
Нажать «✅ Одобрить» → увидеть модалку с ссылкой на конструктор
Нажать «📋 Скопировать готовое сообщение для Telegram» — проверить что скопировалосьДолжно быть: «Привет, [имя]! Твоя анкета одобрена 🎉 Вот ссылка на конструктор...»
Закрыть модалку. Перейти на вкладку «📄 Страницы» — увидеть тестового мастера со статусом «Черновик»
Нажать «📋 Ссылка на конструктор» у мастера — проверить что сообщение скопировалось
В шапке: кнопки «📘 Гайд для мастера», «📋 Скопировать ссылку на гайд», «📋 Скопировать ссылку на анкету», «📕 Моя инструкция» — все работают

🏗D. Тест роли «Мастер» — конструктор

Точка входа: ссылка из модалки одобрения (или из вкладки «Страницы» → «📋 Ссылка на конструктор»)
Открыть ссылку на конструктор из скопированного сообщенияДолжен открыться /mg/build?m=master-N&t=TOKEN с прогресс-баром и первым блоком «Кто ты»
Блок 1 — Hero: заполнить имя, фразу о деле. Фото можно пропустить.Прогресс-бар обновляется. Нажать «Далее →»
Блок 2 — О тебе: написать 2-3 предложения биоНажать «Далее →»
Блок 3 — Предложение: название оффера, описание, 2-3 причины выбратьНажать «Далее →»
Блок 4 — Связь: выбрать «Форма на странице» (чтобы потом протестить заявку участника)Написать текст кнопки, например «Записаться»
Нажать «Посмотреть, как выглядит» — откроется превью страницы в новом окне
Нажать «Опубликовать»Должен появиться экран «🎉 Готово! Твоя страница живёт» с публичной ссылкой
Открыть публичную страницу /mg/m/master-N — проверить что всё отображается: имя, фраза, био, предложение, кнопка CTA

👤E. Тест роли «Участник» — запись к мастеру

Точка входа: публичная ссылка мастера /mg/m/master-N
Открыть публичную страницу мастера /mg/m/master-N
Проверить: имя, фраза, описание, предложение, 3 причины — всё на месте
Нажать CTA «Записаться» → должна развернуться форма (имя, контакт, сообщение)
Заполнить тестовую заявку → нажать «Отправить»Должно появиться: «🌿 [Имя мастера] получит твою заявку и скоро свяжется!»
Вернуться в /mg/admin → вкладка «📨 Заявки участников» → увидеть новую заявку (realtime)
Проверить фильтр: выбрать мастера в dropdown — показывает только его заявки

📱F. Мобильный тест

Открыть /mg/join на телефоне (375px) — форма читаемая, кнопки тапабельные
Открыть публичную страницу мастера на телефоне — всё на месте, кнопка CTA крупная
Открыть /mg/admin на телефоне — базовая работоспособность (вкладки, карточки)

✅G. Финальная проверка

Все чистые URL работают: /mg/ /mg/join /mg/admin /mg/guide /mg/help
Старые URL недоступны: /mg/vitrina.html → 404, /mg/cabinet.html → 404
Нигде нет слова «клиент» — только «участник»
Нигде в UI нет имени «Инна» — только «команда МГ» / «админ»
Гайд для мастера (/mg/guide) — открывается, актуален, контакт команды внизу
Нет ошибок в консоли браузера (F12 → Console)
Удалить тестовые данные из Supabase (если не нужны для демо)Table Editor → mg_applications → удалить тестовую строку. То же в mg_master_pages и mg_leads
