# Проект: МГ — платформа мастер-групп

## Контекст

- **Название:** МГ-платформа (внутренний продукт Нейро Бабки СССР)
- **Задача:** Заменить Telegram как место жизни мастер-группы. Браузерная платформа, где каждый мастер ведёт свою 5-недельную группу — витрина, кабинет, участники, материалы, задания, кейсы, чат.
- **Почему:** Telegram блокируется, участники теряются, связь рвётся. Нужна независимая площадка, доступная везде где есть интернет.
- **Этап:** MVP — Supabase + чистый HTML, без фреймворков.

## Три интерфейса

1. **Витрина мастера** (`vitrina.html?m=<slug>`) — публичная страница, форма записи.
2. **Кабинет мастера** (`cabinet.html?m=<slug>&t=<token>`) — приватный, гейт по токену (SHA-256 hash).
3. **Админ-панель** (`admin.html`) — Инна видит всех мастеров, участников, кейсы.

## Стек

- Frontend — HTML + inline CSS + Vanilla JS, Supabase JS v2 через CDN.
- Backend — Supabase (те же project `my-architecture`, pooler eu-west-2).
- Realtime — `supabase_realtime` publication на `mg_participants` и `mg_messages`.
- Hosting — Beget, `demo.ideidlyabiznesa1913.ru/mg/`.

## Важно

- Все таблицы начинаются с `mg_` чтобы не конфликтовать с beauty.
- Secret-токен кабинета хранится как SHA-256 хэш — сырой токен никогда не попадает в БД.
- RLS открыта широко для MVP — публичное чтение/вставка. UPDATE/DELETE запрещены.
