# Brief: МГ-платформа — конструктор бизнес-визиток

**Фаза:** 3 — Design / Plan.
**Playbook:** [product-creation.md](../../knowledge/playbooks/product-creation.md) раздел 2/Фаза 3.
**Критерий готовности:** Кодыч/Системщик читают этот файл + `BACKEND-PLAN.md` и могут стартовать Build без уточнений.

---

## Обзор

Добавляем в существующий v1 МГ-платформы **поток анкета → модерация → конструктор → публикация**, не снося существующие файлы. Результат — каждая участница мастер-группы УБ может собрать свою бизнес-визитку за 20 минут и получить рабочую ссылку.

**Вход в систему для нового мастера:**
```
Лендинг /mg/ → «Стать мастером» → apply.html → отправка →
экран «сохрани ссылку» → ожидание одобрения →
ссылка в Telegram от Инны → build.html (конструктор) →
8 вопросов → Опубликовать → published-экран с ВАУ →
мастер-страница /mg/m/<slug>
```

## 1. Экраны v2

### 1.1 `index.html` (обновление)

**Что было:** лендинг МГ-платформы с описанием «три интерфейса» и демо-ссылками на Инну.

**Что меняется:**
- Новая секция в начале после hero: **«Стать мастером»** с крупной CTA «Подать заявку → apply.html»
- Подзаголовок: «Собери свою бизнес-визитку за 20 минут — одна ссылка, никаких блокировок»
- Переработанный hero с акцентом на «обойди блокировки мессенджеров»
- Старая секция с демо-ссылками (Инна как мастер) сохраняется ниже — для обратной совместимости

**Что не трогаем:** существующую логику загрузки, брендбук, footer.

### 1.2 `apply.html` (новый)

**Цель:** форма анкеты кандидата в мастера. Результат — сохранённая анкета + экран «сохрани ссылку статуса».

**Поля анкеты** (помечены `*` — обязательные):

1. ФИО `*`
2. Контакт (Telegram @username / телефон / email — одно поле, подсказка «любой, куда с тобой связаться») `*`
3. Город
4. Ниша твоего бизнеса `*` (select: музыка / стиль и красота / психология / эзотерика / тело и здоровье / творчество / коучинг / бизнес-консультирование / образование / другое)
5. Чем ты занимаешься в своём бизнесе `*` (textarea, подсказка «2-3 предложения простым языком»)
6. Что ты делаешь в мастер-группе УБ (textarea, optional — «это для контекста, можно пропустить»)
7. Чего ты хочешь от своего приложения `*` (textarea, «что должно произойти после того как ты покажешь это клиенту?»)
8. Опыт ведения своего бизнеса (select: меньше 1 года / 1-3 года / больше 3 лет)
9. Согласие на обработку персональных данных `*` (checkbox, нельзя отправить без)

**Кнопка «Отправить»** — вызывает RPC `mg_submit_application(payload)`, получает обратно `{master_number, application_token}`.

**Экран «Анкета отправлена»** (после успешного submit):

```
🌿 Спасибо, [имя]! Заявка принята.

Инна скоро с тобой свяжется — обычно это пара часов.

⚠️ ВАЖНО: сохрани эту ссылку — она нужна, чтобы открыть твою страницу позже:

[большое поле с URL статуса, выделяется при клике]

Как сохранить:
  [📋 Скопировать ссылку]
  [✉️ Отправить себе в Telegram]   (tg://msg?text=...)
  [💾 Скачать как файл]              (Blob + download)
  [📱 QR-код]                        (inline SVG, открыть на телефоне)
```

**Примечание:** номер мастера НЕ показывается в UI (см. `critique.md` замечание 1). Живёт только в БД.

### 1.3 `status.html?a=<token>` (новый)

**Цель:** кандидат может проверить статус своей анкеты.

**Состояния:**

| Статус БД | Что видит кандидат |
|---|---|
| `submitted` | «⏳ На рассмотрении. Инна проверяет твою заявку. Обычно это пара часов, максимум день.» |
| `under_review` | «👁 Инна сейчас смотрит твою анкету» (переходное состояние) |
| `approved` | «✅ Ура! Одобрено. Вот ссылка на конструктор твоей страницы: [большая кнопка build.html]» |
| `rejected` | «❌ К сожалению, сейчас не получилось. Причина: [moderation_note]. Можешь подать анкету ещё раз.» |

Дополнительно — если approved, показываем также кнопку «Отправить ссылку конструктора мне в Telegram».

### 1.4 `build.html?m=<slug>&t=<token>` (новый)

**Цель:** конструктор страницы мастера. Главная ВАУ-фича.

**Поведение:**
1. При загрузке: читает `slug` и `t` из URL, хэширует токен через SHA-256, сверяет с `token_hash` в `mg_master_pages`. Если несоответствие — ошибка «неверный токен».
2. Читает `page_config` из `mg_master_pages` + справочник вопросов из `mg_config_questions`
3. Рендерит step-wizard с прогресс-баром

**Структура экрана:**
- Шапка: «Собери свою страницу» + прогресс-бар (0-100%)
- Центральная часть: текущий блок вопросов
- Кнопки «← Назад» / «Далее →» (стилизованы по брендбуку)
- Справа (на десктопе) — sticky preview с текущим состоянием страницы. **Бонусная фича**, если времени хватит
- На мобильном: кнопка «Посмотреть, как выглядит» рядом с текущим блоком — открывает `master-page.html?m=<slug>&preview=1` в новом окне

**4 блока вопросов:**

**Блок 1 — Hero (шаг 1 из 4)**
1. **Как тебя зовут?** (`text`, required) — «Отображается на странице большими буквами»
2. **Одна фраза о твоём деле** (`text`, required, max 80 символов) — «Школа гитары для взрослых в центре Москвы»
3. **Ссылка на твою фотку** (`url`, optional) — «Самый простой способ: Яндекс.Диск → Поделиться → Скопировать ссылку. Можно пропустить — поставим кружок с первой буквой твоего имени»

**Блок 2 — О тебе (шаг 2 из 4)**
4. **Расскажи о себе** (`textarea`, required) — «2-3 абзаца, всё что важно знать клиенту. Пример: «Играю на гитаре 15 лет, преподаю 7. Занималась в Гнесинке. Веду индивидуальные уроки и группы в центре Москвы. Помогаю взрослым учиться без страха и быстро.» Счётчик слов мелко справа, без жёсткого лимита.

**Блок 3 — Твоё предложение (шаг 3 из 4)**
5. **Название твоего предложения** (`text`, required) — «Например: Первое занятие бесплатно / Бесплатная диагностика / Пробное занятие»
6. **Опиши предложение** (`textarea`, required) — «Что именно ты предлагаешь клиенту, который кликнул на страницу. Пример: «Приходи на первый урок, это 60 минут, бесплатно. Попробуем твою гитару, посмотрим что умеешь, подберём программу если зайдёт».»
7. **3 причины выбрать тебя** (`list`, минимум 2 строки, каждая до 60 символов) — «По одной строке. Пример: «Индивидуальный подход / Любой уровень / Удобное время»»

**Блок 4 — Как связаться (шаг 4 из 4)**
8. **Куда написать клиенту, который хочет записаться?** (`cta_choice`, required)
   - Radio 1: **Telegram @username** → поле ввода «@alisa_guitar»
   - Radio 2: **WhatsApp номер** → поле ввода «+7 999 ...»
   - Radio 3: **Форма на странице** (клиент оставит имя и контакт, ты увидишь в своём кабинете) → без поля ввода

После блока 4 — экран **«Готово, публикуем?»** с превью полной страницы + кнопкой «Опубликовать».

**Кнопка «Опубликовать»** — вызывает RPC `mg_publish_page(slug, token)`. Проверяет что обязательные поля заполнены (1, 2, 4, 5, 6, 8). Возвращает публичную ссылку.

### 1.5 Экран «Опубликовано» (в `build.html` или отдельный `published.html`)

**После успешной публикации** — экран с четырьмя элементами:

```
🎉 Готово! Твоя страница живёт по этой ссылке:

[БОЛЬШОЕ ПОЛЕ С URL, выделяется при клике]
https://demo.ideidlyabiznesa1913.ru/mg/m/<slug>

  [📋 Скопировать ссылку]       (primary, золотая кнопка)
  [✉️ Отправить себе в Telegram] (вторичная)
  [📱 QR-код]                    (inline SVG)
  [👁 Посмотреть страницу]       (открывает master-page.html)

💬 Отправь ссылку первому клиенту — посмотри, как он отреагирует.
   Если что-то хочешь поправить — [вернуться к конструктору].
```

Это **главный ВАУ-момент для встречи УБ**.

### 1.6 `master-page.html?m=<slug>` (новый, публичный)

**Цель:** публичная страница мастера. Видят участники (клиенты) мастера.

**Особенности:**
- **НЕ видно** никаких номеров мастера, токенов, админ-кнопок
- Читает `page_config` из view `mg_master_pages_public` (где `published = true`)
- Рендерит секции по блокам из JSON:
  - **Hero:** `display_name`, `headline`, `photo_url` (или инициал)
  - **About:** `bio`
  - **Offer:** `offer_title`, `offer_desc`, `benefits[]`
  - **CTA:** большая кнопка `cta_text`, ведёт на `cta_link_value` (tg:// / tel: / форма)
- Если мастер выбрал «форма на странице» — вместо кнопки внешнего перехода разворачивается мини-форма (имя + контакт + сообщение), вызов RPC `mg_submit_lead`
- Поддерживает `?preview=1` — показать даже если `published = false`, но только с валидным токеном

**Дизайн:** брендбук Инны, адаптив 320-1440, мобильный first. На мобильном — секции одна под другой, крупные тапабельные кнопки.

### 1.7 `admin.html` (обновление)

**Что было:** SHA-256 gate, один таб с мастерами v1, CSV-экспорт.

**Что меняется:**
- Новый таб **«Анкеты»** (по умолчанию) — очередь анкет со статусом `submitted` + `under_review`. Карточки раскрываются, кнопки [Одобрить] [Отклонить]. Realtime: новая анкета всплывает без F5.
- Новый таб **«Собранные страницы»** — все `mg_master_pages`, фильтр по published/draft, ссылки на публичные страницы
- Существующие табы v1 (мастера, CSV-экспорт) **сохраняются** отдельной секцией «v1 (legacy)»

**Модерация:**
- Клик на карточку анкеты → разворачиваются все поля
- Текстовое поле «Комментарий модератора» (опционально)
- Кнопка [Одобрить] → RPC `mg_moderate(app_id, 'approved', note, admin_secret)` → возвращается `{slug, master_token}` → админка показывает модалку «Ссылка на конструктор для Алисы» с кнопкой «Скопировать готовое сообщение для Telegram» (текст: «Привет, Алиса! Твоя анкета одобрена. Вот ссылка на конструктор твоей страницы: [link]. У тебя 8 коротких вопросов — минут 20 работы. Успеха!»)
- Кнопка [Отклонить] → RPC `mg_moderate(app_id, 'rejected', note, admin_secret)` → анкета уходит из очереди

## 3. Что НЕ показываем в UI мастера

Из `critique.md`, замечание 1 + «Ты уверен?» 2.2:

- **Номер «Мастер N»** — только в админке Инны. Мастер не видит его в `status.html`, `build.html`, `published-экране`, `master-page.html`.
- **`application_token`** — в URL `status.html`, но не отображается визуально (query param в адресной строке). Мастер не должен его копировать/показывать.
- **`master_token`** (от кабинета/конструктора) — в URL `build.html`, но не отображается визуально. Мастер просто переходит по ссылке от Инны.
- **`slug`** — виден в URL `master-page.html?m=<slug>`, это нормально (публичный идентификатор).

## 4. Что видят участники (клиенты мастера)

Только `master-page.html?m=<slug>` (или `/mg/m/<slug>` если настроим clean URL). Никаких служебных полей. Никаких ссылок на админку.

На странице:
- Имя мастера
- Одна фраза (headline)
- Фото или инициал
- Текст «о себе»
- Предложение (title + desc + benefits)
- Одна большая кнопка CTA (переход на контакт мастера или форма)

## 5. Брендбук

Все новые экраны — строго в палитре Инны из [design-system.md](../../knowledge/standards/design-system.md):

- Фон страниц: `#ffffff` (белый)
- Тёплые секции/карточки: `#d3bfb1` (бежевый)
- Главный акцент/CTA: `#d4af37` (золотой)
- Hover/вторичный: `#c99700` (горчичный)
- Поддержка/success: `#4b7850` (тёмно-зелёный)
- Глубокий (заголовки): `#103206` (глубокий зелёный)
- Основной текст: `#4c4651` (серый)
- Эмоциональный акцент: `#ffc403` (жёлтый, эпизодически — например, confetti при публикации)

Шрифты: Montserrat (тело) + Playfair Display (заголовки).

## 6. Адаптивность

Все новые экраны работают на:
- Мобильный 320-430px (Алиса будет заполнять с телефона)
- Планшет 768-1024px
- Десктоп 1280-1920px

Мобильный first. Кнопки ≥44px по высоте. Шрифт body ≥14px. Никакого горизонтального скролла.
