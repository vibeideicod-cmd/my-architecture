## 1. Цель и аудитория

### Цель сайта

Лендинг HUB — центральная точка входа для клиентов команды "Нейро Бабки СССР".

**Главное целевое действие:** запись на бесплатную 30-минутную диагностику.  
**Вторичное действие:** написать напрямую в Telegram.

Лендинг должен:
- За 3–5 секунд ответить "это для меня?"
- Снять страх "непонятно что происходит и сколько стоит"
- Провести клиента от "зашёл" до "оставил контакт" за 1.5–3 минуты
- Работать как витрина всей архитектуры: HUB → MINI → SYSTEM

### Целевая аудитория

**Основная:** владельцы оффлайн-бизнеса 35–60 лет.
- Кафе, рестораны, кофейни
- Санатории, гостиницы, оздоровительные центры
- Розничные магазины
- Сервисные бизнесы: мастера, салоны, автосервисы

**Портрет клиента:**
- Скептик: уже был негативный опыт с подрядчиками
- Не знает терминов: "таргет", "воронка", "конверсия" — пустые слова
- Ценит время: нет терпения читать лонгриды
- Думает деньгами: первый вопрос — "сколько стоит?"
- Предпочитает звонок или Telegram — не формы

**Три главных вопроса клиента:**
1. Сколько это стоит?
2. Как быстро будет результат?
3. Вы уже такое делали?

## 2. Архитектура продукта (для понимания структуры лендинга)

```
HUB (лендинг-витрина)
  ├── MINI 1 — Первые клиенты (лидогенерация)
  ├── MINI 2 — Коммуникация (скрипты, шаблоны)
  ├── MINI 3 — Контент на 30 дней
  └── SYSTEM — Полный маркетинг под ключ

**MINI-модули** — для тех, кто хочет начать с одного направления.  
**SYSTEM** — флагман, для тех, кто хочет всё и на постоянной основе.

### [5] MINI-МОДУЛИ — Три продукта

**Цель:** Клиент узнаёт свою задачу в одном из модулей.

**Заголовок секции:**
```
MINI-модули — быстрый старт

```

```

**Три карточки:**

---

**MINI 1 — Первые клиенты**
_Лидогенерация_

Что делаем:
- Лид-магнит под вашу нишу
- Форма захвата + автоответ
- Воронка: увидел → оставил контакт → получил оффер

Результат: стабильный поток входящих заявок

CTA: `Начать с этого →`

---

**MINI 2 — Коммуникация**
_Скрипты и шаблоны_

Что делаем:
- Скрипты ответов (WhatsApp, Telegram, телефон)
- Шаблоны для частых ситуаций
- Базовая логика: кто, когда, что написал

Результат: менеджер тратит вдвое меньше времени на переписку

CTA: `Начать с этого →`

---

**MINI 3 — Контент на 30 дней**
_Тексты и план_

Что делаем:
- Контент-план на месяц
- 30 готовых постов под голос бренда
- Адаптация под нужную соцсеть

Результат: публикуете весь месяц, не думая "что написать"

CTA: `Начать с этого →`

### [7] ПРОЦЕСС — Как работаем

**Цель:** Снять страх "непонятно, что происходит после оплаты".

**Заголовок:**
```
Как это работает
```

**5 шагов (нумерованных):**

```
1. Диагностика (бесплатно)
   Разбираемся в вашем бизнесе: аудитория, конкуренты, что уже работает.

2. Стратегия
   Составляем план: что делаем, зачем, в каком порядке.

3. Запуск
   Начинаем с первого модуля или сразу с системы — по вашему выбору.

4. Аналитика
   Смотрим что работает, корректируем курс каждый месяц.

5. Рост
   Масштабируем то, что даёт результат.
```

**Акцент под шагами:**
```
Первый шаг — 30-минутная диагностика. Бесплатно.
```

**CTA под блоком:** `Записаться на диагностику`

---

**Как быстро будет результат?**
Первые данные — через 1–2 месяца. Устойчивый результат — через 3–6.
Маркетинг — это не таблетка, это система. Строим один раз — работает постоянно.

---

**Мы небольшой бизнес — нам это подойдёт?**
Именно для вас. Большие бюджеты не нужны — нужна последовательность.
MINI-модули созданы специально для старта без лишних вложений.

---

**Уже пробовали SMM — не сработало.**
Расскажите нам об этом на диагностике. Чаще всего дело не в канале,
а в отсутствии стратегии или неверно подобранной аудитории.

---

**Можно взять только один модуль?**
Да. Для этого есть MINI-модули — берёте один и смотрите результат.
Понравилось — добавляете следующий.

---

**Как вы отчитываетесь?**
Ежемесячный отчёт с цифрами и выводами.
Никаких "мы работаем, просто пока не видно".

---

**Можно остановить сотрудничество?**
Да. Работаем помесячно, без длинных контрактов и штрафов.

---

**Вы работаете дистанционно?**
Да, со всеми клиентами — из любого города.
Всё через мессенджеры, созвоны и документы онлайн.

## 5. Дизайн-система

### Цвета

Подбренд "Нейро Бабки СССР" — тёплая горчичная система.

```css
:root {
  /* Фоны */
  --bg:          #0a0a0a;   /* основной фон */
  --bg-alt:      #0d0a06;   /* тёплый тёмный — альтернативные секции */
  --bg-card:     #111111;   /* карточки */
  --bg-elevated: #1a1610;   /* поднятые элементы */

  /* Акценты */
  --accent:       #c99700;  /* горчичный — главный акцент */
  --accent-hover: #e6b800;  /* светлее при hover */
  --gold:         #d4af37;  /* золото — детали, иконки */

  /* Текст */
  --text:         #e8e0d8;  /* основной текст */
  --text-warm:    #d3bfb1;  /* тёплый бежевый — подзаголовки */
  --text-muted:   #a89e94;  /* вторичный текст */

  /* Границы */
  --border:       rgba(201, 151, 0, 0.2);
  --border-hover: rgba(201, 151, 0, 0.5);
}
```

**Принципы работы с горчичным:**
- Точечно: подчёркивание ключевого слова, иконки, hover-эффекты
- Не заливка больших блоков — акцент теряет силу
- Рядом работает: тёмно-коричневый, бежевый, медь
- Избегать рядом: синий, фиолетовый, чистый белый

### Шрифты

```css
/* Подключение */
@import url('https://fonts.googleapis.com/css2?family=Montserrat:wght@300;400;500;600;700&family=Playfair+Display:ital,wght@0,400;0,600;0,700;1,400&display=swap');

/* Применение */
h1, h2 { font-family: 'Playfair Display', serif; }
body, p, button { font-family: 'Montserrat', sans-serif; }
```

| Элемент | Шрифт | Вес | Размер |
|---------|-------|-----|--------|
| Hero заголовок | Playfair Display | 700 | clamp(2.4rem, 7vw, 5rem) |
| Заголовок секции | Playfair Display | 700 | clamp(1.8rem, 4.5vw, 3rem) |
| Подзаголовок | Montserrat | 400 | clamp(1rem, 2.2vw, 1.2rem) |
| Основной текст | Montserrat | 400 | clamp(0.9rem, 2vw, 1rem) |
| Метки / бейджи | Montserrat | 600, uppercase | 11px, letter-spacing: 0.12em |
| Кнопки | Montserrat | 700, uppercase | 13–14px, letter-spacing: 0.08em |

### Визуальное настроение

**Ключевое ощущение:** "Опытный специалист в тёплом кабинете". Не давит, не суетится, не кричит. Уверенность через сдержанность.

**Принципы:**
- Щедрые отступы — признак уверенности, не страх пустоты
- Asymmetry intentional — сетка `1.2fr 0.8fr`, не строгие 50/50
- Тёплые оттенки фона (`#0d0a06`) вместо технически-холодного (`#1a1a1a`)
- Serif в заголовках — зрелость, надёжность, история

**Атмосфера:** иронично-серьёзная. Юмор есть, но за ним всегда смысл и экспертиза.

## 7. Обязательные технические элементы

### Анимации (первый релиз — низкая сложность)

| Элемент | Анимация | Приоритет |
|---------|----------|-----------|
| Hero заголовок | Slide-up по словам при загрузке | Обязательно |
| Секции | Fade-up при появлении в viewport | Обязательно |
| Карточки | Stagger (0.1s задержка между) | Обязательно |
| Кнопки | Hover: translateY(-2px) + shadow | Обязательно |
| Навигация | Появление при прокрутке / backdrop-blur | Обязательно |
| Прогресс-бар | Горчичная полоска вверху страницы | Желательно |

### Анимации (второй релиз — средняя сложность)

| Элемент | Анимация | Когда |
|---------|----------|-------|
| Схема HUB→MINI→SYSTEM | Анимированное появление стрелок | V2 |
| Кейсы | Горизонтальный CSS Scroll Snap | V2 |
| Числа в кейсах | Счётчик при появлении | V2 |
| Марквиз | Бегущая строка с именами клиентов | V2 |

### Форма (финальный CTA)

```html
<form>
  <label for="name">Ваше имя</label>
  <input
    type="text"
    id="name"
    name="name"
    autocomplete="name"
    placeholder="Александра"
    required
  />

  <label for="contact">Телефон или Telegram</label>
  <input
    type="tel"
    id="contact"
    name="contact"
    autocomplete="tel"
    placeholder="+7 (999) 000-00-00"
    required
  />

  <button type="submit">Записаться на диагностику</button>
</form>
```

**Правила формы:**
- Метки `<label>` над полями — не placeholder (исчезает при вводе)
- `font-size: 16px` внутри полей — блокирует автозум в Safari
- Высота полей: минимум 52px
- После отправки: сообщение с именем — "Спасибо, [Имя]! Напишем в течение часа."

## 8. Адаптивность

```css
/* Мобильный (приоритет) */
@media (max-width: 480px) {
  /* Одна колонка. Кнопки на всю ширину. Шрифты через clamp. */
}

/* Планшет */
@media (min-width: 768px) {
  /* MINI-карточки 2 колонки */
}

/* Десктоп */
@media (min-width: 1024px) {
  /* MINI-карточки 3 колонки. Схема горизонтально. */
}
```

**Обязательные проверки на мобиле:**
- [ ] Кнопки не меньше 56px высотой
- [ ] Шрифт основного текста не меньше 16px
- [ ] Нет горизонтального скролла
- [ ] Схема HUB→MINI→SYSTEM читается вертикально
- [ ] Форма удобна для заполнения большим пальцем

## 9. CTA — стратегия повторения

CTA появляется **минимум 3 раза** на странице:

| Позиция | Текст кнопки | Логика |
|---------|-------------|--------|
| Hero (блок 2) | "Пройти диагностику" | Первый импульс |
| После процесса (блок 7) | "Записаться на диагностику" | Клиент понял как работает |
| Финальный CTA (блок 10) | "Записаться на диагностику" | Финальный дожим |

Дополнительно: кнопка в sticky navigation (всегда видна).

**Снятие возражений рядом с CTA:**
```
Бесплатно · 30 минут · Без обязательств
```

## 10. Скорость и производительность

- Все стили — inline в `<style>` (один HTTP-запрос)
- Шрифты: `preconnect` + `display=swap` + только `wght@400;600;700`
- Изображения: WebP + `loading="lazy"` + указывать `width` и `height`
- Анимации: только через `transform` и `opacity`
- Цель: LCP < 2.5 сек на 4G

## 12. Что НЕ делать

На основе экспертной оценки из research.md:

- ❌ Hero-заголовок о команде, не о клиенте ("Мы — Нейро Бабки СССР" — плохо)
- ❌ Портфолио без описания задачи и результата
- ❌ Форма с 5+ полями
- ❌ Один CTA в самом конце страницы
- ❌ Стоковые фотографии вместо реальных
- ❌ Технические термины без объяснения (таргет, CTR, ROI, воронка)
- ❌ Скрытые контакты — только в футере
- ❌ Анимации, блокирующие скроллинг
- ❌ Отсутствие ценового ориентира ("напишите, обсудим")
- ❌ Блок "О нас" раньше блока "Услуги"
