## 🏷 Типы проектов (классификация)

Четыре типа по двум измерениям (ветка × назначение):

| Тип | Что значит | Примеры |
|---|---|---|
| **agency-internal** | Собственные продукты агентства «Нейро Бабки СССР» — лендинги бренда, визитки команды | neuro-babki |
| **agency-client** | Клиенты агентства — кто платит за услуги, работа по договору | beauty, yuzhnoberezhniy |
| **curator-internal** | Инструменты для кураторской деятельности Инны — платформы, дашборды, конструкторы | mastergroup |
| **curator-member** | Проекты для участников мастер-групп — персональные визитки, страницы, инструменты | alisa |

Тип проекта указан в его `CLAUDE.md` и в `PLAN.md`. При создании нового проекта из `_template/` — обязательно указать тип в первых строках.

## 📦 Почему сейчас всё в одной папке (историческая причина)

Плоская структура сложилась исторически — папка `clients/` была создана когда все проекты были клиентскими. Потом появились:
- **neuro-babki** — мы сами как «клиент своего агентства» (Шаг 2 реструктуризации, 2026-04-12)
- **mastergroup + alisa** — кураторская ветка (параллельная сессия, 2026-04-14)

Сейчас папка содержит 4 типа проектов. Миграция в `projects/{agency,curator}/{internal,clients}/` — **большая задача с реальным риском**:
- 5+ скриптов деплоя используют пути `clients/<name>/`
- Beauty код привязан к `clients/beauty/` (Supabase)
- Параллельные сессии могут активно работать в этих папках
- Риск поломки путей в коде при `git mv`

**Решение сейчас:** оставить плоскую структуру + этот README как карту + классификацию по типу в каждом `CLAUDE.md`.

**Решение потом:** см. [АРХ-6](../ideas.md) — когда Beauty и МГ стабилизируются, сделать полную реструктуризацию отдельным трек-шагом.

## 🧭 Как пользоваться этой картой

- **Начинаешь новый проект?** → скопируй `_template/`, определи тип (agency-client / curator-internal / etc.), пропиши в `CLAUDE.md`, добавь строку в таблицу выше.
- **Ищешь проект?** → таблицы выше группируют по ветке бизнеса. Клиент агентства ≠ участник мастер-группы, путать нельзя.
- **Работаешь как Директор?** → помни: клиент агентства и проект кураторской ветки имеют разный контекст (договор vs внутренний инструмент), разные правила приёмки, разных людей в процессе.
