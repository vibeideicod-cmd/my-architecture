# Как завести нового клиента из этого шаблона

> Этот файл — инструкция. При создании нового клиента **удали его**.

## Шаги

1. **Скопируй всю папку `_template/`** в `clients/{имя-клиента}/`
   - Имя папки — короткое, латиницей, в kebab-case: `berezki`, `sanatoriy-sever`, `bakery-masha`
   - Пример: `cp -r clients/_template clients/berezki`

2. **Удали этот файл** (`_HOW-TO-USE.md`) из новой папки

3. **Заполни плейсхолдеры во всех файлах.** В каждом файле в квадратных скобках `[...]` — что нужно вписать. Особенно важно:
   - `CLAUDE.md` — контекст клиента, цвета, этап
   - `brief.md` — заполняется Аналитиком после интервью с клиентом
   - `PLAN.md` — начальная фаза (обычно Discovery) + первые задачи
   - `project-log.md` — первая запись в истории
   - `FILES.md` — карта документов клиента со статусами

4. **Добавь клиента в корневой [CURRENT.md](../../CURRENT.md)** в раздел активных направлений, если это текущий фокус

5. **Добавь клиента в корневой [README.md](../../README.md)** в таблицу «Текущие активные клиенты»

6. **Добавь клиента в [clients/README.md](../README.md)** с типом проекта и статусом

7. **Активируй Директора** — любую первую задачу по клиенту запускай через `agents/director.md`, он определит фазу и назначит специалиста

## Что лежит в шаблоне

| Файл / папка | Для чего |
|---|---|
| `CLAUDE.md` | Контекст клиента для Кодыча и агентов |
| `PLAN.md` | Тактический план: текущая фаза, что делаем, что в очереди |
| `brief.md` | Заготовка брифа — заполняется Аналитиком в фазе Research |
| `project-log.md` | Хроника проекта: история задач, файлы, замечания |
| `FILES.md` | Карта файлов клиента: активное / финальное / черновик / устаревшее / справочное |
| `audio/` | Сюда кладём аудиозаписи клиента (Zoom, голосовые). Файлы НЕ коммитятся (тяжёлые) — обрабатываются скиллом [`/transcribe`](../../.claude/skills/transcribe/SKILL.md) |
| `transcripts/` | Транскрипты клиентских аудиозаписей. `.md` коммитятся как история |

## Правила работы с клиентскими файлами

- **PLAN.md** — живой документ, обновляется постоянно. Раздел «Текущая фаза» — источник правды для Директора.
- **project-log.md** — пишется после каждой значимой задачи или решения.
- **brief.md** — пишется один раз в фазе Research, потом только правки если клиент изменил требования.
- **CLAUDE.md** — пишется один раз в начале, потом правки только если меняется контекст.
- **FILES.md** — обновляется каждый раз, когда появляются новые версии, финалы или устаревшие документы.

## Когда клиент завершён

Когда проект сдан и клиент больше не активен — **не удаляй папку**. Она становится кейсом в портфолио. Только в `project-log.md` поставь статус «Завершён» и добавь итоговую запись с результатами.
