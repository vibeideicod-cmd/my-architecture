## 📋 Чеклист по фазам Playbook

**Легенда:** `[ ]` не начато · `[>]` в работе · `[x]` готово · `[!]` блокер · `[~]` отложено осознанно

### Фаза 1 — Discovery (Идея)
**Ответственный:** product-builder
**Артефакт:** [idea.md](idea.md)

- [ ] Задать Инне 6 вопросов одним заходом (тип продукта / формулировка «создаю X для Y чтобы Z» / ЦА / Beget или Vercel / дедлайн / что готово)
- [ ] Заполнить idea.md с разделом «Чего НЕ будет в MVP»
- [ ] Формулировка подтверждена Инной

### Фаза 2 — Research (Исследование)
**Ответственный:** analyst
**Артефакт:** [research.md](research.md)

- [ ] Конкуренты (5-7 примеров с плюсами/минусами)
- [ ] Функции (15-20 с пометками MVP / потом)
- [ ] UX-сценарии (3 типичных сценария «от открытия до результата»)
- [ ] Технологии (сравнение 2-3 вариантов)
- [ ] Монетизация

### Фаза 2.5 — Critique (Критика)
**Ответственный:** product-builder
**Артефакт:** [critique.md](critique.md)
**Техника:** [critique-techniques.md](../../knowledge/prompting/critique-techniques.md)

- [ ] Раскритиковать глазами пользователя (конкретный портрет из ЦА)
- [ ] Раскритиковать глазами UX-дизайнера
- [ ] Раскритиковать глазами конкурента
- [ ] «Ты уверен?» на ключевых утверждениях research.md
- [ ] Список правок внесён в idea.md/research.md

### Фаза 3 — Design / Plan (Чертёж)
**Ответственный:** product-builder → передаёт специалисту
**Артефакты:** [brief.md](brief.md) + [BACKEND-PLAN.md](BACKEND-PLAN.md) *(если нужен бэкенд)*

- [ ] brief.md: экраны, навигация, функции, «чего НЕ делаем в MVP»
- [ ] BACKEND-PLAN.md (если применимо): сущности, роли, RLS, API, realtime
- [ ] Специалист (websites / systems) подтвердил что может стартовать без уточнений

### Фаза 4 — Build (Реализация)
**Ответственный:** websites (фронт) / systems (бэкенд)
**Артефакты:** код проекта + `CLAUDE.md` (карта)
**Промпты:** [templates-landing.md](../../knowledge/prompting/templates-landing.md) / [templates-mini-app.md](../../knowledge/prompting/templates-mini-app.md) / [templates-backend.md](../../knowledge/prompting/templates-backend.md)

- [ ] Стартовый one-shot промпт выполнен, MVP собран
- [ ] CLAUDE.md-карта проекта готова
- [ ] Проверка в Chrome DevTools (375 / 768 / 1440)
- [ ] Для Mini App — проверка мобильных 320-430px
- [ ] Брендбук Инны применён (для внутренних) / палитра клиента (для клиентских)
- [ ] Итерации по правилу «один промпт — одна задача»

### Фаза 5 — Content (Наполнение)
**Ответственный:** content / Инна
**Артефакты:** реальные данные в коде / базе

- [ ] Все «Lorem ipsum» и заглушки заменены
- [ ] Тексты проверены: простой язык, без канцелярита
- [ ] Если данных нет — помечены TODO, Инна знает

### Фаза 6 — QA + Deploy
**Ответственные:** qa → deployer-beget
**Артефакты:** [TESTING.md](TESTING.md), [DEPLOYMENT.md](DEPLOYMENT.md)

- [ ] QA прогон по чеклисту (agents/qa.md)
- [ ] TESTING.md готов (руководство для внешнего тестировщика)
- [ ] Прогон критики «глазами пользователя» на готовом MVP
- [ ] Тест на 3-5 живых людях (два вопроса: «что непонятно?» / «где застряли?»)
- [ ] Правки по обратной связи внесены
- [ ] DEPLOYMENT.md готов
- [ ] Задеплоено на выбранный хостинг
- [ ] Три финальных теста (см. Урок 12): живая ссылка / целевое действие / бот ещё жив через час
- [ ] Финальная приёмка Инной
