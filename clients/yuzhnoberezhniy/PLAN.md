## 🧩 Усиление «Собери рюкзак» — что добавляем

Три обязательные фичи из Lovable-версии + одна UX-идея:

### 🔥 AgeGroup (возрастная группа)
- **Градация:** 7-10 лет / 11-13 лет / 14-17 лет
- **Зачем:** ЮБ работает с детьми 7-17, им нужны разные вещи
- **Как:** data-атрибут `data-age` на каждом пункте, фильтр

### 🔥 Season (сезон)
- **Градация:** Лето / Зима / Межсезонье (весна/осень)
- **Зачем:** ЮБ круглогодичный (СКЛ + мать-дитя). Летом — купальники, зимой — термобельё
- **Как:** data-атрибут `data-season`, фильтр

### 🔥 SpecialNeeds (особые медицинские потребности)
- **Тип:** boolean тумблер
- **Зачем:** СКЛ — ключевое направление, дети приезжают с диагнозами (дыхательная, опорно-двигательная, нервная система). Нужны лекарства, медкарта, специальное питание
- **Как:** отдельная секция «Медицинское» показывается только при включении

### 🆕 QuickStart (отдельный экран онбординга)
- **Что:** перед показом чеклиста — экран с крупными кнопками выбора всех 4 параметров (пол + возраст + сезон + special needs)
- **Зачем:** родитель обязан осознанно выбрать → чеклист точно настроен под его ребёнка. Снимает ощущение «почему тут купальники в декабре»
- **Как:** новая секция `<section id="onboarding">`, скрывает основной чеклист до клика «Начать»

## 🐛 Известные баги / заблокированные задачи

### BUG-YUB-001: /start в боте не отвечает автоматически
**Статус:** Заблокировано инфраструктурой Beget + Telegram
**Симптом:** Когда пользователь открывает `@ryukzak_yub_bot` и нажимает «Start», в чате не появляется приветственное сообщение. `setMyDescription` через Telegram Bot API отрабатывает, но Telegram показывает этот текст ТОЛЬКО в буквально пустых чатах (история полностью чиста). Для новых пользователей работает; для чатов где уже есть /start история — не показывается.
**Что пытались:** PHP webhook на Beget. Два блокера:
1. `demo.ideidlyabiznesa1913.ru` — Telegram не может зарезолвить DNS (ошибка `Failed to resolve host: Temporary failure in name resolution`). С наших резолверов (Google DNS, Cloudflare) работает идеально. Причина неизвестна — проблема на стороне Telegram.
2. `ideidlyabiznesa1913.ru` (main домен) — DNS резолвится, но SSL-сертификат невалиден для Telegram-валидатора (`certificate verify failed`). Требуется выпустить валидный SSL через Beget панель.
**Готовый код для webhook:** [bot/webhook.php](bot/webhook.php) + [../../deploy-yub-bot.sh](../../deploy-yub-bot.sh). Готов к работе как только инфраструктура разблокируется.
**Как разблокировать:**
- Вариант А: Войти в панель Beget → найти настройки SSL для `ideidlyabiznesa1913.ru` → выпустить Let's Encrypt. Или обратиться в поддержку Beget по поводу demo.-субдомена.
- Вариант B: Переехать на другой хостинг для webhook (отвергнуто Инной — только Beget).
**Обходной путь пока не починено:** Menu Button работает нормально, клиент может попасть в TMA через него. Новые пользователи (первое открытие бота) видят описание из `setMyDescription`.
