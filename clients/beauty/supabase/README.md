```
supabase/

    └── 003_seed_anna.sql   — кладёт в базу тестового мастера Анну Смирнову
```

## Порядок применения миграций

Открыть Supabase → **SQL Editor** → **+ New query**, копировать содержимое
файла, нажать **Run**. Строго в этом порядке:

1. `001_schema.sql` — создаёт 6 таблиц
2. `002_policies.sql` — вешает RLS
3. `003_seed_anna.sql` — заполняет Анной Смирновой

После каждого запуска должна появиться зелёная надпись `Success. No rows returned`.

## Таблицы

| Таблица | Что хранит |
|---|---|
| `masters` | Профиль мастера: имя, город, bio, цвет бренда, аватар |
| `categories` | Категории услуг мастера (Маникюр, Педикюр, ...) |
| `services` | Конкретные услуги: название, цена, длительность, описание, теги |
| `service_photos` | Ссылки на фото услуг (в Storage или внешние) |
| `schedules` | Рабочие часы по дням недели |
| `bookings` | Записи клиентов |

## Политики доступа (RLS)

| Таблица | Публично (anon key) | Приватно |
|---|---|---|
| `masters`, `categories`, `services`, `service_photos`, `schedules` | Чтение всего | — |
| `bookings` | Только INSERT (создание записи) | Чтение, правка — запрещено |

Для управления бронями изнутри админки или из SQL Editor используется
**service_role** ключ — он игнорирует RLS.
