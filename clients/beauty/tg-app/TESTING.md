## 1. Что за проект

Telegram Mini App «Beauty Визитка» — каталог услуг бьюти-мастера с
онлайн-записью внутри Telegram. Один бот обслуживает многих мастеров через
deep link `t.me/beauty_vizitka_bot/app?startapp=master_xxx` (multi-tenant
заявлен в [BACKEND-PLAN.md](BACKEND-PLAN.md), но пока бэкенда нет — все
данные mock).

**Стек:** HTML + CSS + Vanilla JS. Никаких фреймворков, бандлеров, CDN.
Telegram WebApp SDK подгружается одним скриптом из `telegram.org/js/telegram-web-app.js`.

**Бот в Telegram:** `@beauty_vizitka_bot`
**Production URL:** `https://beauty-tma-app.vercel.app` (по [BACKEND-PLAN.md:28](../BACKEND-PLAN.md))
**Vercel Root Directory:** `clients/beauty/tg-app`
**Auto-deploy:** push в `main` → Vercel пересобирает за ~60 сек.

## 3. Как запустить

### Вариант A — внутри Telegram (полный тест, рекомендуется)

1. Открыть бота `@beauty_vizitka_bot`.
2. Нажать кнопку меню (либо `/start`) — откроется TMA.
3. Любой push в ветку `main` пересобирает Vercel за ~60 секунд. Можно
   проверять свежие правки прямо в Telegram.

В Telegram работают: `MainButton`, `BackButton`, `HapticFeedback`, `tg.close()`,
`tg.openTelegramLink()`, `tg.initDataUnsafe.user.first_name` (см. парсинг
имени в [js/app.js:799-803](js/app.js)).

### Вариант B — локально в браузере (UI-тесты)

```bash
cd clients/beauty/tg-app
python3 -m http.server 8080
# или
npx serve .
```

Открыть `http://localhost:8080`.

В браузере SDK Telegram отсутствует — `MainButton`, `BackButton`,
`HapticFeedback` молча игнорируются (везде в [app.js](js/app.js) стоит
`if (!tg) return;`). Навигация и весь UI работают, валидации работают —
этого достаточно для проверки логики, вёрстки и таббара.

### Вариант C — production-проверка

Открыть `https://beauty-tma-app.vercel.app` напрямую в браузере или через
бота в Telegram.

### Слоты времени

Источник: PostgreSQL-функция `get_available_slots` в Supabase (миграция [005_slots_function.sql](../supabase/migrations/005_slots_function.sql)). На фронте — async-обёртка [`getAvailableSlots`](js/data.js) вызывает её через `BeautyDB.client.rpc()`.

Логика:
- **Пн–Пт (рабочие дни Анны):** слоты 10:00, 11:00, …, 17:00 (8 слотов, `slot_duration=60` мин из `schedules`).
- **Сб, Вс (выходные):** пустой массив, `is_working=false` в `schedules`.
- **Уже занятые слоты** (пересекающиеся с подтверждёнными `bookings`) не возвращаются — их видно только в Supabase Dashboard.
- **Прошедшие слоты** на сегодняшний день также фильтруются (`slot_ts > now()`).

Если нужно проверить вручную: Supabase → SQL Editor → `select get_available_slots('anna', current_date + 1);`

### Валидация телефона

Источник: [`validatePhone`](js/app.js#L741-L745)

```js
const clean = val.replace(/[\s\-()]/g, '');
return /^(\+7|8)\d{10}$/.test(clean) || /^\d{10,}$/.test(clean);
```

Принимает:

| Ввод                 | Результат |
|----------------------|-----------|
| `+79001234567`       | ✅ |
| `89001234567`        | ✅ |
| `9001234567`         | ✅ (10 цифр без префикса) |
| `+7 (900) 123-45-67` | ✅ (пробелы / `-` / `()` чистятся) |
| `12345`              | ❌ |
| `123`                | ❌ — красная подсветка + `Haptic.error()` |
| пустое               | ❌ — MainButton остаётся disabled |

**MainButton активируется** при вводе строки с `length > 5` (см.
[`onPhoneInput`](js/app.js#L698-L702)) — то есть от 6 символов. Финальная
валидация при сабмите.

## 5. Архитектура навигации

Это самое важное для тестирования — без понимания этой схемы нельзя интерпретировать поведение.

### Один DOM, переключение классами

Все экраны лежат в `#app` одновременно с `position: absolute; inset: 0`.
Активный экран помечается классом `is-active`, остальные — невидимы и
не кликабельны через `pointer-events: none; visibility: hidden`. Анимация
перехода — fade + slide через класс `is-exiting` на 220 мс.

См. [`navigate(screenId)`](js/app.js#L92-L113) и CSS правила
[style.css:96-117](css/style.css).

### Два типа экранов

| Тип | Список | Tabbar | MainButton/BackButton |
|---|---|---|---|
| **Tab-экраны** (верхнеуровневые) | `index`, `master`, `portfolio`, `bookings` | Виден | Скрыты |
| **Flow-экраны** (внутри сценария записи) | `services`, `details`, `booking`, `confirm`, `success` | Скрыт | Активны |
| **Технические** | `loading`, `onboarding` | Скрыт | Скрыты |

Список tab-экранов жёстко прописан как `TAB_SCREENS` в [js/app.js:117](js/app.js).
Видимость таббара пересчитывается из `navigate()` через
[`updateTabbar`](js/app.js#L119-L128).

### Tabbar (нижнее меню)

Добавлен 2026-04-13. HTML — [index.html:355-372](index.html), стили —
[style.css:1184-1244](css/style.css), логика — [js/app.js:115-141](js/app.js).

4 вкладки:

| data-tab    | Иконка | Подпись     | Что инициализирует |
|-------------|--------|-------------|----------------------|
| `index`     | 🏠     | Каталог     | `initScreen_index()` |
| `master`    | 👤     | О мастере   | `initScreen_master()` |
| `portfolio` | 🖼      | Портфолио   | `initScreen_portfolio()` |
| `bookings`  | 📅     | Мои записи  | `initScreen_bookings()` |

Активная вкладка подсвечена цветом `--accent` (бренд мастера) на
полупрозрачной плашке. Все цвета берутся из CSS-переменных
Telegram-темы (`--tg-bg-sec`, `--tg-hint`, `--tg-text`) — тёмная тема
работает автоматически.

### Сценарий записи (flow)

```
index → services → details → booking → confirm → success
                                                    └─→ (back) index
```

Внутри flow таббар спрятан, навигация ведётся через Telegram MainButton
(вперёд) и BackButton (назад). На `success` вместо BackButton — экранная
кнопка «Вернуться в каталог».

## 6. localStorage

Источник: [`BOOKING_KEY`](js/app.js#L219), [`ONBOARDING_KEY`](js/app.js#L816).

| Ключ                          | Когда пишется | Что хранит |
|-------------------------------|---------------|------------|

**Сброс для повторного теста онбординга:** в DevTools → Application →
Local Storage → удалить `beauty-tma-onboarding-v1` и перезагрузить страницу.

**Запись из localStorage** читается на главной (блок «Моя последняя запись»)
и на вкладке «Мои записи». Удаление ключа `beauty-tma-last-booking-v1`
вернёт оба места в empty-state.

## 9. Сквозной сценарий бронирования

Полный путь, который должен пройти любой релиз:

1. Очистить localStorage (DevTools → Application → удалить `beauty-tma-*`).
2. Перезагрузить страницу → должен открыться экран `onboarding`.
3. Нажать «Посмотреть каталог →» → переход на `index`, ключ онбординга записан.
4. Убедиться, что таббар виден, активна «Каталог».
5. Кликнуть на категорию «Маникюр» → `services` (таббар скрылся).
6. Дождаться окончания скелетона (~350 мс) → выбрать «Маникюр гель-лак» → `details`.
7. Нажать MainButton «Выбрать время» → `booking`.
8. Выбрать ближайший рабочий день (не Сб/Вс) → выбрать слот `14:00`.
9. MainButton «Подтвердить время» → `confirm`.
10. Ввести телефон `+7 900 123-45-67` → MainButton «Записаться».
11. Дождаться спиннера 900 мс → `success`.
12. **Проверить детали:** «Маникюр гель-лак», дата · 14:00, «Анна Смирнова», «1 ч 30 мин».
13. Нажать «← Вернуться в каталог» → `index`, таббар активен.
14. Перейти на вкладку «Мои записи» → должна быть карточка только что созданной записи.
15. Перейти на вкладку «О мастере» → данные Анны.
16. Перейти на вкладку «Портфолио» → emoji-плитки с названиями услуг.

**Ожидаемый итог:** все экраны пройдены, запись в localStorage, обе точки её отображения (главная + вкладка «Мои записи») работают.

## 13. История багов и фиксов

### BUG-001 — TMA полностью некликабельна (КРИТИЧЕСКИЙ) ✅ ИСПРАВЛЕНО
**Дата:** обнаружен 2026-04-12, исправлен 2026-04-13 (коммит `a3442e8`).
**Симптом:** ни одна кнопка не реагировала на клики ни в браузере, ни в Telegram. Единственное исключение — кнопка «Поделиться с подругой» на экране `success`.
**Корневая причина:** все 9 экранов лежат друг на друге через `position: absolute; inset: 0`. Раньше неактивные экраны гасились только через `opacity: 0`, но `pointer-events` не сбрасывался. В DOM последним идёт `#screen-success` — он всегда был сверху и перехватывал все клики. «Поделиться с подругой» работала, потому что физически расположена внутри `#screen-success`.
**Фикс:** добавлены правила `pointer-events: none; visibility: hidden` для `.screen` и `.is-exiting`, и `pointer-events: auto; visibility: visible` для `.screen.is-active`. См. [css/style.css:96-117](css/style.css).
**Регрессионный тест:** на каждом экране все интерактивные элементы (категории на `index`, услуги на `services`, слоты на `booking`, поле телефона на `confirm`, вкладки таббара) — кликабельны.

### BUG-002 — `first_name` содержит мусор после `|` ✅ ИСПРАВЛЕНО
**Дата:** обнаружен 2026-04-12, исправлен 2026-04-13 (коммит `a3442e8`).
**Симптом:** на онбординге у пользователя с `first_name = "Инна|Автосистемы контента и коммуникаций"` отображалось всё целиком — «Привет, Инна|Автосистемы контента и коммуникаций!».
**Фикс:** `(user?.first_name || '').split('|')[0].trim()` в [app.js:799-803](js/app.js).
**Регрессионный тест:** ввести в Telegram `first_name` с любой `|` — на онбординге должна быть только часть до разделителя.

## 14. Чеклист «как чинить» (для разработчика)

Если нашли дефект, действуйте в этом порядке:

1. **Воспроизвести в браузере локально** (`python3 -m http.server 8080`) — большая часть багов видна без Telegram.
2. **Открыть DevTools Console** — JS-ошибки гарантированно сломают навигацию (`navigate` логирует «Экран не найден»).
3. **Найти место правки:**
   - Ошибка в данных (текст / цена / список услуг) → [js/data.js](js/data.js).
   - Ошибка вёрстки / адаптива / темы → [css/style.css](css/style.css).
   - Ошибка логики, навигации, валидации → [js/app.js](js/app.js).
   - Структура экранов / новый блок → [index.html](index.html).
4. **Помнить про навигацию:** добавляете новый верхнеуровневый экран — добавьте его id в `TAB_SCREENS` ([app.js:117](js/app.js)) и пункт в tabbar в [index.html](index.html). Иначе при переходе на него останется виден таббар чужого экрана или не появится сам.
5. **Помнить про CSS-перекрытия:** все экраны абсолютно позиционированы. Любой новый экран должен быть `<div class="screen">` с уникальным `id="screen-XXX"`. Класс `.screen` сам обеспечит позиционирование, скрытие, анимации.
6. **Помнить про MainButton/BackButton:** на верхнеуровневых вкладках — `MainButton.hide(); BackButton.hide()`. На flow-экранах — навешивать обработчики через `MainButton.show(text, onClick)` / `BackButton.show(onClick)`. Очистка предыдущего обработчика идёт через `state.mainButtonHandler` / `state.backButtonHandler` в [app.js:21-65](js/app.js).
7. **Прогнать сквозной сценарий из раздела 9** — это minimum acceptance test перед коммитом.
8. **Проверить тёмную тему** — переключить её в Telegram (или в браузере через эмуляцию `prefers-color-scheme: dark`) и убедиться, что ничего не «выпадает» из палитры. Все цвета должны идти через `var(--tg-*)` или `var(--accent)`.
9. **Закоммитить и запушить в `main`** — Vercel пересоберёт за ~60 сек, можно проверять в Telegram.
10. **Обновить этот файл,** если поведение изменилось: добавить новый кейс, обновить таблицы.
