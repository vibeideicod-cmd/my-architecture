# Beauty TMA — Руководство по тестированию

> Для внешнего тестировщика. Всё, что нужно знать — в этом файле.

---

## 1. Что это за проект

Telegram Mini App «Beauty Визитка» — каталог услуг бьюти-мастера с онлайн-записью.
Стек: HTML + CSS + Vanilla JS, без фреймворков, без сборщиков.

Файлы приложения:

```
tg-app/
├── index.html        — точка входа
├── css/style.css     — все стили
├── js/data.js        — mock-данные (MASTER, CATEGORIES, SERVICES, слоты)
└── js/app.js         — вся логика навигации, TG SDK, localStorage
```

---

## 2. Как запустить

### Вариант A — внутри Telegram (полный тест, рекомендуется)

1. Открыть бот `@beauty_vizitka_bot` в Telegram
2. Нажать кнопку меню внизу (или `/start`) — откроется Mini App
3. Production URL: `https://beauty-tma-app.vercel.app`

В Telegram работают: `MainButton`, `BackButton`, `HapticFeedback`, `tg.close()`, `tg.openTelegramLink()`, автозаполнение имени пользователя на онбординге.

### Вариант B — в браузере (для UI-тестов)

```bash
cd clients/beauty/tg-app
python3 -m http.server 8080
# или
npx serve .
```

Открыть `http://localhost:8080`

В браузере Telegram SDK отсутствует: `MainButton`, `BackButton`, `HapticFeedback` не работают. Навигация работает, весь UI виден. Это достаточно для проверки вёрстки и логики экранов.

## 4. localStorage

Приложение использует два ключа:

| Ключ | Когда пишется | Что хранит |
|------|--------------|------------|
| `beauty-tma-onboarding-v1` | При нажатии «Посмотреть каталог» на онбординге | `'1'` |
| `beauty-tma-last-booking-v1` | После успешного подтверждения записи | JSON-объект с записью |

Для сброса онбординга: удалить ключ `beauty-tma-onboarding-v1` через DevTools → Application → LocalStorage.

**Ожидаемое поведение:**

**Слоты после выбора рабочего дня:**
- Сетка из 10 кнопок-времён: 10:00, 10:30, 11:30, 12:00, 12:30, 14:00, 14:30, 15:30, 16:00, 17:00
- Все слоты зелёные (только свободные)

**Если выбрать субботу или воскресенье (принудительно через JS или дождавшись):**
- Блок: иконка 📅, текст «На этот день нет свободных мест»
- Кнопка «ближайшее: [день] · [время] →»
- При нажатии: переключается дата, выбирается первый слот автоматически

**Проверить:**
- [ ] Выходные недоступны для выбора
- [ ] При выборе рабочего дня появляется сетка слотов

- [ ] BackButton → Экран 3

### Экран 5 — Подтверждение (`screen-confirm`)

**Ожидаемое поведение:**
- Карточка с 5 строками: Услуга, Цена, Дата, Мастер, Время
- Поле телефона — пустое (или автозаполнено из `tg.initDataUnsafe.user.phone_number` если есть в TG)
- `MainButton` «Записаться» — disabled если поле телефона пустое или < 6 символов
- После ввода корректного телефона → MainButton активируется

**Проверить валидацию телефона:**
| Ввод | Результат |
|------|-----------|
| `+79001234567` | ✅ принимается |
| `89001234567`  | ✅ принимается |
| `9001234567`   | ✅ принимается (10 цифр) |
| `+7 (900) 123-45-67` | ✅ принимается (очищается до `+79001234567`) |
| `123` | ❌ — красная обводка на поле, поле трясётся (Haptic.error) |
| пустое поле | ❌ — MainButton disabled |

**При нажатии «Записаться»:**
1. MainButton показывает прогресс-индикатор (900 мс эмуляция API)
2. Haptic «success»
3. Запись сохраняется в localStorage (`beauty-tma-last-booking-v1`)
4. Переход на Экран 6

**Проверить:**
- [ ] Карточка содержит правильные данные выбранной услуги, даты, времени
- [ ] Поле с ошибкой сбрасывается при фокусе
- [ ] 900 мс задержка видна как спиннер на кнопке
- [ ] После подтверждения запись появляется в localStorage

---

### Экран 6 — Успех (`screen-success`)

**Ожидаемое поведение:**
- Анимированная SVG-галочка (circle + polyline через stroke-dashoffset)
- Заголовок «Вы записаны!»
- Карточка с деталями: Услуга, Дата, Мастер, Примерно (длительность)
- Кнопка «📆 Добавить в календарь» — ссылка на Google Calendar
- Кнопка «🔗 Поделиться с подругой»
- Кнопка «← Вернуться в каталог»
- `MainButton` «Готово» — active (закрывает TMA через `tg.close()`)
- `BackButton` скрыта

**Google Calendar URL** формируется из `booking.datetime`:
- `action=TEMPLATE`, `text=Запись к Анна Смирнова`, `details=<название услуги>`
- Время передаётся в UTC (без конвертации часовых поясов — ограничение MVP)

**Кнопка «Поделиться»:**
- В Telegram: `tg.openTelegramLink('https://t.me/share/url?url=...')` — открывает нативный шаринг
- В браузере: пробует `navigator.share()`, иначе `window.open()`
- Текст: «Нашла классную штуку — Beauty Визитка, запись к мастеру прямо в Telegram, без звонков! Попробуй 💅»
- URL: `https://t.me/beauty_vizitka_bot`

**Проверить:**
- [ ] Галочка анимируется при открытии экрана
- [ ] Детали записи совпадают с тем, что было подтверждено на Экране 5
- [ ] Ссылка «Добавить в календарь» открывает Google Calendar с нужными данными
- [ ] «Вернуться в каталог» → Экран 1, state сбрасывается (selectedCategory, selectedService, selectedDate, selectedSlot, bookingResult = null)
- [ ] При возврате через «Мои записи» на Экране 1 видна последняя запись

## 6. Сквозной сценарий — полный путь бронирования

Цель: создать запись от начала до конца.

1. Сбросить localStorage: DevTools → Application → Local Storage → удалить все ключи `beauty-tma-*`
2. Открыть приложение → Экран онбординга
3. Нажать «Посмотреть каталог» → Экран 1
4. Выбрать «Маникюр» → Экран 2
5. Дождаться загрузки (350 мс) → выбрать «Маникюр гель-лак» → Экран 3
6. Нажать «Выбрать время» → Экран 4
7. Выбрать ближайший рабочий день → выбрать слот «14:00» → MainButton активируется
8. Нажать «Подтвердить время» → Экран 5
9. Ввести телефон `+79001234567` → нажать «Записаться»
10. Дождаться 900 мс → Экран 6
11. Проверить карточку: Маникюр гель-лак, 1 800 ₽, выбранная дата·время, Анна Смирнова, 1 ч 30 мин
12. Нажать «← Вернуться в каталог» → Экран 1
13. Убедиться, что появился блок «📋 Моя последняя запись» с деталями

**Ожидаемый итог:** все 8 экранов пройдены, запись сохранена в localStorage, блок «Мои записи» отображается.

## 8. Известные ограничения MVP

| Ограничение | Причина |
|-------------|---------|
| Фото услуг не отображаются — галерея показывает эмодзи | Папка `img/svc/` пустая, реальных файлов нет |
| Все слоты всегда свободны | `getMockSlots()` возвращает `available: true` для всех |
| Запись не отправляется никуда | `handleConfirmSubmit` эмулирует API через `delay(900)` |
| Мастер не получает уведомлений | Нет бэкенда, нет интеграции с ботом |
| Телефон обязателен для записи | Требуется > 5 символов в поле для активации MainButton |
| Время в Google Calendar — UTC | `buildCalendarUrl` не конвертирует в часовой пояс пользователя |

## 9. Где смотреть логи

- Браузер: DevTools Console → ошибки навигации логируются через `console.warn('Экран не найден:', screenId)`
- LocalStorage: DevTools → Application → Local Storage → `http://localhost:8080` или домен Vercel
- Telegram: нет прямого доступа к консоли; использовать `eruda` или Telegram → Settings → Experimental → Telegram Bot API
