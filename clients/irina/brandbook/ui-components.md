# UI-компоненты бренда Ирины

> **Зона:** интерфейсные элементы — кнопки, формы, карточки, модалки, навигация, типовые секции лендингов, состояния. Применяется на сайтах, лендингах, мини-аппах в Telegram и VK, личных кабинетах платформ.
> **Принцип:** компонент = **кубик Lego**. Один раз спроектирован → используется везде. Так бренд везде узнаваем, разработка ускоряется, ошибок меньше.
> **Версия:** MVP · 2026-05-01

---

## 1. Главные принципы UI Ирины

1. **Минимум форм, максимум воздуха.** Дизайнер не уважает себя, если интерфейс «забит».
2. **Один путь действия — одна главная кнопка.** На странице может быть много вторичных, но primary-CTA — один.
3. **Скруглённые углы везде.** Бренд Ирины — про обнимание, не про острые рёбра.
4. **Состояния обязательны.** Loading, error, empty, success — без них интерфейс «мёртвый».
5. **Доступность не опция.** Контраст 4.5:1 для текста, фокус видим, alt-тексты обязательны.

## 2. Кнопки

> **Принцип:** 4 типа кнопок, разные роли. Не делать «5-й тип» под каждый случай.

### 2.1. Primary (главная CTA)

| Параметр | Значение |
|---|---|
| Фон | Кислотно-зелёный `#4DAA02` (триггер действия) |
| Текст | Ваниль `#FCFAE1` |
| Радиус | 12px |
| Высота | 48px (десктоп), 44px (мобильный) |
| Padding | 24px по горизонтали |
| Шрифт | Medium (500) или Bold (700), 16-18px |

**Состояния:**
- **Hover** — фон темнеет на 10% (`#3A8202`)
- **Pressed** — фон темнеет на 20% + scale 0.98
- **Disabled** — opacity 40%, cursor not-allowed
- **Focus** — outline 2px Хвоя `#306654` с offset 2px

**Используется:** «Купить», «Записаться», «Отправить», «Начать», «Продолжить» — там где **деньги или регистрация**.

### 2.2. Secondary (вторичная)

| Параметр | Значение |
|---|---|
| Фон | Хвоя `#306654` |
| Текст | Ваниль `#FCFAE1` |
| Радиус | 12px |
| Высота | 48px / 44px |

**Используется:** «Узнать больше», «Подробнее», «Посмотреть кейсы» — там где **просто переход**.

### 2.3. Ghost (контурная)

| Параметр | Значение |
|---|---|
| Фон | прозрачный |
| Граница | 2px Хвоя `#306654` |
| Текст | Хвоя `#306654` |
| Радиус | 12px |
| Высота | 48px / 44px |

**Hover** — фон Хвоя 10% прозрачности.

**Используется:** «Скачать PDF», «Сохранить», «Назад» — там где **поддерживающее действие**.

### 2.4. Text-only (ссылка)

| Параметр | Значение |
|---|---|
| Цвет | Коралл `#FF935E` |
| Подчёркивание | при hover |
| Шрифт | Medium (500), 14-16px |

**Используется:** ссылки в тексте, «Все продукты →», «Читать кейс →».

### 2.5. Что НЕ берём в кнопки ❌

- Цвета вне палитры (синий, красный, фиолетовый кнопки — нет)
- Кнопки без скругления (прямоугольные с прямыми углами)
- 3D-кнопки с тенями (skeumorphism, устарело)
- Иконка вместо текста на главной CTA (текст обязателен — accessibility)

## 3. Формы (input, textarea, select)

### 3.1. Input (текстовое поле)

| Параметр | Значение |
|---|---|
| Фон | Ваниль `#FCFAE1` (или белый `#FFFFFF` если фон ваниль) |
| Граница | 1px Хвоя `#306654` 30% прозрачности |
| Радиус | 12px |
| Высота | 48px |
| Padding | 16px |
| Шрифт | Regular (400), 16px (на мобильном минимум 16px чтобы iOS не зумировал) |
| Цвет текста | Хвоя `#306654` |
| Placeholder | Хвоя `#306654` 50% прозрачности |

**Состояния:**
- **Focus** — граница 2px Коралл `#FF935E`
- **Error** — граница 2px красный (используем `#C84B3A` — не из палитры, но для error безопасно), снизу — текст ошибки 14px
- **Disabled** — фон Ваниль 50%, текст 50%

### 3.2. Textarea

То же что input, но `min-height: 120px`, `resize: vertical`.

### 3.3. Select (выпадающий список)

То же что input. Стрелка вниз — иконка из иконографии (см. `logo-and-graphics.md`), цвет Хвоя.

### 3.4. Checkbox

| Параметр | Значение |
|---|---|
| Размер | 20×20px |
| Радиус | 4px (квадратный, не круглый) |
| Граница (unchecked) | 2px Хвоя `#306654` |
| Фон (checked) | Хвоя `#306654` |
| Иконка-галочка | Ваниль `#FCFAE1` |

### 3.5. Radio

То же что checkbox, но круглый (`border-radius: 50%`). Заполнение — точка Хвоя в центре.

### 3.6. Лейбл (подпись поля)

- Шрифт: Medium (500), 14px
- Цвет: Хвоя `#306654`
- Расположение: над полем, отступ 8px

### 3.7. Подсказка / описание

- Шрифт: Regular (400), 13px
- Цвет: Хвоя 60% прозрачности
- Расположение: под полем, отступ 4px

## 4. Карточки

### 4.1. Карточка продукта (на каталоге / лендинге)

```
┌────────────────────────┐
│ [Иллюстрация / иконка] │  ← 80×80px или больше, в палитре
│                        │
│ Название продукта      │  ← H4, Bold, Хвоя
│                        │
│ Короткое описание      │  ← Body, Regular, Хвоя 80%
│ 1-2 предложения        │
│                        │
│ от 5 000 ₽             │  ← H5, Bold, Коралл
│ [Подробнее →]          │  ← Text-link, Коралл
└────────────────────────┘
```

| Параметр | Значение |
|---|---|
| Фон | Ваниль `#FCFAE1` (на хвоевой странице) или светлая Хвоя 5% (на ванильной) |
| Радиус | 16px |
| Padding | 24px |
| Граница | 1px Хвоя `#306654` 15% прозрачности (опционально) |
| Hover | Лёгкий подъём — `transform: translateY(-2px)`, тень мягкая |

### 4.2. Карточка кейса

Похожа на карточку продукта, но:
- Сверху — большое фото / иллюстрация результата (16:9 или 4:3)
- Заголовок — имя клиента / тип проекта
- Подзаголовок — какая боль закрывалась
- Ссылка «Смотреть кейс →»

### 4.3. Карточка тарифа (для Платформы С.С.С.Р. в визуале)

```
┌────────────────────────┐
│ ⭐ ТАРИФ «РАБОТА»      │  ← H5, Caps, Коралл (если этот тариф рекомендован)
│                        │
│ от 3 000 ₽ / мес       │  ← H2, Bold, Хвоя
│                        │
│ ✓ Доступ к 4 модулям   │  ← Body
│ ✓ AI в контексте       │
│ ✓ Ежемесячный разбор   │
│                        │
│ [Выбрать]              │  ← Primary button
└────────────────────────┘
```

**«Рекомендованный» тариф** — рамка 2px Коралл, плашка сверху «РЕКОМЕНДУЕМ» в Коралле.

## 6. Типовые секции лендингов

> **Принцип:** лендинг Ирины собирается из 6-8 готовых секций. Не «пиши уникальный лендинг каждый раз» — а «выбирай секции, наполняй контентом».

### 6.1. Hero (первый экран)

```
[Заголовок H1 — главный смысл услуги]
[Подзаголовок — короткое уточнение]
[Primary CTA + опционально Secondary CTA]
[Иллюстрация / фото / видео-плейсхолдер справа]
```

| Параметр | Значение |
|---|---|
| Высота | min-height 600px (десктоп), 500px (мобильный) |
| Фон | Ваниль с лёгким паттерном или плоский |

### 6.2. Боли (5 болей клиента)

```
[H2: «Знакомые ситуации?»]
[Сетка 2×3 или 5 в ряд иконок]
[Под каждой — название боли + 1 предложение]
```

Из архитектуры — 5 болей: «Меня не видят», «Нет клиентов», «Нет времени», «Нет структуры», «Хочу всё».

### 6.3. Решение (что Ирина делает)

```
[H2: «Что я делаю»]
[3-4 шага процесса с иконками: Слышу → Создаю → Передаю]
```

Опирается на С.С.С.Р. в визуале (Структура · Стиль · Свобода · Результат).

### 6.4. Кейсы (3-6 штук)

```
[H2: «Что я уже сделала»]
[Сетка карточек кейсов 3 в ряд (десктоп) / 1 в ряд (мобильный)]
```

См. структуру карточки кейса — раздел 4.2.

### 6.5. Тарифы / Цены

```
[H2: «Как мы можем работать»]
[Сетка карточек тарифов 3 в ряд]
```

См. структуру карточки тарифа — раздел 4.3.

### 6.6. Обо мне

```
[Фото Ирины] | [H2: «Я Ирина»]
              [2-3 абзаца её голосом — см. voice.md]
              [Цитата-сигнатура крупно]
              [Soft-CTA: «Связаться»]
```

### 6.7. FAQ

```
[H2: «Частые вопросы»]
[Аккордеон 5-8 вопросов]
```

Каждый вопрос — клик-разворачивается. Иконка стрелки поворачивается.

### 6.8. Финальный CTA

```
[H2: «Готова обсудить твой проект»]
[1 строка приглашения голосом Ирины]
[Primary CTA + Secondary CTA]
```

## 7. Состояния (states)

> **Принцип:** интерфейс без состояний — мёртвый. Каждое действие пользователя должно иметь визуальный отклик.

### 7.1. Loading

| Контекст | Что показываем |
|---|---|
| Загрузка страницы | Skeleton-блоки на месте контента (плавная пульсация) |
| Загрузка данных в карточке | Spinner в центре карточки |
| Отправка формы | Кнопка → текст «Отправляю...» + spinner внутри кнопки, кнопка disabled |

**Skeleton** — серая Хвоя 10%, плавная анимация opacity 30%↔50%.

### 7.2. Empty

Когда у пользователя нет данных (например, в личном кабинете):

```
[Иллюстрация-плейсхолдер]
[Заголовок: «Здесь пока пусто»]
[Подсказка: «Купи свой первый продукт →»]
[Primary CTA]
```

### 7.3. Error

Когда что-то пошло не так:

```
[Иконка-плейсхолдер не агрессивная]
[Заголовок: «Что-то пошло не так»]
[Подсказка: «Попробуй ещё раз или напиши мне в Telegram»]
[Secondary CTA: «Попробовать снова»]
[Text-link: «Написать Ирине»]
```

> ❌ НЕ ставим красные восклицательные знаки и страшные сообщения. Бренд Ирины — про спокойствие.

### 7.4. Success

Когда действие удалось:

```
[Иконка ✓ в Кислотно-зелёном]
[Заголовок: «Готово»]
[Подсказка: «Посмотри в почте — отправила тебе всё»]
[Primary CTA: «Что дальше»]
```

### 7.5. Confirmation (подтверждение действия)

Модалка-подтверждение перед опасным действием:

```
[Заголовок: «Точно отменить заказ?»]
[Подсказка: «Деньги вернутся в течение 3 дней»]
[Secondary: «Не отменять»] [Destructive: «Да, отменить»]
```

**Destructive button** — Коралл `#FF935E` (не красный, в палитре).

## 9. Сетка и отступы

| Параметр | Значение |
|---|---|
| **Контейнер десктоп** | max-width 1200px, padding 32px |
| **Контейнер мобильный** | padding 16px |
| **Колонки** | 12-колоночная сетка, gap 24px (десктоп), 16px (мобильный) |
| **Вертикальные отступы между секциями** | 80-120px (десктоп), 48-64px (мобильный) |
| **Внутренние отступы карточек** | 24px |

**Принцип отступов:** удваивай — 4px → 8px → 16px → 24px → 32px → 48px → 64px → 80px → 120px. Не используй промежуточные числа («22px», «38px») — теряется ритм.

## 11. Адаптивность

| Брейкпоинт | Ширина | Что меняется |
|---|---|---|
| **Mobile** | до 640px | 1 колонка, padding 16px, шрифты меньше на 20-30% |
| **Tablet** | 640-1024px | 2 колонки, padding 24px |
| **Desktop** | 1024-1440px | 3-4 колонки, padding 32px |
| **Large** | >1440px | контейнер max-width 1200px, центрирован |

**Принцип:** делаем mobile-first. Сначала всё на мобильном, потом расширяем на десктоп.

## 12. Mini Apps (Telegram + VK Mini App)

> **Особенности:** окружение мессенджера накладывает ограничения. WebView Telegram использует системный шрифт темы (если не override), VK Mini App имеет свою UI-библиотеку (VKUI), от которой можно отталкиваться.

| Параметр | Telegram TMA | VK Mini App |
|---|---|---|
| Системные шрифты | SF Pro / Roboto | SF Pro / Roboto / VK Sans |
| Высота шапки | 56px | 56px |
| Цвет фона по умолчанию | Можно override через `themeParams` | Через VKUI токены |
| Кнопки | Можем стилизовать по нашему дизайну | Используем компоненты VKUI + кастомизация цветов |

**Главное правило:** в мини-аппах сохраняем триаду цветов (Хвоя + Ваниль + Коралл) и иронию того, что мы внутри мессенджера. Не пытаемся «заменить» родной интерфейс мессенджера полностью.

## 13. Что НЕ берём в UI ❌

- **Тяжёлые тени и градиенты** на кнопках (skeumorphism, устарело)
- **Анимации больше 300ms** (тормозят восприятие)
- **Параллакс-эффекты на лендингах** (модно в 2018, в 2026 отвлекает от смысла)
- **Hover-эффекты с поворотом** (`transform: rotate`) на кнопках — выглядит дёшево
- **Цвета вне палитры** — никаких «синих ссылок Bootstrap»
- **Кнопки без текста** на главной CTA (нарушает accessibility)
- **Карточки без hover** — мёртвый интерфейс
- **Auto-play видео со звуком** — выгоняет пользователя со страницы
- **Pop-up модалки на входе** («Подпишись на рассылку!») — раздражают
