# PDF-продукты и документы — применение бренда Ирины

> **Зона:** все документы, которые продаются как продукты или вручаются клиенту — PDF-визитки, чек-листы, гайды, открытки, сертификаты, брендбуки клиентов, презентации, коммерческие предложения.
> **Принцип:** PDF Ирины — это **визитная карточка дизайнера**. Если PDF выглядит «обычно» — клиент решит, что и работа Ирины «обычная». Каждый документ — мини-кейс её эстетики.
> **Версия:** MVP · 2026-05-01

---

## 1. Главные принципы PDF-продуктов

1. **Всегда в палитре.** Никаких «дефолтных Helvetica + чёрный текст на белом» — это сразу убивает бренд.
2. **Контакты обязательны.** В каждом PDF — ссылки на сайт Ирины и кнопка «Не хочешь сам — закажи у нас».
3. **Стиль ≥ Содержание.** Даже если содержание простое (чек-лист на 1 страницу) — оформление должно быть на уровне.
4. **Печать ≠ экран.** Учитываем, что человек может распечатать. Поэтому работаем в pt, а не px. Используем CMYK где можно.
5. **Один шаблон — много продуктов.** Используем `_md2html.py` (Markdown → HTML → PDF) — это уже готовый рабочий конвейер.

## 2. Технический конвейер

### Текущий рабочий стек

| Этап | Инструмент |
|---|---|
| 1. Содержание | Markdown-файл с продуктом |
| 2. Конвертация | `clients/irina/deliverables/_md2html.py` (Python + python-markdown) |
| 3. CSS-стилизация | Встроенный `<style>` в HTML (брендбук-цвета, шрифты, поля) |
| 4. PDF | Печать HTML в PDF через Chrome headless или принт-диалог |

**Команда:**
```bash
cd clients/irina/deliverables
python3 _md2html.py путь/к/исходнику.md имя-выходного-файла.html "Заголовок документа"
# Затем открыть HTML в Chrome → Print → Save as PDF
```

### Параметры PDF (заложены в шаблоне)

| Параметр | Значение |
|---|---|
| Формат | A4 (210×297mm) |
| Поля | 18mm (верх/низ), 16mm (левое/правое) |
| Шрифт основного текста | Helvetica Neue / Helvetica / Arial fallback |
| Размер body | 10.5pt |
| Межстрочный | 1.55 |
| Цвет текста | `#306654` Хвоя (сейчас в шаблоне `#1c1c1c` — обновим в первом продакшене) |
| Цвет заголовков | Хвоя `#306654` |
| Фон | Ваниль `#FCFAE1` (сейчас в шаблоне `#fff` — обновим) |
| Акцент | Коралл `#FF935E` (сейчас `#c99700` — обновим) |

> ⚠️ **Текущий шаблон `_md2html.py`** содержит цвета **палитры Инны** (`#103206`, `#c99700`, `#f4f0e8`). При первом PDF-продукте Ирины **обновляем CSS в шаблоне на её палитру** (`#306654`, `#FF935E`, `#FCFAE1`). Делается одной правкой в одном месте — потом все будущие PDF автоматически в её палитре.

## 3. Типы PDF-продуктов

### 3.1. PDF-визитка эксперта (M19, от 300₽)

**Структура (1-2 страницы A4):**

```
[Шапка с фото + имя]
[Сигнатура / специализация]
[О себе — 2-3 абзаца голосом эксперта]
[Услуги / что делает — 3-5 пунктов]
[Контакты крупно]
[Логотип Ирины-производителя в подвале мелко]
```

**Цвета:** Ваниль фон + Хвоя текст + Коралл акценты.

### 3.2. Чек-лист (C1-C8, от 199₽)

**Структура (1-3 страницы A4):**

```
[Обложка: название чек-листа крупно + цитата-сигнатура]
[Внутри: список пунктов с разметкой ✅ ❌]
[Колонка справа — комментарии «как должно быть / как не надо»]
[Подвал каждой страницы: контакты + блок «Закажи у нас»]
```

**Цвета:** Ваниль фон + Хвоя заголовки + Коралл «✅» отметки.

### 3.3. Гайд / помогалка (H1-H10, от 199₽ до 499₽)

**Структура (3-8 страниц A4):**

```
[Обложка: название + что получит клиент]
[Содержание]
[Раздел 1: Введение — почему важно]
[Раздел 2-N: Пошаговые инструкции с скриншотами]
[Раздел финал: Чек-лист «всё ли сделал»]
[Подвал: контакты + блок «Закажи у нас»]
```

**Цвета:** Ваниль фон + Хвоя заголовки + Молодая зелень для блоков «✓ Ты сделал» + Коралл для CTA.

### 3.4. Открытка (M18, от 200₽)

**Структура (1 страница, ориентация горизонтальная или квадратная):**

```
[Иллюстрация + поздравительный текст]
[Имя получателя]
[Подпись отправителя]
[Логотип Ирины очень мелко в углу]
```

**Цвета:** под повод (свадьба — Коралл + Ваниль + золото; ДР — Коралл + Лимон; НГ — Хвоя + Ваниль + Коралл; 8 марта — Коралл + Маджента + Ваниль).

### 3.5. Сертификат (M17, от 200₽)

**Структура (1 страница A4 горизонтально):**

```
[Декоративная рамка из палитры]
[«Сертификат» крупно]
[Имя получателя]
[За что: текст события]
[Подпись + печать (если есть)]
[Дата]
[Логотип эксперта-заказчика + Логотип Ирины-производителя]
```

**Цвета:** Ваниль фон + Хвоя текст + Коралл рамка + декоративные мазки в палитре.

### 3.6. Конструктор презентации эксперта (M14, от 1000₽)

Не PDF в чистом виде — это интерактивный конструктор, который **на выходе** даёт PDF презентации.

**Структура выходного PDF:**

```
[Слайд 1: Титульный — фото эксперта + имя + тема]
[Слайд 2-N: Контент по структуре презентации]
[Финальный слайд: «Спасибо за внимание» + контакты]
```

**Цвета слайдов:** на выбор клиента из бренд-наборов Ирины (или с применением её цветов клиента, если есть).

### 3.7. Конструктор коммерческого предложения (M16, от 1000₽)

**Структура выходного PDF (3-5 страниц A4):**

```
[Обложка: название проекта + лого заказчика + дата]
[Стр. 2: Понимание задачи (что услышали)]
[Стр. 3: Предлагаемое решение (что сделаем)]
[Стр. 4: Стоимость и сроки]
[Стр. 5: Контакты + почему мы]
```

**Цвета:** под бренд клиента (если у него есть палитра) или дефолтная палитра Ирины.

### 3.8. Брендбук клиента (B1.4, от 20к)

**Структура (15-30 страниц A4):**

```
[00 Обложка]
[01 Манифест клиента (его сигнатура, миссия)]
[02 Логотип и знак (с правилами охранного поля)]
[03 Цветовая система (палитра + правила 60/30/10)]
[04 Типографика (шрифтовая пара + иерархия)]
[05 Графический язык (формы, паттерны, иконки)]
[06 Применение (соцсети, сайт, печать)]
[07 Антипримеры (что НЕЛЬЗЯ)]
[08 Файлы и форматы (что передаём клиенту)]
```

> Брендбуки клиентов Ирины строятся **по структуре её собственного брендбука** (см. `clients/irina/brandbook/README.md`) — это её фирменная методология.

### 3.9. Презентация для эксперта / выступления

**Структура слайдов** (вертикальная или горизонтальная — на выбор клиента):

- 16:9 для онлайн-выступлений (Zoom, YouTube)
- 4:3 для классических презентаций
- 9:16 для сторис-формата

**Цвета:** обязательно палитра клиента + 1-2 акцента.

## 4. Шапка и подвал PDF (универсальный шаблон)

### Шапка (на каждой странице кроме обложки)

```
[Логотип эксперта/Ирины слева]    [Название документа в центре мелко]    [Номер страницы справа]
```

| Параметр | Значение |
|---|---|
| Высота шапки | 12-15mm |
| Цвет линии под шапкой | Хвоя `#306654` 30% прозрачности, толщина 0.5pt |
| Шрифт | Caption, 8-9pt |

### Подвал (на каждой странице)

```
[Контакты слева]            [Сайт в центре]            [«Не хочешь сам — закажи у нас» справа со ссылкой]
```

| Параметр | Значение |
|---|---|
| Высота подвала | 12-15mm |
| Цвет | Хвоя 60% прозрачности |
| Шрифт | Caption, 8-9pt |
| Ссылки | Коралл `#FF935E` |

## 5. Обложки PDF

> **Принцип:** обложка должна **сразу** показать характер документа. Не «название мелким шрифтом в углу» — а громкое визуальное заявление.

### Структура обложки

```
┌──────────────────────────────────────┐
│ [Логотип Ирины небольшой в углу]    │
│                                      │
│  [Иллюстрация / графический акцент   │
│   занимает 40% высоты]               │
│                                      │
│  НАЗВАНИЕ                            │  ← H1, 32-40pt, Bold, Хвоя
│  ДОКУМЕНТА                           │
│                                      │
│  Подзаголовок / описание             │  ← Body, 14pt, Regular, Хвоя 80%
│  что получит клиент                  │
│                                      │
│  «Слышу, какого человек цвета»       │  ← Italic / Caveat, Коралл
│                                      │
│  [Ваниль фон + триада цветов снизу]  │
└──────────────────────────────────────┘
```

## 6. Иллюстрации в PDF

> **Принцип:** иллюстрации в продуктах Ирины — её преимущество перед конкурентами. Используем — но не переборщить.

### Где иллюстрации обязательны

- Обложка PDF (40-50% высоты)
- Открытки (главный визуальный элемент)
- Брендбуки клиентов (на каждой странице блока «Применение»)
- Коммерческие предложения (1 на блок)

### Где иллюстрации опциональны

- Чек-листы (1 на странице, для эмоции)
- Гайды (по теме блока, 1 на 2-3 страницы)
- Презентации (на главных слайдах)

### Где иллюстраций НЕТ

- PDF-визитки экспертов (фото клиента + графика — не иллюстрация)
- Сертификаты (декоративная рамка вместо иллюстрации)

### Стиль

См. `logo-and-graphics.md` пункт 5 — двойные экспозиции, 3D-объём, мультяшные карикатуры (добрые), мазки сочетающихся цветов.

## 7. Печать (физическая)

> Когда клиент хочет распечатать PDF на бумаге.

### Параметры для типографии

| Параметр | Значение |
|---|---|
| Размер | A4 (стандарт), A5 (открытки), A6 (визитки) |
| Поля для печати | + 3mm bleed с каждой стороны (для типографии) |
| Цветовая модель | CMYK (а не RGB) |
| Разрешение иллюстраций | 300 dpi минимум |
| Шрифты | Embedded в PDF (отправлять только PDF/X-1a или PDF/X-3 для типографии) |

### Бумага (рекомендации Ирины как дизайнера)

| Тип продукта | Бумага |
|---|---|
| Визитки | Матовая 300-350 г/м² |
| Открытки | Матовая или фактурная 250-300 г/м² |
| Сертификаты | Дизайнерская бумага с фактурой 200-250 г/м² |
| Брендбук | Матовая 130-150 г/м² (страницы) + плотная обложка 250+ г/м² |

> 💡 Эту таблицу можно показывать клиенту как часть консультации перед печатью.

## 8. Брендирование (применение бренда заказчика на PDF Ирины)

> **Особенность:** часть PDF-продуктов Ирины — это **продукты эксперта** (визитка, презентация, КП). На них размещён логотип эксперта, а Ирина — мелким копирайтом «дизайн: Ирина Цепаева» в подвале.

### Где Ирина становится «фоном»

- PDF-визитка эксперта → главный — клиент, Ирина в подвале
- Открытка-конструктор → главный — повод, Ирина в углу
- Сертификат → главный — событие, Ирина в подвале
- Конструктор презентации эксперта → главный — клиент, Ирина в подвале
- Конструктор КП → главный — клиент, Ирина в подвале

### Где Ирина — главное

- Чек-листы и гайды самой Ирины (C1-C8, H1-H10) → главный — она, контакты крупно
- Брендбук Ирины (внутренний) → она

### Формула «дизайн: Ирина»

```
Дизайн и разработка: Ирина Цепаева
[ссылка на сайт]   |   [Telegram]
```

| Параметр | Значение |
|---|---|
| Размер шрифта | 8pt |
| Цвет | Хвоя 60% прозрачности |
| Расположение | Подвал, центр или справа |

## 9. Антипримеры — как НЕ делать PDF под Иринин бренд

### ❌ Плохо

- Обложка с заголовком 14pt в углу — выглядит как ворд-документ
- Чёрный текст `#000000` на белом фоне — стерильно, не Ирина
- Times New Roman 12pt — это не дизайнерский PDF, а «отчёт»
- Содержание написано CapsLock-ом всё подряд — кричит
- Иллюстрации из Pinterest без разрешения — юридически и эстетически плохо
- Несколько шрифтов разных стилей (Sans + Serif + рукописный + готический) — шум
- Ярко-синие гиперссылки в стиле «по умолчанию» — выпадает из палитры
- Подвал отсутствует на страницах кроме первой — теряется бренд

### ✅ Хорошо

- Обложка с крупным H1 (32-40pt) и иллюстрацией
- Цвет текста — Хвоя `#306654`, фон — Ваниль `#FCFAE1`
- Шрифты по выбранной паре из `typography.md`
- Заголовки H1-H4 в иерархии, не «всё одинаковое»
- Авторские иллюстрации Ирины
- 2 шрифта максимум (заголовочный + основной)
- Гиперссылки в Коралле `#FF935E`
- Подвал на каждой странице с контактами

## 10. Чек-лист перед сдачей PDF клиенту

- [ ] Цвета по палитре Ирины (или клиента, если это его бренд)
- [ ] Шрифты — по типографической паре, не дефолтные системные
- [ ] Заголовки имеют иерархию H1 > H2 > H3 (не одного размера)
- [ ] Body 10.5pt, межстрочный 1.55 (по шаблону `_md2html.py`)
- [ ] Поля A4: 18mm/16mm
- [ ] На каждой странице (кроме обложки) — шапка и подвал
- [ ] Подвал содержит: контакты Ирины ИЛИ контакты эксперта (если это продукт эксперта)
- [ ] В чек-листах/гайдах есть блок «Не хочешь сам — закажи у нас»
- [ ] Иллюстрации в высоком разрешении (если для печати — 300 dpi)
- [ ] Гиперссылки активны (кликаются в PDF)
- [ ] Шрифты embedded в PDF (отправляется не «ссылка на шрифт», а сам шрифт)
- [ ] Если для печати — bleed 3mm + цветовая модель CMYK
