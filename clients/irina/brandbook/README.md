# Брендбук Ирины Цепаевой — рабочая карта

> **Назначение:** внутренний рабочий брендбук для команды агентов. Не маркетинговый PDF на показ клиентам — это будет сгенерировано отдельно по запросу.
> **Принцип:** модульный. Каждый агент берёт **только нужный модуль(и)**, а не тянет весь брендбук. Экономит токены, ускоряет работу, фокусирует.
> **Версия:** MVP-5 · 2026-05-01
> **Статус:** базовый рабочий минимум собран. Расширяется по мере появления конкретных задач.

---

| Модуль | Что внутри | Размер |

**Итого MVP-5 ≈ 2150 строк.** Каждый агент в среднем тянет 2-3 модуля = 500-1000 строк за вызов.

## 🎯 Главные принципы бренда (краткая выжимка)

> Если у агента нет времени читать модули — это **минимум**, который он должен помнить.

### 1. Сигнатура

> **«Слышу, какого человек цвета.»**

Это не слоган — это рабочий принцип. Ирина **слышит** клиента → видит его в цветах → переводит в визуал. Любой текст, любой продукт под Ирину должен звучать через эту призму.

### 2. Триада цветов бренда

**Сосновая хвоя** `#306654` + **Ваниль** `#FCFAE1` + **Тёплый коралл** `#FF935E`

Все три цвета на одном макете = безошибочно «это Ирина». Полная палитра — в `palette.md`.

### 3. Голос — личностный, не корпоративный

«Мне нравится», «я люблю», «то есть», «всегда по-разному». Никаких «мы предлагаем», «компания осуществляет», «эффективные решения». Полный гид — в `voice.md`.

### 4. Методология — С.С.С.Р. в визуале

**Структура · Стиль · Свобода · Результат.** Это рабочий процесс Ирины. Любой продукт встраивается в эту систему. Подробно — в `architecture.md` Слой 1.4.

### 5. Юридическая мантра самозанятой

Ирина **«изготавливает дизайн / разрабатывает веб-приложения / разрабатывает чат-боты / верстает / иллюстрирует»**. Запрещены слова: «провожу аудит», «консультирую», «разрабатываю стратегию», «сопровождаю». Подробно — `legal-okved.md`.

## 🔄 Что добавляется по мере появления задач

Это «следующая волна» модулей — НЕ делаем вперёд, делаем когда появилась живая задача:

| Модуль | Триггер для создания |
|---|---|
| `social.md` | Когда соберём VK-упаковку Ирины (в плане на 2026-05-01) |
| `ui-components.md` | Когда сделаем первый лендинг Ирины или TMA |
| `documents.md` | Когда оформим первый PDF-продукт (визитка, чек-лист, гайд) |
| `dont.md` (антипримеры) | Накапливается из ошибок по ходу — каждый раз когда что-то пошло «не так», добавляем сюда |

**Принцип:** документ растёт вместе с задачами, а не «на бумаге заранее». Так каждый модуль остаётся **рабочим**, а не теоретическим.

## 🔁 Как обновлять

1. **Каждый модуль — отдельный файл.** Меняешь только нужный, не правишь весь брендбук.
2. **Версионируется через git.** История правок видна в `git log clients/irina/brandbook/<file>.md`.
3. **Сверочные встречи** Инны и Ирины — по итогам обновляем модули и помечаем версию (v1.1, v1.2 и т.д.) в шапке файла.
4. **Если правишь правило, которое противоречит существующему** — сначала обсуди с Ириной, она дизайнер и её слово ведущее по визуалу/UX (см. `feedback_irina_designer.md` в памяти).
