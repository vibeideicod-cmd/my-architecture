# Research: цифровая визитка Ирины Цепаевой

> **Назначение:** входные данные для product-builder и Кодыча (Websites) при сборке цифровой визитки.
> **Цель визитки:** за 2 секунды показать «супер дизайнер» визуально, до прочтения текста.
> **Контекст:** Ирина — дизайнер с 5+ лет опыта, специализация — брендбуки, фирменный стиль, лендинги, Telegram/VK Mini App, AI-помощники. ЦА — собственники готового бизнеса из 8 ниш (инфобизнес, финансы, бьюти, коучи, фитнес, торговля, риэлторы, санатории).
> **Палитра проекта:** коралл `#FF935E` · хвоя `#306654` · ваниль `#FCFAE1` · лимон `#F8EB00` · маджента `#FF2EC4` · неон `#13F740`. Шрифты — Playfair Display + Montserrat. TOV — на «вы», тёплый премиум-эксперт.
> **Версия:** v1.0 · 2026-05-03 · Analyst
> **Метод:** WebSearch + WebFetch по реальным источникам, никаких выдуманных URL.

Из 5 направлений Playbook (конкуренты, функции, UX, технологии, монетизация) для одностраничной визитки сворачиваем до 3 наиболее релевантных: конкуренты-бенчмарки, UX-сценарии, приёмы-тренды.

## 1. Конкуренты и бенчмарки

Подобраны 7 примеров: 5 западных лауреатов Awwwards 2025–2026, 2 русских ориентира. Для каждого: ссылка, «вау»-приёмы, что подойдёт Ирине, что не брать.

### 1.1. Mark Vogelaar — `markvogelaar.com`

**Awwwards:** Personal Design Portfolio, Honorable Mention.
**Вау:**
- Один доминирующий цвет на всём сайте (ярко-красный `#ff213b`) держит идентичность: страница узнаётся за 1 секунду по цвету.
- Минималистичный фон, акцентный цвет работает «маркером бренда» — ту же логику Ирина может применить с `#FF935E` коралл/`#306654` хвоя.
- Сделан на Framer — много встроенных микроанимаций без перегруза.

**Что подойдёт Ирине:**
- Принцип «один цвет = бренд». На Иринином сайте такой цвет — коралл `#FF935E` или хвоя `#306654` (одна доминанта в визуальном впечатлении первого экрана, а не равная мешанина).
- Чистый минималистичный фон + цветной акцент — ровно её формула «ваниль 60-70% + хвоя/коралл».

**Что не брать:**
- У Vogelaar холодная сдержанность — он не обогревает страницу. У Ирины наоборот, тёплое премиум: к одному цвету добавляем мягкие коралловые радиальные градиенты (по `visual-language.md` §2.1).

### 1.2. Artiom Yakushev — `art-yakushev.com`

**Awwwards:** Portfolio Honors, декабрь 2025; Site of the Day.
**Вау:**
- Hero — крупный заголовок «As a digital designer, I help brands…», разбитый на несколько строк с визуальным ритмом. Никакого слайдера.
- В hero есть «текущее время в Ереване (18:32 am)» — мелкая человеческая деталь, делает страницу живой.
- Соцссылки в hero (Instagram, Behance, LinkedIn) — посетитель сразу понимает, куда писать.
- Скролл — постепенное «распаковывание» проектов с мягкими fade-in.

**Что подойдёт Ирине:**
- **Заголовок-манифест в 2-3 строки** в hero вместо слайдера. У Ирины: «Слышу, какого человек цвета. Делаю бренды и интерфейсы для тех, кто работает 24/7» (на Playfair Display, по типошкале §5.2 visual-language).
- **«Живая» деталь в hero** — у Ирины это может быть текущее время / город / статус «беру 1 проект в месяц». Это и сигнатура, и сигнал «я конкретный человек, не агентство».
- **Ссылки на Telegram/VK прямо в hero** — без скролла до футера. Это критично для конверсии (см. UX-сценарий 1).
- **Sequential reveal на скролле** — уже описан в `visual-language.md` §6.2 (page-load orchestration), берём.

**Что не брать:**
- У Yakushev почти нет цвета (белый + чёрный + минимальные акценты). У Ирины наоборот — палитра должна быть видна. Минимализм да, но тёплый минимализм с ванилью + хвоей + коралловой акцентной массой.
- ALL CAPS в навигации — у Ирины запрещён CAPS LOCK на длинных фразах (`dont.md` §4). Только короткие шильдики и кнопки.

### 1.3. Elliott Mangham — `elliott.mangham.dev`

**Awwwards:** Portfolio Honors, ноябрь 2025; 18× Site of the Day.
**Вау:**
- Профессиональное фото в hero + краткое позиционирование «Creative Web Developer» + статус доступности «available April 2026».
- Прогресс-бар загрузки (0%–80%) при первом заходе — премиум-приём «не обычный сайт».
- Двойной CTA в hero: «Schedule call» + «Cost estimate» — две скорости для разных типов лидов.
- Список клиентов (Disney, Universal Music) и счётчик наград как social proof выше первого экрана.

**Что подойдёт Ирине:**
- **Фото Ирины в hero** — критично. Лицо бренда видно сразу. Сейчас в `visual-language.md` §3.6 описан Inset-glow на портрете — это и есть применение.
- **Статус доступности** — «беру 1 проект в мае», «следующий слот — июнь». Создаёт ощущение премиум-эксперта (не «возьмётся за всё»).
- **Двойной CTA**: «Написать в Telegram» (быстрый разговор) + «Скачать презентацию» / «Посмотреть кейсы» (для тех, кто пока изучает). См. UX-сценарий 3.

**Что не брать:**
- Loading прогресс-бар 0-80% — на одностраничной визитке избыточно, тормозит первое впечатление. Заменяем page-load orchestration’ом (sequential reveal без явного «лоадера»).
- Длинный список логотипов клиентов на западный манер — у Ирины ЦА часто не публичная (бьюти-мастер из Краснодара, риэлтор из Сочи), показывать названия не получится. Заменяем на формат «8 ниш бизнеса × число выполненных проектов».

### 1.4. Gavin Schneider Productions — `gsproductions.co.za`

**Awwwards:** Portfolio Honors, февраль 2026; Developer Award; Site of the Day.
**Вау:**
- Полноэкранное фото проекта в hero — главным сразу становится визуальный результат, а не фото автора.
- Минималистичная навигация: только название + переключатель «List / Grid».
- Hover на проекте подсвечивает мета (клиент, фотограф) — никаких декоративных оверлеев.
- Сетка кейсов — асимметричная, поэтому глаз не залипает, а ведёт.

**Что подойдёт Ирине:**
- **Полноэкранный главный кейс** в начале блока «Работы» — после hero сразу одна большая работа во всю ширину, потом сетка. Это зона «вау»: посетитель ещё не доскроллил до подробностей, но уже увидел уровень.
- **Асимметричная сетка кейсов** — тренд 2026 (см. §3.4). Вписывается в палитру: ванильный фон, коралловые подписи, скруглённые карточки 16-20px (`ui-components.md`).
- **Минималистичная навигация** — только логотип/имя слева и Telegram/VK справа. Без меню «Услуги/Цены/Контакты» во весь экран — для одностраничника лишнее.

**Что не брать:**
- У Gavin белый фон и чёрный текст — холодная редактура. У Ирины фон ванильный, текст хвоя — те же принципы, но в её палитре.

### 1.6. Bruno Simon — `bruno-simon.com`

**Awwwards:** Portfolio Honors, декабрь 2025; Site of the Month.
**Вау:**
- 3D-гоночный мир, которым посетитель управляет автомобилем для исследования портфолио. Three.js + Rapier (физика) + Howler.js (звук).
- Геймификация: достижения, гоночная схема, «Whispers» (сообщения от других посетителей до 30 символов).
- Полностью open source под MIT.

**Что подойдёт Ирине:**
- **Принцип «портфолио — это опыт, а не каталог»** — да. У Ирины это реализуется не 3D-машинкой, а интерактивной палитрой («Слышу, какого человек цвета»: скроллим — фон меняет оттенок коралл → хвоя → маджента в зависимости от секции). Один точечный wow-эффект.

**Что НЕ брать (категорически):**
- 3D-движок и геймификация — это дорого, тяжело, и противоречит образу «тёплый премиум-эксперт». Bruno строит образ программиста-аттракциона; Ирина строит образ дизайнера, к которому идут за вкусом, не за фейерверком.
- Ставим как пример «куда не надо» — слишком много шоу, мало смысла для бизнес-клиента.

## 2. UX-сценарии

3 типичных пути захода на цифровую визитку дизайнера. Для каждого — что заставляет написать, что критично, что бесит.

### 2.1. Сценарий 1 — «Получил ссылку в Telegram/WhatsApp от знакомого»

**Контекст:** к Ирине порекомендовали через знакомого. Человек в чате видит «Вот хорошая дизайнерша — irinatsepaeva.ru». Кликает с телефона, в одной руке.

**Что делает:**
1. Жест: открывает на телефоне, сразу скроллит большим пальцем.
2. Сканирует hero за 2-3 секунды: видит фото, имя, что делает, цвет страницы.
3. Свайпает 1-2 экрана: ищет «свежие работы» и «как написать».
4. Если зацепило — сразу нажимает на иконку Telegram, не возвращаясь к чату.
5. Если не зацепило — сворачивает вкладку, возвращается в исходный чат.

**Что работает:**
- Hero, который грузится за < 2 секунд (мобильный 4G).
- Лицо Ирины + 1 строка позиционирования + кнопка Telegram **в первом экране без скролла**.
- Sticky-bar внизу с Telegram/VK иконками в фирменных цветах (`#229ED9`/`#0077FF`) — поэтому кнопка «написать» под пальцем всегда.

**Что бесит:**
- Pop-up «Подпишись» через 5 секунд (запрещён в `dont.md` 6.4).
- Auto-play видео со звуком (запрещён там же).
- Cookie-баннер во весь экран без понятной кнопки «принять».
- Hero без фото — «непонятно, кто это».
- 5 секунд лоадера прежде чем что-либо появится.

**Критично для конверсии:**
- **Ссылка на Telegram должна быть native — `tg://resolve?domain=irina_…` для мобильных, fallback на `https://t.me/irina_…`.** Без копирования ника руками.
- Sticky-меню «Написать» в нижней трети экрана на скролле.
- В hero — статус доступности («Беру 1 проект в мае») для премиум-впечатления.

---

### 2.2. Сценарий 2 — «Пришёл с поста в VK или Telegram-канале»

**Контекст:** Ирина запостила кейс с CTA «подробности на сайте». Аудитория — её подписчики, тёплая, но ещё не готова писать «в лоб».

**Что делает:**
1. Кликает по ссылке из поста, ожидая увидеть **тот самый кейс** или развёрнутый рассказ о нём.
2. Если попадает на главную, а не на конкретный кейс — фрустрация, скролл вниз искать.
3. Если кейс есть — изучает фото/процесс, потом возвращается наверх к hero.
4. Решает: «нравится её стиль в целом» → смотрит другие работы → пишет в Telegram, или: «нет, не моё» → закрывает.

**Что работает:**
- Якорные ссылки на кейсы (`/#case-beauty-tma`, `/#case-brandbook-sanatorium`) — пост в VK ведёт прямо на нужный кейс, не на главную.
- Кейс — большой полноэкранный блок: «До → После», 2-3 фото процесса, краткий комментарий Ирины.
- В конце кейса — кнопка «Хочу так же» с переходом в Telegram + автотекст «Пришёл по кейсу [название]».

**Что бесит:**
- Пост обещает «подробности», а сайт — общая визитка без подробностей.
- Кейс открывается в новой вкладке через лайтбокс — на мобильном это неудобно.
- Между постом и сайтом разный тон («вы» в посте → «ты» на сайте, или наоборот) — обрыв доверия.

**Критично для конверсии:**
- Каждый крупный кейс на визитке имеет уникальный анкор-URL.
- TOV на сайте = TOV в соцсетях («вы» + тёплый премиум, по `tov-source.md`).
- В кейсе понятно, **кто заказывал и какую задачу решали** — даже если бренд клиента не назван («бьюти-салон в Москве», «риэлтор премиум-сегмента»).

---

### 2.3. Сценарий 3 — «Холодный посетитель из поиска или рекламы»

**Контекст:** человек загуглил «дизайнер брендбук Краснодар», попал на сайт Ирины. Доверия нет, есть скепсис.

**Что делает:**
1. На hero оценивает: «выглядит ли это профессионально».
2. Скроллит вниз быстро, ищет: что делает + цены + кейсы + отзывы.
3. На блоке услуг — пытается понять «делает ли она то, что мне надо».
4. На блоке кейсов — оценивает уровень и «выглядит ли как мой бизнес».
5. Если пошло — медленнее изучает конкретный кейс.
6. Решение «писать или нет» — обычно после 60-90 секунд на сайте.
7. Если пишет — обычно в форме «расскажите о…» (не сразу в Telegram).

**Что работает:**
- Чёткий блок «Что я делаю» — 4 продукта линейки (брендбук, лендинг, мини-апп, AI-помощник) с короткой формулировкой и от какой суммы / срока.
- Социальное доказательство: 2-3 отзыва с фото клиента и его нишей (бьюти-мастер из Москвы, тренер из Сочи).
- FAQ: «как мы работаем», «что входит в оплату», «сколько правок» — снимает базовые возражения.
- Двойной CTA: «Написать в Telegram» (быстро) + «Заполнить бриф» (для тех, кто хочет подумать).

**Что бесит:**
- Цены прячутся за «уточняйте у менеджера» — для холодного посетителя это red flag.
- 5 страниц лендинга пустых обещаний без конкретики.
- «Свяжитесь со мной любым удобным способом» — нет конкретного канала.
- Кейсы только в виде ссылок на Behance — холодному посетителю лень переходить.

**Критично для конверсии:**
- Конкретные диапазоны цен («Брендбук от 50 000 ₽», «Лендинг от 30 000 ₽»).
- Кейсы прямо на странице (не только ссылки наружу).
- Альтернативный CTA для тех, кто стесняется писать в Telegram — форма-бриф с понятными полями.
- Юр.чистота формулировок Ирины («изготавливаю», «разрабатываю» — не «провожу аудит» — по `clients/irina/CLAUDE.md`).

### 2.4. Сводный список «что критично для конверсии в написал»

1. **Hero загружается за < 2 секунд** на мобильном 4G — иначе уходят.
2. **Фото Ирины + имя + 1 строка позиционирования + Telegram-кнопка** видны без скролла.
3. **Sticky-bar внизу** с Telegram/VK на мобильном на всех экранах кроме hero.
4. **Якорные ссылки на кейсы** для перехода с постов соцсетей.
5. **Двойной CTA**: быстрый (Telegram) + медленный (бриф / скачать презентацию).
6. **Конкретные цифры в услугах** — диапазоны цен, не «договорная».
7. **Юридически чистый словарь Ирины** («изготавливаю», «разрабатываю»), не путать с консультантским словарём Инны.

## 3. Приёмы-тренды 2025–2026 для цифровой визитки дизайнера

Источники: Awwwards (примеры выше), Figma resource library «Web Design Trends 2026», Envato «Web design trends 2026», Muzli Blog «Web Design Trends 2026», Showit, Colorlib «Portfolio Design Trends 2026». Тренды отсортированы по применимости к Ирине.

### 3.1. Типографика как hero-элемент (тренд №1 для дизайнеров)

**Что в тренде:**
- Variable fonts с кинетической анимацией.
- Drop caps (буквица в первом абзаце) — возвращение редакторского приёма.
- Editorial-вёрстка: типографика «несёт страницу», изображения — поддержка.
- Бесшовный микс serif + sans (Playfair + Montserrat — ровно эта пара).

**Подходит Ирине:**
- **Drop cap «И» в первом блоке** после hero — Playfair Display Bold, цвет коралл, размер 5-6 строк. Это и буквица, и микро-логотип одновременно.
- **Текст-перелив коралл → маджента на одном слове H1** — уже описан в `visual-language.md` §2.6.
- **Variable font для Playfair Display** — есть в Google Fonts, бесплатно, даёт плавную анимацию веса при hover на главном слове.

**НЕ подходит:**
- Кинетическая типографика «летит во все стороны» — это молодёжный хаос (Gen Z), не премиум.
- Drop cap не применять на каждом абзаце — только в первом блоке.

---

### 3.2. Mesh-градиенты + warm neutrals — главный цветовой тренд 2026

**Что в тренде:**
- Mesh-градиенты вместо плоской заливки (Figma blog 2026).
- Muted earth tones: оливковый, taupe, бежевый, тёплый серый.
- Сочетание ярких неоновых акцентов с приглушённой базой («бренд жив, но не кричит»).
- Цветной шум поверх градиента (noise overlay) — убирает «пластиковость».

**Подходит Ирине:**
- **Hero-радиальный mesh** «Закат-коралл» (`visual-language.md` §2.1) — точно в тренде.
- **Ваниль + хвоя как warm neutrals** — попадание в тренд один-в-один.
- **Маджента + неон-зелёный как точечные wow-акценты** — Ирина уже задумала эту схему, тренд её подтверждает.
- **Noise overlay 0.05** на ванильном фоне (`visual-language.md` §4.1) — обязательно.

**НЕ подходит:**
- Радужные градиенты из 5+ цветов (запрещены в `dont.md` 3.3).
- Холодный синий/бирюзовый mesh (вне палитры).

---

### 3.3. Scroll-trigger анимации с задержкой (sequencing)

**Что в тренде:**
- Page-load orchestration: hero появляется лесенкой 60-100ms между элементами.
- IntersectionObserver на скролл — fade + slide на 16-20px.
- Сайты со scroll-trigger анимациями имеют +30% session time (исследование Figma 2026).

**Подходит Ирине:**
- **Page-load sequencing в hero** — уже описан в `visual-language.md` §6.2. Берём.
- **`.in-view` через IntersectionObserver** — для каждой секции после hero (`visual-language.md` §6.4). Берём.

**НЕ подходит:**
- Параллакс на каждой секции (запрещён в `dont.md` 6.3).
- Анимации > 300ms (запрещены там же).

---

### 3.4. Асимметричная сетка + scroll-snap (тренд для портфолио)

**Что в тренде:**
- Asymmetric grid: главное изображение слегка off-center, текст наезжает на край.
- Scroll-snap по секциям на десктопе — каждая секция «приклеивается» при скролле.
- Полноэкранные секции на мобильных, асимметрия только на десктопе.

**Подходит Ирине:**
- **Асимметричная сетка кейсов** — после первого полноэкранного главного кейса 2-3 ряда работ в шахматной разнонаклонной сетке (большой кейс + 2 средних + 1 широкий). Это «вау» без шоу.
- **Scroll-snap по крупным секциям** на десктопе (hero / кейсы / услуги / контакт) — даёт ощущение «премиум-журнала».

**НЕ подходит:**
- Жёсткий grid Bootstrap-стиля — не Иринин язык (`visual-language.md` §4.3 запрещает).
- Scroll-snap на мобильном — мешает обычному скроллу.

---

### 3.5. Custom cursor — точечно, только на hero

**Что в тренде:**
- Кастомный курсор (магнит, кружок, инверсия) — обязательный приём премиум-портфолио.
- Используется на hero и на ховере по интерактивным элементам.
- На touch-устройствах отключается автоматически.

**Подходит Ирине:**
- **Коралловый кружок 24×24px на hero** — уже описан в `visual-language.md` §6.5. Решение «использовать или нет» — за Ириной, но тренд за.
- **Magnetic effect на главной CTA-кнопке** — кнопка «притягивается» к курсору на 5-10px при приближении. Премиум-микро-деталь.

**НЕ подходит:**
- Custom cursor на всём сайте — отвлекает от чтения текста.
- Курсор-эмодзи или анимированный гиф — слишком игриво для премиум-эксперта.

---

### 3.6. AI-стиль и интерактив без шоу

**Что в тренде:**
- AI-помощники прямо на странице (мини-чат в углу).
- Интерактивные демки (палитра, превью брендбука).
- «Опыт, а не каталог» — посетитель что-то делает, а не только листает.

**Подходит Ирине (точечно):**
- **«Слышу, какого человек цвета» как интерактив** — на отдельной секции посетитель выбирает 3 эмоции / прилагательных, а сайт показывает 3-4 палитры из её работ, попадающие в эти эмоции. Это и «вау», и сигнатура бренда, и AI-демка одновременно. Можно реализовать на чистом JS без AI на бэкенде.
- **AI-помощник Ирины** — отдельный модуль, не в первой версии визитки. Указано как будущее в `clients/irina/CLAUDE.md` Решение 2.

**НЕ подходит:**
- 3D-аттракцион Bruno Simon-стиля — не Иринин формат.
- Полноэкранный AI-чат «привет, я AI-помощник, чем помочь» как hero — холодный tech, не тёплый премиум.

---

### 3.7. Тренды, которые НЕ берём для Ирины

Из общих 2026-трендов (Figma, Envato, Muzli) для Ирины **отказываемся** от:

- **Brutalism / broken grids** — модно, но молодёжный хаос (Envato 2026). У Ирины тёплый премиум.
- **Glitch-эффекты, VHS-noise** — холодный tech, не её бренд.
- **Y2K-эстетика, неоновая хирома** — слишком тренд-зависимо, бренд должен жить 3+ года.
- **Полностью тёмная тема** — Иринин фон ванильный, не тёмный (`visual-language.md` §0).
- **Heavy 3D, Three.js, WebGL** — дорого, медленно, перегруз.
- **Auto-play видео в hero** — запрет в `dont.md`.
- **Маскотов и AI-сгенерированных персонажей** — у Ирины человек, не маскот.

## 4. Резюме для product-builder и Кодыча

### 4.1. Топ-3 приёма, которые точно берём в визитку Ирины

**1. Hero как 1-экран-всё-видно: фото Ирины + манифест на Playfair Display + статус доступности + двойной CTA в Telegram/VK.**
*Обоснование:* в UX-сценарии 1 (ссылка в Telegram) человек принимает решение «писать или нет» в первые 2-3 секунды, не прокручивая. Если в первом экране нет лица + статуса + кнопки «написать» — конверсия падает. Этот приём подтверждается всеми 5 Awwwards-портфолио (Yakushev, Mangham, Mark Vogelaar, Khodkina, Gavin) и трендом 2026 «Hero как декларация бренда».

**2. Один доминирующий цвет первого экрана + radial mesh-градиент «Закат-коралл» + drop cap «И» в первом блоке после hero.**
*Обоснование:* Mark Vogelaar показывает, что один доминирующий цвет = идентичность за 1 секунду. У Ирины этим цветом становится Коралл `#FF935E` поверх ванильной базы. Drop cap — типографический тренд 2026, который ровно подходит Playfair Display и одновременно работает как мини-логотип «И». Mesh-градиент уже описан в `visual-language.md` §2.1, тренд 2026 (warm mesh + earth tones) подтверждает.

**3. Асимметричная сетка кейсов + полноэкранный главный кейс сразу после hero + якорные URL для каждого кейса.**
*Обоснование:* Gavin Schneider Productions использует именно эту схему и берёт Awwwards. Асимметрия попадает в тренд 2026 «editorial layout». Якорные URL критичны для UX-сценария 2 (приход с поста соцсетей) — без них Ирина теряет тёплых лидов из VK/Telegram.

### 4.2. 2-3 предостережения «вот этого не делаем»

**1. НЕ делаем 3D-аттракцион / геймификацию / Three.js в стиле Bruno Simon.**
*Обоснование:* Bruno Simon показывает, что это работает для разработчика-аттракциона. Ирина — тёплый премиум-эксперт, к которому идут за вкусом и слухом, не за фейерверком. 3D противоречит её TOV «слышу, какого человек цвета». Плюс это дорого, медленно и невозможно поддерживать на чистом HTML+CSS (правило корня).

**2. НЕ делаем холодный минимализм Vogelaar/Mangham-стиля (белый + чёрный + один цветной акцент).**
*Обоснование:* у Ирины должна быть видна вся триада палитры на первом экране (ваниль + хвоя + коралл). Если фон чисто-белый и текст чисто-чёрный — мы теряем тёплое премиум и попадаем в общий «дизайнер-шаблон». Используем ваниль `#FCFAE1` как фон и хвою `#306654` как текст по `visual-language.md`.

**3. НЕ делаем CAPS LOCK на длинных фразах, GIF-loader, pop-up «подпишись через 5 секунд», auto-play видео.**
*Обоснование:* всё это запрещено в `dont.md` §4 и §6.4. Awwwards-примеры (Yakushev) показывают CAPS только в коротких шильдиках 1-3 слова — для Ирины такое допустимо, но не в манифесте hero.

### 4.3. Рекомендация по hero-секции (главный экран)

**Структура hero визитки:**

1. **Фон:** radial-gradient «Закат-коралл» (`visual-language.md` §2.1) поверх ваниль `#FCFAE1` + noise overlay 0.05.
2. **Слева (60% ширины на десктопе, всё на мобильном):**
   - Манифест H1 на Playfair Display 700, 36-72px (clamp): «Слышу, какого человек цвета.»
   - Под ним — drop cap-style второй H2 на Montserrat 500, 22-32px: «Делаю бренды и интерфейсы для тех, кто работает 24/7.»
   - Статус доступности маленьким Montserrat 400 14px цвет хвоя 60%: «Беру 1 проект в мае. Следующий слот — июнь.»
3. **Справа (40% ширины на десктопе, скрыто или ниже на мобильном):**
   - Фото Ирины в овальной рамке с inset-glow (`visual-language.md` §3.6).
4. **Под манифестом — двойной CTA:**
   - Primary: «Написать в Telegram» (фон акид `#4DAA02`, иконка Telegram `#229ED9` слева).
   - Secondary: «Скачать презентацию» (бордер хвоя, текст хвоя, без фона).
5. **Page-load orchestration** (`visual-language.md` §6.2): манифест H1 → 60ms → подзаголовок → 140ms → статус → 220ms → CTA → 300ms.
6. **Custom cursor** коралловый кружок 24×24px только в зоне hero — *опция, на согласование с Ириной*.
7. **Sticky-bar внизу мобильного** с иконками Telegram/VK на всех экранах кроме hero.

**Что главное — за 2 секунды посетитель должен понять:**
1. *Это супер-дизайнер* — по типографике (Playfair Display + Montserrat), палитре (ваниль + хвоя + коралл) и фото.
2. *Это конкретный человек, а не агентство* — по фото, имени и статусу доступности.
3. *Можно написать прямо сейчас* — по кнопке Telegram в первом экране без скролла.
