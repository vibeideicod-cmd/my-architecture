## 1.3. Голос бренда vs голос юридический

**Голос наружу (на лендингах, в соцсетях, в текстах):**
> «Слышу клиента → создаю под него → передаю готовый продукт.»

**Голос в чеках, оферте и договорах** — только разрешённые виды деятельности самозанятой:

| Юр.конструкция | Что покрывает |
|---|---|
| **«Изготовление дизайна»** | Брендбук, фирстиль, визитка, иллюстрации, макеты |
| **«Разработка веб-приложения»** | Лендинг, квиз, мини-сайт, мини-приложения внутри Telegram и VK |
| **«Разработка чат-бота»** | Бот для записи, бот-прогрев, AI-помощник под клиента |
| **«Вёрстка веб-страницы»** | HTML-вёрстка лендингов, цифровых визиток |
| **«Иллюстрация и художественное оформление»** | Иллюстрации, серии, баннеры |

**Запрещённые слова:** «провожу аудит», «консультирую», «разрабатываю стратегию», «управляю проектом», «сопровождаю внедрение». Подробно — `legal-okved.md`.

# ЧАСТЬ 3. КАТАЛОГ ПРОДУКТОВ — ПУЛ ИДЕЙ К ВЫБОРУ

## 🌟 Главный принцип всех продуктов — «вау, как круто»

> **Правило (от Инны):** любой продукт Ирины должен **кричать о технологичности**, а не быть будничным. Не просто бот / калькулятор / шаблон — а **игровой, увлекающий, с эффектом «вау»**. Клиент должен хотеть похвастаться: «смотри что я сделал за копейки», «смотри какой инструмент я даю своим ученикам».

**Что это значит на практике:**
- Любой калькулятор / трекер / дневник → **геймификация** (баллы, уровни, серии, бейджи)
- Визуальный результат — анимации, прогресс-бары, празднования достижений
- Не «введи цифры → получи цифры», а «введи → увидь визуальный результат → захоти показать друзьям»
- Все продукты — **с шильдиком «работает на нейросетях / ИИ»** (там где это правда), это маркер новизны
- Личный кабинет везде, где есть смысл — кабинет тренера + кабинет ученика, кабинет эксперта + кабинет его клиентов

**Что НЕ берём в линейку:**
- Будничные финансовые калькуляторы «для себя» (личный бюджет, налоги, бюджет свадьбы) — это считать цифры, скучно, не Иринина зона
- Скучные шаблоны без интерактива
- Боты, где «спросил — ответил» без эмоции

### 🎮 Интерактивные платформы с геймификацией (для тренеров / коучей / преподавателей)

> **Логика (от Инны):** у тренеров и коучей частые проблемы — низкая доходимость до встреч, ученики не делают ДЗ, теряется вовлечённость. ## 🧭 Навигация каталога на сайте — два среза

> **Логика (от Инны):** на сайте каталог продуктов даётся в двух разрезах одновременно. Клиент сам выбирает удобный.

### Срез 1. По типу продукта (стандартная навигация)

Категории: Визитки → Брендбуки → Лендинги → Боты → Мини-приложения → Гайды → Калькуляторы → Шаблоны → Платформа

### Срез 2. По боли клиента — лестница решений

При выборе боли клиент видит **полную лестницу решений** под эту боль — от микро-продукта до флагмана:

```
Боль «Меня не видят»
   ↓
   🟠 Микро (за 199-1000₽): Гайды по самопрезентации, генератор аватара,
      шаблон визитки, AI-генератор палитры
   ↓
   🟡 Среднее (5-40к): Мини-визитка, упаковка одной соцсети,
      визитка в TMA, мини-курс «Бренд за 7 дней»
   ↓
   🟢 Крупное (60-200к): Полный брендбук + упаковка соцсетей,
      ребрендинг, флагман
   ↓
   🚀 Флагман (300к+): Полная визуальная экосистема
```

Это нужно отразить на странице каждого лендинга боли (Слой 5 архитектуры). Не один продукт — а **лестница** с понятным переходом «купи дёшево → попробуй среднее → возьми крупное».

## ⚠️ Открытые вопросы перед запуском

### Экономика микро-продуктов 🔴

Прежде чем запускать любой микро-продукт за 100-500₽, **через Financial-агент посчитать юнит-экономику**. Вопросы:

- **Себестоимость операции:** комиссия платёжной системы (~3%), стоимость API-вызовов (для AI-генераторов), хостинг — сколько остаётся с одного оплаченного продукта в 200₽?
- **Нижний порог цены:** при какой цене продукт **уходит в минус**? (если AI-вызов стоит 50₽, а продукт за 200₽ → плохая экономика)
- **Минимальный тираж:** сколько штук нужно продать в месяц чтобы микро-направление окупало хостинг + поддержку?
- **Группировка:** возможно, выгоднее **подписка на доступ ко всем микро-инструментам сразу** (например, 999₽/мес = безлимит на все генераторы), чем разовые продажи

**Задача Financial:** просчитать модель для типового микро-продукта (палитра-генератор / гайд-самоучка / калькулятор) и дать «зелёный свет» / «красный» / «нужен рефакторинг цены».

### Технический формат реализации 🟡

Часть микро-продуктов проще как **отдельные мини-сайты** (один лендинг = один продукт), часть — как **разделы на одной платформе** «Сделать сейчас». Решение принимается через Systems-агент + DevOps/Infra при сборке Слоя 5.

### Юнит-экономика средних продуктов 🟡

Боты-администраторы 30-40к — нужно посчитать через Pricing-Specialist:
- сколько часов команды на типовой бот?
- какая маржа на одну продажу?
- какая модель сопровождения после сдачи (подписка / разовый / гибрид)?
