# Бриф: цифровая визитка Ирины Цепаевой

> **Тип продукта:** одностраничная цифровая визитка (HTML+CSS+JS, без фреймворков)
> **Этап MVP:** один HTML-файл на корне `stydiyatsi.ru`
> **Будущие этапы:** адаптация под Telegram Mini App (этап 2) и VK Mini App (этап 3)
> **Автор брифа:** product-builder
> **Получатель:** Кодыч (Websites)
> **Версия:** v1.1 · 2026-05-03 (после Critique)
> **Цель:** Кодыч читает этот документ + `visual-language.md` и стартует без уточняющих вопросов.

---

## 0. Ключевая задача

Собрать одностраничную цифровую визитку Ирины Цепаевой, которая за **2 секунды** показывает «супер дизайнер» визуально (палитра + типографика + фото + статус), до того как посетитель прочитает текст. Главная конверсия — нажатие на Telegram или VK в hero или в закрепе мобильного.

**Что строим:** HTML-страницу под URL `https://stydiyatsi.ru/`. Этап 1 MVP — без портфолио, без цен, без AI-помощника, без формы заявки. Только манифест, объяснение подхода, интерактивная демка «Слышу цвет», ссылка на студию VK и связь.

**Зачем:** Ирина сейчас собирает клиентов через сарафан и пост в VK. Визитка — короткое доказательство «я современный дизайнер с авторским подходом», на которую можно дать ссылку из любого мессенджера и не объяснять руками.

## 1. Целевая аудитория и сценарии

ЦА Ирины — самодостаточные собственники готового бизнеса из 8 ниш (инфобизнес, финансовые консультанты, бьюти, коучи, фитнес, торговля, риэлторы, санатории). Тёплые, не холодные. Главное — пришли по рекомендации или с поста в соцсетях, где Ирина уже задала первый контекст.

### Сценарий 1 — пришёл по ссылке из мессенджера (главный для MVP)

Контекст: знакомый кинул ссылку «Вот хорошая дизайнерша — посмотри». Открывает с телефона одной рукой, в одной руке.

**Что критично для конверсии:**
- Hero грузится за < 2 сек на 4G
- Лицо Ирины + 1 строка позиционирования + статус доступности видны без скролла
- Кнопка Telegram **в первом экране** (нативный `tg://` для мобильных, fallback `https://t.me/…`)
- Sticky-bar внизу с Telegram + VK на всех экранах кроме hero
- Нет pop-up «подпишись» через 5 сек, нет cookie-баннера во весь экран, нет лоадера 5 сек

### Сценарий 2 — пришёл с поста в VK

Контекст: подписчик группы Студии увидел пост и кликнул «подробнее». Тёплый, но ещё не готов писать.

**Что критично:**
- Тон на сайте = тон в VK-группе (на «вы», тёплый премиум, без канцелярита)
- Блок «Студия в VK» отдельно (не смешан со «связаться») — посетитель из VK хочет увидеть, что Студия живая, и вернуться в неё
- Якорные URL (`#about`, `#color-listen`, `#contact`) под будущие посты-ссылки

### Сценарий 3 — холодный гость из поиска

Контекст: загуглил «дизайнер брендбук», попал на визитку. Скепсис, мало доверия.

**Что критично для MVP (без портфолио):**
- Чёткий блок «Что я делаю» с 3 цветными карточками (брендбуки / цифровые продукты / упаковка соцсетей) — понимает, делает ли Ирина его задачу
- Интерактивная демка «Слышу цвет» — вместо кейсов даёт прожитый опыт авторского подхода
- Юр.чистый словарь Ирины («изготавливаю / разрабатываю / верстаю / реализую»)
- НЕТ цен — для MVP это решение Инны (объяснение в Critique)
- Альтернатива «писать в Telegram прямо сейчас» = «посмотреть студию в VK» (медленный путь)

## 2. Экраны / секции

7 секций, по порядку прокрутки. Все на одной странице, между ними якорные URL.

### 2.1. Hero (`#hero`)

**Назначение:** за 2 секунды показать «это супер дизайнер с авторским подходом». Главная конверсия.

**Что лежит:**
- **Манифест H1** на Playfair Display 700, размер по `--fs-h1` (clamp 36→72px): «Слышу, какого человек цвета.»
- **Подзаголовок H2** на Montserrat 500, по `--fs-h3`: «Создаю фирменный стиль и интерфейсы из разговора.» (без слов «провожу аудит / консультирую / сопровождаю» — это словарь Инны)
- **Статус доступности** Montserrat 400 14px, цвет `--pine` 60%: «Открыт 1 слот на май. Старт следующего — июнь.» Слева от статуса — точечный индикатор `--neon` 8×8px.
- **Портрет Ирины** в овальной/круглой рамке справа на десктопе, сверху на мобильном. Применяется inset-glow по §3.6 visual-language.md.
  - **Fallback (если фото нет на момент сборки MVP):** монограмма «И» в круглой рамке `--vanilla` на mesh-фоне Hero, размер совпадает с местом портрета. Под монограммой мелкая подпись Montserrat 400 13px цвет `--pine` 60%: «Портрет скоро». НЕ имитируем фото и НЕ ставим заглушку из стока.
- **Один CTA под манифестом** (после Critique — убрали Secondary VK из hero, чтобы не размывать фокус):
  - Primary: «Написать в Telegram» — фон через градиент `--acid` §2.4, иконка Telegram оригинальная `#229ED9` слева. Magnetic effect 5-10px при приближении курсора (только на десктопе).
  - VK ушёл в `#contact` и в sticky-bar мобильного.

**Какие приёмы applied (visual-language.md):**
- §2.1 Hero-фон: radial mesh-градиент «Закат-коралл» поверх `--vanilla`
- §3.5 Цветной drop-shadow `filter: drop-shadow(0 6px 20px rgba(255,147,94,0.35))` под H1
- §3.6 Inset-glow на портрете
- §6.2 Page-load orchestration: H1 → 60ms → H2 → 140ms → статус → 220ms → CTA → 300ms → портрет → 380ms (потолок 300ms на саму анимацию, задержки могут быть длиннее)
- §6.5 Custom cursor — коралловый кружок 24×24px ТОЛЬКО в зоне `<section id="hero">`, отключается на touch и при `prefers-reduced-motion`
- §4.1 Noise overlay 0.05 — глобально на body, в hero выглядит органично

**Что у пользователя в голове:** «Какая тёплая страница. Кто это. Можно ли написать. Что она делает.»

**Назначение:** объяснить авторский подход в 3-4 предложениях, дать почувствовать TOV.

Где-то там лиловый, где-то — глубокий зелёный с тёплым закатом. Превращаю «сделайте красиво» в чёткую визуальную систему, которая работает на бизнес-задачи.»

**Какие приёмы applied:**

|---|---|

`scroll-behavior: smooth;` на `html`.

### 3.2. Sticky-bar мобильный

**Условие появления:**
- Только на мобильных (`@media (max-width: 768px)`)
- Появляется когда пользователь проскроллил мимо hero (после 100vh) — fade-in 200ms
- Скрывается при возврате в hero

**Что внутри:**
- Фиксированный footer внизу экрана, высота 56px + `padding-bottom: env(safe-area-inset-bottom)` для iOS (чтобы home-bar не закрывал иконки), фон `--vanilla` с тенью сверху `0 -4px 12px rgba(48,102,84,0.15)`.
- 2 круглые иконки 44×44px (Apple HIG минимум для тач):
  - Telegram `#229ED9`, ведёт на `https://t.me/irina_onlane_gesing`
  - VK `#0077FF`, ведёт на `https://vk.com/irina_1369`
- Между иконками — мелкая подпись Montserrat 500 13px цвет `--pine`: «Написать»

**Семантика:** `<nav role="navigation" aria-label="Связаться">`, не `<aside>` — это навигационный элемент, а не сторонний контент.

**На десктопе sticky-bar НЕ показываем** — там CTA в hero и в финале достаточно.

### 3.3. Scroll-snap (опция)

- На десктопе (`@media (min-width: 1024px) and (hover: hover)`) — `scroll-snap-type: y proximity` на `<main>`, `scroll-snap-align: start` на каждой `<section>`. **Proximity, не mandatory** — не блокируем естественный скролл.
- **Исключение:** `<section id="manifest">` имеет `scroll-snap-align: none` — она 80vh высотой и `proximity` на ней «прилипает» некрасиво в Safari/Firefox.
- На мобильном — отключён.
- При `prefers-reduced-motion: reduce` — отключён.
- **QA-проверка обязательна** в Safari + Firefox + Chrome на десктопе. Если в любом из браузеров скролл «дёргает» — отключаем snap полностью.

### 3.4. Header

## 6. Функции (интерактив)

### 6.1. Custom cursor (hero)

- Только в зоне `<section id="hero">`. На остальных секциях — стандартный курсор.
- SVG inline 24×24px, заливка `--coral` (`#FF935E`), `cursor: url(…) 12 12, pointer`.
- На touch (`@media (hover: none)`) — `cursor: auto`.
- При `prefers-reduced-motion: reduce` — `cursor: auto`.
- Стандартные интерактивные элементы (`<a>`, `<button>`) внутри hero — поверх pointer'а как обычно.

**Fallback при QA-конфликте:** custom cursor + magnetic CTA в одной зоне могут конкурировать визуально. На QA после первой сборки проверяем: если перегруз — **custom cursor отключаем первым** (это менее значимый из двух приёмов), magnetic CTA сохраняем как ключевой премиум-сигнал.

### 6.2. Magnetic CTA

Только на главной кнопке Telegram в hero, только на десктопе с hover.

```
- При движении мыши в радиусе 120px от центра кнопки — кнопка смещается на (dx, dy) = ((mouseX - centerX) * 0.15, (mouseY - centerY) * 0.15), но не больше ±10px по каждой оси.
- transition: transform 200ms cubic-bezier(.22,.61,.36,1)
- При выходе мыши из зоны — возврат transform: translate(0,0) за 200ms
- На touch и reduced-motion — отключено.
```

### 6.3. Scroll-snap

- `scroll-snap-type: y proximity` на `<html>` (не mandatory, чтобы не блокировать привычный скролл).
- `scroll-snap-align: start` на каждой `<section>`.
- Только на десктопе ≥1024px с `(hover: hover)`.
- Отключается на mobile и reduced-motion.

### 6.4. Sticky-bar мобильный

См. §3.2 этого документа.

### 6.5. Page-load orchestration (hero)

```
.hero > * { opacity: 0; animation: rise 300ms cubic-bezier(.22,.61,.36,1) forwards; }
.hero > h1 { animation-delay: 60ms; }
.hero > h2 { animation-delay: 140ms; }
.hero > .status { animation-delay: 220ms; }
.hero > .cta-row { animation-delay: 300ms; }
.hero > .portrait { animation-delay: 380ms; }
```

При reduced-motion — все элементы видны сразу, без анимации.

### 6.6. IntersectionObserver на остальных секциях

```
const io = new IntersectionObserver((entries) => {
  entries.forEach(e => { if (e.isIntersecting) e.target.classList.add('in-view'); });
}, { threshold: 0.15 });
document.querySelectorAll('.reveal').forEach(el => io.observe(el));
```

CSS:
```
.reveal { opacity: 0; transform: translateY(20px); transition: opacity 250ms, transform 250ms; }
.reveal.in-view { opacity: 1; transform: none; }
```

В `#services` — лесенка delay 0/120/240ms между карточками через `nth-child`.

#### Алгоритм матчинга

```
function pickPalette(selectedEmotionIds) {
  // Считаем для каждой палитры, сколько выбранных эмоций совпадает с её matches
  const scored = PALETTES.map(p => ({
    palette: p,
    score: selectedEmotionIds.filter(id => p.matches.includes(id)).length
  }));

  // Берём палитру с максимальным score; при равенстве — первую попавшуюся
  scored.sort((a, b) => b.score - a.score);
  const top = scored[0];

  // Если score = 0 (ни одна эмоция не совпала) — отдаём дефолтную forest-coral
  if (top.score === 0) return PALETTES[0];

  return top.palette;
}
```

**Что Кодыч решает сам:**
- Точный размер портрета на десктопе vs мобильном
- Точные грид-параметры карточек услуг (gap, padding)
- Иконки-капли в карточках услуг — генерирует SVG inline (3 капли в `--coral` / `--pine` / `--magenta`)
- Иконку-фавикон (монограмма «И» в `--pine` на `--vanilla`)
- Точные паузы между чипами эмоций (gap)

**Что НЕ решает сам — спрашивает:**
- Если возникает противоречие между brief'ом и visual-language.md — приоритет у visual-language.md (это бренд-источник)
- Если возникает желание добавить секцию, которой нет в §2 — НЕ добавлять, спросить product-builder
- Если хочется сделать цены/портфолио/форму — НЕ делать, это явный запрет в §8

**Чего НЕ выдумывает Кодыч:**
- ❌ Числа клиентов («50+ брендов», «20 экспертов» и т.п.) — Ирина не подтвердила цифры, не пишем
- ❌ Названия клиентов — нет утверждённого списка
- ❌ Отзывы и цитаты клиентов — нет публичных
- ❌ Регалии («лауреат», «эксперт года») — нет
- ❌ Год основания студии — не подтверждён
- ❌ Любые конкретные кейсы и фото работ — у Ирины нет публикуемых на момент MVP
