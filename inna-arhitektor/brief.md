## 1. Резюме

**Продукт:** персональная посадочная Инны Архитектор = лид-машина с мгновенным результатом.
**Позиционирование:** *«Собираю автосистемы продаж экспертам и оффлайн-бизнесу»*.
**Аудитория v1:** эксперты (инфобиз) и владельцы оффлайн-бизнеса, которым Инну рекомендовали или кто нашёл через контент.
**Платформы v1:** браузер (по ссылке) + Telegram Mini App. VK Mini App — **отложен в v1.1**.
**Метрика:** 30–45% посетителей оставили контакт = «хорошо». 45%+ = вау.

**Главный механизм:** посетитель заполняет 3–5 полей → видит превью своей упаковки → для скачивания файла оставляет контакт (TG или email) → получает результат + ссылку.

## 2. Визуальное направление

**Архетип:** **Редакционный** (из 12 в [`agents/websites.md`](../agents/websites.md)).
**Обоснование:** аудитория — владельцы бизнеса и эксперты, читают много, ценят ясность и плотную типографику. Тон — экспертный-сдержанный с теплотой (гибрид без сленга). Редакционный = точная типографика + строгий порядок секций + минимум декора, премиальность через акцент.

**Палитра (утверждена 2026-04-20):**
```
--bg:        #103206   /* тёмно-зелёный, основа — НЕ чёрный */
--accent:    #c99700   /* горчичный — главный акцент */
--accent-2:  #e86c3a   /* оранжевый — hover/CTA */
--text:      #e8e0d8   /* светло-бежевый */
--text-mute: #a89e94   /* приглушённый */
--border:    rgba(201,151,0,.18)
```

**Шрифты:** Montserrat (body) + Playfair Display (Hero-заголовки).

**Ограничения:**
- Без маркеров «Нейро Бабки СССР» (имя/логотип/подпись).
- Без чёрного `#0a0a0a` (Инна: «чтобы не зажмурился при открытии»).
- Сложность кода по архетипу: редакционный = минимум CSS, точная типографика, ноль декоративных техник (см. правило в `agents/websites.md:132`).

## 3. Структура страницы (сверху вниз)

### 3.1 Hero
- Фото Инны (справа или фоном, crop квадрат 1:1, ~240×240 на мобиле / ~360×360 на десктопе) — **TBD приложит Инна**
- Имя: «Инна Архитектор» (Playfair, крупно)
- Подзаголовок: *«Собираю автосистемы продаж экспертам и оффлайн-бизнесу»* (Montserrat 500)
- **Одна главная CTA-кнопка:** *«Собери свою упаковку за 60 секунд»* → скролл к Конструктору
- Нейтральная формулировка (не «упаковку эксперта»), чтобы оффлайн-бизнес не отвалился

### 3.2 Триггеры скорости (вместо кейсов)

**Идея:** 3–5 карточек, каждая = «формат — за сколько — чем сделано». Акцент на скорости и технологиях.

**Драфт (цифры подтвердит Инна):**

| Формат | Срок | Как делается |
|---|---|---|
| Визитка-упаковка эксперта | **2 часа** | Нейроагенты + vibe-coding |
| Сайт-визитка / лендинг | **1 сутки** | Нейроагенты + готовые шаблоны |
| Telegram Mini App под бизнес | **3–5 дней** | TMA SDK + Supabase |
| ИИ-ассистент в отдел продаж | **1–2 недели** | Каскад скилов + интеграция CRM |
| Автоворонка в мессенджере | **неделя** | Скилл-цепочка + бот |

Подзаголовок блока: *«Что и как быстро собираю»*. Под таблицей микротекст: *«Скорость — за счёт связки нейроагентов и собственных скилов. Не пишу с нуля — собираю из проверенных модулей»*.

### 3.3 Конструктор упаковки (лид-машина)

**Механика:**

1. **Форма на 3 поля обязательных + 2 опциональных:**
   - *Как тебя зовут* (обязательно)
   - *Что ты делаешь* — одной фразой (обязательно)
   - *Главное достижение / цифра* (обязательно)
   - *Кому ты помогаешь* (опционально)
   - *Куда писать* (TG / Instagram / сайт — опционально)

2. **Превью в реальном времени** под формой (или справа на десктопе). Собирается карточка-упаковка с теми же стилями что у Hero Инны. **Дебаунс 300ms** чтобы не лагало на Android.

3. **Кнопка «Забрать готовое»** — только после того как все 3 обязательных поля заполнены. При клике — **открывается модалка с гейтом**:
   - Один input с переключалкой: **TG username** (`@...`) или **email**
   - Кнопка «Отправь мне результат»
   - Под кнопкой: *«Отправлю HTML-файл, можно прикрепить к любому сообщению»*

4. **После ввода контакта:**
   - Лид сохраняется в Supabase (таблица `inna_leads`)
   - На Telegram Инне падает уведомление в ЛС (через простой вебхук в TG-бот)
   - Посетителю отдаётся: (а) ссылка на страничку с его упаковкой `https://<домен>/u/<slug>`, (б) возможность скачать HTML-файл через `Blob + URL.createObjectURL`
   - **Мягкий пуш:** *«Спасибо! Если хочешь больше такого — подпишись на канал»* + кнопка TG-канала Инны

## 4. Платформенная специфика

### Браузер (v1, главный)
Полный функционал. Self-contained HTML-файл (один `index.html`) с inline CSS/JS + Google Fonts + Supabase CDN. Deploy на Beget под отдельным путём/поддоменом (**TBD домен**).

### Telegram Mini App (v1, второстепенный)
Тот же HTML + `/js/tma-init.js` (из Beauty-паттерна). Бонусы TMA:
- `tg.initDataUnsafe.user.username` — **префилл поля «как тебя зовут»** и сразу известен контакт → в модалке гейта показываем «Мы узнали тебя как @username, подтверди или поменяй». Конверсия ×2.
- `tg.MainButton` — глобальная кнопка «Забрать готовое» вместо inline.
- Haptic feedback на submit.

Отдельный файл `tma.html` собирается скилом из того же source.

### VK Mini App (v1.1, отложен)
Заготовка `vk.html` в `templates/` пустая. Когда активируем — берём из research.md VK Bridge + `VKWebAppGetEmail` (нативная модалка вместо формы).

## 5. Технологии

| Слой | Что | Почему |
|---|---|---|
| Фронт | Vanilla HTML + inline CSS + vanilla JS | Корневое правило репо (без фреймворков и CDN-либ) |
| Шаблоны | Строки HTML со `{{placeholders}}` + простой `replace()` | Не тащим mustache/handlebars |
| Хранение лидов | Supabase (как в MG) | Уже настроен в репо, RLS работает |
| Уведомления Инне | TG bot webhook (простой POST в `/sendMessage`) | Быстрее Google Sheets, нативно для Инны |
| Сборка HTML-файла для посетителя | `new Blob + URL.createObjectURL + <a download>` | Без backend, чисто в браузере |
| Деплой | Beget через `deploy-inna.sh` (копия `deploy-mg.sh`) | Стандарт репо |

## 6. Чего НЕ делаем в MVP (v1)

- ❌ VK Mini App (отложен в v1.1)
- ❌ Выбор цветов посетителем (фиксируем палитру Инны, цвета = upsell)
- ❌ Загрузка фото посетителя (монограмма/инициалы-плейсхолдер)
- ❌ Редактирование после публикации (нет secret-токена, нет кабинета)
- ❌ Выбор из 6 форматов (презентация/кейс/пост/сторис/лендинг/рассылка) — это следующий продукт
- ❌ Tripwire «купи за 500₽» на thank-you — мягко пушим в TG-канал вместо
- ❌ Оплата на странице
- ❌ Аналитика внутри (используем Яндекс.Метрику отдельно)
- ❌ Кейсы с цифрами (убрали в критике — нет чистых чисел)

## 7. Скилл `inna-architect` — что соберёт skill-conductor CREATE

```
inna-arhitektor/skills/inna-architect/
├── SKILL.md                     # Главный файл: когда триггерится, что делает
├── references/
│   ├── platforms.md             # Отличия browser / tma / vk
│   ├── archetype.md             # Редакционный архетип + палитра + шрифты
│   └── content-schema.md        # JSON-схема входных данных (5 полей формы)
├── templates/
│   ├── browser.html             # Self-contained для браузера
│   ├── tma.html                 # + TG SDK init
│   └── vk.html                  # Пустая заготовка, v1.1
└── evals/
    └── evals.json               # 3 тест-кейса: минимум / полный / edge-case
```

**Триггеры для SKILL.md:**
- «собери визитку/упаковку Инны», «сгенерируй страницу Инны Архитектора», «собери HTML для Инны Архитектор» → скилл активируется.
- НЕ триггерится на визитки клиентов агентства (у них своя палитра).

**Вход скилла:** JSON по схеме:
```json
{
  "platform": "browser" | "tma" | "vk",
  "visitor": {
    "name": "...",
    "role": "...",
    "achievement": "...",
    "audience": "...",
    "contact": "..."
  },
  "inna": {
    "tg_channel": "...",
    "tg_direct": "...",
    "email": "...",
    "photo_url": "..."
  }
}
```

**Выход скилла:** готовый HTML-файл в `output/<platform>/index.html`.
