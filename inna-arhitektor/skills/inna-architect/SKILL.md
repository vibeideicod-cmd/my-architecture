# Инна Архитектор — генератор посадочной

## Обзор

Этот скилл превращает JSON с данными Инны (фото, контакты, TG-канал) и данными посетителя (если есть) в готовый HTML-файл посадочной «Инна Архитектор — автосистем и коммуникаций». HTML самодостаточен: inline CSS + inline JS + Google Fonts + Supabase через CDN. Никаких сборок, никаких фреймворков.

Суть посадочной — **лид-машина**: посетитель заполняет форму → видит превью своей упаковки → оставляет контакт → получает HTML-файл.

**Источник правды:** [brief.md](../../brief.md) в корне ветки `inna-arhitektor/`. При любом противоречии между SKILL.md и brief.md — приоритет brief.md.

## Когда использовать

✅ Триггеры:
- «собери визитку Инны», «обнови посадочную Инна Архитектор»
- «сгенерируй HTML для браузерной версии Инны»
- «собери TG Mini App для Инны»
- «поменяй данные на посадочной Инны и пересобери»

❌ НЕ использовать:
- Для клиентских визиток (beauty, yuzhnoberezhniy, alisa, mastergroup) — у них своя палитра и брендинг, этот скилл их сломает
- Для рассылок, постов, сторис, лендингов курсов — это другие будущие скилы, не этот
- Для VK Mini App в текущей версии — `vk.html` пока пустая заготовка (v1.1)

## Параметры скилла

### Платформа (обязательная)
`platform`: `"browser"` | `"tma"` | `"vk"` (для v1 работают browser и tma; vk пустая заготовка).

### Данные Инны (обязательные для рендера)
```json
{
  "inna": {
    "tg_channel": "https://t.me/...",
    "tg_direct":  "https://t.me/...",
    "email":      "...@..." | null,
    "photo_url":  "/clients/neuro-babki/assets/inna-portrait.jpg"
  }
}
```

Если какого-то поля нет — подставляй заглушку `TBD` в HTML (но в консоль вываливай warning).

### Триггеры скорости (редактируемые)
Массив из 3–5 пунктов формата «формат — срок — технология»:
```json
{
  "speed_triggers": [
    {"format": "Визитка-упаковка эксперта", "time": "2 часа", "tech": "Нейроагенты + vibe-coding"},
    {"format": "Сайт-визитка / лендинг", "time": "1 сутки", "tech": "Нейроагенты + готовые шаблоны"},
    {"format": "Telegram Mini App", "time": "3–5 дней", "tech": "TMA SDK + Supabase"},
    {"format": "ИИ-ассистент в отдел продаж", "time": "1–2 недели", "tech": "Каскад скилов + CRM"},
    {"format": "Автоворонка в мессенджере", "time": "неделя", "tech": "Скилл-цепочка + бот"}
  ]
}
```

### Данные посетителя (опциональные, для pre-generated demo)
```json
{
  "visitor": null | {
    "name": "...",
    "role": "...",
    "achievement": "...",
    "audience": "...",
    "contact": "..."
  }
}
```

Если `visitor: null` — посадочная рендерится пустая, конструктор работает интерактивно на странице.
Если `visitor: {...}` — рендерится пре-заполненный вариант (используется для демо или для ссылки `/u/<slug>` после сабмита).

### Данные Supabase (обязательные)
```json
{
  "supabase": {
    "url": "https://<project>.supabase.co",
    "anon_key": "eyJ...",
    "table": "inna_leads"
  }
}
```

Если не передано — конструктор работает, но форма ничего не сохраняет (dev-режим). В production без Supabase нельзя.

## Жёсткие правила

1. **Палитра фиксирована.** Не используй `#0a0a0a` чёрный, не используй любые посторонние цвета. Только из `references/archetype.md`.
2. **Без маркеров «Нейро Бабки СССР»** нигде. Проверяй HTML после генерации: нет ли подписи «СССР», нет ли «Нейро Бабки», нет ли ссылок на `/hub/` или `/Irina/`.
3. **Mobile-first.** Базовые стили — для 375px. Desktop через `@media (min-width: 768px)`. Не наоборот.
4. **Архетип Редакционный.** Минимум CSS-декора. Ноль grain overlay, ноль floating blobs, ноль множественных градиентов. Типографика и пустое пространство — главные инструменты.
5. **Один HTML-файл на платформу.** Не делим на external css/js. Всё inline.
6. **Fонты только Google Fonts** (Montserrat + Playfair Display). Fallback — системный sans-serif.

## Последовательность работы (для агента)

### Шаг 1. Проверь вход
- Достаточно ли данных в `inna.*`? Если нет — укажи warning и подставь TBD.
- Платформа выбрана правильно (`browser` | `tma` | `vk`)?
- Supabase-креды переданы (если нужен production-режим)?

### Шаг 2. Выбери шаблон
- `browser` → `templates/browser.html`
- `tma` → `templates/tma.html`
- `vk` → `templates/vk.html` (v1: пустая заготовка, верни 501 Not Implemented)

### Шаг 3. Подставь placeholders
Шаблоны используют формат `{{key}}` — подставляй простым `replace()`. Плейсхолдеры:
- `{{INNA_PHOTO_URL}}`, `{{INNA_TG_CHANNEL}}`, `{{INNA_TG_DIRECT}}`, `{{INNA_EMAIL}}`
- `{{SPEED_TRIGGERS_HTML}}` — генерируется из массива `speed_triggers` через функцию-рендер
- `{{SUPABASE_URL}}`, `{{SUPABASE_ANON_KEY}}`, `{{SUPABASE_TABLE}}`
- `{{VISITOR_PREFILL_JSON}}` — сериализованный `visitor` или `null`

Плейсхолдер должен быть заменён **один раз**. Если плейсхолдер остался в финальном HTML — это баг генерации.

### Шаг 4. Валидация
Перед выдачей HTML проверь:
- В файле нет ни одного `{{...}}` — все плейсхолдеры подставлены
- В файле нет строк `Нейро Бабки`, `СССР`, `<tg: Telegram>`
- HTML валиден (открыть через `python -c "import html.parser; ..."` или просто проверить теги)
- Размер файла < 150KB

### Шаг 5. Сохрани
Путь:
- `browser` → `../../output/browser/index.html`
- `tma` → `../../output/tma/index.html`
- `vk` → `../../output/vk/index.html`

### Шаг 6. Выдай отчёт
- Путь к сгенерированному файлу
- Список warning'ов (TBD-подстановки)
- Инструкция: как открыть (browser — напрямую в Chrome; TMA — загрузить на hosted URL и подключить через BotFather)

## Частые ошибки

**Ошибка:** HTML содержит `{{VISITOR_PREFILL_JSON}}` как строку.
**Причина:** забыл подставить плейсхолдер. Сериализуй `visitor` через `JSON.stringify()` и вставь.
**Фикс:** добавь в pipeline шаг «проверка-нет-оставшихся-плейсхолдеров».

**Ошибка:** страница выглядит как AI slop — grain, blobs, много градиентов.
**Причина:** агент добавил декор из «красивых» примеров других проектов.
**Фикс:** архетип Редакционный = минимум декора. Удали blobs и grain, верни чистую типографику.

**Ошибка:** на мобиле форма лагает при вводе.
**Причина:** нет debounce на preview-рендер.
**Фикс:** `debounce(updatePreview, 300)` на каждом `input` события.

## Скрипт-помощник

`scripts/generate.sh` принимает JSON на stdin и платформу как аргумент, рендерит HTML:
```bash
cat input.json | bash scripts/generate.sh browser
# → выводит HTML в stdout
```
Для Claude-агента скрипт НЕ обязателен — агент может делать подстановку напрямую из шаблона, читая `templates/<platform>.html` и выполняя `replace()`. Скрипт полезен когда мы хотим регенерацию из CLI (например, в `deploy-inna.sh`).
