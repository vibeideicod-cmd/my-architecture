# Веб-визитка Инны Андрейченко — каркас «Кокпит»

Одностраничная визитка. Один файл `index.html` — всё внутри (HTML + CSS + JS). Без CDN кроме Google Fonts. Открывается в любом современном браузере без бэкенда: моковые эндпоинты возвращают синтетические данные.

- **Концепция:** «Кокпит» — полноэкранный портрет на тёмно-зелёном `#103206` с золотом `#D4AF37`.
- **Главная конверсия:** запись на консультацию (`#schedule`).
- **Промежуточная конверсия:** «Аудит за 60 секунд» с захватом контакта (`#audit`).
- **Источники правды:**
  - Бренд-код: `agent-runtime/outputs/2026-05-04-brandbook-visitka-inna.md`
  - Юр.пакет: `agent-runtime/outputs/2026-05-04-yurpaket-visitka-inna.md`

- сохранить как `assets/inna-hero.jpg` или `assets/inna-hero.webp` (тогда поправить расширение в `index.html`, переменная `probe.src`).

## 2. Плейсхолдеры — что подменить (передаётся Copywriter)

Плейсхолдеры записаны в формате `{{KEY}}`. Поиск по файлу: `\{\{[A-Z0-9_]+\}\}`. Группы — ниже.

### 2.1. Мета и шапка
| Ключ | Что |
|---|---|
| `{{META_TITLE}}` | `<title>` страницы (до 60 символов) |
| `{{META_DESCRIPTION}}` | `<meta name="description">` (до 160 символов) |
| `{{SITE_URL}}` | URL сайта целиком (используется в Политике и в QR-блоке для PDF) |

### 2.2. Hero
| Ключ | Что |
|---|---|
| `{{HERO_EYEBROW}}` | Микро-надпись над H1 (ALL CAPS, 1–4 слова) |
| `{{HERO_H1}}` | Главный заголовок |
| `{{HERO_H1_ITALIC}}` | Хвост заголовка курсивом золотом (1–2 слова) |
| `{{HERO_H2}}` | Подзаголовок (одно предложение, до 18 слов) |
| `{{HERO_CTA_PRIMARY}}` | Текст главной кнопки («Записаться на разговор») |

### 2.3. С.С.С.Р.
| Ключ | Что |
|---|---|
| `{{SSSR_EYEBROW}}` | Микро-надпись |
| `{{SSSR_H2}}` | Заголовок секции |
| `{{SSSR_INTRO}}` | Один абзац-вступление (до 30 слов) |
| `{{SSSR_TIP_1..4}}` | Tooltip на узел SVG (Структура / Стратегия / Свобода / Результат) — до 12 слов |
| `{{SSSR_CARD_1..4_TITLE}}` | Заголовок карточки (1 слово: «Структура» и т.д.) |
| `{{SSSR_CARD_1..4_TEXT}}` | Текст карточки (до 25 слов) |

### 2.4. Цифры
| Ключ | Что |
|---|---|
| `{{N1..N4_VALUE}}` | Числовое значение для счётчика (только цифры, например `120`) |
| `{{N1..N4_SUFFIX}}` | Суффикс рядом с числом (`+`, `%`, `мес`) — пустой допустим |
| `{{N1..N4_LABEL}}` | Подпись под числом |

### 2.5. Услуги
| Ключ | Что |
|---|---|
| `{{SERVICES_EYEBROW}}` / `{{SERVICES_H2}}` | Шапка секции |
| `{{SVC_1..4_TITLE}}` | Заголовок карточки |
| `{{SVC_1..4_TEXT}}` | 1–2 строки описания |

### 2.6. Аудит за 60 секунд
| Ключ | Что |
|---|---|
| `{{AUDIT_EYEBROW}}` / `{{AUDIT_H2}}` / `{{AUDIT_INTRO}}` | Шапка стартового экрана |
| `{{AUDIT_START_CTA}}` | Кнопка «Начать аудит» |
| `{{Q1..Q6_TEXT}}` | Текст вопроса (одно предложение) |
| `{{Q1..Q6_OPT_1..4}}` | 4 варианта ответа на каждый вопрос |
| `{{AUDIT_RESULT_EYEBROW}}` | Микро-надпись на экране результата |
| `{{AUDIT_GAPS_LABEL}}` | Подпись «Главные дыры:» |
| `{{AUDIT_GAP_PLACEHOLDER}}` | Дефолтный текст одного пункта дыр (до подключения логики 3 пунктов) |
| `{{AUDIT_SUCCESS_TITLE}}` / `{{AUDIT_SUCCESS_TEXT}}` / `{{AUDIT_SUCCESS_CTA}}` | Экран «Спасибо» после отправки |

**Скоринг (в коде):** сумма очков по 6 вопросам (1–4 каждый) → 6–12 «Операционный», 13–18 «Системный», 19–24 «Архитектурный». Если copywriter переформулирует уровни — поправить в JS-функции `audit.score()`.

## 3. Моковые эндпоинты — на что заменить, когда n8n заработает

Все моки помечены в `index.html` комментарием `// MOCK:`.

### 3.1. POST `/api/audit-lead`
- **Где:** `auditForm.addEventListener('submit', ...)`.
- **Сейчас:** `console.log('[mock POST /api/audit-lead]', payload)` + переход на success-экран.
- **Нужно:** раскомментировать `await fetch('/api/audit-lead', ...)` и заменить URL на `https://n8n.<домен>/webhook/audit-lead`.
- **Payload:**
  ```json
  {
    "name": "Имя",
    "contact": "@nick или vk.com/...",
    "channel": "telegram|vk",
    "answers": {"q1":1,"q2":3,"q3":2,"q4":4,"q5":2,"q6":3},
    "level": "Операционный|Системный|Архитектурный",
    "consent_given_at": "2026-05-04T08:30:00.000Z",
    "consent_text_version": "v1"
  }
  ```
- **n8n должен:** записать в базу (Моя База Клиентов), отправить уведомление в личный TG Инны, отправить пользователю результат в его TG/VK.

### 3.2. GET `/api/slots`
- **Где:** функция `fetchSlots()`.
- **Сейчас:** возвращает массив дней на 14 будних дней вперёд, шаг 60 минут с 10:00 до 19:00 МСК. Случайно помечает ~25% слотов занятыми (для демо).
- **Нужно:** заменить тело `fetchSlots()` на `return fetch('https://n8n.<домен>/webhook/slots').then(r => r.json())`.
- **Ожидаемый формат ответа:**
  ```json
  [
    {
      "date": "2026-05-05",
      "slots": [
        {"time":"10:00","iso":"2026-05-05T07:00:00.000Z","available":true},
        {"time":"11:00","iso":"2026-05-05T08:00:00.000Z","available":false}
      ]
    }
  ]
  ```
  В `renderSchedule(days)` поле `day.date` — JS-Date. Когда придёт ISO-строка — обернуть в `new Date(day.date)` либо нормализовать на стороне n8n.
- **n8n должен:** взять занятость из календаря Инны, склеить с шаблоном (будни 10–19), отдать. Имена пользователей в названия событий не выводим (см. п.6 рисков юр.пакета).

### 3.3. POST `/api/book`
- **Где:** `bookForm.addEventListener('submit', ...)`.
- **Сейчас:** `console.log('[mock POST /api/book]', payload)` + success-экран с датой/временем.
- **Нужно:** раскомментировать `await fetch('/api/book', ...)` и заменить на `https://n8n.<домен>/webhook/book`.
- **Payload:**
  ```json
  {
    "name": "Имя",
    "contact": "@nick или vk.com/...",
    "channel": "telegram|vk",
    "slot_iso": "2026-05-05T07:00:00.000Z",
    "consent_given_at": "2026-05-04T08:30:00.000Z",
    "consent_text_version": "v1"
  }
  ```
- **n8n должен:** проверить, что слот ещё свободен (race condition), создать событие в календаре, выдать ссылку на Zoom, написать пользователю в выбранный канал, прислать Инне краткую сводку.

### 3.4. Hero-индикатор «Свободно N слотов»
Сейчас считается на лету из ответа `/api/slots`: суммирует свободные слоты на ближайшие 7 дней. Никаких отдельных эндпоинтов не нужно.

## 4. Что уже работает (без бэкенда)

- адаптив (мобильный first, breakpoints 768 / 1024),
- IntersectionObserver на все секции (fade-in 300 мс) — кроме hero, он статичный,
- IntersectionObserver на счётчики «Цифры» (накручивает при появлении),
- хедер с пульсирующей точкой «свободно N слотов»,
- SSSR-виджет: 4 узла + tooltip по hover/focus,
- Аудит за 60 секунд: 6 вопросов, прогресс-бар, скоринг → 3 уровня, форма с обязательным согласием, success-экран,
- Расписание: 14 дней, моковые слоты, модалка бронирования с обязательным согласием, success-экран,
- `<dialog>` с полным текстом Политики (открывается из 3 точек: 2 чекбокса + футер),
- `window.print()` через ссылку «Скачать PDF» в финальном блоке,
- `@media print` с инвертированной палитрой (`#FFF2E9` фон, `#103206` текст), снятым grain, скрытым интерактивом, видимым QR-блоком, `@page { margin: 18mm 16mm }`,
- film grain по всей странице (SVG fractalNoise inline, без сетевого запроса),
- доступность: `aria-pressed` на toggle-кнопках, `aria-label` на иконки, `<dialog>` для модалок, `tabindex` на узлах SSSR,
- `prefers-reduced-motion` — выключает все transition/animation,
- защита от системной тёмной темы — `<meta name="color-scheme" content="dark">`.

## 5. Чего НЕТ намеренно (по запретам в задаче)

- никакой аналитики (Метрика/GA),
- никакого cookie-баннера (нет cookies — нет баннера),
- никакого параллакса, видеофона, auto-play,
- никаких упоминаний инструментов (vibe coding/AI/нейросетей) в коде или комментариях,
- никаких `rgba(0,0,0,…)` теней — только тени из палитры,
- никаких внешних JS-библиотек.

## 6. Перед публикацией (чек-лист)

1. Подложить `assets/inna-hero.jpg` (от Nano Banana).
2. Получить от Copywriter все `{{...}}`-плейсхолдеры разделов 2.1–2.9 и поправить найти/заменить.
3. Заполнить юр.реквизиты раздела 2.10 — Инна вручную (ФИО / ИНН / ОГРНИП / адрес / email / дата редакции Политики).
4. Подменить QR-плейсхолдер в `.pdf-qr` — вставить настоящий SVG QR-кода на финальный URL сайта.
5. Подключить n8n — подменить три мока (раздел 3) на боевые URL.
6. Прогнать через `/seo` для дозаполнения мета-тегов.
7. Проверить, что чекбокс согласия не предзаполнен и отправка форм заблокирована до галочки (уже так — но перепроверить).
8. Прогнать `/qa` (чек-лист на AI slop, доступность, печать в PDF из Chrome/Safari).
9. Опубликовать через `/deploy`.
