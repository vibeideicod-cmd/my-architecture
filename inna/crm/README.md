## Зачем нужна CRM сейчас

Без CRM маршрут `HUB → Карта точек роста → контакт → follow-up` остаётся ручным. Квиз уже может показать результат, но лид не фиксируется как объект работы.

CRM нужна не как “большая система управления продажами”, а как первый операционный слой:

1. принять контакт из `karta-rosta.html`;
2. сохранить ответы и первичный результат;
3. показать Инне, кто пришёл и что у него болит;
4. дать статус follow-up;
5. не потерять клиента при переходе в платный AI-аудит / сопровождение.

## MVP-решение

Для первого запуска не строим сложный кабинет.

**Минимум:**
- одна база лидов;
- карточка лида;
- статусы;
- источник заявки;
- ответы квиза;
- ручной follow-up;
- экспорт/просмотр для Инны.

**Не делаем в MVP:**
- сложные роли пользователей;
- автоворонки на 20 шагов;
- платёжки;
- AI-генерацию PDF;
- интеграцию со всеми каналами сразу;
- красивый клиентский кабинет.

## Первый рабочий маршрут

```text
karta-rosta.html
  ↓ POST /api/leads
CRM сохраняет лид
  ↓
Инна видит карточку в статусе new
  ↓
ручной follow-up в Telegram/VK/WhatsApp/Email
  ↓
статус меняется на contacted / qualified / audit-offered / closed
```

## Критерий готовности MVP

CRM MVP считается готовой, когда:

- [ ] `karta-rosta.html` отправляет лид в API;
- [ ] API валидирует имя/контакт/канал;
- [ ] ответы квиза сохраняются;
- [ ] в базе есть статус лида;
- [ ] Инна может увидеть список лидов;
- [ ] Инна может открыть карточку и понять следующий шаг;
- [ ] при ошибке API квиз честно показывает fallback на Telegram.
