## AI-помощник Инны — диалоговые тексты

### Первое сообщение (приветствие)

> «Привет! Я — AI-помощник Инны. Помогу разобраться, что у тебя в бизнесе и какое решение Инны подойдёт.»

**Юридический статус:** AI представляется честно, не выдаёт себя за Инну. Решение зафиксировано в `inna/architecture.md` Слой 5, п.5.3.

### Тон

- Говорит на «ты» (как Инна в соцсетях)
- Простой и дружеский язык — как объясняешь знакомому (правило из CLAUDE.md)
- Без канцелярита и сложных оборотов
- Никаких «вайбкод», «нейросети как фундамент» — только «аудит, сопровождение, экспертиза» (правило бренда)

### Сценарий вопросов (4-6 шт.)

Точная формулировка — задача Copywriter-агента. Структурно:
1. Чем занимаешься? (определяет нишу — 1 из 8)
2. Что сейчас болит больше всего? (определяет боль — 1 из 5)
3. Уточняющий вопрос по боли
4. Готов ли вкладываться в развитие? (определяет сегмент A/B/C)
5. Финал: предложение лендинга / продукта / встречи / Платформы С.С.С.Р.

Маршрутизация: см. `inna/architecture.md` Слой 5, п.5.3.

## Лендинги по болям — структура

5 страниц, по одной на боль (синхронно с Слоем 2). Каждый лендинг строится по единому шаблону:

| Блок | Что внутри |
|---|---|
| H1 | Боль клиента в его словах |
| Подзаголовок | Решение в одной фразе |
| Что болит / 3-4 пункта | Расшифровка боли (узнавание) |
| Решение Инны | 2-3 продукта из этой боли (вход → основное → флагман) |
| Кейсы (мини-блок) | 2-3 кейса с фильтром по нишам этой страницы |
| CTA | «Поговорить с AI-помощником Инны» / «Записаться на разбор» |

Тексты под каждый лендинг — задача Copywriter-агента. Источники для текстов:
- `inna/audience.md` — портреты, боли, язык клиента
- `inna/product-line.md` — продукты под каждую боль
- `inna/positioning.md` — позиционирование, антипримеры, голос бренда

## ⚖️ Юридическая лексика для сайта (ОКВЭД 70.22)

**Все публичные тексты сайта HUB** — оферта, чеки, кнопки, описания услуг — должны соответствовать ОКВЭД 70.22 «Консультирование по вопросам коммерческой деятельности и управления».

### Правило одной формулы

**Аудит → Рекомендации → Сопровождение внедрения.** Всё, что мы оказываем — укладывается в эту формулу.

### Чек-лист: что говорим / что не говорим

| ✅ Используем | ❌ Запрещено в публичном |
|---|---|
| Провожу аудит | Делаю / создаю / разрабатываю |
| Даю рекомендации | Изготавливаю / произвожу |
| Сопровождаю внедрение | Программирую / пишу код |
| Архитектор / Эксперт / Консультант | Разработчик / Программист / Вайбкодер |
| Экспертная консультация | Услуга по разработке |
| Абонентское консультационное обслуживание | Подписка на ПО / SaaS |
| AI-помощник Инны (инструмент) | AI-сервис / AI-продукт (как отдельный товар) |

### Формулировки для тарифов и продуктов

- **Разовая работа** → «Экспертная консультация» или «Сопровождение внедрения»
- **Подписка на Платформу** → «Абонентское консультационное обслуживание»
- **VIP / индивидуальная работа** → «Индивидуальное экспертное сопровождение»
- **Лид-магниты, чек-листы** → «Информационный материал» или «Экспертный аудит-черновик»

### Формулировки для кнопок CTA

| Кнопка | Юр. суть |
|---|---|
| «Поговорить с AI-помощником Инны» | Предварительная квалификация запроса |
| «Записаться на разбор» | Запрос на экспертную консультацию |
| «Получить аудит» | Запрос на бесплатный экспресс-аудит |
| «Оформить подписку» | Заключение договора на абонентское консультационное обслуживание |

### Что писать про AI и автоматизации

**Нельзя:** «Мы делаем вам AI-консультанта», «Подписка на нашу AI-систему», «Купите наш бот».

**Можно:** «Провожу аудит и рекомендую AI-инструменты под ваш бизнес», «Сопровождаю внедрение AI-помощника в вашу команду», «Экспертное сопровождение работы с AI».

### AI-помощник Инны на сайте — юр. позиция

AI-помощник на сайте HUB **не является продуктом**. Это **инструмент квалификации запросов** (как форма обратной связи, только умнее). Он помогает Инне определить, какая консультация нужна посетителю, и записаться на неё.

В первом сообщении AI должен честно сообщать: «Я — AI-помощник Инны». Это закрывает риски по защите прав потребителей и по обязательной маркировке AI.

Подробнее об AI-помощнике: `inna/architecture.md` Слой 5, п.5.3.

### Если сомневаетесь

Любая публичная формулировка → проверить через 3 вопроса:
1. Подходит под формулу «Аудит → Рекомендации → Сопровождение»?
2. Нет в списке запрещённых слов?
3. Можно вписать в чек или оферту по ОКВЭД 70.22?

Если хоть одно «нет» — переформулировать или вынести из публичного.

Полные правила бренда: `inna/positioning.md` + `inna/architecture.md` Слой 1, п.1.6.
