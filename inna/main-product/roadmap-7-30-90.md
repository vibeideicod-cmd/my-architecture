# Roadmap 7/30/90 — главный продукт «Бизнес-архитектура под ключ»

Дата: 2026-05-17  
Статус: первая дорожная карта реализации на Client Zero  
Client Zero: Инна

## Цель roadmap

Понять, как из идеи продукта перейти к рабочей реализации:

> не “сделать CRM” и не “дописать HUB”, а собрать доказанный продуктовый маршрут, который потом можно повторять для Ирины и клиентов.

---

## 7 дней — доказать первый рабочий маршрут

Цель недели:

```text
Человек пришёл
→ понял, что делает Инна
→ прошёл Карту точек роста
→ оставил контакт
→ попал в понятный статус/follow-up
→ получил следующий оффер
```

### P0 на 7 дней

| Шаг | Что сделать | Почему |
|---|---|---|
| 1 | Утвердить компонентную карту продукта | Чтобы не строить отдельные куски вслепую |
| 2 | Переписать главный смысл HUB под актуальную роль Инны | Сейчас корневой HTML ещё частично старый |
| 3 | Довести «Карту точек роста» до честного MVP | Уже есть HTML, нужно проверить UX/тексты/результат |
| 4 | Решить юридический минимум для контакта | Без согласия на ПД публичный сбор контакта рискован |
| 5 | Сделать минимальный статус лида | Не обязательно полная CRM, но лид не должен теряться |
| 6 | Сформировать оффер после карты | После бесплатного входа должен быть понятный платный следующий шаг |
| 7 | QA маршрута | Проверить как реальный человек |
| 8 | Запланировать market research лидеров рынка | Чтобы продукт не был собран в вакууме |

### Что НЕ делаем в первые 7 дней

- не строим полную Платформу С.С.С.Р.;
- не делаем все 5 лендингов по болям;
- не запускаем VK/TG;
- не подключаем сложный AI;
- не делаем полную CRM, если достаточно временного статуса/follow-up.

### Выход 7 дней

| Артефакт | Статус на выходе |
|---|---|
| `main-product/` | главный продукт описан как система |
| `index.html` | HUB говорит актуальным языком Инны |
| `karta-rosta.html` | квиз можно пройти |
| CRM/follow-up | есть минимум: ручной или API |
| Оффер AI-аудита | есть мост в платный продукт |
| QA | маршрут проверен |

---

## 30 дней — превратить маршрут в продаваемый продукт

Цель месяца:

> На Инне работает MVP главного продукта, который можно показывать как живую витрину и переносить на Ирину.

### Что должно появиться за 30 дней

| Блок | Что сделать |
|---|---|
| HUB | Главная + минимум 1 лендинг боли |
| Карта точек роста | Улучшить результат, добавить PDF/текстовую сводку |
| CRM | Подключить сохранение лидов, статусы, ручной follow-up |
| AI-база | Минимум: 5 болей, продукты, FAQ, tone |
| Юр.минимум | Политика ПД, согласие, аккуратные формулировки |
| Офферы | Платный AI-аудит + сопровождение внедрения |
| Контент | Первые материалы для VK/TG ведут на HUB |
| Ирина | Начать зеркалирование подхода на её контур |
| Market research | Разобрать лидеров рынка и аналоги: бизнес-архитекторы, AI-автоматизаторы, CRM-интеграторы, агентства под ключ |

### Выход 30 дней

К концу 30 дней должно быть можно сказать:

> “Вот как продукт работает на мне. Вот вход. Вот карта. Вот CRM. Вот следующий платный шаг. Вот что будет у клиента.”

## 90 дней — упаковать в повторяемый клиентский продукт

Цель 90 дней:

> Превратить Client Zero в коммерческий продукт, который можно продавать и вести по стандарту.

### Что должно быть за 90 дней

| Блок | Что сделать |
|---|---|
| Методология | Финальная версия playbook для клиентов S/M/L |
| Шаблоны | `brief.md`, `architecture.md`, `CORE-ASSETS-AUDIT.md`, `IMPLEMENTATION-ROADMAP.md`, `QA-REPORT.md` |
| Витрина | HUB показывает продукт “Бизнес-архитектура под ключ” |
| Кейсы | Инна + Ирина как первые живые примеры |
| CRM | Рабочая база клиентов и статусы |
| AI | AI-помощник отвечает на базе знаний Инны |
| Платформа | Решение: строить/не строить первую версию С.С.С.Р. по факту спроса |
| Продажи | Пакеты S/M/L, цены, условия, юр.формулировки |

### Выход 90 дней

Коммерческий продукт должен иметь:

1. понятное название;
2. понятную цену/пакеты;
3. понятный процесс работы;
4. список артефактов на выходе;
5. первый рабочий кейс;
6. инфраструктуру для ведения клиента;
7. QA-критерии “готово / не готово”.

---

## С чего начинаем прямо сейчас

Не с CRM как кода.

Первый следующий шаг:

> Проверить P0 первого маршрута: HUB → Карта точек роста → контакт → статус/follow-up → оффер AI-аудита.

Если на этой проверке главный провал — CRM, строим CRM.  
Если главный провал — юридический минимум, делаем ПД.  
Если главный провал — непонятный оффер после карты, делаем оффер.

## Текущий предварительный приоритет

По P0-аудиту `p0-route-audit-2026-05-17.md`:

1. **Оффер после Карты точек роста** — DONE: `paid-ai-audit-offer.md` + оффер-блок в `karta-rosta.html`.
2. **Юр.минимум для сбора контакта** — DONE в MVP: чекбокс, политика, версия согласия.
3. **Перепаковка HUB** — DONE в MVP: верх, схема, флагман и финальный CTA обновлены под актуальную роль Инны.
4. **Минимальный статус лида** — DONE: `karta-rosta.html` отправляет payload в CRM endpoint и показывает fallback.
5. **CRM API** — DONE: production endpoint, `/admin`, test lead, status update, daily backup.
6. **Market research** — следующий содержательный слой после QA: проверить лидеров рынка и аналоги, чтобы уточнить упаковку и отличие.

Именно поэтому CRM нужна, но не обязательно первая.
