# Components Map — из чего состоит продукт «Бизнес-архитектура под ключ»

Дата: 2026-05-17  
Статус: первая карта составляющих главного продукта

## Главный принцип

Компоненты не строятся все сразу. Сначала определяем, какие компоненты нужны конкретному клиенту, потом собираем roadmap.

У малого бизнеса будет 30-40% этой карты.  
У большого — почти вся карта.

## Минимальный состав MVP главного продукта

Чтобы доказать продукт на Инне, достаточно:

1. Карта бизнеса Инны.
2. Inventory + deep audit.
3. Компонентная карта продукта.
4. Первый маршрут `HUB → Карта точек роста → CRM/follow-up`.
5. Минимальный CRM-статус.
6. Юр.минимум для сбора контактов.
7. Roadmap 7/30/90.
8. QA-проверка маршрута.

## Что может “вылезти” дальше

Как ты правильно сказала, CRM может оказаться не первым реальным блокером. После карты компонентов возможны другие P0:

- нет юр.согласия на сбор ПД;
- HUB говорит старым языком;
- нет ясного оффера после Карты точек роста;
- нет roadmap после архитектуры;
- нет статуса клиента в папке;
- AI-база пустая;
- невозможно объяснить продукт за 30 секунд.

Поэтому следующий шаг — не кодить дальше CRM, а собрать `client-zero-plan.md`.
