## Слой 3: Внутренняя кухня

> **VS Code · нейро-команда · база знаний · внутренние инструменты**

Рабочая инфраструктура. Всё что живёт в VS Code: агенты, база знаний, шаблоны, плейбуки. Юридически — внутренние инструменты для оказания консультационных услуг.

### 3.1. Нейро-команда (29 агентов)

Живёт в `my-architecture/agents/`. Директор читается перед любым ответом — он маршрутизирует задачи к нужному агенту.

**29 активных агентов** в 10 департаментах: маркетинг, продажи, продукт, аналитика, брендинг, планирование, финансы, юридический, инфраструктура, качество.

Полный список агентов, их роли и маршрутизация задач: `agents/routing-table.md`

**Как работает команда:** Директор управляет только 10 руководителями департаментов. Руководитель департамента знает своих специалистов и скиллы. Директор + руководитель + специалист закрывают 95% задач.

**В роадмапе:** Client Success · Accountant · Partner.

### 3.2. Принципы работы команды

**Директор — маршрутизатор верхнего уровня.** Все задачи входят через Директора. Он управляет только **10 руководителями департаментов** — не каждым специалистом и не каждым скиллом напрямую. Руководитель департамента знает своих специалистов и их инструменты — и принимает решения внутри своего домена.

**Принцип иерархии:**

```
Директор
   ↓ управляет только 10 руководителями
Руководитель департамента
   ↓ управляет своими специалистами и скиллами
Специалист (агент) со своими скиллами
   ↓ выполняет конкретную задачу
Результат
```

**Скилл = инструмент агента.** Скилл живёт отдельным файлом (чтобы можно было переиспользовать), но **принадлежит конкретному агенту**. В каждом агент-файле есть раздел `### Скиллы` со ссылками на его инструменты. Директор не держит каталог скиллов в голове — он знает только руководителей департаментов.

**Два режима работы:**

| Режим | Когда | Как |
|---|---|---|
| **Линейный** | Стандартная задача | Директор → руководитель департамента → специалист + скилл → результат |
| **Параллельный** | Сложная задача, несколько доменов | Директор → несколько руководителей одновременно → Директор собирает результат |

**Типичный рабочий цикл:**

```
Задача от Инны
   ↓ Директор читается первым (обязательно)
   ↓ Определяет домен → передаёт руководителю департамента
   ↓ Руководитель выбирает нужного специалиста и/или скилл
   ↓ Специалист выполняет через свой скилл
   ↓ Директор контролирует качество (QA-чек)
   ↓ Результат Инне
```

**Правила взаимодействия:**
- Директор читается **до** любого ответа на задачу — не после
- Директор знает только руководителей департаментов; руководитель знает своих специалистов и скиллы
- Агент работает только в своём домене; выход за рамки — через Директора
- При конфликте приоритетов (контент vs продажи, бренд vs юридика) — Директор принимает решение

- **Конвертирует** — записывает на консультацию / отдаёт чек-лист / ведёт на продукт

**Сценарий работы AI-консультанта:**

```
Один AI-консультант "Инна-помощник" (или нейтральное имя)
   ↓ задаёт 4-6 вопросов
   ↓ внутри определяет сегмент
   ↓ запускает нужный сценарий:
       → Для эксперта → ведёт к "Платформа С.С.С.Р."
       → Для офлайн-бизнеса → ведёт к "AI-аудит"
       → Для крупного клиента → ведёт к "Нейро-архитектура"
   ↓ финал: запись на встречу / получение материала / покупка
```

### 5.4. Нишевые лендинги — принцип
