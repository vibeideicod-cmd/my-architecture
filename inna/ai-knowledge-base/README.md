# AI-knowledge-base — клиент-ориентированный слой базы знаний 11.05 v1

> **Парный документ к** `inna/architecture-2026-05-11-v1.md` Слой 5.3 «Один AI — 5 контекстов».
> **Зачем существует:** AI-помощник Инны отвечает посетителю HUB. Его «голова» = база знаний. Если на месте базы — внутренняя методология Director (`knowledge/`) — AI отвечает «прочитай Phase 2 из product-creation.md», клиент в шоке. Нужен **отдельный слой** на языке эксперта-клиента.
> **Статус:** каркас. Наполнение — задача Content + Copywriter + AI-Builder, до запуска Этапа 4 серверной.
> **Триггер:** AI-помощник не запускается без этой папки заполненной.

---

## 1. Зачем нужен отдельный слой

### 1.1. Что НЕ подходит как база знаний AI

| Источник | Почему НЕ подходит |
|---|---|
| `knowledge/` (вся папка) | Внутренняя методология команды — playbooks Director, агент-руководства, Phase 1-7 продукт-конструкции. Язык методологический, не клиентский |
| `agents/*.md` | Описания агентов команды Инны — не публичный контент |
| `inna/architecture-*.md` | Архитектурный документ — для команды, не для клиента |
| `inna/key-decisions.md` | Внутренние решения — клиент не должен видеть |
| `clients/*/` | Чужие данные — не наш AI должен про них рассказывать |

### 1.2. Что подходит и пишется заново

| Что | Зачем AI это знает | Откуда берём |
|---|---|---|
| **5 болей** в словах эксперта-клиента | AI диагностирует боль за 1-2 вопроса | `inna/audience.md` + `inna/positioning.md` + правки Copywriter |
| **Продуктовые карточки** (8 продуктов из `product-line.md` + Платформа) | AI предлагает конкретный продукт под боль | `inna/product-line.md` — выжимка под клиента |
| **FAQ** (30-50 вопросов) | AI отвечает на типовые вопросы без передачи Инне | Сбор из реальных диалогов (МГ, Zoom, чат-сообщество) |
| **Кейсы** (Beauty, Южнобережный, Ирина, Инна сама) | Социальное доказательство по нишам | `clients/*/case-study.md` — выжимка под публику |
| **Прайс-вилки** | AI отвечает «от 50к до 300к» когда уместно | `inna/product-line.md` |
| **«Как мы работаем»** — методология С.С.С.Р. в 1 минуту | AI отвечает на «как у вас всё устроено» | `inna/positioning.md` + `knowledge/methodology/` (адаптация) |

## 3. Шаблон карточки боли

```markdown
---
pain: 01
name: Тащу всё на себе
status: для AI-помощника HUB
---

## В словах клиента
«Я делаю всё сама — переписки, посты, рассылки, видео, ответы клиентам. Хочу делегировать, но боюсь что без меня всё развалится.»

## Узнавание боли (что отвечает AI)
Когда клиент описывает похожую ситуацию — AI отвечает в формате:
1. Узнавание («звучит как ситуация когда ты тащишь всё сама»)
2. Зеркало («это часто бывает у экспертов которые работают через сарафан»)
3. Мост к решению («есть несколько способов снять это — расскажу что подходит именно тебе?»)

## Связанные продукты
- Вход: лид-магнит «5 рутин, которые можно отдать AI» → AI-диагностика
- Mid: AI-помощник для рутины (бот ответов / автоответы) — от 30к
- Flagman: Нейро-архитектура под ключ — от 300к

## Что НЕ отвечать
- Не давать готовое решение в первом ответе (преждевременная продажа)
- Не упоминать конкурентов по имени
- Не сравнивать цены
- Не использовать слова «курс / обучение / урок» (ОКВЭД 70.22)
- Не обещать «100% результат за N дней»

## Юр-плашка
Все ответы AI попадают под формулу «Аудит → Рекомендации → Сопровождение внедрения» по ОКВЭД 70.22.
```

## 4. Шаблон карточки продукта

```markdown
---
product: ai-diagnostika
name: AI-диагностика
type: bridge (мост от лид-магнита к платному)
price: бесплатно (для AI на HUB) / встроена в AI-помощника
---

## Что это в одной фразе
4-6 вопросов через AI → понимаешь где у тебя точка роста и какой продукт подойдёт.

## Что получает клиент
- Понимание своей ниши и боли (из 8 ниш + 5 болей)
- Понимание этапа развития (C/B/A — по методологии С.С.С.Р.)
- Конкретное предложение продукта Инны (с ценой и описанием)
- Возможность записаться на платный аудит или забрать лид-магнит

## Когда AI предлагает
- Клиент только зашёл на HUB и не знает с чего начать
- Клиент сказал «расскажи что у вас есть» — это рано для продукта
- Клиент сказал «помоги выбрать» — точно AI-диагностика

## Когда AI НЕ предлагает
- Клиент уже готов покупать конкретный продукт — не предлагаем диагностику, ведём в продукт
- Клиент задаёт юридический или административный вопрос — отвечаем по сути
- Клиент жалуется — сначала эмпатия, потом решение

## CTA
«Готов(а) пройти AI-диагностику? Это 4-6 вопросов, займёт ~3 минуты — а в финале я покажу что подойдёт именно тебе.»

## Юр-плашка
AI-диагностика = инструмент квалификации, не самостоятельный продукт. Не образовательная услуга. ОКВЭД 70.22.
```

## 5. Юр.фильтр — что НИКОГДА не отвечает AI

По `feedback_no_education_license` + `feedback_okved_law_filter_always`:

### Запрещённые слова
- курс, обучение, школа, тренинг, урок, преподаватель, ученик, академия, мини-курс, диплом, сертификат об обучении
- «гарантирую N клиентов за N дней» (нарушение защиты прав потребителей)
- «секрет миллионных продаж» (инфоцыганская лексика, не наша ЦА)

### Разрешённые формулировки
- консультация, аудит, разбор, сопровождение, методические материалы
- экспертная сессия, чек-лист, шаблон, видео-разбор, диагностика
- «помогаем разобраться», «строим под твою задачу», «адаптируем под нишу»

### Обязательная маркировка
- При первом сообщении: «Я — AI-помощник Инны, виртуальный ассистент, не сама Инна»
- При сборе контакта: «Согласен(на) на обработку персональных данных»
- Перед выдачей советов: «Это рекомендация, не персональное юридическое/медицинское заключение»

## 6. Содержание FAQ (30-50 вопросов)

| Категория | Примеры вопросов |
|---|---|
| **Что делаете** | «Что такое архитектор автосистем?» / «Это IT-консалтинг?» / «Вы делаете сайты?» |
| **Для кого** | «Подойдёт ли мне если я психолог?» / «А если у меня бьюти?» / «У меня нет команды — подойдёт?» |
| **Как работаем** | «Сколько времени занимает аудит?» / «Что входит в платный аудит?» / «А если мне не подойдёт?» |
| **Цены** | «Сколько стоит?» / «А мини-продукт есть?» / «Какие тарифы Платформы?» |
| **Гарантии** | «Если не сработает — вернёте деньги?» / «А если я не доведу до конца?» |
| **Технологии** | «Какие нейросети используете?» / «Это безопасно с моими данными?» / «А вы российский сервис?» |
| **Юр.вопросы** | «Какой ИП у вас?» / «Договор подпишем?» / «А налоги?» |
| **Партнёр Ирина** | «Кто такая Ирина?» / «Она тоже консультирует?» |
| **Кейсы** | «Покажете примеры?» / «Кто из клиентов вырос?» / «А санатории работали?» |

Финальный список — задача Sales-агента (он видит реальные вопросы клиентов) + Content-агента (формулирует ответы).
