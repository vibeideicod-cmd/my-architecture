## Короткий вердикт

Контур Инны не надо строить с нуля. Уже есть:
- стратегическая база;
- два HTML-варианта HUB;
- интерактивная визитка;
- пакет первой воронки;
- каркас AI-базы;
- концепт платформы.

Главный разрыв: **есть витрина и тексты, но ещё нет единой рабочей цепочки “человек пришёл → прошёл карту/диагностику → контакт попал в базу → AI/Инна выдали следующий шаг”.**

## Главные выводы

1. **Основной кандидат на production-HUB — корневой `index.html`.** Он крупнее, богаче и содержит больше блоков, чем `hub/index.html`.
2. **`hub/index.html` сейчас лучше считать дублем/старой версией**, пока не принято решение о production-пути.
3. **Первый продуктовый вход должен быть “Карта точек роста”, а не “бесплатная диагностика”.** Это уже зафиксировано в `inna/моя-воронка-1/terminology.md`.
4. **Формы пока декоративные.** В `index.html` заявка не отправляется наружу, а в `inna/visitka/index.html` backend-вызовы закомментированы.
5. **AI-помощник нельзя запускать на пустой базе.** Сначала минимум: 5 болей, продуктовые карточки, FAQ-ядро, tone.
6. **Визитка уже может жить как production-актив**, но перед воронкой нужно синхронизировать её CTA с новым входом.

## Решение по запуску

Запускать VK/Telegram пока рано. Сначала нужен минимальный рабочий путь:

```
HUB → Карта точек роста → результат → контакт → ручной/CRM follow-up → предложение следующего шага
```

После этого можно:
1. обновлять VK;
2. публиковать закреп;
3. вести трафик на HUB;
4. зеркалить систему на Ирину.

## Следующий рабочий шаг

Начать с **P0 “единый production-HUB + CTA Карта точек роста”**.

Конкретно:
1. принять корневой `index.html` как основную версию для доводки;
2. заменить публичную “бесплатную диагностику” на “Карту точек роста”;
3. оставить “диагностику” только как внутреннюю/AI-логику, если нужно;
4. решить, куда ведёт CTA до подключения CRM: Telegram `@neuro_strateg_inna`, `@AI_avtosistem` или форма с ручной обработкой.
