# Создаём бота-копирайтера: мультиагентный подход

## Суть (3-5 предложений)

Пошаговая инструкция для создания Telegram-бота, который пишет в вашем стиле на основе загруженных примеров. Подход использует мультиагентную архитектуру: каждый агент отвечает за одну задачу (копирайтинг, редактирование, контент-план, тестирование), что даёт точнее результат, чем одна универсальная модель. Боту необходимо знание ваших паттернов речи, контент-плана на три ветки (экспертная, личная, продуктовая) и интеграция с OpenRouter для выбора оптимальной модели под каждую задачу.

## Ключевые знания

1. **Четырёхэтапная архитектура проекта** — сначала создаёшь команду агентов в agents.md, затем Claude строит всю архитектуру, потом добавляешь ключи API и тестируешь, в конце итеративно доработаешь. Пропустить этап с agents.md = хаос.

2. **Одиннадцать специализированных агентов** — Оркестратор (координация), Архитектор (структура), Декомпозитор (разбиение задач), Тестировщик, Диспетчер (маршрутизация), Аналитик стиля (Tone of Voice), Ресёрчер, Копирайтер, Критик, Редактор, Контент-планер. Каждый отвечает за одну работу — ИИ плохо понимает многосоставные задачи.

3. **Выбор модели по задаче** — Claude Opus 4.6 для архитектуры и сложного кода, Claude Sonnet 4.6 для творческих задач и копирайтинга, Claude Haiku 4.5 для быстрых проверок и маршрутизации. Температура зависит: низкая (0.1–0.3) для логики, высокая (0.7–0.8) для творчества.

4. **Контент-план на три ветки** — Экспертная (40%, тренды и мнения), Личная (40%, истории и доверие), Продуктовая (20%, офферы и кейсы). Каждый пост должен попадать в слой аудитории по БМПХС: Боли, Мечты, Проблемы, Хотелки, Страхи.

5. **OpenRouter вместо прямого API** — один ключ даёт доступ к Claude, GPT-4.1, Gemini, Llama и другим. Задаёшь лимит ($5–10) и срок действия, чтобы контролировать расходы.

6. **Загрузка обучающего контента** — выгружаешь посты из Telegram Desktop (целиком синяя иконка с сайта telegram.org) в JSON или MD, потом загружаешь файл в чат бота. Боту нужны примеры твоего стиля.

7. **Tone of Voice из голосовых** — транскрибируй голосовые сообщения и отдай Claude: он найдёт паттерны речи точнее, чем ты самому себе опишешь.

8. **Ошибка = скриншот в Claude** — получил ошибку в боте? Скриншот → Claude Code → он сам починит. Не трать время на долгое описание.

9. **Голосовые промпты экономят время** — Super Whisper или бесплатный Handy, назнач горячую клавишу. Нажал → сказал → отпустил. В разы быстрее печати.

10. **Мелкие задачи = лучший код** — не отправляй "напиши весь бот", а разбей на мелкие куски. Декомпозитор это делает автоматически, но ты тоже можешь.

## Промпты

### Промпт 1 — Создание команды агентов

```
Создаём копирайтера в Telegram, который будет писать моим языком на основе выгруженных файлов из моего Telegram-канала.

Файлы могут быть загружены в виде MD или JSON.

Он должен уметь:
- Создавать и читать контент-план
- Редактировать посты: делать более человечными, короткими, хлёсткими
- Писать в моём стиле на основе загруженных примеров

В первую очередь создай команду агентов в файле agents.md:
- Оркестратор — координирует всех
- Архитектор — планирует структуру проекта
- Декомпозитор — разбивает задачи на мелкие подзадачи
- Тестировщик — проверяет работу всех агентов

Для каждого агента укажи: роль, модель, температуру.
Также выдай структуру агентов с описанием.
```

### Промпт 2 — Запуск разработки

```
Отлично, теперь запускай команду в работу.
Полностью спроектируй и создай сервис.
Скажи, куда добавить ключи для OpenRouter и для бота Telegram.
```

### Промпт 3 — Контент-план с тремя ветками

```
Создай файл контент-план.md, подвязанный к кнопке «Контент-план».
Сохрани его в папку проекта.

В контент-плане:
- Проанализируй все посты, которые я загрузил как примеры стиля
- Разбей контент-план на три ветки:
  1. Экспертная — взгляд на рынок, тренды, экспертные мнения
  2. Продуктовая — про мои продукты и услуги
  3. Личная — истории, ошибки, процессы, закулисье
- Продающая ветка — в меньшей пропорции, чем остальные
- Проанализируй целевые аудитории, разбей на подсегменты
- Исходя из их болей, мечт, проблем, хотелок и страхов (БМПХС) создай план на ближайшие две недели
- По кнопке «Контент-план» в Telegram показывай 3 ближайших темы.
```

## Что может усилить архитектуру

- **Мультиагентная декомпозиция** — каждый агент за свою задачу, а не монолит. Это паттерн, который можно тиражировать на другие проекты (не только копирайтинг, но и разработку, аналитику, исследования).

- **Tone of Voice как вход в ML-систему** — идея извлекать паттерны речи из голосовых и текстов и использовать их как instruction для моделей универсальна и может работать в других задачах персонализации.

- **Трёхслойный контент-план (экспертная/личная/продуктовая)** — продолжаемый принцип балансировки контента для любого информационного канала или проекта (соцсети, новостной сайт, блог).

- **БМПХС как фреймворк сегментации аудитории** — можно применять не только в контент-плане, но и в дизайне продуктов, формировании оффера, создании скрипта продаж.

- **OpenRouter как абстракция над моделями** — паттерн независимости от конкретного провайдера; можно переключаться между Claude, GPT, Gemini без переписывания кода.
