# inbox-cl20/ — Обработанные материалы от Claude 20

Сюда Claude 20 (браузер) пушит обработанные файлы. Директор (Claude 100, VS Code) забирает их и интегрирует в knowledge/.

**Правила для Claude 20:**
- Пушить ТОЛЬКО в эту папку — в другие папки доступа нет
- Один файл = один материал (урок, встреча, идея)
- Формат — всегда MD, структура ниже
- Имена файлов: `тип-YYYY-MM-DD-тема.md`

---

## Формат файла

Каждый файл должен следовать этой структуре:

```markdown
---
тип: урок / встреча / идея / статья / кейс
источник: Нейроуниверситет / встреча МГ / свои заметки / внешний курс
дата: 2026-04-16
автор: Алексей Южанинов / участники МГ / Инна / другое
---

# [Название материала]

## Суть (3-5 предложений)
О чём этот материал, главная идея.

## Ключевые знания
Пронумерованный список конкретных знаний, техник, фреймворков.
Только практически полезное — теорию без применения пропускать.

1. **[Название знания]** — описание, как применять
2. **[Название знания]** — описание, как применять
...

## Промпты (если есть)
Готовые промпты из материала — скопировать как есть, в блоках кода.

## Фишки участников (для встреч МГ)
Кто чем поделился — кратко, по пунктам.
- **[Имя/ник]:** что рассказал, какая фишка
- **[Имя/ник]:** что рассказал, какая фишка

## Что может усилить нашу архитектуру
Мнение Claude 20: какие из этих знаний могут быть полезны для
проекта my-architecture. Не решения — только сигналы для Директора.

## Сырые данные (опционально)
Если Инна попросила сохранить оригинал — цитаты, фрагменты транскрипции.
```

---

## Примеры имён файлов

```
урок-2026-04-16-бот-копирайтер.md
встреча-2026-04-15-мг-zoom.md
идея-2026-04-16-квиз-санаторий.md
статья-2026-04-14-промтинг-бизнес.md
```

---

## Что НЕ делает Claude 20

- Не принимает решений по архитектуре
- Не создаёт/меняет файлы в knowledge/, agents/, clients/
- Не удаляет файлы
- Не придумывает факты — только то, что есть в материале
