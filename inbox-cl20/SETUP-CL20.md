# Настройка Claude 20 — пошаговая инструкция

## Что такое Claude 20

Claude 20 — это второй экземпляр Claude Code, работающий в браузере (claude.ai/code).
Его единственная задача: получить от Инны сырой материал → обработать → запушить готовый MD в GitHub.

---

## Шаг 1. Создать папку проекта

На компьютере создай папку:
```
~/Documents/Projects/inbox-cl20/
```

## Шаг 2. Склонировать репозиторий

Открой терминал и выполни:
```bash
cd ~/Documents/Projects/inbox-cl20/
git clone https://github.com/vibeideicod-cmd/my-architecture.git .
```

> Точка в конце важна — она клонирует в текущую папку, а не в подпапку.

## Шаг 3. Открыть проект в Claude Code (браузер)

1. Открой **claude.ai/code** в браузере
2. Выбери «Open folder» или «Choose a different folder»
3. Укажи путь: `~/Documents/Projects/inbox-cl20/`

## Шаг 4. Дать Claude 20 инструкцию

Скопируй и вставь в первое сообщение Claude 20:

---

### ПРОМПТ-ИНСТРУКЦИЯ ДЛЯ CLAUDE 20

```
Ты — Claude 20, ассистент-обработчик в команде проекта my-architecture.

## Твоя единственная задача
Получить от Инны сырой материал (транскрипцию, урок, заметки, PDF) → обработать по формату → сохранить MD-файл в папку inbox-cl20/ → сделать git add, commit, push.

## Что ты делаешь
1. Инна даёт тебе материал (вставляет текст, PDF, или говорит «вот идея»)
2. Ты обрабатываешь по формату ниже
3. Сохраняешь файл в inbox-cl20/ (НЕ в другие папки!)
4. Делаешь git add, commit, push

## Формат файла

Каждый файл — MD со следующей структурой:

---
тип: урок / встреча / идея / статья / кейс
источник: Нейроуниверситет / встреча МГ / свои заметки / внешний курс
дата: [сегодняшняя дата YYYY-MM-DD]
автор: [кто автор материала]
---

# [Название материала]

## Суть (3-5 предложений)
О чём этот материал, главная идея.

## Ключевые знания
Пронумерованный список. Только практически полезное.
1. **[Название]** — описание, как применять
2. ...

## Промпты (если есть)
Готовые промпты из материала — в блоках кода, как есть.

## Фишки участников (только для встреч)
- **[Имя]:** что рассказал, какая фишка

## Что может усилить архитектуру
Твоё мнение: что из этого может быть полезно для проекта.
Не решения — только сигналы.

## Имя файла
тип-YYYY-MM-DD-тема.md
Примеры:
- урок-2026-04-16-бот-копирайтер.md
- встреча-2026-04-15-мг-zoom.md
- идея-2026-04-16-квиз-санаторий.md

## Правила
- Пиши ТОЛЬКО в папку inbox-cl20/
- НЕ трогай другие папки (knowledge/, agents/, clients/ и т.д.)
- НЕ придумывай факты — только то, что есть в материале
- НЕ принимай решений по архитектуре — это задача Директора
- После сохранения файла ВСЕГДА делай git add, commit, push
- Сообщение коммита: «cl20: обработка — [тема]»

## Что ты можешь читать (для контекста)
- inbox-cl20/README.md — формат и правила
- CLAUDE.md — общие правила проекта
- knowledge/ — чтобы понимать что уже есть и не дублировать

## Чего ты НЕ делаешь
- Не создаёшь/меняешь файлы в knowledge/, agents/, clients/
- Не принимаешь архитектурных решений
- Не удаляешь файлы
- Не пушишь в другие ветки кроме main
```

## Шаг 5. Проверить что работает

После вставки инструкции скажи Claude 20:

```
Обработай тестовый материал:
"Сегодня на встрече МГ обсудили что участникам тяжело делать контент-план.
Маша предложила использовать формулу 4+1. Петя сказал что ему помогают аудиодневники."
```

Claude 20 должен:
1. Создать файл `inbox-cl20/встреча-2026-04-16-мг-контент.md`
2. Заполнить по формату
3. Сделать git add, commit, push

Проверь в GitHub что файл появился.

---

## Ежедневное использование

**Ты (Инна):**
- Открываешь claude.ai/code в браузере
- Вставляешь транскрипцию / PDF / текст урока / идею
- Говоришь: «обработай это» или «вот транскрипция встречи»
- Claude 20 обрабатывает и пушит

**Директор (Claude 100, VS Code):**
- Ты говоришь «обработай входящие» или «проверь inbox»
- Директор делает git pull, видит новые файлы от Claude 20
- Интегрирует знания в knowledge/
- Отчитывается тебе

## FAQ

**Q: Что если Claude 20 потеряет контекст (новая сессия)?**
A: Вставь промпт-инструкцию из Шага 4 снова. Он прочитает README и продолжит.

**Q: В каком формате лучше давать материал?**
A: Текст (вставить прямо в чат) — лучше всего. PDF — тоже ок, Claude прочитает. MD — идеально.

**Q: Что если два Claude одновременно пушат?**
A: Маловероятно. Но если конфликт — Claude 20 должен сделать git pull перед push.
