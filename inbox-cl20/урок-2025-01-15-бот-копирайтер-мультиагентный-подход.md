# Создаём бота-копирайтера. Мультиагентный подход

## Суть
Мультиагентный подход разбивает сложный проект на несколько ИИ-агентов, каждый отвечает за одну задачу. Это даёт управляемость процессом, более точный код и сокращение количества итераций доработки. За 4 этапа (Команда → Архитектура → Реализация → Докрутка) создаётся рабочий Telegram-бот, который пишет посты в вашем стиле на основе загруженных примеров.

## Ключевые знания

1. **Мультиагентный подход точнее, чем одна большая задача** — ИИ плохо понимает многосоставные задачи. Когда каждый агент отвечает за одно, результат точнее. Декомпозитор разбивает на мелкое, остальные выполняют.

2. **4 этапа любого проекта** — 
   - Команда: создать agents.md (Оркестратор, Архитектор, Декомпозитор, Тестировщик с ролями и моделями)
   - Архитектура: Claude строит всю структуру, API, файлы
   - Реализация: добавить ключи, запустить, проверить базовую работу
   - Докрутка: тестируем → описываем что не нравится → Claude правит → бесконечный цикл

3. **Выбор модели по задаче** — Opus (архитектура, дорого, максимум качества), Sonnet (тексты, код, творчество, оптимум), Haiku (быстрые проверки, маршрутизация, дёшево). Для копирайтера основная работа на Sonnet.

4. **11 агентов бота** — Оркестратор (0.3), Архитектор (0.7), Декомпозитор (0.3), Тестировщик (0.1), Диспетчер (0.2), Аналитик стиля (0.5), Ресёрчер (0.4), Копирайтер (0.8), Критик (0.4), Редактор (0.6), Контент-планер (0.5). Температура 0.1–0.3 для логики, 0.4–0.8 для творчества.

5. **Контент на три ветки: Экспертная (40%) + Личная (40%) + Продуктовая (20%)** — 
   - Экспертная: взгляд на рынок, тренды
   - Личная: истории, ошибки, закулисье
   - Продуктовая: услуги, кейсы, цены (меньшая доля)
   - Используйте БМПХС (Боли, Мечты, Проблемы, Хотелки, Страхи) для каждой целевой группы

6. **Telegram Desktop (с сайта telegram.org) выгружает посты в JSON** — это обучение для бота. Синяя иконка целиком, не из App Store. Заливаете примеры → бот анализирует стиль → пишет так же.

7. **OpenRouter объединяет все модели в одном ключе** — Claude, GPT, Gemini, Llama вместе. Ключ в Settings → API Keys, задайте лимит ($5–10) для контроля расходов.

8. **Скрытый файл .env хранит ключи (Telegram + OpenRouter)** — на Mac видны по ⌘+⇧+.. Если .env.example — просто переименуйте. Закрывайте окно ключа только после копирования.

9. **Ошибка в боте? Скриншот в Claude Code** — скриншот → вставить → Claude чинит → перезапуск. Цикл тестирования.

10. **Голосовые промпты быстрее (Super Whisper или Handy)** — горячая клавиша → сказали → отпустили. Handy бесплатный.

11. **Claude Code vs Codex vs Cursor** — Claude Code в Pro (лучший), Codex (OpenAI, $20), Cursor (объединяет модели).

## Промпты

### Промпт 1 — Команда агентов

```
Создаём копирайтера в Telegram, который будет писать моим языком на основе
выгруженных файлов из моего Telegram-канала.
Файлы могут быть загружены в виде MD или JSON.

Он должен уметь:
- Создавать и читать контент-план
- Редактировать посты: делать более человечными, короткими, хлёсткими
- Писать в моём стиле на основе загруженных примеров

В первую очередь создай команду агентов в файле agents.md:
- Оркестратор — координирует всех
- Архитектор — планирует структуру проекта
- Декомпозитор — разбивает задачи на мелкие подзадачи
- Тестировщик — проверяет работу всех агентов

Для каждого агента укажи: роль, модель, температуру.
Также выдай структуру агентов с описанием.
```

### Промпт 2 — Запуск разработки

```
Отлично, теперь запускай команду в работу.
Полностью спроектируй и создай сервис.
Скажи, куда добавить ключи для OpenRouter и для бота Telegram.
```

### Промпт 3 — Контент-план на три ветки

```
Создай файл контент-план.md, подвязанный к кнопке «Контент-план».
Сохрани его в папку проекта.

В контент-плане:
- Проанализируй все посты, которые я загрузил как примеры стиля
- Разбей контент-план на три ветки:
  1. Экспертная — взгляд на рынок, тренды, экспертные мнения
  2. Продуктовая — про мои продукты и услуги
  3. Личная — истории, ошибки, процессы, закулисье
- Продающая ветка — в меньшей пропорции, чем остальные
- Проанализируй целевые аудитории, разбей на подсегменты
- Исходя из их болей, мечт, проблем, хотелок и страхов
  создай план на ближайшие две недели

По кнопке «Контент-план» в Telegram показывай 3 ближайших темы.
```

## Что может усилить архитектуру

- Интеграция с парсингом соцсетей — автоматическое подтягивание актуальных топиков в контент-план
- Tone of Voice из голосовых сообщений — транскрибировать и анализировать для точности стиля
- A/B-тестирование вариантов постов — несколько версий и анализ реакций
- Переиспользование контента — один пост в несколько каналов
- Реитерирующий контент-план — обновление на основе трендов и залипаемости постов
