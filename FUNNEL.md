## 2. Юр.рамка — две независимые единицы

| | **Инна Андрейченко** | **Ирина Цепаева** |
|---|---|---|
| Юр.форма | ИП | Самозанятая (НПД, ФЗ-422) |
| ОКВЭД / виды | **70.22** Консультирование | **74.10** Дизайн + **62.01** Разработка ПО + **90.03** Иллюстрация |
| Что говорит публично | «провожу аудит, даю рекомендации, сопровождаю внедрение» | «изготавливаю, разрабатываю, верстаю, реализую» |
| Запрещённые слова | делаю, создаю, разрабатываю, программирую | провожу аудит, консультирую, разрабатываю стратегию, сопровождаю внедрение |
| База клиентов | «Моя база клиентов» (VPS, vibe coding) | Своя (Notion / таблица / мини-CRM) |
| Договоры | Свой шаблон ИП-клиент | Свой шаблон самозанятая-клиент |
| Чеки | Через ИП | Через «Мой налог» |

**Закулисье ↔ юр.фасад:**
- Закулисье (нигде не пишем в документах): вдвоём ходят на встречи, обмениваются работами, проверяют друг друга.
- Юр.фасад: каждая работает с клиентом **напрямую по своему договору**. Между Инной и Ириной никаких договоров нет (ни поручения по ПД, ни рамочного партнёрства). Партнёрских процентов между ними тоже нет.

**Главное:** клиент видит «слаженную команду», но юридически это **две независимые сделки**.

## 4. AI-диагностика — два AI-помощника

### Архитектурное решение #2 (`clients/irina/architecture-decisions.md`)

**Один движок (Claude API) — два AI с разными настройками.** Каждый живёт на своём сайте, обучен на своих материалах, говорит своим тоном.

| Параметр | AI-помощник Инны | AI-помощник Ирины |
|---|---|---|
| Где живёт | На HUB | На сайте Ирины |
| Обучен на | `inna/positioning.md`, `inna/product-line.md`, методология | `clients/irina/tov-source.md`, `clients/irina/positioning.md` (когда будет), портфолио |
| Тон | Голос Инны | Голос Ирины («слышу, какого человек цвета», «передаю готовый продукт») |
| Рекомендует | Продукты Инны → если нужна реализация → маршрутизирует на Ирину | Продукты Ирины → если нужна архитектура/стратегия → маршрутизирует на Инну |
| Юр.статус | «AI-помощник Инны» — представляется как AI | «AI-помощник Ирины» — представляется как AI |
| Реализация | ❌ Не реализован — главный блокер по `ARCHITECTURE-ROADMAP.md` | ❌ Не реализован — после AI-помощника Инны |

### Сценарий квалификации (4-6 вопросов)

**AI-помощник Инны** (источник: `inna/website.md` + `inna/architecture.md` Слой 5):
1. Чем занимаешься? → определяет нишу (1 из 8)
2. Что сейчас болит больше всего? → определяет боль (1 из 5)
3. Уточняющий вопрос по боли
4. Готов ли вкладываться в развитие? → определяет сегмент A / B / C
5. Финал: предложение лендинга боли / продукта / совместной встречи / Платформы С.С.С.Р.

**AI-помощник Ирины** (источник: `clients/irina/architecture.md` Слой 5.3):
1. Расскажи о себе и задаче (открытый)
2. Какая боль (1 из 5)
3. Какая ниша (1 из 8)
4. Размер запроса (микро / средний / крупный — определяет слой 1/19/80)
5. Финал: раздел «Сделать сейчас» / Платформа С.С.С.Р. в визуале / лендинг по боли / запрос на флагман

### Кросс-маршрутизация

- AI-помощник Инны: «Под вашу задачу подойдёт реализация — это работа Ирины Цепаевой, моего партнёра. Хотите, познакомлю?»
- AI-помощник Ирины: «Похоже, вам нужна стратегия и методология — это работа Инны Андрейченко, с которой мы работаем в связке. Хотите, передам контакт?»

**Куда сохраняется результат квалификации:** в свою базу клиентов через N8N (вебхук от AI → база). У каждой — своя.

### 5.3. Партнёрские бандлы (`clients/irina/architecture-decisions.md` Решение 4)

**Бандл = маркетинговая упаковка, юридически — 2 контракта.**

Пример: «Упаковка эксперта — 50к» = брендбук Ирины (30к) + аудит позиционирования Инны (20к). Клиент видит один продукт, подписывает 2 договора, платит каждой напрямую.

| Бандл | Зона Инны (консультация) | Зона Ирины (реализация) | Боль |
|---|---|---|---|
| Упаковка эксперта | Аудит позиционирования | Брендбук + визитка + соцсети-визуал | Меня не видят |
| Старт онлайн | Стратегия воронки | Лендинг + квиз + бот-запись | Нет клиентов |
| Автосистема | Архитектура автоматизации | AI-помощник + воронка-вёрстка | Нет времени |

## 7. Точка соединения двух контуров — совместная встреча

После квалификации (AI-помощник любого из двух сайтов) — если клиент **прошёл фильтр** (сегмент A у Инны / слой 80 у Ирины) — запускается **совместная встреча** Инна + Ирина (закулисье).

**Что не описано в существующих файлах** — будет добавлено сюда по итогам обсуждения с Инной:
- Где встреча: Zoom / офлайн / TG-звонок?
- Кто формально ведёт встречу?
- Как клиент узнаёт что приходят обе?
