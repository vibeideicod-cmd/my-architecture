## 🅰 СЦЕНАРИЙ А — Anthropic заблокировал твой аккаунт

### Что произошло (понять)

- В VS Code в Claude Code пишет «authentication failed» или «account suspended»
- На `claude.ai` тебя выкидывает или пишет «ваш аккаунт заблокирован»
- Письмо от Anthropic: «we have suspended your account due to...»

### Что НЕ страшно

✅ Все твои файлы на месте — на Mac, на VPS, на GitHub.
✅ 31 агент команды на месте — это .md-файлы, никак не зависят от Anthropic.
✅ n8n + Baserow работают — они на VPS, не на Anthropic.
✅ Roo Code установлен и настроен на бесплатный Gemini.

### Что делать (шаг за шагом)

**Шаг 1 — Открой VS Code (если он закрыт)**
- Иконка в Dock, или Spotlight (Cmd+Space) → набери «Visual Studio Code» → Enter.

**Шаг 2 — Найди Roo Code в боковой панели**
- Слева в VS Code вертикальная панель с иконками.
- Найди иконку **кенгуру** (тёмная фигурка). Это Roo Code.
- Кликни на неё — справа откроется панель Roo Code.

**Шаг 3 — Проверь что Gemini подключён**
- Сверху в панели Roo Code нажми **иконку шестерёнки** (⚙️).
- Должно быть выбрано: **Провайдер API = Google Gemini**, **Модель = gemini-2.5-flash**.
- Если поле «API-ключ» пустое — иди в раздел **«Если ключ слетел»** ниже.
- Если всё на месте — нажми **стрелку «←»** сверху, чтобы вернуться в чат.

**Шаг 4 — Напиши задачу прямо в чат**
- Снизу в панели Roo Code есть поле «Введите сообщение...».
- Напиши то, что собиралась делать сегодня. Например:
  > «Привет. Anthropic меня забанил, теперь работаем через тебя. Прочитай `STATE.md` в проекте `my-architecture` и расскажи где мы остановились.»
- Roo Code прочитает файлы и продолжит работу.

**Шаг 5 — Если задача сложная и Gemini Flash не справляется**
- Внизу панели нажми на надпись «Architect» (или какой режим там стоит) → выбери **«Code»** для написания кода или **«Orchestrator»** для сложных задач.
- Если совсем не справляется — переключи модель на **Gemini 2.5 Pro**: шестерёнка → Модель → `gemini-2.5-pro`.
- Если и это не помогло — переходи к **Шаг 6**.

**Шаг 6 — Если нужна модель помощнее (платно, но мощно)**
- В Roo Code → шестерёнка → провайдер API → выбери **«OpenAI»**.
- API-ключ возьми с `platform.openai.com` (раздел API keys, там у тебя $14.94 на балансе).
- Модель: `gpt-5` или `gpt-4o`.
- Сохрани. Работаешь.

### Если Gemini API-ключ слетел

1. Открой в браузере **`aistudio.google.com/apikey`**.
2. Войди аккаунтом **`vibeideicod@gmail.com`**.
3. Нажми кнопку копирования рядом с существующим ключом ИЛИ создай новый кнопкой **«+ Create API key»**.
4. Вставь в Roo Code: шестерёнка → API-ключ → `Cmd+V` → сохрани.

### Подача апелляции (отдельно от работы)

Если думаешь что бан **ошибочный** — напиши:
- Email: `support@anthropic.com`
- Тема: «Account Suspension Appeal»
- Тело: коротко, по-английски: какой email аккаунта, когда забанили, попроси пересмотреть.
- Срок ответа: **обычно 3-7 дней**, иногда дольше.

### Подписка $100/мес — что с ней

- Если бан **окончательный** — отмени подписку Claude Max, чтоб не списывали ещё месяц. Зайди на `claude.ai` → Settings → Billing → Cancel.
- Если **временный** — подожди ответа Anthropic, не отменяй.

## 🅱 СЦЕНАРИЙ Б — Mac пропал (украли, сломался, утопила)

### Что произошло

- Физически нет MacBook
- Или Mac не загружается
- Или диск повреждён

### Что НЕ страшно

✅ **Все твои данные есть на VPS** в зашифрованном бэкапе (mac-2026-XX-XX_XX-XX.tar.gz.gpg).
✅ Бэкапы делаются каждый день в 22:00 — потеряешь максимум **24 часа** работы.
✅ FileVault включён — данные на Mac зашифрованы, посторонний не прочитает.

### Что делать

**Шаг 1 — Достань новый Mac**
- Любой свежий Mac. Идеально — тот же MacBook Air M1/M2 чтоб не переучиваться.
- Войди в **свой Apple ID** (`vibeideicod@gmail.com` или какой у тебя).

**Шаг 2 — Установи нужные программы**
- **VS Code** — `code.visualstudio.com` → скачай → установи.
- **Homebrew** (менеджер программ для терминала) — открой Терминал → вставь:
  ```
  /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  ```
- **GnuPG** (для расшифровки бэкапа) — в терминале: `brew install gnupg`.
- **Расширения VS Code**: Claude Code (если ещё работает), Roo Code, Pencil.

**Шаг 3 — Достань SSH-ключ**

Это самое важное и трудное место. **Без SSH-ключа ты не зайдёшь на VPS, чтобы скачать бэкап.**

Варианты:
- **Если в iCloud Keychain включена синхронизация** — ключ может прилететь сам. Проверь: открой Связку ключей → ищи «SSH key».
- **Если ты записывала где-то отдельно** — например в 1Password, Bitwarden, или скопировала в личный Telegram «Избранное».
- **Если ничего нет** — иди в панель Beget Cloud (`cp.beget.com`) → войди логином/паролем (восстанови если нужно) → найди свой VPS Cheerful Marik → раздел «**Настройки**» → **«SSH-ключи»** → можешь добавить новый ключ или сбросить root-пароль и зайти по паролю.

**Шаг 4 — Скачай бэкап с VPS**

В терминале:
```
ssh root@45.9.41.80 "ls -lh /root/claude-backups/"
```
Увидишь список архивов. Самый свежий — внизу. Скачай его:
```
scp root@45.9.41.80:/root/claude-backups/mac-САМЫЙ_СВЕЖИЙ.tar.gz.gpg ~/Downloads/
```

**Шаг 5 — Расшифруй бэкап**

Тебе понадобится **passphrase** (тот длинный пароль типа `1c0ec007dea3fc7f5b76b2f4d2dc9561`). Где он:
- На бумажке у тебя дома (помнишь, я просила записать?).
- ИЛИ в iCloud Keychain (если синхронизировался).

В терминале:
```
gpg -d ~/Downloads/mac-САМЫЙ_СВЕЖИЙ.tar.gz.gpg > ~/Downloads/restored.tar.gz
```
Введи passphrase.

**Шаг 6 — Распакуй**

```
cd /
tar -xzf ~/Downloads/restored.tar.gz
```

Архив развернётся точно туда, откуда брался — `Users/innaandreychenko/Documents/Projects/`, `Users/innaandreychenko/.claude/` и т.д.

**Шаг 7 — Проверь что всё на месте**

- Открой VS Code, выбери папку `~/Documents/Projects/my-architecture/`.
- Должны быть видны все твои файлы, агенты, история.
- Открой `STATE.md` — увидишь где остановилась.

**Шаг 8 — Восстанови подключения**

- В Claude Code или Roo Code — заново вставь API-ключи.
- В Связку ключей добавь passphrase бэкапа: открой терминал, выполни:
  ```
  security add-generic-password -a "$USER" -s "claude-backup-key" -w "ТВОЯ_PASSPHRASE" -U
  ```
- Запусти `~/Backups/backup-mac-to-vps.sh` чтобы убедиться что бэкап продолжает работать.

