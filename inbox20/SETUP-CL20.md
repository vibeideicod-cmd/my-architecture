# Как настроить Claude 20 — пошаговая инструкция

---

## Шаг 1. Открой Claude Code в браузере

1. Открой браузер (Chrome)
2. Перейди по адресу: **claude.ai/code**
3. Войди под своим аккаунтом Anthropic (тот же что для обычного claude.ai)

Ты увидишь интерфейс похожий на чат, но это Claude Code — он умеет создавать файлы и пушить в GitHub сам.

---

## Шаг 2. Подключи GitHub

При первом входе claude.ai/code попросит подключить GitHub:

1. Нажми **«Connect to GitHub»** (или «Подключить GitHub»)
2. Откроется страница GitHub — нажми **«Authorize»** (Разрешить)
3. Вернёшься в claude.ai/code

Если не попросил — найди настройки (значок шестерёнки) → GitHub → Connect.

---

## Шаг 3. Выбери наш репозиторий

1. В claude.ai/code нажми **«Open project»** или **«Select repository»**
2. Найди в списке: **vibeideicod-cmd/my-architecture**
3. Выбери его
4. Ветка: **main**

Теперь Claude 20 видит все файлы нашего проекта.

---

## Шаг 4. Дай инструкцию Claude 20

Скопируй весь текст ниже и вставь в первое сообщение:

```
Ты — Claude 20, ассистент-обработчик в проекте my-architecture.

Прочитай файл inbox-cl20/README.md — там твой формат и правила.

## Твоя единственная задача
Я (Инна) даю тебе сырой материал → ты обрабатываешь → сохраняешь MD-файл в папку inbox-cl20/ → делаешь git add, commit, push.

## Что ты делаешь
1. Я вставляю текст (транскрипцию, урок, заметки, идею)
2. Ты обрабатываешь по формату из inbox-cl20/README.md
3. Сохраняешь файл в inbox-cl20/ (имя: тип-YYYY-MM-DD-тема.md)
4. Делаешь git add, commit -m "cl20: обработка — [тема]", push

## Формат файла

---
тип: урок / встреча / идея / статья / кейс
источник: [откуда материал]
дата: [YYYY-MM-DD]
автор: [кто автор]
---

# [Название]

## Суть (3-5 предложений)
Главная идея.

## Ключевые знания
1. **[Название]** — как применять
2. ...

## Промпты (если есть)
Готовые промпты в блоках кода.

## Фишки участников (только для встреч)
- **[Имя]:** что рассказал

## Что может усилить архитектуру
Сигналы для Директора (Claude 100).

## Правила
- Пиши ТОЛЬКО в папку inbox-cl20/
- НЕ трогай другие папки (knowledge/, agents/, clients/)
- НЕ придумывай факты
- После сохранения ВСЕГДА делай git add, commit, push
- Коммит: «cl20: обработка — [тема]»

Подтверди что понял и готов к работе.
```

---

## Шаг 5. Проверь что работает

После того как Claude 20 подтвердит — напиши ему:

```
Обработай тестовый материал:
Сегодня на встрече МГ обсудили что участникам тяжело делать контент-план.
Маша предложила использовать формулу 4+1. Петя сказал что ему помогают аудиодневники.
```

Claude 20 должен:
1. Создать файл `inbox-cl20/встреча-2026-04-16-мг-контент.md`
2. Сделать git add, commit, push
3. Сказать тебе что готово

**Как проверить:** открой GitHub → vibeideicod-cmd/my-architecture → папка inbox-cl20 → должен появиться новый файл.

---

## Ежедневное использование

1. Открой **claude.ai/code** в браузере
2. Выбери проект **my-architecture** (если не выбран)
3. Вставь материал: транскрипцию, урок, идею, заметки со встречи
4. Claude 20 обработает и запушит сам
5. Когда работаешь со мной (Claude 100) в VS Code — скажи **«обработай входящие»** — я заберу всё из inbox-cl20/

---

## Если что-то пошло не так

**Claude 20 не видит репозиторий:**
→ Проверь подключение GitHub в настройках claude.ai/code

**Claude 20 не может запушить:**
→ Скажи ему: «сделай git pull и попробуй снова»

**Claude 20 потерял контекст (новая сессия):**
→ Вставь инструкцию из Шага 4 снова. Он прочитает README и продолжит.

**Claude 20 лезет в другие папки:**
→ Скажи: «Стоп. Ты пишешь ТОЛЬКО в inbox-cl20/. Прочитай inbox-cl20/README.md»
