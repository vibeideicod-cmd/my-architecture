# Создание собственного субагента (на примере YouTube-анализатора)

> Из курса по работе с Claude Code. Дополнения от Claude помечены значком 💡.

---

## О чём эта инструкция

Это **практический пример** того, как добавить нового субагента в уже существующую агентную систему. На примере YouTube-анализатора, который скачивает видео, транскрибирует его и делает выжимку. Шаги — те же, что для любого другого субагента, поэтому используйте этот пример как шаблон.

---

## Задача субагента YouTube Analyzer

Что должен уметь:
1. Принимать ссылку на YouTube-ролик.
2. Скачивать видео.
3. Транскрибировать его (звук → текст).
4. Делать краткую выжимку: ключевые инсайты, выводы, итог.
5. Встроиться в пайплайн так, чтобы после него можно было запустить `report-generator` (для PDF-отчётов).

---

## Шаги создания

### Шаг 1. Запустить новый диалог в Claude Code

Это важно — **новый**, чистый диалог. Если будете создавать субагент в перегруженном диалоге, могут закрасться ошибки из контекста.

```
/clear
```
Или просто откройте новый чат в расширении.

### Шаг 2. Описать задачу субагента

Простой, но конкретный промпт:

```
Создай нам ещё одного субагента. Его задача — скачивать видео с YouTube, транскрибировать его в текст и делать краткую выжимку по этому видео.

В выжимке должны быть:
- О чём конкретно это видео
- Несколько самых важных инсайтов, упомянутых в видео
- Ключевые выводы
- Краткий итог

Также этот агент должен встраиваться в нашу агентную систему так, чтобы после него можно было цепочкой запускать других агентов, например report-generator для создания отчётов по этому YouTube-видео.

Для скачивания YouTube-видео используется CLI-инструмент yt-dlp.
```

> 💡 **Дополнение от Claude:** обратите внимание — автор курса **прямо указал инструмент** (yt-dlp). Это важно. Если бы не указал, Claude мог выбрать другой подход. Когда у вас есть предпочтения по инструментам — сразу их называйте. Это экономит время и защищает от «творчества».

### Шаг 3. Что сделает Claude Code

В курсе показан реальный процесс:

1. **Анализирует, что уже установлено.** Claude Code проверяет, есть ли в системе нужные инструменты:
   - **Whisper** (от OpenAI) — для транскрипции аудио в текст. Бесплатный, локальный.
   - **FFmpeg** — для работы с видео и аудио форматами.
   - **yt-dlp** — для скачивания YouTube-видео.

2. **Сообщает, что есть.** В курсе у автора Whisper и FFmpeg уже были, а yt-dlp не было.

3. **Устанавливает недостающее.** Claude Code сам ставит yt-dlp через стандартную команду установки (например, `brew install yt-dlp` на Mac).

4. **Создаёт файл субагента.** Например, `.claude/agents/youtube-analyzer.md`.

5. **Обновляет CLAUDE.md** — добавляет нового агента в список и маршрутизацию.

### Шаг 4. Проверить результат

После создания обязательно откройте созданный файл `.claude/agents/youtube-analyzer.md` и проверьте:

```yaml
---
name: youtube-analyzer
description: Скачивает видео с YouTube, транскрибирует и делает краткую выжимку с ключевыми инсайтами и выводами.
tools: Bash, Read, Write
model: claude-sonnet-4-6
color: red
---

# YouTube Analyzer

## Зависимости (CLI tools)
- yt-dlp (для скачивания)
- whisper (для транскрипции)
- ffmpeg (для конвертации аудио)

## Алгоритм
1. Получить URL видео от пользователя или Router.
2. Создать папку agent-runtime/shared/youtube/<video_id>/
3. Скачать видео через yt-dlp в эту папку.
4. Извлечь аудио в формате mp3 через ffmpeg.
5. Транскрибировать аудио через whisper.
6. Прочитать транскрипт.
7. Сделать выжимку по структуре:
   - О чём видео (1 абзац)
   - Ключевые инсайты (3-5 пунктов)
   - Выводы (2-3 пункта)
   - Краткий итог (1 абзац)
8. Сохранить выжимку в agent-runtime/shared/youtube/<video_id>/summary.md
9. Вернуть путь к summary.md.
```

> 💡 **Дополнение от Claude:** проверка обязательна. Иногда Claude может:
> - Прописать ненужные tools (избыточные права).
> - Забыть очистку временных файлов после работы.
> - Не обработать ошибки (что если видео недоступно?).
>
> Поправьте такие места руками.

### Шаг 5. Тестирование

Запустите новый диалог и протестируйте:

```
Возьми вот этот YouTube-ролик: <ссылка>
Транскрибируй его и сделай выжимку с использованием youtube-analyzer.
```

Что проверить:
- Скачал ли видео.
- Прошла ли транскрипция.
- Корректна ли структура выжимки.
- Сохранился ли результат в `agent-runtime/shared/`.

### Шаг 6. Тестирование цепочки

Если задача работать в пайплайне — протестируйте связку:

```
Возьми этот YouTube-ролик, транскрибируй и сделай отчёт в PDF.
```

Должны последовательно отработать:
1. **YouTube Analyzer** — транскрибирует, делает выжимку.
2. **Report Generator** — берёт выжимку, делает PDF.

## Реальный пример из курса

В курсе автор взял реальное видео на английском языке про хобби нобелевских лауреатов.

**Промпт:**
```
Транскрибируй это видео с использованием агента, сделай отчёт и выжимку по нему.
<ссылка на YouTube>
```

**Результат через 2 минуты — PDF с такой структурой:**
- **Об авторе видео:** выпускник MIT, прошёл путь от бездомного подростка до советника миллиардных компаний.
- **Тема:** как экраны и соцсети разрушают мозг.
- **Ключевые инсайты:**
  - 70 часов экранного времени в неделю.
  - Толерантность дофамина — нужно всё больше стимуляции.
  - Подмена переживаний съёмкой для подписчиков.
- **Исследование:** 20-летнее исследование 773 нобелевских лауреатов. У них в 3 раза больше серьёзных хобби и в 9 раз выше вероятность формального обучения ремёслам/искусству.
- **Фреймворк VIPE** — 4 категории хобби: Vital force, Inquiry, Belonging, Expression.
- **Правила:** «правило трёх» (попробуйте хобби 3 раза, прежде чем решить), «не публикуйте хобби в соцсетях» (это убивает игру).

Качество выжимки — отличное. И всё это **за пару минут**, без участия пользователя.

> 💡 **Дополнение от Claude:** из практики: **YouTube Analyzer особенно полезен для длинных подкастов и лекций**. Двухчасовое видео → выжимка из 5 пунктов. Экономит огромное количество времени, когда хочется быстро понять суть, не тратя 2 часа на просмотр. Я сам бы такое использовал ежедневно — на мне же не остановишь, я модель.

## Ускорение через Supadata

Локальный Whisper медленный — особенно на длинных видео и слабых машинах. Можно использовать **онлайн-сервис Supadata** (supadata.ai):

- Бесплатный тариф — до 100 транскрипций в месяц.
- Работает через API, в разы быстрее локального Whisper.
- Поддерживает множество языков.

**Как переключить субагент:**

1. Зарегистрироваться на supadata.ai.
2. Получить API-ключ.
3. Добавить ключ в `.env`:
   ```
   SUPADATA_API_KEY=your_key_here
   ```
4. Сказать Claude Code:
   ```
   Внеси в субагент youtube-analyzer изменения: вместо локального Whisper используй Supadata API. Документация: https://docs.supadata.ai. Используй ключ из .env.
   ```
5. Claude Code обновит инструкцию агента.

> 💡 **Дополнение от Claude:** **обязательно** добавьте `.env` в `.gitignore` и в `deny`-список `settings.json`. С API-ключами в коде — катастрофа. Если случайно загрузите в публичный репозиторий, ключ будет украден за минуты (есть боты, которые сканируют GitHub). Всегда — только через переменные окружения.

## Принципы создания субагента (общие, не только для YouTube)

### 1. Сначала продумайте интерфейс

- **Что на входе?** URL? Файл? Текст? Параметры?
- **Что на выходе?** Файл? Структурированный JSON? Текст?
- **Где промежуточные данные?** В `shared/`?

### 2. Выберите модель под роль

- Анализ, синтез, рассуждения → **Opus**.
- Простая обработка по чёткому алгоритму → **Sonnet**.
- Массовые быстрые операции → **Haiku**.

### 3. Минимальные tools

В `tools` указывайте **только нужное**. Меньше — безопаснее и предсказуемее.

### 4. Chain of Thought в инструкции

Пошаговый алгоритм. Не «сделай выжимку» — а «1. скачай → 2. транскрибируй → 3. выдели инсайты → 4. сохрани → 5. верни путь».

### 5. Обработка ошибок

В инструкции явно опишите, что делать, если:
- Входные данные некорректные.
- Внешний сервис недоступен.
- Файл не создался.

Например: «Если транскрипция упала с ошибкой, верни Router сообщение: "Не удалось транскрибировать видео по причине X. Рекомендую попробовать другой источник."».

### 6. Логирование

В каждый шаг алгоритма добавьте короткое сообщение в лог. Это критично для отладки.

## Чек-лист добавления нового субагента

1. **[ ] Очистите контекст** (`/clear` или новый диалог).
2. **[ ] Опишите задачу** в одном промпте, чётко и конкретно.
3. **[ ] Укажите предпочитаемые инструменты**, если есть.
4. **[ ] Уточните встройку в пайплайн** — кому он передаёт результат, кто его вызывает.
5. **[ ] Дайте Claude Code сделать.**
6. **[ ] Проверьте созданный файл агента.** Откройте, прочитайте, поправьте если надо.
7. **[ ] Проверьте обновления в `CLAUDE.md`.** Добавлен ли в маршрутизацию?
8. **[ ] Проверьте установку зависимостей.** Если Claude установил CLI-инструменты — убедитесь, что они работают (`which yt-dlp`).
9. **[ ] Тестируйте отдельно.** Запустите субагент на простом примере.
10. **[ ] Тестируйте в цепочке.** Запустите промпт, который провоцирует пайплайн с новым агентом.
11. **[ ] Документируйте.** Если делитесь системой — добавьте описание в README.
