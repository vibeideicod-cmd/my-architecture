# Подключение Claude Code к Telegram и мобильному приложению

> Из курса по работе с Claude Code. Дополнения от Claude помечены значком 💡.

---

## О чём эта инструкция

Если вы хотите управлять своей агентной системой **с телефона**, не открывая компьютер — есть два официальных способа:

1. **Telegram-бот** — переписываетесь с Claude Code прямо в Telegram, как с обычным ботом. Удобно для команды (можно расшарить).
2. **Мобильное приложение Claude** + функция **Remote Control** — нативное приложение Anthropic, которое подхватывает сессии с компьютера. Удобно для личного использования.

В курсе показаны оба способа.

---

## Способ 1. Подключение через Telegram-бота

### Что нужно

- Установленный Claude Code (терминал или расширение в VS Code).
- Аккаунт Telegram.
- Установленный Bun (это рантайм для JavaScript, как Node.js, но быстрее).

### Шаг 1. Создать бота в BotFather

1. Откройте Telegram, найдите бота `@BotFather`.
2. Нажмите **Start** (или команда `/start`).
3. Команда `/newbot`.
4. Введите **название бота** — любое читаемое (например, "My Claude Code").
5. Введите **никнейм бота** — обязательно должно заканчиваться на `bot` (например, `myclaudecode_bot`).
6. BotFather пришлёт **bot token** — длинная строка вида `123456789:ABCdef...`.
7. **Сохраните этот токен** — он понадобится дальше. Никому не давайте его — это полный доступ к боту.

> 💡 **Дополнение от Claude:** обращайтесь с bot token как с паролем. Если он попадёт в чужие руки — человек получит полный контроль над ботом, сможет рассылать от его имени и читать переписку. **Не коммитьте его в git, не вставляйте в публичные документы, не публикуйте в скриншотах.**

### Шаг 2. Установить плагин Telegram в Claude Code

1. Зайдите на официальный сайт Claude Code (`claude.com/code` → раздел документации) и найдите **инструкцию по подключению Telegram**.
2. Там есть готовая команда для установки плагина — скопируйте её.
3. В Claude Code (терминал или расширение) **вставьте команду** в чат.
4. Claude Code предложит «Install for you» — подтвердите.
5. Плагин Telegram установится.

### Шаг 3. Подключить токен бота

1. На странице с инструкцией есть команда вида:
   ```
   /telegram-token "YOUR_BOT_TOKEN"
   ```
2. Замените `YOUR_BOT_TOKEN` на свой токен от BotFather.
3. Введите команду в Claude Code.
4. Claude сохранит токен и попробует связать аккаунт с ботом.

### Шаг 4. Установить Bun

Bun нужен как рантайм для работы плагина. Установка:

**На macOS:**
```bash
curl -fsSL https://bun.sh/install | bash
```

**На Windows:**
```powershell
powershell -c "irm bun.sh/install.ps1 | iex"
```

После установки **перезапустите Claude Code** — Bun должен подхватиться.

### Шаг 5. Активировать channels-режим

В Claude Code введите:
```
/channels
```

Claude переключится в режим работы через каналы (channels) — это режим, в котором он принимает сообщения извне (от Telegram).

### Шаг 6. Привязать ваш Telegram-аккаунт

1. Откройте созданного бота в Telegram.
2. Нажмите **Start** (`/start`).
3. Бот пришлёт сообщение с просьбой авторизоваться — там будет команда вида `/connect <ID>` или подобная.
4. Скопируйте эту команду из сообщения бота.
5. Вернитесь в Claude Code и **вставьте её в чат**.
6. Claude Code привяжет ваш Telegram-аккаунт к текущей сессии.
7. На вопрос «I am yes» / «подтвердить» → подтвердите.

Готово — теперь вы можете писать боту в Telegram, и сообщения будут приходить в Claude Code.

### Шаг 7. Проверка

В Telegram-боте напишите:
```
Привет
```

В Claude Code должно появиться сообщение «Привет». Claude ответит, и ответ придёт обратно в Telegram.

> 💡 **Дополнение от Claude:** обратите внимание — этот бот **связан с конкретным диалогом** Claude Code, который сейчас активен. Если переключитесь на другой диалог — связь нарушится. Это ограничение текущей версии. Автор курса упоминает, что работает над более продвинутым решением.

## Использование Telegram-бота

После подключения вы можете:

- **Задавать любые вопросы** Claude Code прямо из Telegram.
- **Запускать агентов** — например, «Запусти deep-researcher по теме криптовалют».
- **Запускать Agent Teams** — «Сделай финансовый прогноз через агентную команду».
- **Парсить сайты** — «Спарси новости с Hacker News».
- **Получать отчёты** — Claude может генерировать PDF, и в Telegram придёт ссылка или файл.

### Пример из курса

В Telegram бот спросили:
```
Какие агенты у тебя есть?
```

Ответ:
```
У меня 8 агентов:
- router
- deep-researcher
- parser
- news-digest
- doc-analyzer
- report-generator
- meeting-notes
- youtube-analyzer
```

После этого можно дать боту любое задание, которое выполнит вся эта команда.

> 💡 **Дополнение от Claude:** реальная польза в том, что вы можете запустить тяжёлый ресёрч, **закрыть телефон и заняться другими делами**. Через 5-10 минут бот вернётся с PDF-отчётом. Бот фактически становится точкой входа в вашу автоматизацию — без необходимости открывать ноутбук.

### Ограничения текущей версии

- Один Telegram-бот = один активный диалог Claude Code на компьютере.
- Если компьютер выключен / Claude Code не запущен — бот не работает.
- Нет возможности переключаться между разными проектами через бота.

### Решение этих ограничений

- **Запускайте Claude Code на отдельном VPS-сервере** (а не на личном ноутбуке) — он будет работать 24/7.
- **Сделайте отдельного бота для каждого проекта** — у вас может быть несколько ботов.
- Следите за обновлениями — Anthropic и сообщество работают над более продвинутыми интеграциями.

## Способ 2. Мобильное приложение Claude + Remote Control

Этот способ — проще и работает для **личного использования**.

### Что это

У Anthropic есть **официальное мобильное приложение Claude** (iOS / Android). В нём есть функция **Remote Control**, которая позволяет вашему телефону «подцепиться» к терминальному Claude Code на компьютере.

### Шаг 1. Установить приложение

- iOS: App Store → "Claude by Anthropic"
- Android: Google Play → "Claude by Anthropic"

### Шаг 2. Залогиниться

В приложении **залогиньтесь тем же аккаунтом**, что и на компьютере (Pro/Max/etc.). Это критично — Remote Control работает только если оба клиента под одним аккаунтом.

### Шаг 3. Активировать Remote Control в Claude Code

В терминальном Claude Code на компьютере введите:
```
/remote-control
```

Появится **QR-код** и кнопка **Connect Session**.

### Шаг 4. Подключить телефон

Два варианта:

**A. Через QR-код:**
1. На телефоне в приложении Claude → меню → Code.
2. Кнопка «Connect via QR code».
3. Сканируете QR-код с экрана компьютера.

**B. Через Connect Session:**
1. В Claude Code на компьютере нажимаете «Connect Session».
2. На телефоне в приложении Claude → меню → Code.
3. Текущая сессия появится автоматически.

### Шаг 5. Использовать

Теперь в приложении Claude на телефоне есть раздел **Code**. В нём дублируется всё, что происходит в Claude Code на компьютере.

- Пишете на телефоне → отправляется в Claude Code на компьютере.
- Claude отвечает → ответ приходит и на телефон, и в Claude Code.

> 💡 **Дополнение от Claude:** Remote Control работает синхронно — каждый раз, когда вы отправляете сообщение с телефона, оно появляется и в Claude Code на компьютере (можно даже видеть, как Claude печатает ответ). Это значит, **компьютер должен быть включён**, и Claude Code в нём — тоже.

### Условия работы Remote Control

- Компьютер с Claude Code должен быть **активен** в момент использования с телефона.
- Сессия должна быть открыта.
- Желательно, чтобы Claude Code не был свернут / в режиме сна.

### Решение для 24/7 работы

Чтобы пользоваться с телефона в любой момент, нужно, чтобы Claude Code на компьютере был всегда «жив». Варианты:

1. **VPS-сервер.** Снимаете виртуальный сервер (Linode, DigitalOcean, отечественные провайдеры), устанавливаете на него Claude Code, оставляете запущенным. К нему подключаетесь с телефона через Remote Control.
2. **Домашний сервер / NAS.** Если есть дома мощный ПК или NAS — можно держать его всегда включённым с запущенным Claude Code.
3. **Не выключать ноутбук.** Самое простое, но энергозатратное.

> 💡 **Дополнение от Claude:** для серьёзной работы с агентной системой 24/7 — VPS лучший вариант. На стандартный VPS (4 ГБ RAM, 2 vCPU) Claude Code легко уместится. Вы получаете доступ к своей агентной системе из любой точки мира, с любого устройства.

## Сравнение Telegram vs Remote Control

| Параметр | Telegram-бот | Remote Control |
|---|---|---|
| **Сложность установки** | Средняя (несколько шагов) | Простая (отсканировать QR) |
| **Кому удобно** | Команда (можно дать доступ нескольким людям) | Лично вам |
| **Нативность** | Через Telegram (3rd party) | Официальное приложение Claude |
| **Привязка к диалогу** | Один диалог за раз | Все ваши сессии |
| **Передача файлов** | Через Telegram | Через приложение |
| **Лимит сообщений** | По API Telegram | По подписке Claude |
| **Что нужно держать включённым** | Компьютер с Claude Code | Компьютер с Claude Code |

### Когда выбрать что

- **Лично вам, для личного контроля** → Remote Control (приложение Claude).
- **Для команды, чтобы делегировать** → Telegram-бот.
- **Для делегирования сотрудникам, чтобы они не имели прямого доступа к вашему ПК** → Telegram-бот.
- **Если планируете работать с разными проектами с телефона** → Remote Control (он работает с разными сессиями, Telegram пока нет).

> 💡 **Дополнение от Claude:** идеально — иметь **оба способа**. Remote Control — для оперативной работы лично с вами. Telegram-бот — как «точка входа» для рутинных команд (запустить мониторинг, получить дайджест), которыми могут пользоваться коллеги.
