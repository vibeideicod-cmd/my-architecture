# Протокол MCP — подключение внешних сервисов

> Из курса по работе с Claude Code. Дополнения от Claude помечены значком 💡.

---

## Что такое MCP и зачем он нужен

**MCP (Model Context Protocol)** — это протокол, разработанный Anthropic, который позволяет Claude Code общаться с внешними сервисами: Google Sheets, Notion, Slack, GitHub, YouTube, Google Calendar, Google Drive, и сотнями других.

Аналогия из курса: **MCP — это USB-хаб**. Ваш AI-агент — это компьютер. Через MCP он подключается к разным «устройствам» (внешним сервисам), и протокол даёт агенту понимание, что это за устройство и как с ним работать.

---

## Какую проблему решает MCP

### Без MCP: HTTP-запросы вручную

Если хотите, чтобы Claude работал с Google Sheets, можно «жёстко» прописать в инструкциях:
- URL endpoint API.
- Структуру запроса.
- Структуру ответа.
- Авторизацию.
- И так для **каждого** действия (создать лист, добавить строку, форматировать).

В Google Sheets API — **38 разных эндпоинтов** (создать лист, добавить строку, столбец, форматирование, и т. д.). Описать каждый — кошмарно много работы и токенов.

### С MCP: одна команда — и всё работает

MCP-сервер уже содержит:
- Все эндпоинты сервиса.
- Описание, как ими пользоваться.
- Структуру данных.
- Авторизацию через OAuth.

Claude автоматически «понимает», какие инструменты доступны, что они делают и какие параметры принимают.

> 💡 **Дополнение от Claude:** ключевое преимущество MCP — стандартизация. Вы один раз настраиваете сервер, а все совместимые AI-инструменты (Claude Code, Claude Desktop, Cursor, Antigravity и др.) сразу могут им пользоваться. Это «универсальный USB» для AI.

## Где найти готовые MCP-серверы

Те же маркетплейсы, что и для скилов:
- **Smithery** (smithery.ai) — есть отдельная вкладка MCP.
- **Skills the Sage** — там тоже доступны MCP.

В Smithery можно искать по категориям и сервисам: Google Sheets, Notion, GitHub, YouTube, Calendar, Slack, Drive, и так далее.

## Как установить MCP-сервер

### Способ 1. Через Claude Code (агентом, в один клик)

Это самый простой способ.

1. На странице MCP-сервера (например, Google Sheets в Smithery) есть **раздел «For agents»** — там готовый промпт.
2. Скопировать этот промпт.
3. Вставить в Claude Code (например, в чат в терминале).
4. Нажать Enter.
5. Claude Code сам выполнит установку — настроит, подключит, всё пропишет.

> 💡 **Дополнение от Claude:** этого способа не было раньше — это новая фича. Используйте именно его, если можете. Никаких ручных команд, никаких подводных камней.

### Способ 2. Через терминал вручную

1. На странице MCP в Smithery есть **раздел «For humans»** с готовой shell-командой.
2. Найти строку под Claude Code, нажать «Copy».
3. Открыть терминал (можно прямо в Claude Code через `!команда`, можно отдельно).
4. Вставить команду.
5. Нажать Enter.
6. Подождать пару секунд.

### Способ 3. Прописать руками в settings.json

В `~/.claude/settings.json` есть раздел `mcpServers`. Туда можно прописать сервер с указанием его endpoint и параметров. Это нужно, если:
- Сервер не в маркетплейсе (например, ваш собственный).
- Нужна тонкая настройка.

Этот способ уже для опытных пользователей — обычно достаточно способов 1 и 2.

## Авторизация в подключённом сервисе

После установки MCP сервер ещё не подключён к вашему аккаунту в этом сервисе. При первом вызове произойдёт следующее:

1. Вы пишете в Claude Code что-то вроде «создай Google-таблицу».
2. MCP-сервер видит, что нет авторизации.
3. Claude выдаёт **ссылку** для авторизации.
4. Зажимаете **Cmd (на Mac)** или **Ctrl (на Windows)** и кликаете на ссылку — она откроется в браузере.
5. На сайте маркетплейса (Smithery) вы логинитесь.
6. Выбираете аккаунт Google, который хотите подключить.
7. Подтверждаете доступ.
8. Возвращаетесь в Claude Code, пишете «готово».
9. Claude Code проверяет соединение и продолжает.

> 💡 **Дополнение от Claude:** при авторизации через Smithery вы фактически даёте маркетплейсу доступ к своему Google-аккаунту. Это нормальная практика для OAuth, но имейте в виду — у Smithery есть техническая возможность видеть запросы. Для критически чувствительных данных лучше развернуть свой собственный MCP-сервер локально (продвинутый сценарий).

## Управление установленными MCP

### Команда `/mcp` в Claude Code

Посмотреть, какие MCP сейчас установлены:
```
/mcp
```

Покажет список всех активных серверов с их именами, статусом (подключён / не подключён) и количеством инструментов.

### В расширении VS Code

В меню Claude Code внутри VS Code можно зайти в раздел MCP — там тоже виден список. **Но в расширении показываются не все MCP** — это известная проблема. В терминале список полный.

> 💡 **Дополнение от Claude:** ещё одна причина работать в терминале, а не в расширении — он показывает реальное состояние MCP. Если в расширении кажется, что MCP «нет», запустите `/mcp` в терминале — скорее всего, всё на месте.

## Минусы MCP — большой расход контекста

Это главный недостаток MCP. И его обязательно нужно понимать.

### Проблема

При каждом запуске диалога Claude Code загружает в контекст **описания всех инструментов всех установленных MCP**, чтобы знать, чем он может пользоваться.

Например, у Google Sheets MCP — 38 инструментов. Их описания занимают примерно **2 000 токенов**. Если у вас установлено **10 разных MCP** — это около **20 000 токенов**, которые попадают в контекст **с каждым запросом**, даже если вы не пользуетесь этими сервисами.

Можно проверить расход через `/context` — увидите, сколько занимают MCP.

### Когда MCP оправдан

- Вы пользуетесь сервисом **часто**, ежедневно (например, постоянно работаете с Google Sheets).
- В рамках одного диалога нужны разные действия в одном сервисе (читать, писать, форматировать).

### Когда MCP НЕ оправдан

- Вы пользуетесь сервисом **раз в неделю** или реже.
- Достаточно одного-двух действий, не всех 38.
- Хотите экономить контекст в больших проектах.

## Альтернатива MCP — скилы для конкретных API

В тех же маркетплейсах можно найти **скилы**, которые делают то же самое, но **подгружаются в контекст только при необходимости**.

Например, для Apify в Smithery есть и MCP-сервер, и скил. В описании скила прямо написано: «сделан, чтобы заменить тяжёлые MCP-протоколы на лёгкое подключение через скил».

**Разница:**

| MCP | Скил с API |
|---|---|
| Описание инструментов в контексте **всегда** | Описание в контексте **только когда вызван** |
| 2 000+ токенов «налогом» каждый диалог | 0 токенов в обычное время |
| Полная функциональность сразу | Нужно прописать использование явно или через хороший description |
| Подключение в 1 клик | Чуть сложнее настроить |

> 💡 **Дополнение от Claude:** автор курса намекает, что MCP — частично устаревающий протокол. Это спорно — Anthropic активно его развивает. Но факт остаётся: для редко используемых сервисов скил **существенно** экономит контекст. Стратегия: ставьте MCP только для **постоянно** используемых сервисов (1–3 шт.), для остального — скилы.

## Лайфхак: превратить MCP в скил

Если вы установили MCP, поняли, что им редко пользуетесь, но не хотите терять функциональность, можно **сделать из MCP скил**. Тогда подгрузка будет только по требованию.

### Как это сделать

1. Установите официальный скил **Skill Creator** от Anthropic (есть в Smithery).
2. В Claude Code напишите:
   ```
   У нас установлен MCP Google Sheets. Из этого MCP сделай мне скил. Проанализируй документацию этого MCP и создай скил, который будет учитывать всю эту документацию, при этом не будет загружать контекст каждый раз, когда мы не пользуемся этим MCP. Используй скил Skill Creator.
   ```
3. Claude Code запустит Skill Creator и сгенерирует:
   - `SKILL.md` с описанием.
   - Папку `references/` со списком всех инструментов API.
4. После этого можно **отключить MCP** — все возможности останутся в скиле, но без «налога» на контекст.

> 💡 **Дополнение от Claude:** это работает с любым MCP. Если у вас 5–10 MCP, и большинство нужны редко — переведите их в скилы. Контекст разгрузится в разы. Только не забудьте отключить или удалить сам MCP после создания скила, иначе будут оба сразу.

## Какие сервисы стоит подключать через MCP

Самые популярные и полезные (по совокупности отзывов и опыта автора курса):

| Сервис | Что даёт |
|---|---|
| **Google Sheets** | Работа с таблицами: создание, редактирование, форматирование |
| **Google Calendar** | Просмотр событий, создание встреч |
| **Google Drive** | Поиск, чтение, создание документов |
| **Notion** | Работа с базами и страницами Notion |
| **YouTube** | Парсинг видео, транскрипты, метаданные |
| **GitHub** | Работа с репозиториями, PR, issues |
| **Slack** | Чтение сообщений, отправка в каналы |
| **Chrome DevTools** | Автоматизация браузера (см. отдельную инструкцию) |
| **RSS-feeds / новости** | Регулярный мониторинг |

> 💡 **Дополнение от Claude:** правило выбора простое: если работаете с сервисом **каждый день** — ставьте MCP. Если **раз в неделю и реже** — ставьте скил. Если **разово** — даже скил не нужен, можно один раз попросить Claude использовать обычный API.

## Чек-лист подключения MCP

1. Открыть Smithery → раздел MCP.
2. Найти нужный сервис.
3. Скопировать промпт «For agents» или команду «For humans».
4. Установить (через Claude Code или терминал).
5. При первом вызове — авторизоваться по ссылке в браузере.
6. Проверить через `/mcp`, что сервер активен.
7. Если используете редко — рассмотреть конвертацию в скил через Skill Creator.
