# Agent Teams — режим агентных команд с коммуникацией

> Из курса по работе с Claude Code. Дополнения от Claude помечены значком 💡.

---

## Что такое Agent Teams и чем они отличаются от субагентов

**Agent Teams** — это новейшая экспериментальная функция Claude Code, которая позволяет субагентам **общаться друг с другом** напрямую, не только через главного агента.

### Сравнение

| Обычные субагенты | Agent Teams |
|---|---|
| Изолированы друг от друга | **Могут общаться между собой** |
| Передают результат только главному агенту | Могут запрашивать данные у других субагентов |
| Один путь данных: subagent → main → subagent | Двусторонняя коммуникация |
| Дёшево по токенам | **Очень дорого по токенам** |
| Подходит для большинства задач | Только для сложных, комплексных задач |

### Когда нужны именно Agent Teams

Если у субагентов могут быть **противоречия** или **необходимость переуточнения**, обычный режим не справится. Например:

- **Финансовый ресёрч:** parser спарсил данные, news-digest нашёл новости, deep-researcher делает анализ. Если data conflict (например, разные источники дают разный курс валют) — кто-то должен это резолвить. В обычном режиме они не могут согласовать данные напрямую.
- **Юридический ресёрч:** один агент анализирует законы, второй прецеденты. Им нужно «договориться» о трактовке.
- **Дизайн системы:** один агент проектирует БД, второй API. Им нужно согласовать схемы.

> 💡 **Дополнение от Claude:** в 90% задач Agent Teams **не нужны**. Обычные субагенты решают почти всё. Agent Teams — это «когда вы упёрлись в потолок обычных субагентов и видите, что им явно не хватает коммуникации». Не включайте режим из любопытства — он буквально жжёт лимиты.

## Включение Agent Teams

### Шаг 1. Прописать в settings.json

В `~/.claude/settings.json` (глобальный) или `.claude/settings.json` (локальный для проекта) добавить параметр:

```json
{
  "experimental": {
    "agentTeams": true
  }
}
```

Это **экспериментальная функция**. Включается только через `settings.json` — ни через `/config`, ни как-то ещё.

### Шаг 2. Прописать в CLAUDE.md

В главной инструкции проекта явно указать, что используется режим Agent Teams:

```markdown
## Режим работы

Этот проект использует Agent Teams (экспериментальный режим). Субагенты должны коммуницировать между собой при необходимости — запрашивать данные, согласовывать противоречия, передавать промежуточные результаты.
```

### Шаг 3. В промпте указать, что нужна командная работа

При запуске задачи явно написать:

```
Запусти агентную команду. Агенты должны обмениваться информацией между собой. Если одному агенту нужна дополнительная информация — он может запросить её у другого.
```

> 💡 **Дополнение от Claude:** если включить только `agentTeams: true` в settings, но не указать в промпте, что нужна командная работа — Claude может работать в обычном режиме субагентов. Поэтому **обе** настройки важны.

## Пример из курса: финансовый ресёрч

В курсе автор демонстрирует мощь Agent Teams на сложной задаче — анализ курса доллара к рублю на ближайшие месяцы.

### Промпт

```
Сделай детальный анализ по курсу валют на сегодняшний день и на ближайшее будущее. Очень важно оценить, что происходит на рынке, оценить новости по рынку и сделать краткий прогноз по курсу доллара на ближайшие несколько месяцев, насколько это возможно точно.

Для глубокого анализа используй парсинг сайтов, которые я тебе дам ниже. Также посмотри новости текущей мировой повестки и новости по Российской Федерации именно в финансовом секторе.

Чтобы агенты обменивались информацией между собой. Если одному агенту необходима дополнительная информация — он может запрашивать эту информацию у другого агента, чтобы прогноз был максимально точным.

В итоге мне важно получить детальный отчёт как от финансового аналитика по курсу доллара по отношению к российскому рублю на ближайшие несколько месяцев. Также пропиши вероятные сценарии развития событий и проценты вероятности каждого, а также свою рекомендацию — стоит ли сейчас доллары покупать или продавать.

Сайты для парсинга:
https://banki.ru
https://www.moex.com
```

### Что произошло

1. **Главный агент** прочёл задачу.
2. Запустил **News Digest** для сбора новостей.
3. Запустил **Deep Researcher** для аналитического ресёрча.
4. Запустил **Parser** для парсинга банковских и биржевых сайтов.
5. **Все три субагента работали параллельно**, при этом **общались друг с другом**:
   - Parser спросил у News Digest: «Какие сейчас крупные события на рынке нефти?»
   - News Digest передал данные.
   - Deep Researcher запросил у Parser свежие котировки.
   - Parser ответил.
6. **Каждый агент окрашен в свой цвет** (parser жёлтый, news-digest синий, deep-researcher зелёный, main бирюзовый, report-generator фиолетовый) — для удобства навигации в терминале.
7. После всех итераций главный агент собрал результаты, передал в **Report Generator**.
8. Report Generator создал PDF-отчёт.

### Результат

Полноценный финансовый отчёт с:
- Executive Summary.
- Текущим состоянием рынка.
- Ключевыми рыночными индикаторами.
- Фундаментальным анализом.
- Прогнозами от различных аналитиков (Goldman Sachs, ЦБ РФ, Минфин).
- Тремя сценариями (оптимистичный/базовый/пессимистичный) с указанием **вероятностей** в процентах.
- Помесячным прогнозом.
- Триггерами и ключевыми событиями.
- Стратегией хеджирования.
- Списком источников (он нашёл их сам).

> 💡 **Дополнение от Claude:** обратите внимание — **никто из субагентов не был «специализирован под финансы»**. Это были общие агенты: ресёрч, парсинг, новости. Но в Agent Teams они «договорились» и совместно собрали отчёт уровня финансового аналитика. Это и есть мощь режима — **синергия независимых ролей**.

## Как переключаться между агентами в Agent Teams

В терминальном Claude Code:

| Действие | Что делает |
|---|---|
| **Стрелка вниз** | Перебор активных субагентов |
| **Enter** | Войти в выбранного |
| **Escape** | Вернуться к main |

В каждом субагенте видно его **внутренние сообщения** — в том числе **то, что он получает от других субагентов**.

В курсе автор показывает диалог Parser:
- «Жду результатов от Deep Researcher и News Digest...»
- «Получил финансовые данные от News Digest»
- «Передал свои данные в Deep Researcher»
- «Подтвердил готовность данных»
- «Задача номер один выполнена»

Это реальный обмен в стиле «чата команды». Очень наглядно.

> 💡 **Дополнение от Claude:** **в расширении VS Code этого не видно**. Только в терминале. Если планируете работать с Agent Teams — обязательно ставьте Tmux (см. ниже) и используйте терминальный Claude Code.

## Визуализация через Tmux

### Что это

**Tmux** (Terminal Multiplexer) — программа, которая позволяет запускать несколько терминальных сессий внутри одного окна. С Claude Code в режиме Agent Teams Tmux даёт **визуально разделить**, в каком окне какой субагент работает.

### Установка

**На Mac:**
```bash
brew install tmux
```

**На Linux:**
```bash
sudo apt install tmux
```

**На Windows:** Tmux требует WSL (Windows Subsystem for Linux).

### Использование

1. В обычном терминале запускаете:
   ```
   tmux
   ```
   Появляется зелёная строка снизу — вы в режиме Tmux.

2. Запускаете Claude Code:
   ```
   claude
   ```

3. Когда главный агент запускает команду в Agent Teams, **Claude Code сам спавнит окна Tmux** для каждого субагента.

4. Вы видите все окна одновременно в одном терминальном окне.

### Что нужно прописать в settings.json

Чтобы Claude Code запускал агентов в Tmux-окнах, в `settings.json` надо добавить специфичные настройки. Точные параметры — в Telegram-канале автора курса.

> 💡 **Дополнение от Claude:** Tmux — продвинутый инструмент. Если вы новичок — освоение займёт время. Но для Agent Teams он невероятно полезен: вместо переключения стрелками между скрытыми вкладками вы видите всех агентов **одновременно**. Это качественно меняет восприятие работы системы.

## Главный минус Agent Teams — расход токенов

### Почему

В обычных субагентах среда изолирована — контекст одного не виден другому. В Agent Teams агенты **обмениваются контекстом**. Это значит:

- Каждое сообщение между ними — токены.
- Передача данных — токены.
- Ответы на запросы — токены.
- Переуточнения — токены.

Расход в **разы выше**, чем у обычных субагентов.

### Когда оправдано

- Подписка **Max** или **API-ключи без жёсткого лимита**.
- Задача **критически важная**, где качество выше денежной экономии.
- Сложность задачи такая, что без коммуникации агентов не справиться.

### Когда не оправдано

- **Pro-подписка (20 $/мес)** — лимиты сожрутся за один запуск.
- Задачу можно решить через обычный пайплайн.
- Хочется попробовать «чтобы посмотреть» — лучше посмотрите на видеоразборах.

> 💡 **Дополнение от Claude:** реалистично — на Pro-подписке Agent Teams включать **смысла нет**. Это инструмент для тех, кто на Max или API. Если вы на Pro — освойте обычные субагенты, доведите систему до ума, и только потом, когда апгрейднетесь до Max, экспериментируйте с Agent Teams.

## Сравнение: обычный режим vs Agent Teams (для одной и той же задачи)

В курсе показано: **на той же задаче с финансовым прогнозом** обычный пайплайн дал бы существенно более слабый результат, потому что:

- Без коммуникации Parser не знал бы, какие именно котировки актуальны.
- News Digest не знал бы, какие конкретные темы важны для парсера.
- Deep Researcher не мог бы запросить уточнений.

Каждый бы сделал свою часть в изоляции, и в финальном отчёте были бы пробелы. Координатор (главный агент) мог бы их заметить, но это потребовало бы дополнительных раундов запуска.

В Agent Teams всё это произошло **в одном раунде** — субагенты сами доспросили друг у друга нужное.

> 💡 **Дополнение от Claude:** если ваша задача — **исследовательская**, где смежные темы переплетены, и одно зависит от другого — Agent Teams даст качественно лучший результат. Если задача **линейная** (спарси → проанализируй → сделай отчёт) — обычные субагенты справятся не хуже, и в разы дешевле.

## Чек-лист использования Agent Teams

- [ ] У вас подписка Max или API без жёсткого лимита.
- [ ] Задача требует коммуникации между специалистами разных областей.
- [ ] В `settings.json` прописано `experimental.agentTeams: true`.
- [ ] В `CLAUDE.md` указано, что система работает в режиме Agent Teams.
- [ ] В промпте явно сказано: «агенты должны обмениваться информацией».
- [ ] Работаете в **терминальном** Claude Code (не в расширении).
- [ ] (Опционально) Установлен Tmux для визуализации.
- [ ] Готовы к большему расходу токенов.
- [ ] Все субагенты в проекте корректно работают в одиночку (это не «волшебная палочка», которая исправит плохие инструкции).

---

## Что важно понимать

Agent Teams — это **не магия**. Это инструмент, который позволяет уже работающим субагентам общаться. Если у вас субагенты сами по себе плохо работают — Agent Teams сделает только хуже (они будут путать друг друга).

Сначала **доведите до ума обычных субагентов**, протестируйте каждого отдельно, потом цепочки, и только потом — Agent Teams.
