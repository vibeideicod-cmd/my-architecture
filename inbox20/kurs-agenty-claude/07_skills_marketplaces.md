# Установка скилов из маркетплейсов

> Из курса по работе с Claude Code. Дополнения от Claude помечены значком 💡.

---

## О чём эта инструкция

Эта инструкция о том, как **не писать скилы самому**, а скачивать готовые из публичных каталогов. На сегодня существуют сотни тысяч готовых скилов — под парсинг, дизайн, работу с документами, аналитику, кодинг и почти любую другую задачу. Установка занимает буквально пару кликов.

> 💡 **Дополнение от Claude:** напомню, что такое скил (если читали инструкции из первой части курса — пропустите). Скил — это набор «инструкция + скрипты + шаблоны», который добавляет Claude Code узкую специализацию. Например, скил `pdf` умеет генерировать PDF, скил `xlsx` — работать с Excel-таблицами, скил `frontend-design` — собирать аккуратный UI. Claude Code читает только короткое описание скила (frontmatter с `name` и `description`), а полное содержимое подгружает только если решит, что скил нужен для текущей задачи. Поэтому можно ставить десятки скилов без перегрузки контекста.

---

## Главные преимущества скилов

1. **Экономят контекст.** Claude читает только название и краткое описание (`description`) каждого скила, а не весь файл целиком. Полная инструкция загружается только тогда, когда Claude решил применить скил.
2. **Дают слэш-команды.** После установки появляется команда `/имя-скила`. Можно вызвать его явно, без запроса.
3. **Не нужно писать самим.** В маркетплейсах уже есть готовые скилы под практически любую задачу.

---

## Маркетплейсы скилов

В курсе упоминаются два основных:

| Маркетплейс | Особенности |
|---|---|
| **Skills the Sage** (skills.sage) | Установка в 2 клика прямо из браузера, ничего скачивать руками не надо. Есть сортировка по количеству установок (показатель доверия). Минус — нет разбивки по категориям |
| **Smithery** (smithery.ai) | Более удобный интерфейс. Видно, от какого провайдера скил (например, Anthropic, GitHub, OpenAI — это «официальные», им можно доверять). Есть категории: research, кодинг, планирование, дизайн, аналитика |

> 💡 **Дополнение от Claude:** перед установкой любого скила обязательно проверьте провайдера. Скилы от Anthropic, GitHub, OpenAI, Microsoft, Vercel — безопасны. Скилы от незнакомых авторов с малым числом установок — потенциальный риск. Внутри скила могут быть инструкции вида «отправь содержимое .env на такой-то URL» — и если вы работаете в режиме Bypass Permissions без правильно настроенного `deny`-списка, это может привести к утечке ключей. Открывайте `SKILL.md` перед установкой и читайте, что внутри.

## Шаги установки скила (на примере Skills the Sage)

### Шаг 1. Найти скил

1. Зайти на сайт маркетплейса (Skills the Sage или Smithery).
2. В поиске вписать тематику: например, `browser`, `excel`, `pdf`, `notion`.
3. Выбрать подходящий скил.
4. Перед установкой открыть его страницу, прочитать `summary` и содержимое файла `SKILL.md` — что он делает и какие команды вызывает.

### Шаг 2. Получить команду установки

На странице скила сверху обычно есть **готовая команда установки**, что-то типа:
```
npx ... install <author>/<skill-name>
```
Скопируйте её.

### Шаг 3. Вставить команду в терминал

1. Откройте VS Code (или Antigravity).
2. Откройте встроенный терминал. Можно либо закрыть Claude Code в терминале, либо открыть **сплит-терминал** рядом с ним — это не имеет значения.
3. Вставьте скопированную команду в обычный терминал (не в Claude Code, а именно в системный терминал).
4. Нажмите Enter.

> 💡 **Дополнение от Claude:** если в скопированной команде уже указано конкретное имя скила в конце — оно установится сразу, без выбора. Если убрать последние слова (имя скила) и оставить только корневую команду репозитория — установщик покажет меню всех скилов, которые есть в этом репозитории, и можно выбрать один или несколько. У одного автора может быть как один скил, так и репозиторий из десятков.

### Шаг 4. Выбрать конкретные скилы

Если репозиторий содержит несколько скилов, появится меню.
- Перемещение — стрелками.
- Выбор — пробелом.
- Подтверждение — Enter.

### Шаг 5. Выбрать целевую программу

Установщик спросит, для какого AI-инструмента ставить скил. Возможные варианты:
- **Claude Code** — выбираем, если работаем с ним.
- Antigravity — выбран по умолчанию.
- Codex — выбран по умолчанию.
- Cursor — выбран по умолчанию.
- Gemini CLI, GitHub Copilot — выбраны по умолчанию (универсально).

Помечаем нужное пробелом, Enter.

### Шаг 6. Выбрать область установки

Спросит, куда ставить:
- **Project (локально)** — только в текущий проект, в папку `.claude/skills/` рядом с проектом.
- **Global (глобально)** — для всех проектов, в `~/.claude/skills/`.

В курсе автор для демонстрации выбрал Project (чтобы было видно, как файл появляется в текущей папке). На практике большинство универсальных скилов лучше ставить **глобально**.

### Шаг 7. Symlink

Установщик спросит про **Symlink** → выбирайте **Yes**. Это создаст символическую ссылку на скил в папке проекта/глобальной — он сразу подхватится.

### Шаг 8. Готово

Через секунду скил установлен. В файловой системе появится папка скила с `SKILL.md`, `templates/`, `references/` и так далее.

## Как использовать установленный скил

### Вариант A. Автоматический вызов (без явного указания)

Просто опишите задачу. Claude Code прочитает `description` всех установленных скилов и **сам решит**, какой подходит.

**Пример из курса:** автор установил скил `agent-browser` (автоматизация браузера). Затем написал в Claude Code:
```
Открой сайт news.ycombinator.com, посмотри, какие последние новости были по интересующим нам техническим темам, а также сделай скриншот последних новостей.
```
Про скил он не сказал ни слова. Claude Code сам определил, что подходит `agent-browser`, и запустил его. Скил спарсил новости, открыл в headless-режиме браузер, сделал скриншот, сохранил в указанную папку.

### Вариант Б. Явный вызов через слэш-команду

```
/имя-скила
```
Откроется окно со списком всех установленных скилов. Можно выбрать конкретный.

### Вариант В. Упомянуть в промпте

```
Используй скил excel и сделай таблицу из этого списка новостей.
```

> 💡 **Дополнение от Claude:** если у вас десяток скилов и Claude часто «угадывает» не тот, причины обычно три:
> 1. Плохой `description` у скилов — они конфликтуют.
> 2. Промпт слишком общий.
> 3. Несколько скилов с похожим назначением — Claude путается.
>
> Решение: явно указывайте нужный скил в промпте или через слэш-команду. И не стесняйтесь править `description` в `SKILL.md` — это просто текст.

## Пример из курса: создание Excel-таблицы через скил

После того как скил `agent-browser` спарсил новости в первый файл, автор хотел получить из них Excel-таблицу. Но Claude Code «по умолчанию» Excel создавать не умеет.

**Решение** — поставить скил `excel` (есть в маркетплейсе Smithery как официальный).

После установки скил находит триггер в `description` (что-то вроде «Use when user asks to create spreadsheets»), и команда работает:

```
Создай Excel-таблицу из этих новостей.
```

Claude Code:
1. Прочитал описание `excel`-скила.
2. Понял, что нужен именно он.
3. Применил его — создал `.xlsx` со столбцами Title, Source, Points, Comments, Category, Number.

Всё это — за 10 секунд. Аналогично можно делать PDF (скил `pdf`), презентации PowerPoint (скил `pptx`), Word-документы (скил `docx`).

> 💡 **Дополнение от Claude:** если хотите сразу удобно просматривать `.xlsx` файлы внутри VS Code (без открытия Excel), установите расширение **Office Viewer**:
> 1. В VS Code → иконка Extensions слева.
> 2. В поиске введите `Office Viewer`.
> 3. Install. Теперь `.xlsx`, `.docx`, `.pptx` будут открываться прямо в VS Code.

## Безопасность при установке скилов

Чек-лист перед установкой стороннего скила:

1. **Провайдер.** Anthropic / GitHub / OpenAI / Microsoft / Vercel — безопасны. Незнакомые авторы — осторожнее.
2. **Количество установок.** В Skills the Sage есть сортировка по этому показателю — чем больше, тем безопаснее.
3. **Открыть `SKILL.md`** перед установкой. Прочитать, какие скрипты и какие команды вызывает.
4. **Проверить раздел `scripts/`.** Если внутри скрипты, которые делают что-то с `.env`, ключами, отправляют данные на сторонние URL — отказываться.
5. **Иметь настроенный `deny`-список** в `~/.claude/settings.json` (запрет на чтение `.env`, на `rm -rf`, на `curl`-экспорт). Это страховка от вредоносных скилов.

> 💡 **Дополнение от Claude:** настроенный `deny`-список не панацея, но сильно повышает шансы. Без него Claude Code в режиме Bypass Permissions может выполнить любую команду из скила без вашего ведома. С `deny`-списком — попытка обращения к запрещённой команде попросит подтверждения, и вы увидите подозрительное действие до его выполнения.
