# Установка Chrome DevTools MCP — автоматизация браузера

> Из курса по работе с Claude Code. Дополнения от Claude помечены значком 💡.

---

## Что это и зачем нужно

**Chrome DevTools MCP** — это MCP-сервер, который даёт Claude Code полный контроль над браузером Chrome. С ним Claude может:

- Открывать любые сайты.
- **Кликать на конкретные элементы** (кнопки, ссылки).
- Заполнять формы (логин, пароль, поиск).
- Делать скриншоты.
- Парсить динамические сайты, которые без браузера не работают (одностраничные приложения, сайты с JavaScript-рендерингом).
- Эмулировать мобильное устройство.
- Снимать сетевые запросы.

> 💡 **Дополнение от Claude:** ключевое отличие Chrome DevTools MCP от простого скила-парсера — он работает не «снаружи» через HTTP-запросы, а **через настоящий браузер**. Это значит, что он видит сайт глазами пользователя: с JS-рендерингом, скрытыми элементами, pop-up'ами, авторизацией. Парсер на основе HTTP-запросов не сможет, например, добавить товар в корзину Wildberries — это требует кликов и работы с DOM. Chrome DevTools MCP сможет.

## Установка

### Шаг 1. Найти команду установки

В курсе используется готовая команда установки Chrome DevTools MCP. Её можно найти:

1. На сайте маркетплейса **Smithery** (smithery.ai) — раздел MCP.
2. В поиске вписать: `Chrome DevTools` или `puppeteer`.
3. Найти соответствующий сервер.
4. Открыть страницу.
5. В разделе **«For agents»** скопировать промпт, либо в **«For humans»** скопировать команду установки для Claude Code.

### Шаг 2. Установить через Claude Code

Самый простой способ:
1. Скопировать промпт из «For agents».
2. Открыть Claude Code в терминале.
3. Вставить промпт.
4. Нажать Enter.
5. Claude Code сам установит MCP.

Альтернативно — через `!`-режим в чате терминального Claude Code:
1. В чате Claude Code вписать `!`.
2. Вставить shell-команду установки.
3. Enter.

### Шаг 3. Проверить установку

```
/mcp
```
В списке должен появиться `chrome-devtools`.

> 💡 **Дополнение от Claude:** при установке Chrome DevTools MCP может потребоваться установить браузер Chromium — это произойдёт автоматически. Это довольно объёмная установка (200+ МБ), поэтому первый запуск может быть медленным.

## Пример использования (из курса)

В курсе автор демонстрирует работу на сайте Wildberries.

### Промпт

```
Используя MCP Chrome, найди в этом сайте https://wildberries.ru витамин D3 5000 и добавь его в корзину.
```

### Что делает Claude Code

1. **Открывает браузер** (Chromium в headless или visible режиме).
2. **Переходит на сайт** wildberries.ru.
3. **Использует поиск** — пишет в строку «витамин D3 5000».
4. **Просматривает результаты** — находит подходящую карточку товара.
5. **Делает скриншот** — чтобы понять структуру страницы (когда нет очевидной кнопки).
6. **Переходит в карточку товара**.
7. **Находит кнопку «Добавить в корзину»** и кликает на неё.
8. **Возвращает результат**: «Товар добавлен в корзину».

### Дальше — продолжение диалога

```
Перейди в корзину и посмотри все товары, что там есть.
```

Claude Code:
1. Кликает на иконку корзины.
2. Парсит содержимое корзины.
3. Возвращает список: товар, количество, цена, скидка.

## Какие задачи решает Chrome DevTools MCP

| Сценарий | Пример |
|---|---|
| **Парсинг сложных сайтов** | Сайты с авторизацией, динамическим контентом, бесконечной прокруткой |
| **Автоматизация действий** | Заполнить форму, нажать кнопку, добавить в корзину |
| **Тестирование UI** | Проверить, как сайт выглядит в разных условиях, снять скриншоты |
| **Мониторинг конкурентов** | Регулярно проходить по сайтам конкурентов, собирать данные |
| **E2E-тесты** | Автоматизировать пользовательские сценарии для проверки сайта |
| **Проверка backend** | Делать клики и смотреть, как запросы уходят на бэк (через DevTools network) |

> 💡 **Дополнение от Claude:** для **разработки сайтов** Chrome DevTools MCP — невероятно полезен. После того как Claude Code собрал ваш сайт, вы можете попросить его «открой свой сайт в Chrome, проверь визуально все секции, сделай скриншоты, найди проблемы». Это закрывает большую часть ручного QA. Особенно ценно для адаптивности — можно эмулировать мобильное устройство.

## Альтернативы Chrome DevTools MCP

Есть и другие MCP / скилы для работы с браузером:

| Инструмент | Особенности |
|---|---|
| **Chrome DevTools MCP** | Полный контроль через DevTools Protocol, нативная интеграция с Chrome |
| **Playwright MCP** | Кроссбраузерный (Chrome + Firefox + Safari), быстрее на сложных сценариях |
| **Puppeteer MCP** | Аналог, более старый |
| **Скил agent-browser** | Лёгкий, без MCP «налога» на контекст, для базовых задач |

> 💡 **Дополнение от Claude:** для большинства задач хватает либо **Chrome DevTools MCP**, либо **скила agent-browser**. Скил легче по контексту, но менее гибкий. Для разовых задач (парсинг, скриншот) — скил. Для регулярной автоматизации — MCP.

## Безопасность и осторожности

Chrome DevTools MCP даёт Claude **очень мощный** инструмент. Нужно понимать риски:

1. **Браузер с сохранёнными сессиями.** Если запустить Chrome DevTools MCP в окружении, где у вас уже залогинены банки, рабочие сервисы — Claude технически может «зайти» и в них.
2. **Опасность ошибки.** Claude может случайно отправить форму, оплатить заказ, удалить пост, нажав не туда.
3. **Параллельные команды.** При использовании в субагентах несколько процессов могут конкурировать за один браузер.

**Рекомендации:**
- Используйте **отдельный профиль Chrome** для автоматизации, без сохранённых паролей и сессий.
- На действиях с **деньгами и удалением данных** — обязательно с подтверждением (`Ask before edits` или явные `deny` в `settings.json`).
- Для **продакшена** не используйте этот MCP без человеческого надзора.

> 💡 **Дополнение от Claude:** в `settings.json` можно ограничить, какие сайты Claude может открывать через WebFetch. Аналогично можно настроить и для Chrome DevTools MCP — например, разрешить только определённые домены. Это сильно снижает риски.

## Ограничения

1. **Не работает в облачных IDE без браузера.** Если вы используете Claude Code на удалённом сервере без GUI — Chrome нужно запускать в headless-режиме, не везде это удобно.
2. **Скорость.** Каждое действие требует «подождать загрузки» — заметно медленнее, чем HTTP-парсинг.
3. **Жрёт оперативку.** Chrome — тяжёлое приложение. На слабых машинах может тормозить.
4. **Контекст.** Описания инструментов Chrome DevTools — большие. На каждый запрос «налог» в контексте.

## Чек-лист использования

- [ ] MCP установлен (`/mcp` показывает `chrome-devtools`).
- [ ] Использован отдельный профиль Chrome без важных сохранённых сессий.
- [ ] В `settings.json` добавлен `deny`-список на критичные действия (деньги, удаление).
- [ ] Промпт сформулирован чётко, с указанием конкретных действий.
- [ ] Если задача регулярная — оформлено в субагент с понятной инструкцией.
- [ ] Если задача разовая — рассмотрен лёгкий скил `agent-browser` как альтернатива.
