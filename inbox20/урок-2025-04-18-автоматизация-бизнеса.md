# Как устроен бизнес и что автоматизировать — Урок 18

## Суть (3-5 предложений)
Любой бизнес — от ларька до корпорации — это набор повторяющихся процессов по одной схеме: Лид → Контакт → Сделка → Выполнение → Аналитика. Между каждым шагом спрятаны десятки мелких ручных действий, которые «съедают» время команды. Деньги в автоматизации — не в сложных красивых решениях, а в простых повторяющихся задачах. Автоматизация должна измеряться конкретно: сколько часов или денег экономится в день. Если цифры нет — процесс выбран неправильно.

## Ключевые знания

1. **Универсальная схема любого бизнеса** — применять как карту при первом разговоре с клиентом: Лид (заявка/звонок) → Контакт (первое общение, квалификация) → Сделка (договор, оплата) → Выполнение (услуга/поставка) → Аналитика (отчёты, улучшения). Любой бизнес укладывается в эти 5 этапов.

2. **Тест: подходит ли процесс для автоматизации** — три критерия, все три должны быть «да»:
   - Повторяется регулярно — ежедневно, еженедельно, при каждой новой заявке
   - Занимает ощутимое время — от 15 минут в день и больше
   - Выполняется по шаблону — каждый раз делается одинаково

3. **Четыре зоны, где прячется автоматизация** — чеклист при аудите клиента:
   - **Работа с лидами:** первый ответ на заявку, приветствие, распределение по менеджерам, напоминания, квалификация клиента
   - **Работа с документами:** составление договоров, КП, типовые письма и ответы, выставление счетов, акты
   - **Работа с данными:** перенос между системами, заполнение таблиц, выгрузка из CRM, синхронизация баз, обработка форм
   - **Аналитика и отчёты:** ежедневные отчёты, сбор данных для презентаций, мониторинг показателей, дайджесты, отчёты для клиентов

4. **Матрица приоритетов — что автоматизировать первым:**
   - Первый ответ на заявку: 20–50 раз/день, 1–2 часа → **Высокий приоритет**
   - Составление договора: 3–5 раз/день, 30–60 мин → Средний
   - Ежедневный отчёт: 1 раз/день, 20–40 мин → Средний
   - Перенос данных между таблицами: 2–3 раза/день, 15–30 мин → Базовый

5. **Главный принцип ROI** — автоматизация считается в деньгах или часах. Если не можешь назвать цифру — процесс выбран неправильно. Нельзя автоматизировать «то, что прикольно сделать» (красивый дашборд, голосовой бот) без измеримого эффекта.

6. **Лучший способ найти процессы** — спросить сотрудников. Если кто-то говорит «я каждый день делаю одно и то же» — это идеальный кандидат для автоматизации.

## Промпты (если есть)

**Промпт 1 — Анализ бизнеса, поиск процессов:**

```
Я хочу найти процессы в моём бизнесе, которые можно автоматизировать.
Мой бизнес: [ОПИШИТЕ СВОИМИ СЛОВАМИ — чем занимаетесь, сколько человек в команде, какие основные задачи выполняются каждый день]

1. СХЕМА БИЗНЕСА
- Нарисуй простую схему моего бизнеса — от первого касания клиента до завершения сделки.
- Разбей на этапы (5–7 шагов).

2. АНАЛИЗ ПРОЦЕССОВ
- Для каждого этапа выпиши: что конкретно делается вручную?
- Кто это делает? Сколько времени занимает?

3. КАНДИДАТЫ НА АВТОМАТИЗАЦИЮ
Найди 5 процессов, которые:
- повторяются регулярно
- занимают больше всего времени
- выполняются по шаблону (не требуют творчества)

4. ПРИОРИТЕТЫ
Расставь эти 5 процессов по приоритету — что автоматизировать первым, чтобы получить максимальный эффект за минимальные усилия.
Сохрани результат в файл business-analysis.md
```

**Промпт 2 — Детальное описание одного процесса:**

```
Я хочу автоматизировать вот этот процесс:
[ОПИШИТЕ ПРОЦЕСС — например: "Каждый раз когда приходит заявка с сайта, я вручную пишу ответное сообщение в Telegram, копирую данные клиента в таблицу Google Sheets и ставлю задачу себе на звонок"]

Помоги мне описать этот процесс в формате, который подходит для автоматизации:

1. ЧТО ЗАПУСКАЕТ ПРОЦЕСС? (триггер — что происходит в начале)
2. ПОШАГОВЫЕ ДЕЙСТВИЯ — распиши каждый шаг: кто делает, что делает, сколько времени, какие данные нужны
3. ЧТО НА ВЫХОДЕ? — что должно получиться в конце процесса?
4. ГДЕ МОЖНО СЛОМАТЬСЯ? — типичные ошибки и исключения
5. ПРЕДЛОЖЕНИЕ ПО АВТОМАТИЗАЦИИ — как это можно автоматизировать с помощью Claude Code + Telegram Bot / Google Sheets?

Сохрани в файл process-[название].md
```

## Что может усилить архитектуру

- Промпт 1 — готовый инструмент для первой сессии с клиентом: до технического разговора провести бизнес-аудит и сразу получить карту процессов и точки входа.
- Тест из трёх вопросов использовать как фильтр при получении любого запроса — не браться за то, что не проходит тест.
- Матрица приоритетов — готовая структура для предложения клиенту: начинать с «первого ответа на заявку» (максимальный ROI), а не с красивых решений.
- Урок 19 — следующий шаг уже известен: автоматизация заявок с сайта → Telegram + Google Sheets.
