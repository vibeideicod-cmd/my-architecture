# Market Research — Анализ рынка

## Обзор

Запускается Analytics-Head → Analyst по запросу Директора или Product-Builder. Используется редко — только когда нужно ответить на вопрос «есть ли рынок». Результат: `clients/<имя>/market-research.md` или `knowledge/research/<тема>.md` если исследование общее.

## Порядок работы

### Шаг 1. Определи объект исследования

Уточни перед стартом:
- Что именно исследуем (ниша, продукт, сегмент)
- Гео: Россия / СНГ / международный
- Глубина: поверхностный обзор (30 мин) или глубокий анализ (2-3 часа)
- Цель: принять решение «входить / не входить», найти точку входа, подготовить питч

### Шаг 2. Объём и динамика рынка

```
- Объём рынка (TAM): [примерная оценка в рублях/долларах/единицах]
- Доступный сегмент (SAM): [сколько из этого реально достичь]
- Реалистичная доля (SOM): [сколько можно взять за 1-2 года]
- Динамика: [рынок растёт / стагнирует / падает — на сколько % в год]
- Источники оценок: [откуда данные]
```

Если точных данных нет — строй оценку снизу вверх: сколько потенциальных клиентов × средний чек × частота покупки.

### Шаг 3. Структура рынка

- Кто основные игроки (не детально — это /competitor-research)
- Есть ли лидер рынка который «съедает» большую долю?
- Рынок фрагментирован (много мелких) или консолидирован (несколько крупных)?
- Есть ли сезонность?

### Шаг 4. Тренды

Три категории трендов:
- **Технологические**: что меняет способ потребления продукта
- **Поведенческие**: как меняются запросы и привычки ЦА
- **Регуляторные**: что меняется в законах, что открывает или закрывает возможности

### Шаг 5. Барьеры входа

```
- Финансовые: [минимальные вложения для старта]
- Технологические: [что нужно уметь / иметь]
- Репутационные: [сложно ли войти без имени на рынке]
- Регуляторные: [лицензии, сертификаты, ограничения]
- Временные: [сколько нужно времени чтобы стать заметным]
```

### Шаг 6. Вывод: входить или нет

Итоговый ответ в одном из трёх форматов:
- **Входить сейчас** — рынок растёт, барьеры управляемые, есть незанятая ниша
- **Входить с оговорками** — рынок есть, но [конкретные условия для входа]
- **Не входить** — потому что [конкретная причина: насыщение, регуляторика, временные затраты vs потенциал]

### Шаг 7. Оформи результат

```markdown
## Анализ рынка: [Ниша/Продукт]
**Дата:** [дата]
**Гео:** [Россия/СНГ/другое]

### Объём рынка
[TAM / SAM / SOM + динамика]

### Структура
[Игроки, фрагментация, сезонность]

### Тренды
[Технологические / Поведенческие / Регуляторные]

### Барьеры входа
[По 5 параметрам]

### Вывод
[Входить / С оговорками / Не входить + обоснование]
```

Финальная фраза: «Анализ рынка готов. Передаю в работу.»

## KPI

Скилл отработал хорошо если:
- Есть хоть какая-то оценка объёма рынка — даже приблизительная с обоснованием метода
- Тренды реальные, с примерами — не «рынок растёт потому что ИИ везде»
- Вывод однозначный — нет «с одной стороны / с другой стороны»
- Источники данных указаны

## Рекомендуемая модель

**Opus** — стратегический анализ, нужно выстраивать логику и делать нетривиальные выводы из ограниченных данных
