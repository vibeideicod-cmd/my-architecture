# Транскрипция аудио — Слой 1 пайплайна

Превращает любой аудиофайл в текстовый `.md` через Deepgram REST API.
Без анализа, без разбора — просто сырой транскрипт.

## Когда тебя вызывают

- Инна положила файл в `inbox-inna/audio/` и сказала «обработай» / «транскрибируй»
- Инна дала путь к файлу в чате
- Инна дала ссылку на YouTube-видео без субтитров (после `yt-dlp` скачивания)
- В рамках Слоя 3 (`/client-onboarding-audio`) — как первый шаг

## Чего ты НЕ делаешь

- НЕ разбираешь транскрипт на смысловые блоки — это Слой 2 (`/audio-debrief`)
- НЕ раскладываешь факты по файлам клиента — это Слой 3 (`/client-onboarding-audio`)
- НЕ вытаскиваешь идеи / правки / задачи — это работа Director-а или Слоя 2 после твоего отчёта
- НЕ удаляешь исходный аудиофайл — он остаётся для возможного переслушивания

## Зависимости

Перед запуском убедись:
- ✅ `ffmpeg` установлен (`which ffmpeg` должно показать путь)
- ✅ Ключ Deepgram в Связке ключей под именем `deepgram-api-key`
- ✅ `jq` установлен (стандартно есть на macOS)

Если что-то отсутствует — стоп, скажи Инне что не хватает, дай команду установки. Не пытайся транскрибировать без ключа.

## Шаги работы

### Шаг 1 — определи источник и клиента

**Спроси у Инны если не очевидно:**
- Это про конкретного клиента? (если да — какого)
- Какая короткая тема? (2-4 слова — для имени файла)

**Не спрашивай если из контекста ясно:**
- Имя файла уже содержит тему (например `2026-05-08-ирина-визитка.m4a`)
- Файл лежит в `clients/<имя>/audio/` — клиент очевиден
- Инна сразу написала «это про клиента X, тема Y»

**Куда уйдёт транскрипт:**
- Клиент указан → `clients/<имя>/transcripts/<дата>-<тема>.md`
- Клиент не указан → `transcripts/<дата>-<источник>-<тема>.md`

### Шаг 2 — подготовь wav через ffmpeg

```bash
AUDIO_INPUT="<путь_к_исходнику>"
AUDIO_WAV="/tmp/audio-$(date +%s).wav"

ffmpeg -i "$AUDIO_INPUT" -vn -ar 16000 -ac 1 -c:a pcm_s16le -y "$AUDIO_WAV"
```

Если ffmpeg выдал ошибку «Invalid data found» — формат не поддерживается, покажи Инне ошибку и спроси что делать.

### Шаг 3 — отправь в Deepgram

```bash
DEEPGRAM_API_KEY=$(security find-generic-password -s deepgram-api-key -w)

if [ -z "$DEEPGRAM_API_KEY" ]; then
  echo "Ключ deepgram-api-key не найден в Связке ключей"
  exit 1
fi

curl -sS -X POST "https://api.deepgram.com/v1/listen?model=nova-3&language=ru&punctuate=true&smart_format=true&paragraphs=true&diarize=true&utterances=true" \
  -H "Authorization: Token $DEEPGRAM_API_KEY" \
  -H "Content-Type: audio/wav" \
  --data-binary @"$AUDIO_WAV" \
  -o /tmp/deepgram-response.json
```

Параметры расшифрованы в [knowledge/methodology/audio-pipeline.md](../../../knowledge/methodology/audio-pipeline.md) — там полная таблица. Не выбрасывай `diarize` и `utterances` — они нужны для встреч с несколькими спикерами.

**Проверь HTTP-статус:**
```bash
ERROR=$(jq -r '.err_code // empty' /tmp/deepgram-response.json)
if [ -n "$ERROR" ]; then
  echo "Deepgram вернул ошибку: $ERROR"
  cat /tmp/deepgram-response.json
  exit 1
fi
```

### Шаг 4 — извлеки текст и метаданные

```bash
DURATION=$(jq -r '.metadata.duration' /tmp/deepgram-response.json)
SPEAKERS=$(jq -r '[.results.utterances[].speaker] | unique | length' /tmp/deepgram-response.json)
PARAGRAPHS=$(jq -r '.results.channels[0].alternatives[0].paragraphs.transcript' /tmp/deepgram-response.json)
```

**Если спикеров > 1** — собери транскрипт с метками:
```bash
UTTERANCES=$(jq -r '.results.utterances[] | "**Speaker \(.speaker):** \(.transcript)\n"' /tmp/deepgram-response.json)
```

**Если 1 спикер** (диктофон / голосовое) — используй просто `$PARAGRAPHS`.

### Шаг 5 — сохрани .md по шаблону

```markdown
---
дата: 2026-05-08
источник: ирина-визитка.m4a
длительность: 18 минут
формат: m4a
спикеров: 2
---

# Ирина — обсуждение визитки

## Саммари

<5-10 строк выжимки. Опиши:
- О чём аудио в одну фразу
- Кто говорит (если есть имена в контексте)
- Ключевые факты, цифры, имена, даты
- Главные решения / задачи которые прозвучали
- Если есть незакрытые вопросы — отметь>

## Транскрипт

<если 1 спикер — paragraphs.transcript;
если > 1 — utterances со Speaker 0/1/...>
```

**Имя файла:** `<YYYY-MM-DD>-<источник>-<тема>.md`. Если клиент известен — `<YYYY-MM-DD>-<тема>.md` (источник и так в папке клиента).

**Саммари пиши сам** — Deepgram его не делает. Прочитай транскрипт, выпиши главное в 5-10 строк. Не пересказывай дословно, а вытащи суть. Если что-то не понятно — отметь в саммари «нужно уточнение у Инны: ...».

### Шаг 6 — почисти временные файлы

```bash
rm -f "$AUDIO_WAV" /tmp/deepgram-response.json
```

Исходный аудиофайл (`$AUDIO_INPUT`) **не трогай** — он остаётся в `inbox-inna/audio/` или `clients/<имя>/audio/` для возможного переслушивания. Инна сама решит когда удалить.

### Шаг 7 — отчёт Инне

Сообщи в чате коротко:

```
Транскрипт готов: <путь_к_файлу>.md
Длительность: <минут>, спикеров: <число>
Тема: <короткая>

Саммари: <2-3 предложения из шапки>

Что делаем дальше?
```

Не пытайся сам решать что делать дальше — это решение Инны. Возможные варианты которые можешь предложить:
- Запустить разбор через `/audio-debrief`
- Передать содержание конкретному агенту (например content-у для поста)
- Оставить как есть для будущей работы
- Если это онбординг клиента — сделать раскладку по 6 файлам клиента (Слой 3)

## Особые случаи

### Файл больше 2 GB
Deepgram не примет. Подели через `ffmpeg -t <секунд>` на части. Транскрибируй каждую отдельно, сшей результаты в один `.md`.

### Аудио на английском
В curl-команде смени `language=ru` на `language=en`. Если Инна говорит «там русский и английский» — используй `language=multi`.

### Длинная Zoom-запись (> 1 часа)
Saber всё равно транскрибируй целиком — Deepgram умеет долгие файлы. Просто запрос займёт больше времени (до 5-10 минут). Скажи Инне что обработка идёт.

### YouTube без субтитров
Сначала скачай аудио через `yt-dlp`:
```bash
yt-dlp -x --audio-format m4a -o "/tmp/yt-%(title)s.%(ext)s" "<URL>"
```
Дальше — обычный пайплайн с `m4a` на входе.

### Нет ffmpeg / нет ключа Deepgram
Стоп. Скажи Инне что нужно установить или положить. Не пытайся обойти.
