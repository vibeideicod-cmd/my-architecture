# Client Audit — Полный аудит нового клиента

## Обзор

Запускается Product-Builder или Директором в самом начале работы с новым клиентом. Даёт команде полную картину за одну сессию. Результат: `clients/<имя>/audit.md`.

## Порядок работы

### Шаг 1. Собери базовую информацию

Задай клиенту (или найди самостоятельно если клиент уже описан):
- Имя / название бизнеса, сфера, гео
- Сколько лет на рынке
- Сколько человек в команде (если есть)
- Основной продукт / услуга — что продают прямо сейчас

### Шаг 2. Аудит продукта

```
- Что продаёт: [конкретный продукт/услуга, формат, длительность, формат работы]
- Цена: [текущая цена / диапазон]
- Как продаёт: [звонок, переписка, встреча, автоматически]
- Результат для клиента: [что получает в итоге]
- Гарантии: [есть ли, какие]
```

### Шаг 3. Аудит аудитории

Кто реальный клиент (не «целевая аудитория» в теории):
- Возраст, пол, локация
- Как приходят (откуда узнают)
- Что говорят когда покупают (почему выбрали)
- Что говорят когда отказывают (почему не купили)
- Средний цикл сделки (от первого контакта до оплаты)

### Шаг 4. Аудит каналов

```
| Канал | Статус | Подписчики/охват | Что публикуют | Результат |
|---|---|---|---|---|
| VK | жив/мёртв/нет | ... | ... | ... |
| Telegram | жив/мёртв/нет | ... | ... | ... |
| Сайт | есть/нет | трафик | ... | ... |
| WhatsApp/чат-бот | есть/нет | ... | ... | ... |
```

### Шаг 5. Аудит конкурентов (экспресс)

Не полный анализ — только срез:
- 2-3 прямых конкурента, которых называет сам клиент
- Чем клиент отличается (по мнению клиента)
- Что конкуренты делают заметно лучше

Если нужен полный анализ → запускай `/competitor-research` отдельно.

### Шаг 6. Текущие боли клиента

Прямо спроси или выяви из разговора:
- Главная проблема которую хочет решить
- Что уже пробовал (и почему не сработало)
- Какой результат считает успехом через 3 месяца

### Шаг 7. Оформи audit.md

```markdown
## Аудит клиента: [Имя/Название]
**Дата:** [дата]
**Кто проводил:** Client Audit skill

### Бизнес
[Краткое описание — ниша, гео, размер, срок]

### Продукт
[Что продаёт, цена, формат, результат для клиента]

### Аудитория
[Кто реальный клиент, как приходит, цикл сделки]

### Каналы
[Таблица каналов со статусом]

### Конкуренты (экспресс)
[2-3 конкурента + точка отличия]

### Боли клиента
[Главная проблема + что уже пробовал + критерий успеха]

### Выводы для команды
[3-5 наблюдений которые влияют на стратегию работы с клиентом]
```

Финальная фраза: «Аудит готов. Передаю директору для определения следующего шага.»

## KPI

Скилл отработал хорошо если:
- Все 6 блоков заполнены (нет «неизвестно» без объяснения почему)
- Блок «Выводы для команды» содержит инсайты, не пересказ разделов
- Файл `clients/<имя>/audit.md` создан и готов к использованию командой
- Выявлены скрытые боли — не только то, что клиент назвал сам

## Рекомендуемая модель

**Sonnet** — диалог с клиентом + интерпретация — нужна способность улавливать контекст и задавать уточняющие вопросы
