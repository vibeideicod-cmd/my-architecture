Ты проверяешь SEO-метатеги в HTML-файле и дописываешь недостающие.

## Как вызывать

`/seo [путь к файлу]`

Если путь не указан — спроси: "Какой файл проверить?"

## Шаг 1. Прочитай файл

Найди блок `<head>` и проверь наличие каждого тега.

## Шаг 2. Составь отчёт

| Тег | Статус | Значение (если есть) |
|-----|--------|----------------------|
| `<title>` | ✅ / ❌ | ... |
| `<meta name="description">` | ✅ / ❌ | ... |
| `<meta name="robots">` | ✅ / ❌ | ... |
| `<meta property="og:title">` | ✅ / ❌ | ... |
| `<meta property="og:description">` | ✅ / ❌ | ... |
| `<meta property="og:url">` | ✅ / ❌ | ... |
| `<meta property="og:image">` | ✅ / ❌ | ... |
| `<meta property="og:type">` | ✅ / ❌ | ... |
| `<meta name="twitter:card">` | ✅ / ❌ | ... |
| `<meta name="twitter:title">` | ✅ / ❌ | ... |
| `<meta name="twitter:description">` | ✅ / ❌ | ... |

## Шаг 3. Предложи дополнение

Если есть недостающие теги — предложи готовый блок для вставки сразу после `<meta name="viewport">`.

**Правила для текстов:**
- `description`: 120-160 символов, конкретно, без штампов
- `og:description` = тот же текст что description
- `og:type` = "website"
- `twitter:card` = "summary_large_image"
- Стиль: простой язык, обращение "вы", глаголы вместо существительных

Бери за основу существующий `<title>` и содержимое h1, первых абзацев.

Если `og:url` неизвестен — заглушка `https://DOMAIN`, пометка "Замените DOMAIN".
Если `og:image` неизвестен — заглушка `/images/og-cover.jpg`, пометка "Добавьте обложку".

## Шаг 4. Спроси подтверждение

"Вставить теги в файл?"

Если да — вставь блок сразу после `<meta name="viewport">` через Edit.

## Примеры

**Вход:** `/seo clients/ukotel/index.html`
**Выход:** Таблица 11 тегов со статусами ✅/❌ + готовый `<meta>`-блок для вставки с заполненными title, description (148 символов), og:* и twitter:*

## Если данных не хватает

Если путь к файлу не указан — спроси: «Какой файл проверить?»

Если `og:url` неизвестен — ставь заглушку `https://DOMAIN` с пометкой «Замените DOMAIN».
Если `og:image` нет — заглушка `/images/og-cover.jpg` с пометкой «Добавьте обложку».

Не останавливайся из-за отсутствия og:image или og:url — выдай блок с заглушками.

## Что передаём дальше

Инне — отчёт (таблица тегов) + готовый блок `<meta>` для вставки.

После подтверждения («да», «вставь») — вставляет блок в файл через Edit сразу после `<meta name="viewport">`.

Финальная фраза скилла: «Вставить теги в файл?»

## KPI

Скилл отработал хорошо если:
- Все 11 тегов проверены и статус проставлен
- Недостающие теги предложены готовым блоком (не описанием)
- description укладывается в 120-160 символов

## Рекомендуемая модель

**Haiku** — рутинная проверка по чеклисту, детерминированный результат
