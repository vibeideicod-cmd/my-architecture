# Competitor Research — Анализ конкурентов

## Обзор

Разовый глубокий анализ конкурентов под конкретный продукт или клиента. Запускается Analytics-Head → Analyst. Результат: `clients/<имя>/competitor-research.md` с таблицей конкурентов и выводами для стратегии.

## Порядок работы

### Шаг 1. Уточни параметры

Перед стартом определи:
- Ниша и продукт (что именно анализируем)
- Гео (Россия / СНГ / международный рынок)
- Уровень конкурентов (прямые = тот же продукт для той же ЦА, косвенные = альтернативные решения той же задачи)

### Шаг 2. Найди конкурентов

Минимум 5 прямых конкурентов. Источники:
- Поиск Яндекс/Google по ключевым запросам ЦА
- ВКонтакте, Telegram — тематические сообщества и каналы
- Маркетплейсы и агрегаторы ниши
- Upwork / HeadHunter (если B2B)
- Кто упоминается в отзывах и сравнениях

### Шаг 3. Проанализируй каждого

По 6 параметрам для каждого конкурента:

```
**[Название конкурента]**
- Продукт/услуга: [что продают]
- ЦА: [кто их клиент]
- Цена: [ценовой диапазон или модель]
- Сильные стороны: [почему выбирают]
- Слабые стороны: [на что жалуются, где дыры]
- Каналы: [где продвигаются, какой масштаб]
```

### Шаг 4. Сведи в таблицу

```markdown
| Конкурент | ЦА | Цена | Сила | Слабость | Канал |
|---|---|---|---|---|---|
| ... | ... | ... | ... | ... | ... |
```

### Шаг 5. Выводы для клиента

Три блока:
1. **Белые пятна** — что никто не закрывает, куда можно зайти
2. **Точка отличия** — как клиент может выделиться относительно конкурентов
3. **Риски** — где конкуренты сильны, с кем лобовое столкновение нежелательно

### Шаг 6. Оформи результат

Файл `clients/<имя>/competitor-research.md`:

```markdown
## Анализ конкурентов: [Название проекта]
**Дата:** [дата]
**Ниша:** [ниша]

### Конкуренты
[Таблица по 5+ конкурентам]

### Детальные карточки
[По каждому из 6 параметров]

### Выводы
**Белые пятна:** [что не занято]
**Точка отличия для клиента:** [конкретная рекомендация]
**Риски:** [с кем лучше не конкурировать напрямую]
```

Финальная фраза: «Анализ конкурентов готов. Передаю в работу.»

## KPI

Скилл отработал хорошо если:
- Минимум 5 прямых конкурентов проанализированы
- Нет пустых ячеек в таблице — каждый параметр заполнен реальными данными, не предположениями
- Блок «Выводы» даёт конкретную рекомендацию, а не пересказ таблицы
- Результат сохранён в `clients/<имя>/competitor-research.md`

## Рекомендуемая модель

**Sonnet** — нужно находить, сопоставлять и интерпретировать разрозненные данные; Haiku может не уловить тонкие различия в позиционировании
