# Digital Audit — Анализ цифрового присутствия

## Обзор

Запускается Analytics-Head → Product-Analyst или напрямую по запросу Директора. Разовый срез: что есть, что работает, что мертво, что срочно чинить. Результат: `clients/<имя>/digital-audit.md`.

## Порядок работы

### Шаг 1. Карта присутствия

Составь полный список всех цифровых точек клиента:

```
| Канал | Ссылка | Статус | Последняя активность |
|---|---|---|---|
| Сайт | url | есть/нет | дата |
| VK | url | живой/мёртвый/нет | дата |
| Telegram | url | живой/мёртвый/нет | дата |
| WhatsApp/чат | - | есть/нет | - |
| YouTube | url | живой/мёртвый/нет | дата |
| Авито/2ГИС | url | есть/нет | дата |
| Другое | url | ... | ... |
```

Статус: **живой** = публикации за последние 30 дней, **мёртвый** = нет публикаций 30+ дней, **нет** = канала не существует.

### Шаг 2. Аудит сайта

Если сайт есть:
- Главный посыл первого экрана — понятно ли что предлагают за 5 секунд?
- Есть ли чёткий CTA (звонок, заявка, мессенджер)?
- Мобильная версия — работает, не работает, сломана?
- Скорость загрузки — быстро / медленно (субъективно)
- Форма связи / кнопки — кликабельны, ведут куда надо?
- Есть ли SEO-признаки: заголовки, описания страниц?

### Шаг 3. Аудит соцсетей

Для каждого живого канала:
```
**[Канал]**
- Подписчики/охват: [количество]
- Частота постов: [раз в неделю примерно]
- Что публикуют: [тип контента]
- Вовлечённость: [лайки/комменты — высокая/средняя/низкая]
- Есть ли продажи через канал: [да/нет/неизвестно]
- Главная проблема канала: [конкретно]
```

### Шаг 4. Аудит коммуникаций

- Как клиенты пишут/звонят — какой канал основной?
- Скорость ответа (если известно)
- Есть ли скрипт/шаблон ответа или всё вручную?
- Есть ли CRM или только переписки в личке?

### Шаг 5. Аудит рекламы

- Запускали ли рекламу когда-нибудь? Какую?
- Текущие рекламные кампании — есть / нет?
- Если были: какой результат, почему остановили?

### Шаг 6. Диагноз и приоритеты

```markdown
### Что работает
[Что реально приносит клиентов или трафик прямо сейчас]

### Что мертво и тянет вниз
[Брошенные каналы, устаревший сайт, сломанные кнопки]

### Критические дыры
[Чего нет совсем, но должно быть — например, нет CTA на сайте]

### Приоритеты (топ-3 действия)
1. [Самое срочное — что даст быстрый эффект]
2. [Второе по важности]
3. [Третье]
```

### Шаг 7. Оформи результат

Файл `clients/<имя>/digital-audit.md` по структуре шагов 1-6.

Финальная фраза: «Цифровой аудит готов. Передаю в работу.»

## KPI

Скилл отработал хорошо если:
- Карта присутствия полная — нет каналов которые пропустили
- Блок «Приоритеты» содержит конкретные действия, не общие слова
- Найдена хотя бы одна критическая дыра (сломанное, отсутствующее, мёртвое)
- Результат сохранён в `clients/<имя>/digital-audit.md`

## Рекомендуемая модель

**Sonnet** — нужно анализировать и интерпретировать, делать выводы из разрозненных данных
