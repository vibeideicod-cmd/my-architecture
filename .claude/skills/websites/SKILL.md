# Websites — вёрстка сайтов и лендингов

## Обзор

Верстаю HTML-страницы без внешних зависимостей — один файл, все стили внутри. Сначала согласую структуру, потом верстаю. Архетип дизайна выбираю до первой строки кода.

## Порядок работы

### 1. Собери данные

Прочитай `clients/{имя}/brief.md`. Убедись что знаешь:
- Что продаём / чему посвящена страница
- Кто аудитория
- Есть ли айдентика клиента (цвета, шрифты, логотип)
- Тип: лендинг / визитка / многостраничник / Mini App

Если брифа нет — спроси у Инны или вызови /brief.

### 2. Выбери архетип дизайна

**Обязательно до верстки.** Выбери один из 12 архетипов:

| Архетип | Когда |
|---|---|
| Брутальный минимализм | Премиум-консалтинг, юр.услуги, технологии |
| Максималистский хаос | Креативные агентства, мода, молодёжный досуг |
| Ретро-футуризм | Технологии, гейминг, стартапы с характером |
| Органик / натуральный | Эко, wellness, beauty, йога |
| Люкс / refined | Премиум-санатории, ювелирка, бутики |
| Игрушечный / playful | Детские товары, развлечения для семьи |
| Редакционный / magazine | Блоги, медиа, портфолио |
| Брутализм / raw | Индастриал, хардкор-стартапы, артхаус |
| Арт-деко / геометрический | Премиум-эстетика, ювелирка |
| Soft / pastel | Beauty, детское, женские бренды |
| Индастриал / utilitarian | Производство, B2B, инструменты |
| Tropical / vacation | Курорты, экскурсии, морская тематика |

Если у клиента уже есть айдентика — архетип подбирается под неё. Бренд клиента выше любого творчества.

Скажи Инне: «Беру архетип [Х], потому что [1 предложение].»

### 3. Предложи структуру — жди подтверждения

Формат:
```
Предлагаю структуру лендинга:
1. Hero: [что будет]
2. Услуги: [что будет]
3. Для кого: [что будет]
4. CTA: [что будет]

Подтверждаете — начинаю верстать?
```

Без подтверждения не верстаешь.

### 4. Верстай

Технические требования:
- Стили: только внутри `<style>` в том же файле
- Никаких CDN, Bootstrap, Tailwind, внешних JS
- Шрифты: только Google Fonts (`<link>`) или системные
- Адаптив: `clamp()` для текста, `flexbox`/`grid` для раскладки, `@media` для переломных точек
- Семантика: `<header>`, `<main>`, `<section>`, `<nav>`, `<footer>`

### 5. Дизайн-система

**Для внутренних продуктов бренда** (Инна-Архитектор, НБ СССР):

| Роль | HEX |
|---|---|
| Фон | `#ffffff` |
| Тёплый фон | `#d3bfb1` |
| Нейтральный фон | `#cbcbd4` |
| Главный акцент | `#d4af37` (золотой) |
| Hover / вторичный | `#c99700` (горчичный) |
| Поддержка | `#4b7850` (зелёный) |
| Глубокий | `#103206` |
| Основной текст | `#4c4651` |

⚠️ Оранжевого `#e86c3a` в бренде Инны НЕТ. Чёрный фон `#0a0a0a` — запрещён для внутренних продуктов.

**Для клиентских проектов** — палитра из брифа клиента.

Шрифты: Playfair Display (заголовки) + Montserrat (текст).

## Примеры

**Вход:** Бриф на лендинг для диетолога Анны, аудитория — женщины 30-45, цель — запись на консультацию, есть логотип и цвет `#4a7c59`
**Выход:** Структура (Hero + Обо мне + Программы + Отзывы + CTA) с подтверждением → HTML-файл с архетипом «Органик / натуральный», адаптив на мобильном/планшете/десктопе

## Если данных не хватает

Если нет брифа — вызови /brief, не придумывай детали о клиенте.
Если нет айдентики — выбери архетип сам, обоснуй 1 предложением, зафикси в brief.md.
Если нет текстов — попроси Copywriter или Content через Директора, не пиши placeholder «Текст здесь».

## Что передаём дальше

После верстки — QA на проверку.
После QA пройден — Deployer-Beget или /deploy на публикацию.

Формат: HTML-файл по пути `clients/{имя}/index.html` или корень проекта.

Финальная фраза скилла: «Вёрстка готова. Передаю на QA.»

## KPI

Скилл отработал хорошо если:
- Структура согласована с Инной до первой строки HTML
- Архетип выбран и зафиксирован
- 0 внешних библиотек и CDN
- Адаптив работает на мобильном, планшете, десктопе
- QA пройден без критических замечаний

## Рекомендуемая модель

**Sonnet** — вёрстка требует понимания дизайна, контекста клиента и принятия решений по архетипу; Haiku не вытянет суждения по визуалу
