Ты запускаешь деплой сайта на хостинг Beget.

## Шаг 1. Проверь .env

Прочитай файл `.env` в корне проекта.

Убедись, что заполнены все переменные:
- `BEGET_USER` — логин на Бегете
- `BEGET_HOST` — SSH-хост
- `BEGET_PATH` — путь на сервере

Если `.env` не существует или переменные пустые — скажи:
"Нет данных для деплоя. Нужно заполнить `.env`:
- BEGET_USER — логин на Бегете
- BEGET_HOST — SSH-хост (панель Бегета → SSH/FTP)
- BEGET_PATH — /home/логин/домен/public_html
- DOMAIN — домен сайта"

И остановись — не запускай скрипт.

## Шаг 2. Запусти деплой

Если `.env` в порядке — запусти:

```bash
./deploy-beget.sh
```

## Шаг 3. Сообщи результат

Если успешно — скажи: "Готово. Сайт обновлён на [DOMAIN]."
Если ошибка — покажи текст ошибки и предложи что проверить.

---

## Дополнительные команды

Если Инна говорит "проверь доступ" — запусти:
```bash
ssh ${BEGET_USER}@${BEGET_HOST} "echo OK"
```

Если Инна говорит "что задеплоится" — dry-run:
```bash
rsync -avzn --exclude='.git/' --exclude='.env' --exclude='*.md' --exclude='*.sh' --exclude='agents/' --exclude='knowledge/' --exclude='ideas/' --exclude='.claude/' --exclude='.DS_Store' ./ ${BEGET_USER}@${BEGET_HOST}:${BEGET_PATH}/
```

## Что НЕ делать

- Не деплоить без заполненного .env
- Не коммитить .env в git
- Не удалять файлы на сервере вручную без запроса Инны

## KPI

Скилл отработал хорошо если:
- Сайт доступен по домену после деплоя
- 0 ошибок rsync или SSH в выводе
- .env не попал в коммит и не улетел на сервер
