# YouTube-сценарий

Готовит сценарий YouTube-видео для клиентов в нишах с YouTube-присутствием
(инфобизнес, онлайн-образование, финансовые консультанты, коучи и психологи).

## Когда тебя вызывают

- Клиент попросил сценарий YouTube-видео под конкретный продукт
- Клиент дал ссылку на чужое видео и просит «такое же, но про моё»
- Нужно содрать структуру с успешного ролика конкурента и сделать своё

## Два режима работы

### Режим 1 — Разбор чужого ролика → свой сценарий

**Вход:** ссылка на чужое видео + что мы продаём + кто наша ЦА.

**Шаги:**
1. **Транскрипция** — через `knowledge/methodology/video-transcripts.md`:
   - Уровень 1: YouTube субтитры через WebFetch (бесплатно, для большинства роликов)
   - Уровень 2: Claude 20 в браузере через `inbox-cl20` (для коротких роликов)
   - Уровень 3: Deepgram/Supadata (когда разморозим скилл `/client-onboarding-audio`)
2. **Анализ структуры** — выдели 6 блоков чужого ролика (Хук, Open Loops, основные точки, CTA)
3. **Адаптация** — те же блоки под нашего клиента, его продукт, его ЦА
4. **Сценарий** — выдай по структуре ниже

### Режим 2 — Сценарий с нуля

**Вход:** тема + формат (экспертный / разбор кейса / интервью / ответы на вопросы) + хронометраж + продукт + ЦА.

**Шаги:**
1. Уточни **главную мысль** ролика (что зритель должен запомнить через сутки)
2. Сформулируй **хук** под боль ЦА
3. Распиши 6 блоков структуры
4. Выдай сценарий

## Структура сценария — 6 блоков

```
1. ХУК (0:00-0:30) — вступление с триггером
2. OPEN LOOPS — 2-3 крючка чтобы зритель досмотрел
3. СТРУКТУРА (основные точки) — 4-5 опорных смыслов с таймкодами
4. CTA — призыв к действию (подписаться / перейти / записаться)
5. МЕТА — название ролика, превью-идея, описание
6. ГОТОВЫЙ СЦЕНАРИЙ — шпаргалка для съёмки БЕЗ суфлёра
```

### Формат блока

```
⏱ БЛОК [номер] · [хронометраж]
Ключевые смыслы: [1-2 фразы что доносим]
Опорные точки:
  • [точка 1 — конкретно что говорим]
  • [точка 2]
  • [точка 3]
🪝 Open loop: «[конкретная фраза-крючок]»
🔗 Переход к следующему блоку: «[фраза-связка]»
⚠ Адаптировать: [места где клиент должен подставить свои кейсы / цифры / имена]
```

## Стиль клиента

**Перед стартом** прочитай `clients/<имя>/writing-style.md` — там зафиксирован
тон, длина, эмодзи, словарь клиента. Сценарий пишется в этом стиле.

Если файла нет — это **первая контент-задача для клиента**, попроси у Инны 2-3
примера в стиле клиента (или вытащи из его существующих видео / соцсетей) и
создай `writing-style.md` (см. раздел «Сбор стиля клиента» в `agents/content.md`).

**Регистр и пол** адаптируются под клиента. Никаких дефолтных «ребят», «короче»,
«блин» — только если это правда стиль конкретного клиента.

## Принципы

- 4-5 опорных точек на блок — **не текст для зачитывания**, а шпаргалка для
  съёмки без суфлёра. Клиент говорит своими словами по точкам
- **Open loops** под воронку клиента — крючки должны вести к продукту, не
  абстрактные «расскажу позже»
- **⚠ Адаптировать** в каждом блоке где клиент подставляет свои кейсы / цифры /
  имена — место для персонализации
- **Запреты:** filler-фразы из `knowledge/methodology/tov-anti-patterns.md`
  («Конечно!», «Давай разберёмся!»), хайп-эмодзи (🙌 👏 🚀 🔥), штампы «в мире
  где...»

## Где берёшь знания

- `clients/<имя>/writing-style.md` — стиль конкретного клиента
- `clients/<имя>/brief.md` — продукт, ЦА, цели
- `knowledge/methodology/video-transcripts.md` — инструменты транскрипции
  чужого видео для Режима 1
- `knowledge/methodology/tov-anti-patterns.md` — что нельзя использовать в
  тексте сценария

## Что передаём дальше

Финальный сценарий → `clients/<имя>/youtube/<дата>-<тема>.md`. Клиент или Ирина
оформляют тизер / превью отдельно (это не зона youtube-scenario).

## KPI

Сценарий отработал хорошо если:
- Клиент может снять видео по нему **без суфлёра** — точки понятны, своими
  словами раскрываются
- Open loops чёткие, ведут к продукту клиента
- Все «⚠ Адаптировать» места обозначены — клиент знает где вставить свои
  цифры / кейсы / имена
- Хронометраж ±20% от запрошенного

## Связки с другими скиллами и агентами

- **`/discovery`** — если клиент просит сценарий в рамках нового продукта,
  сначала Discovery → idea.md → потом сценарий с правильным фокусом
- **`agents/content.md`** — общий стиль клиента (writing-style.md), посты
  под YouTube-выпуск (как анонс)
- **`agents/copywriter.md`** — описание под видео и продающий блок (текст в
  описании ролика для перехода на лендинг)
- **`agents/qa.md`** — финальная проверка сценария на отсутствие filler-фраз,
  хайп-эмодзи, на читаемость без суфлёра
