# Сигналы непонимания + готовые реплики

> Справочник для скилла /discovery. Используется во время интервью —
> когда клиент сигналит непонимание, размытость или копирование без понимания.
>
> **Лимит ловли — ≤3 сигналов за интервью.** После третьего фиксируй
> в маркерах «нечётко по такому-то вопросу» и иди дальше — не дожимай.

---

## Таблица сигналов

| Сигнал клиента | Реплика-уточнение |
|---|---|
| «Как у [конкурент]» | «Покажи 2-3 примера, что именно у них тебе нравится — конкретный экран или функция?» |
| «Что-то такое модное» | «Давай сузим — модное это что: анимации, тёмный фон, минимализм, AI-фишки? Что из этого тебе откликается?» |
| «Всё, что есть на рынке» | «MVP = одна вещь, которую делаем сильно. Остальное — на этап 2. Если выбрать одну функцию, без которой продукт не имеет смысла — какая?» |
| «Не знаю, как это назвать» | «Опиши на пальцах — что человек делает шаг за шагом? С названием разберёмся.» |
| «Ну такое» | «Покажи на примере — открой сайт или приложение, которое для тебя ‘такое’. Разберём по нему.» |
| «Я думаю...» | «Это пока гипотеза или решение? Если гипотеза — давай зафиксирую как открытый вопрос, чтобы не строить на нём всё ТЗ.» |
| «Как-то так» | «Давай я уточню как мы это понимаем: [твоё толкование]. Так?» |
| «Удобный интерфейс» | «‘Удобный’ для кого и в каком сценарии? Разверни — что должен сделать пользователь за 5 секунд?» |
| «Современный дизайн» | «‘Современный’ это: тёмный фон / много пространства / большой шрифт / анимации / 3D? Что из этого?» |
| «Качественный» | «Качественный — по чему меряем: скорость, дизайн, точность ответов, отсутствие багов? Где в первую очередь?» |
| «Для всех» | «Этот продукт работает только если у нас есть конкретный ‘первый клиент’. Опиши его — кто, сколько лет, чем занимается, где живёт?» |
| «Простой и красивый» | См. [conflicts.md](conflicts.md) — это часто ложный конфликт. |

## Принцип формулировки

**❌ Плохо:** «Расскажи, как ты понимаешь X» — звучит как проверка, клиент закрывается.
**✅ Хорошо:** «Давай я уточню, как мы это понимаем: [моё толкование]. Так?» — открытое уточнение.

**❌ Плохо:** «Это слишком размыто, дай конкретику» — оценочно, клиент чувствует себя глупо.
**✅ Хорошо:** «Сузим до одной вещи — какая важнее всего?» — направляет, не оценивает.

**❌ Плохо:** «Ты не понимаешь, что говоришь» (даже мягко) — никогда.
**✅ Хорошо:** «Покажи на примере — у тебя есть готовый аналог, который можно открыть?»

---

## Лимит ловли

Не больше **3 сигналов** за интервью. Если ловлю в 4-й раз:

1. Зафиксируй в маркерах: «Размыто по [тема] — уточним на следующем раунде»
2. Продолжай интервью с тем, что есть
3. В ТЗ в секции «Открытые вопросы для разработки» перечисли все маркеры

**Принцип:** лучше выдать ТЗ с явными открытыми вопросами, чем превратить интервью в допрос и потерять клиента.

## Когда сигнал — это просто стиль речи

Не каждое «как-то так» — реальная размытость. Иногда это просто разговорная привычка. Проверь:

- Если за «как-то так» следует конкретика по сути — не лови, это слово-паразит.
- Если за «как-то так» нет ничего конкретного — это сигнал, реагируй.

Принцип: ловим **смысловую** размытость, не лексическую.
