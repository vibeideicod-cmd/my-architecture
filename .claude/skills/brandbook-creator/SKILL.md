# Brandbook Creator — документация бренд-стандартов

## Обзор

Беру бренд-платформу от Branding и превращаю её в рабочий документ. Цвета с кодами, шрифты с правилами, голос бренда с примерами — так, чтобы любой дизайнер и копирайтер сделали всё правильно без дополнительных объяснений. Никогда не принимаю решения по палитре без подтверждения официального источника.

## Порядок работы

### 1. Перед стартом выясни:

1. Чей бренд — наш (Инна/Ирина) или клиентский
2. Что уже есть — палитра, шрифты, логотип, предыдущие версии
3. Цель — полный брендбук, бренд-код, или конкретный раздел
4. Формат выдачи — MD-файл, HTML, или структура для Ирины

### 2. Правило палитры (КРИТИЧНО)

**Для нашего бренда** — официальная палитра только из PDF «26 04 Палитра Инны ВИЗУАЛ»:

| Цвет | HEX | Применение |
|---|---|---|
| Тёмно-зелёный | `#103206` | Фоны, основной |
| Зелёный | `#4B7850` | Акценты |
| Золотой | `#D4AF37` | Выделение |
| Жёлтый | `#FFDD4C` | CTA, кнопки |
| Кремовый | `#FFF2E9` | Текст на тёмном |
| Неоново-зелёный | `#13F740` | Акценты (осторожно) |
| Насыщенный зелёный | `#4DAA02` | Вторичный акцент |
| Ярко-розовый | `#FF2EC4` | Особые акценты |

Брендбук Ирины v3.0 содержит другие коды — **не использовать без явного указания Инны**.

**Для клиентского бренда** — берём только из материалов клиента, не привносим нашу палитру.

### 3. Структура полного брендбука

```
## Брендбук: [Название бренда]

### 1. Бренд-ядро
[Миссия, позиционирование, ЦА — из Branding]

### 2. Цветовая система
| Цвет | HEX | HSL | RGB | Применение |
|---|---|---|---|---|
| Основной | ... | ... | ... | Фоны, крупные блоки |
| Акцентный | ... | ... | ... | CTA, кнопки |
| Текст | ... | ... | ... | Основной текст |

Правильно / неправильно: [примеры сочетаний]

### 3. Типографика
| Шрифт | Вес | Размер | Использование |
|---|---|---|---|
| Montserrat | 700 | clamp(28px, 5vw, 48px) | H1 |
| Montserrat | 600 | clamp(22px, 4vw, 36px) | H2 |
| Montserrat | 400 | clamp(16px, 2.5vw, 20px) | Body |

CSS-переменные:
```css
:root {
  --color-bg: #103206;
  --color-accent: #FFDD4C;
  --font-heading: 'Montserrat', sans-serif;
  --font-body: 'Montserrat', sans-serif;
}
```

### 4. Логотип

### 5. TOV — Tone of Voice
[Как звучит бренд: словарь, запрещённые слова, примеры]

### 6. Примеры применения

### Чеклист готовности

```

### 4. Бренд-код (компактный формат, 1-2 страницы)

```
## Бренд-код: [Название]

Суть бренда: [одно предложение]

Цвета: [основной #hex] / [акцент #hex] / [текст #hex]
Шрифт: [название] [вес]
Голос: [3 прилагательных]
Нельзя: [3 запрещённых приёма]
```

### 5. TOV-стандарт

```
## TOV-стандарт: [Бренд]

### Голос бренда
[Тон: экспертный / дружеский / тёплый наставник]

### Словарь (используем)
[10-15 характерных слов и оборотов]

### Словарь (избегаем)
[Канцелярит, шаблоны, чужой стиль]

### Примеры
| Неправильно | Правильно |
|---|---|
| «Мы рады предложить вам» | «Вот что подойдёт для вашей задачи» |
```

## Примеры

**Вход:** Нужен бренд-код для нового клиента — психолог, палитра: бежевый #F5EDE3 и терракота #C67C52, шрифт Cormorant, тон тёплый и профессиональный
**Выход:** Бренд-код на 1 страницу с HEX + RGB + CSS-переменными, типографической шкалой для H1/H2/body, TOV: словарь «присутствие / бережно / пространство», запрещено: «работаем», «услуги», «пройдём путь вместе».

## Если данных не хватает

Если нет официальной палитры — не придумывай коды. Скажи: «Нужен источник: PDF от дизайнера или ссылка на существующий брендбук».

Если нет TOV — задай 3 вопроса: «Как бренд говорит — формально или по-свойски? Чего избегает? На кого похож из известных брендов?»

Если задача на наш бренд — сначала проверь `knowledge/brand/brandbook.md`, не изобретай заново.

## Что передаём дальше

Websites — CSS-переменные и UI-сниппеты для вёрстки.
Content / Copywriter — TOV-стандарт и словарь.
QA — аудит страниц на соответствие брендбуку.

Формат: структурированный MD-файл с CSS-переменными + чеклист готовности.

Финальная фраза скилла: «Брендбук готов. Передаю [Websites/Content/QA].»

## KPI

Скилл отработал хорошо если:
- Палитра взята из официального источника — не из памяти и не из брендбука Ирины v3.0
- Клиентский бренд не смешан с нашей палитрой
- CSS-переменные готовы к встройке без правок
- TOV содержит «неправильно / правильно» — не абстрактные описания

## Рекомендуемая модель

**Opus** — создание брендбука требует понимания визуальной системы, точности в цветовых кодах и умения формулировать TOV; ошибка в палитре = несоответствие бренду во всех будущих материалах
