# AI-Builder — строитель AI-консультантов

## Обзор

Создаю Продукт №4: AI-консультант, обученный на конкретном бизнесе клиента. Квалифицирует лидов, отвечает на вопросы, ведёт к покупке. Самопродаётся — клиент сначала видит AI Инны в работе и хочет такой же. Цена: 30-100к за создание + 5-15к/мес подписка.

## Порядок работы

### Этап 1: Бриф (интервью с клиентом)

6 вопросов — задавать блоками по 2:
1. Ниша и продукты: что продаёте, для кого, какие цены?
2. Частые вопросы лидов: топ-10 вопросов на входе?
3. Возражения: топ-5 и как с ними работаете?
4. Сценарии: как идеальный клиент приходит к покупке?
5. Что нельзя: какие темы AI не должен касаться?
6. Тон: формально / дружески / экспертно?

Результат: `clients/{имя}/brief.md` с данными для AI.

### Этап 2: Архитектура сценариев

```
Входящий посетитель
  ↓ Приветствие + 2-3 квалификационных вопроса
  ↓ Определение сегмента
  → Сценарий А: [тип клиента 1] → [нужный продукт]
  → Сценарий Б: [тип клиента 2] → [нужный продукт]
  → Сценарий В: [тип клиента 3] → [нужный продукт]
  ↓ Финал: запись / материал / покупка
```

### Этап 3: База знаний клиента

Структура файлов в `clients/{имя}/ai-knowledge/`:
- `about.md` — кто клиент, что делает, УТП
- `products.md` — все продукты, цены, что включено
- `faq.md` — ответы на частые вопросы
- `objections.md` — работа с возражениями
- `cases.md` — кейсы и результаты клиентов
- `contacts.md` — как записаться, как оплатить

### Этап 4: Системный промпт AI

```
Ты — AI-помощник компании [Название].
Твоя роль: [что делает этот AI]
Тон общения: [формальный / дружеский / экспертный]
Что ты знаешь: [ссылка на базу знаний]
Чего ты не делаешь: [ограничения]
Финальная цель каждого разговора: [запись / покупка / материал]
```

Правило: AI клиента не отвечает вне рамок базы знаний — лучше «не знаю, уточните у менеджера», чем галлюцинация.

### Этап 5: Реализация и тест

- Технология: Claude API (claude-sonnet-4-6 или выше, предпочтительно) или GPT API
- Интеграция: JS-виджет на сайт или через n8n webhook
- Тест: 20 тестовых диалогов по всем сценариям + крайние случаи (агрессивный лид, вопросы вне ниши, попытки сломать AI)
- Деплой виджета: Beget (production) / Vercel (тест)

### Этап 6: Ежемесячное ведение (подписка)

- Анализ диалогов: что спрашивают, где AI «тупит»
- Обновление базы знаний под новые продукты / акции
- Добавление новых сценариев
- Ежемесячный отчёт клиенту: диалоги, конверсии, улучшения

Подписка — конкретный объём: «обновление N сценариев + отчёт», не абстрактная «поддержка».

## Примеры

**Вход:** Диетолог Мария, 3 программы питания (3к/6к/12к), аудитория — женщины 30-45 с лишним весом, главные вопросы — «дорого» и «не хватает времени готовить», тон — «как подруга-врач»
**Выход:** Сценарий A (бюджет до 3к) → «Онлайн-старт»; Сценарий Б (есть время) → «Программа Баланс»; Сценарий В (нет времени) → программа с готовыми списками. Системный промпт: «Ты — AI-помощник диетолога Марии. Говоришь тепло, как подруга-врач, без медицинского занудства. Не ставишь диагнозы. Твоя цель — помочь понять какая программа подойдёт и записаться». База знаний: 6 файлов по структуре. Тест: 20 диалогов, 3 крайних случая (вопрос про лекарства → «это к врачу»).

## Если данных не хватает

Если клиент не может назвать топ-10 вопросов — задай: «Вспомните последние 5 входящих сообщений от лидов. Что они спрашивали дословно?»

Если нет кейсов и результатов — не выдумывай. Пиши в `cases.md` только реальные истории или оставь раздел пустым с пометкой «добавим после первых 3 клиентов».

Если непонятен тон — предложи 3 варианта: «формальный эксперт», «дружеский партнёр», «тёплый наставник» — пусть клиент выберет.

## Что передаём дальше

Websites — координируем интеграцию виджета AI на сайт клиента.
Systems — консультируемся по технической интеграции (n8n, webhook).
QA — передаю готового AI-консультанта на тестирование сценариев.

Формат: системный промпт + 6 файлов базы знаний + описание сценариев + инструкция по интеграции виджета.

Финальная фраза скилла: «AI-консультант готов. Передаю на QA и интеграцию.»

## KPI

Скилл отработал хорошо если:
- Пройдены все 6 этапов: бриф → архитектура → база знаний → промпт → тест → деплой
- 20 тестовых диалогов пройдены без галлюцинаций и выхода за рамки базы знаний
- Крайние случаи протестированы (агрессия, вопросы вне ниши)
- Подписка: конкретный объём работ, не «поддержка»

## Рекомендуемая модель

**Opus** — создание AI-консультанта требует проектирования сценариев, понимания психологии клиента и умения создавать системные промпты; ошибка в архитектуре сценариев = нерабочий продукт
