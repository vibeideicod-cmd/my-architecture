Ты проверяешь SEO-метатеги в HTML-файле и дописываешь недостающие.

## Как вызывать

`/seo [путь к файлу]`

Если путь не указан — спроси: "Какой файл проверить?"

## Шаг 1. Прочитай файл

Найди блок `<head>` и проверь наличие каждого тега из списка ниже.

## Шаг 2. Составь отчёт

Выведи таблицу:

| Тег | Статус | Значение (если есть) |
|-----|--------|----------------------|
| `<title>` | ✅ / ❌ | ... |
| `<meta name="description">` | ✅ / ❌ | ... |
| `<meta name="robots">` | ✅ / ❌ | ... |
| `<meta property="og:title">` | ✅ / ❌ | ... |
| `<meta property="og:description">` | ✅ / ❌ | ... |
| `<meta property="og:url">` | ✅ / ❌ | ... |
| `<meta property="og:image">` | ✅ / ❌ | ... |
| `<meta property="og:type">` | ✅ / ❌ | ... |
| `<meta name="twitter:card">` | ✅ / ❌ | ... |
| `<meta name="twitter:title">` | ✅ / ❌ | ... |
| `<meta name="twitter:description">` | ✅ / ❌ | ... |

## Шаг 3. Предложи дополнение

Если есть недостающие теги — предложи готовый блок для вставки сразу после `<meta name="viewport">`.

**Правила для текстов тегов:**
- `description`: 120-160 символов, конкретно, без штампов ("уникальный", "широкий спектр услуг" — запрещено)
- `og:description` = тот же текст что description
- `og:type` = "website" (по умолчанию)
- `twitter:card` = "summary_large_image"
- Стиль: простой язык, обращение "вы", глаголы вместо существительных

Возьми за основу текст уже существующего `<title>` и содержимое страницы (h1, первые абзацы) — сформулируй описание из того, что есть.

Если `og:url` неизвестен — поставь заглушку `https://DOMAIN` и напиши: "Замените DOMAIN на реальный адрес".
Если `og:image` неизвестен — поставь заглушку `/images/og-cover.jpg` и напиши: "Добавьте обложку для соцсетей".

## Шаг 4. Спроси подтверждение

"Вставить теги в файл?"

Если да — вставь блок сразу после строки с `<meta name="viewport">` с помощью Edit.
